Wrap a ChunkDemuxer inside ManifestDemuxer and set up time keeping

ManifestDemuxer will operate by owning a ChunkDemuxer and controlling
the access to it so that manifest parsers can add data. That means
ManifestDemuxer will need to keep a state of pending seeks, playback
rate, and current media time. This CL adds the inner ChunkDemuxer and
the time keeping, which relies on the ChunkDemuxer's outputting of
DecoderBuffers. A followup CL will add integration with the manifest
parsers and the seek/data loop.
