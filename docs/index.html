Fix the risk of division-by-zero crashes and resolve UNSAFE_TODO

If 0 is entered in the DIP text box, `size_` will still be updated. This
causes a division-by-zero crash during rendering. Update `size_` only
after valid non-zero input is received.

Additionally, an UNSAFE_TODO has been removed, and strtoul is replaced
with base::HexStringToUInt.
