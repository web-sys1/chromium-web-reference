[UIA] Fix readonly attribute on generated newlines

When the text range spans only a generated newline, we should return
the readonly text attribute value of the common anchor instead of the
one the endpoints' anchor.

This change is needed because Narrator appears to look at the
readonly value of that generated newline character to determine if
it is an interesting node Scan Mode should stop on. Without this,
Narrator's Scan mode gets stuck in the web page.
