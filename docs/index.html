[PA] Fix PartitionAllocGetSlotStartInBRPPool crash on slot start

Fix bug introduced by crrev.com/c/3436945. It assumed that
PartitionAllocGetSlotStartInBRPPool was only called with a pointer
within an allocated object, and neglected the fact that it was also
called from CheckThatSlotOffsetIsZero. This CL restores the previous
behavior of PartitionAllocGetSlotStartInBRPPool.
