Remove NGLayoutResult::IsSingleUse().

Sole fragments (i.e. when nodes don't break into multiple fragments)
will now be cached, also when they are inside a block fragmentation
context. We already had code in place to miss the cache if fragmentainer
size or fragmentainer offset changes. Added a few unit tests.

This is in preparation for another CL, where the SingleUse flag would no
longer be conditioned on simply being inside block fragmentation.
Removing the flag will ensure consistency, and make the logic simpler.
And as a "bonus" we'll sporadically hit the cache even when
block-fragmented. :)
