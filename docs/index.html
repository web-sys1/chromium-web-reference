android: LibraryLoader: also record thread time

This is based on a random guess.

Introducing a measurement of based on currentThreadTimeMillis() for the
time (CPU time) it takes to load the library. The measurements based on
uptimeMillis() already exist (ChromiumAndroidLinker.*LoadTime2). These
two kinds of measurements together may help confirm or reject the
hypothesis that library loading in the zygote is slow/fast because of
CPU cycle starvation.

Needed for a temporary investigation. Does not seem like a metric worth
maintaining for a long time.
