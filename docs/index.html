> [Fuchsia] Enable YUV_420_BIPLANAR usage on Fuchsia
>
> Previously YUV_420_BIPLANAR were used on Fuchsia only for
> FuchsiaVideoDecoder. GpuMemoryBufferVideoFramePool was using a pair of
> R8 and RG88 textures for YUV frames. In this change:
>  1. Implemented GetSupportedFormatsForTexturing() in Scenic and Flatland
>     ozone platforms
>  2. Updated GetPlatformSpecificTextureTarget() to return 0 on Fuchsia,
>     to satisfy DCHECKs in the existing code.
>  3. Updated gpu::FeatureInfo to detect YUV format support on Fuchsia.
>  4. GpuMemoryBufferVideoFramePool now passes ycbcr_info from the source
>     frame to the frames on the output.
>  5. VideoResourceUpdater now sets ycbcr_info for video frames where it's
>     not set by the decoder.
