PA: Remove the self pointer from shim functions

tl;dr: Removes the `self` table pointer from shim functions.
Uses a named table pointer instead.

Background:

OptimizeAllocatorDispatchTable [1] copies the shim function
pointers from the next table to the top table, and introduced
the following bug.

```
void ShimFunc(self, size, context) {
  MakeSideEffect();

  self->next->alloc_function(self->next, size, context);
}

TableA = { ShimFunc };

TableB = { nullptr };  // Inherits TableA.
```

With the above setting, ShimFunc will be called twice.
1. ShimFunc(TableB, size, context) as TableB->alloc_function
2. ShimFunc(TableA, size, context) as TableA->alloc_function
This is not the expected behavior. This pattern is fixed in this
patch as follows:

```
void ShimFunc(size, context) {  // No `self`.
  MakeSideEffect();

  // `TableA` is explicitly specified.
  TableA->next->alloc_function(size, context);
}

TableA = { ShimFunc };

TableB = { nullptr };  // Inherits TableA.
```

The following setting is also confusing.

```
void ShimRealloc(self, size, context) {
  if (size == 0)
    return self->free_function(self, size, context);

  // Do the job.
}

TableA = { ShimRealloc };

TableB = { nullptr };  // Inherits TableA.
```

With the current impl, ShimRealloc is called with TableB instead
of TableA, where ShimRealloc is originally defined. ShimRealloc
may expect TableA->free_function instead of TableB->free_function.

This patch removes the self argument and removes the related
confusions.
