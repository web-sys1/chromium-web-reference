AW SafeMode: disable SafeMode if prefs are missing

This ensures SafeMode will auto-disable itself if we're ever in either
of the following conditions when a client queries SafeMode state:

1. The timestamp pref is missing. This case should already be working
   because the timestamp defaults to Jan 1st, 1970 which should be
   expired on nearly every device. This just adds a test to verify this
   behavior in case the implementation changes in the future.
2. The actions list pref is missing (implemented and tested).
