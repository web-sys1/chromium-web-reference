[functions] Use AtomicString for attributes in CycleElem

The attribute name is already an AtomicString at both relevant
call sites. This reduces indirection noise when digging the value
out of the variant, and removes any concerns regarding the safety
of holding on to a raw pointer here.

This is a minor cleanup to prepare for detecting cycles
for local variables in @function rules.
