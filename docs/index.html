[iOS] Log metrics for ThemeSyncableServiceIOS

This change adds three new UMA histograms to track the lifecycle and
behavior of the new ThemeSyncableServiceIOS:
- IOS.ThemeSync.InitialSyncState: The state of the Sync server (empty,
  has data, error) during the initial setup.
- IOS.ThemeSync.RemoteThemeAction: The Sync client's decision when
  evaluating an incoming remote theme (applied, ignored due to policy,
ignored because it already matches, or error).
- IOS.ThemeSync.StopAction: When the user's pre-sync local theme is
  successfully restored because sync was halted, disabled, or cleared.

This will help monitor the health, adoption, and edge-case behavior of
the new Sync service.
