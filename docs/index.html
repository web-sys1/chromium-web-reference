[MAGI] Add test harness protocol, schemas, and runner script

This CL introduces a comprehensive "unit" test harness for the MAGI
multi-agent protocol to prevent regressions and backsliding in prompt
instructions and state machine transitions.

Key features:
 1.  **Test Protocol (SKILL_TEST.md)**: Defines the methodology for
     testing each phase independently using mock inputs.
 2.  **Structured Schemas (magi_test_schemas.json)**: Defines
     Scenario and TestCase structures supporting parameterization
     and expert injection (mock reviews).
 3.  **Phase-Specific Tests**: Initial test cases for Phase 1
     (Scaffolding) and Phase 5 (Review/Consensus) in dedicated JSON
     files.
 4.  **Harness-Agnostic Runner (run_magi_tests.py)**: A Python script
     to execute tests, providing full automation for Jetski (via
     agentapi) and a flexible fallback for generic CLI environments.
 5.  **Test Data (testdata/)**: Checked-in dummy files and a mock
     BUILD.gn to allow real compilable validation of agent-generated
     code.

Once the harness lands I am going to add more tests to fill the suite
out.
