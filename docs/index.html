Original change's description:
> [bluetooth] Fix crash in BluetoothRemoteGattCharacteristic.
>
> Fix a crash in the BluetoothRemoteGattCharacteristic destructor where
> pending notify commands are cancelled.
> BluetoothRemoteGattCharacteristic::pending_notify_commands_ referred
> to a nullptr resulting in a crash when calling
> NotifySessionCommand::Cancel().
>
> This is a reentrency bug which is caused by the the StartNotifySession
> error callback deleting its connection and destroying the
> BluetoothRemoteGattCharacteristic instance before it's pending
> notify command queue (pending_notify_commands_) has its front item
> popped.
>
> The ultimate cause of this bug is the way that pending_notify_commands_
> is used. It is a queue where the front item is the notify command
> currently being executed. The problem is that when the command
> is finishing (i.e. OnStopNotifySessionSuccess/Error) the front command
> is *moved* from the front of the queue into a local variable, leaving
> a nullptr in the queue until the callback has finished executing. This
> is done to prevent a new notify command from being executed in a
> reentrant call before the current notify command handler has finished.
>
> This fix pops the item from the queue before calling the success/error
> callbacks, and adds a guard value |notify_command_running_| to prevent
> a new notify command from executing until the current one has finished.
>
> Now that a null pointer isn't left in the queue, it was converted
> from pointers to structs for simplicy and efficient memory use.
> Additionally a sequence checker was added for sequence violation
> detection.
>
