[clang-plugin] Check discouraged types for blink data members

In blink renderer code, WTF container types are preferred to std
container types. Especially, member variables of std container types
or GURL are strongly discouraged [1]. Currently
audit_non_blink_usages.py checks for such usages, but it's too broad
(e.g. it also checks for std::vector for parameters or local variables
in a Web* implementation), and we often need to ignore the warnings
with 'git cl upload --bypass-hooks' which also ignores other checks.

The clang plugin (behind flag check-blink-data-member-types of
find-bad-constructs plugin) checks blink renderer data members for
discouraged types. It supports an annotation to suppress the check
if the usage is necessary.

[1] https://chromium.googlesource.com/chromium/src/+/main/third_party/blink/renderer/README.md#type-dependencies
