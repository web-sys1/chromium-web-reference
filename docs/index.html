Refactor PasswordSpecFetcher from ios/chrome to ios/components

This is so it may be shared with ios/web_view, which is not allowed to
depend on ios/chrome transitively.

ios/web_view is built once and may be used by any number of clients. To
support this use case, the API Key used for the password spec fetcher is
now passed into the initializer.
