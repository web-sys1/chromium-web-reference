CertVerifierServiceFactory: Speculative fix for mojo DCHECK if UpdateCRLSet called during shutdown

Mojo expects that the callback will be run unless the corresponding
binding has been closed. The CertVerifierServiceFactory binding is
essentially a singleton and will not be closed during shutdown, so the
ThreadPool can be shutdown while the Mojo methods could still be run.
