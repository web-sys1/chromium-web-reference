> WebXR: add enterprise policy for immersive-ar sessions
>
> This CL adds new enterprise policy that is used to gate access to
> immersive-ar sessions in WebXR Device API. The enterprise policy is
> per-profile, and is implemented by propagating the pref value from
> browser process to blink via WebPreferences.
>
> Additionally, the CL also introduces new AndroidBrowserTest that
> exercises the added glue code, with the assumption that the enterprise
> policy code that controls the pref is correct.
>
> Other changes:
> - add manifest entry in test jinja template in order for them to work on
>   newer Android versions
> - add DVLOGs in WebXR code paths to assist with debugging
