arc: Introduce kOutOfProcessVideoDecoding flag.

This CL introduces the kOutOfProcessVideoDecoding to decide whether we
should start utility processes to perform video decoding on behalf on
ARC++/ARCVM.

We use the feature (disabled by default) in
gpu_arc_video_service_host.cc to start the aforementioned utility
processes and request the binding of an
arc.mojom.VideoAcceleratorFactory. Because of this, we must declare the
type of sandbox this interface requires. For now, we use kUtility as a
placeholder. In a future CL, we define our own sandbox type.
