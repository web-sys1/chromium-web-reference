preload: Fix prerender doesn't work with same urls with different target_hint

Currently, PrerendererImpl::PrerenderInfo doesn't consider target_hint
and only push one prerender at a time in
`PrerendererImpl::ProcessCandidatesForPrerender`. This CL makes
`PrerendererImpl::ProcessCandidatesForPrerender` push one prerender per
target_hint value, and updates the dedup methods in
`PrerendererImpl::MaybePrerender` to avoid deduplicating prerenders with
a same url but different target_hint.
