Refactor externally managed installation flow

This CL has two main goals:
* Improving the readability of the installation flow and making
future changes easier to implement (DD [1]).
* Enabling in-place update of placeholder apps (DD [2]).

This is done by eliminating the split between installation tasks
and commands. This CL introduces a core `ExternalAppResolutionCommand`
command that is scheduled with a minimal lock when an externally
managed app should be installed. It loads all metadata and acquires
additional locks as needed and delegates functionality to sub jobs.
This part of the change implements no functionality change.

The 2nd part of this change implements in-place updates of placeholder
apps by checking in the `ExternalAppResolutionCommand` whether the app
id has changed and performs a migration if that is the case.
