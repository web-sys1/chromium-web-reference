Correctly uninstall the KM-TEST Loopback Adapter after using it.

The SandboxedNetworkChangeNotifierBrowserTest adds a test
"Microsoft KM-TEST Loopback Adapter" to verify network change
notifier behavior on Windows.

However, on certain platform configurations, simply removing the
device node did not correctly trigger the PnP manager to spot that
the device was no longer present, causing the network interface to
remain after the test had concluded (until the machine rebooted).

This CL updates the test to correctly call DiUninstallDevice to
match the SetupDiCallClassInstaller call, to correctly uninstall
the device completely.

This was tested manually by remoting into the bots, and verifying
that the network interface always gets removed after test finishes,
so as a result, this CL also re-enables the test.

BUG=1487750
