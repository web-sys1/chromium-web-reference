LegacyWebRtcVideoFrameAdapter: Don't convert to I420 when scaling NV12.

We had previously made it so that already-mapped NV12 frames can be
scaled without being converted to I420 but had forgotten to override the
legacy adapter's CropAndScale() which defaulted to "ToI420() + scaling".

This caused a bug when a multi-layered LibvpxVp8Encoder is used which
performs scaling prior to mapping. Not expecting the pixel format to
change, the encoder would drop all frames resulting in
https://crbug.com/1195144.

This CL fixes that by overriding CropAndScale(). The fix has been
verified both manually in Google Meet and in a unit test that is added.
