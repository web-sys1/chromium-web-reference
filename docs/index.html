ChromeOS: TabManager: Explicitly Skip Discarded Tabs

It's possible for a tab to be discarded and then killed again in
LowMemoryKillImpl(). Usually the KillTab() call will just fail for the
tab that is already discarded, but this behavior is subtle and hard to
understand.

Discarded tabs should be explicitly skipped and logged as such so the
memory logs are more meaningful.
