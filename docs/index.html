[ChromeCompanero] Add runtime data_deps on chromecompaneros for tests

Adds runtime data_deps on
//chrome/common/request_header_integrity/internal:chromecompaneros for
unit_tests and browser_tests source sets when is_chrome_branded is true.

Because libchromecompaneros is dynamically loaded at runtime via
ScopedNativeLibrary without compile-time linking, GN requires declaring
it in data_deps so that the build system compiles and stages the shared
library into the output directory ($root_out_dir) and packages it inside
Android test APKs (such as unit_tests_apk).

Importantly, this avoids adding data_deps directly to
request_header_integrity, which would cause production Android Clank
builds (like trichrome_chrome_google_bundle) to package multiple native
libraries and fail JNI trichrome linker scripts
(write_native_libraries_java.py: Multiple libraries not supported when
using chromium linker).
