Revive DiskCacheBackendTest.FileSharing on Windows

At [1], I replaced disk_cache::DeleteCacheFile calls to base::DeleteFile
calls. There are subtle difference between them (especially on older
Windows), the former returns true and the latter returns false when
failing to get a file information. See [2] for details. That also failed
DiskCacheBackendTest.FileSharing.

This change accepts the difference and fixes the test. By checking the
existing disk_cache::DeleteCacheFile calls in [1],

 - BlockFiles::RemoveEmptyFile
   The return value is used only for reporting UMAs and adding LOGs.
 - EntryImpl::DeleteData
   The return value is used only for reporting UMAs and adding LOGs.
 - SimpleSynchronousEntry::CloseSparseFile
   The return value is not used.
 - SimpleCacheDeleteFile(1)
   The target path is just created, so there should be no file handles
   that opens the path.
 - SimpleCacheDeleteFile(2)
   This is used only when the renaming failed, and we accept that the
   operation can be flaky in this case, so we're fine with the behavior
   difference.

the behavior difference doesn't matter.
