[Privacy Hub] Location - handle system components

As a part of the Privacy Hub efforts, This CL adds the location
handling logic for the ChromeOS system components. Users can now control
the system-level location permission that affects the whole system
(similar to Android's location permission). This user preference is
owned by the primary active user and is applied to the entire session.

This CL adds support for this toggle for all system components, namely:
WizardController, NightLightClient, GeolocationController and
TimeZoneResolver.
Currently the switch doesn't stop these clients to ping GLS when off,
but prohibits sending the wifi and cellular signal data to limit the
resolution to approximate-level (that is IP-based location).
