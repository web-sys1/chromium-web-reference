Move WebUI creation code from WebContentsImpl to NavigationRequest

Currently WebUI creation goes from RFHM -> RFH -> WebContents. This
means creation of WebUI depends on the existence of RFH, which won't
work well if an RFH is created only after OnResponseStarted due to
speculative RFH creation ban when a pending commit RFH exists, as
WebUI creation is needed to get to OnResponseStarted on WebUI
navigations.

We want to decouple WebUI creation from speculative RFH creation,
so that we can create a WebUI for a NavigationRequest and keep it in
the NavigationRequest until a RFH has been picked (and created, if the
RFH is a speculative RFH) for the navigation.

This CL is the first step towards that, where we move the WebUI
creation code from WebContentsImpl to NavigationRequest, and call the
WebUI creator function directly from RenderFrameHostManager instead of
through the RenderFrameHost. The created WebUI is currently still
immediately set to be owned by the RenderFrameHost, to keep the
changes in this CL small.

See doc: https://docs.google.com/document/d/1ke5ZlCgQC5O0vt7FopVnOKW9asz-ZRFLl3XRpE8tLk8/edit#
