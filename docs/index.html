Return -1 from OnStateUpdate if no frame will be produced.

Currently, this code returns the current parent sequence ID. But, we
want to be able to immediately apply any state requests if no frame
will be produced. The current behaviour will still need to wait for the
next frame to apply the request.

This is needed for fixing the case where we need to ack a origin change
in bounds in ozone/wayland, where no frame will be produced.
