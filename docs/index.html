Implement double press drag selection for views::Textfield.

Allows users to select text and adjust the selection in one double
press and drag movement. On the second tap down, the closest word is
selected. Then, starting a dragging movement (without lifting the
second press) expands/shrinks the selection. This is a part of a
planned ChromeOS feature (see design doc: go/cros-touch-text-editing).

To do this, record tap_down_count for ET_GESTURE_TAP_DOWN events. We
compute this using the same GestureDetector state variables as the
existing tap_count of ET_GESTURE_TAP events, so that the tap_down_count
and tap_count of corresponding TAP_DOWN and TAP events match.

Then, in Textfield, select text and update the selection dragging
state based on the tap down count. The selection behaviour for taps
should be similar to what is currently in Textfield::OnGestureEvent,
except that double taps and triple taps will change the selection when
the tap is pressed down, rather than when the tap is released.
