TextFragmentHandler: only generate for non-empty visible text range

Update the check before starting text fragment generation to ensure the
selection corresponds to a visible, non-empty text range within the
frame. This prevents generating links when the selection contains only
invisible or empty text. Consider the case:
<input type="file">
<script>
  document.execCommand('selectall');
</script>

If the user opens the context menu at this point, the selection has no
visible text, which previously could trigger a `DCHECK`. This change
avoids that crash by ensuring text fragment generation only runs for a
non-empty visible text range.
