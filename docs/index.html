[omnibox][3p-aim] Update button text, tooltip, and visibility on change

User's can toggle their DSE while the browser is still running. 3p AIM
configs can change from google <-> 3P <-> invalid (not all DSE's support
AIM button). When the config changes, the button text, tooltip, and
visibility are updated. Affects existing and new buttons (i.e. when
opening new browser window).

Buttons are initialized regardless of config validity on window
creation, but they're hidden if needed. This allows changing the config
from invalid -> to valid to work correctly.

Both the button (which is actually a page action) view and controller
instantiateion are guarded by null aim button service check. Because
while the config can change between null (invalid) to non-null (valid);
the service cannot. A null service (e.g. on special ASH profiles) won't
suddenly become non-null, so the page action won't be needed on these
profiles.
