[CrOS Hotspot] Remove OnHotspotStateFailed() from observer

This CL removes the OnHotspotStateFailed() from
CrosHotspotConfigObserver because as discussed with the platform team,
instead of using the error code in hotspot state, SetTetheringEnabled
now will return a result string to indicate success or not including
error information. So, we don't need to observe OnHotspotStateFailed.
See crrev/c/4027263 for platform related change.
