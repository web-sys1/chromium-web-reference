Image descriptions should check security origin, not document url.

The image descriptions feature is limited by protocol: only allowed
on http, https, file, and data urls. However, checking the document
url isn't the best way to determine this, because an iframe created
by a parent document without a "src" will appear to have a url of
"about:blank", but it's actually controlled by its parent document.

The solution is to check the security origin, which returns the
correct protocol of the owning document if different.

This enables the image description feature to work in Google Docs,
among other places.

Bug: 1211002
AX-Relnotes: Fixes issue where automatic image descriptions weren't being generated in some iframes.
