webview/nonembedded: Add reset experiments action

When a user enables any experimental flags in the
dev UI, the developer UI service will become a
foreground service and have a constant notification
warning the user they have flags enabled that
may cause issues.

Adding an action that resets the flags
so that users have a clear and easy way to do
this.

Resetting the flags could have issues with state
if the flags fragment was already open because
we'd need to sync up that state from the service.

Using the same flags intent and adding a reset
experiments intent so that the user also
sees where this leads. This also deals with
that state issue.

Still letting the flags fragment control the resetting
because if we managed the reset from the MainActivity,
we'd need to wait for the service connection to finish
before we could open the fragment which would block UI.
We'd also be modifying flag state from more than one place
would could be confusing.

Decided to use the same flag icon for the action. This
won't be visible from Android N but it could still show
up in places. The flag seems like an appropriate icon for
this.
