Sort inspectable views on chrome://extensions page

When more than one inspectable view exists for an extension, the
chrome://extensions page currently shows just the first view and links
to the details page to show more. The backend was sending the service
worker last which meant it was often multiple clicks to access.

This could have been fixed on the backend, but it feels like it would
be likely to regress there as this is a UI concern and it would not
be clear that the order is important. Consequently, I fixed this here
instead.
