Fix dangling pointers in TestPrintRenderFrame.

PrintBrowserTest owns TestPrintRenderFrame objects, which in turn has
raw_ptrs to some //content objects. These pointers hang around for too
long and point to objects for a tab after the tab went away during tear
down. Fix this issue by freeing the TestPrintRenderFrame objects in
PrintBrowserTest::TearDownOnMainThread() before all the tabs close.
