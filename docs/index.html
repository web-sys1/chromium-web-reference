Fix bug in overlay promotion when using fullscreen overlay strategy

Background: `last_successful_strategy_` is tracked to figure out if we
can remove the main plane. This is possible if there is an opaque
overlay that covers the entire display. (FullscreenOverlayStrategy)

Additionally, after crrev.com/c/5817538, we temporarily disable overlay
promotion for a certain number of frames if we see a capture request.

Cause of flickering during recording: When we started screen capture for
a fullscreen ARC++ app, the first frame would get promoted using
fullscreen overlay strategy. However, the next consecutive frames would
be skipped due to temporal disable of overlays. But unfortunately,
`last_successful_strategy_` would have stale Fullscreen strategy since
it was reset/updated in
`OverlayProcessorUsingStrategy::AttemptWithStrategies()` and would not
get called since overlays were disabled.

Therefore we would incorrectly remove the primary plane, even though the
fullscreen quad was not getting promoted causing flickering.
