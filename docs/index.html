[Omnibox] Clipboard can set and detect URL mime type

On Android S, the system clipboard can tell apps what type of data is in
the clipboard. But Android do not recognize the scheme "chrome://", so
when a URL start with "chrome://" in the system clipboard, Android S
will tell Chrome that it contents text.

To make Chrome know if "Chrome://" is in the clipboard, this CL needs to
update 2 things:
1. ClipboardRecentContent can recognize scheme "Chrome://".
2. Clipboard.java will set mime type to URL when copy a URL to the
   clipboard. And also check the mime type when checking the content
   type of the clipboard.
