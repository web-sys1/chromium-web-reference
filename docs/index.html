[FedCM] Clean up more using declarations in content/browser/webid/

Clean up using declarations in header and source files under
content/browser/webid/ to follow Chromium style guidelines.

Specifically:
- Remove namespace-scoped using declarations (like IdentityProviderDataPtr
  and IdentityRequestAccountPtr) from header files, replacing them with
  explicit types (scoped_refptr<IdentityProviderData> and
  scoped_refptr<IdentityRequestAccount>).
- Move using declarations in header files to class scope where they are
  intended to be public helper aliases (e.g. in NetworkRequestManager,
  EmailVerificationRequest, UserInfoRequest).
- Move using declarations in .cc files inside the namespace or class
  scope to avoid polluting the global namespace.
- Clean up unused or redundant using declarations.

This makes the code cleaner and conforms to the C++ style guide.
