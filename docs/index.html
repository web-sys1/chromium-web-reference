dpwa: Fix CreateInstallInfoForCreateShortcut

With features::kCreateShortcutIgnoresManifest enabled, a shortcut
to a manifested pages is supposed to behave exactly like the
manifest says, except the current URL should be used as start-url,
and the manifest_id should be deleted (to avoid clashes and updates).

Before this CL, the tests matching *WebAppIntegration_30SiteWco_112SiteWcoShown_*
would fail (5 tests in total) with this feature enabled, because
CreateInstallInfoForCreateShortcut did not copy over display_override.

BUG=1216457
TEST=Enable kCreateShortcutIgnoresManifest, run browser_tests
--gtest_filter=*WebAppIntegration_30SiteWco_112SiteWcoShown_*
