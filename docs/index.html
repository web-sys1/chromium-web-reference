Fix crash when manually opening DefaultStoreChanged bubble

The crash happened because of a CHECK failure. Check assumes that the
bubble was triggered automatically and thus that the default store
pref value must be different than the current account store opt in
status. That is not true if the user closed and then manually reopened
this bubble, because at this point the default store pref was already
changed.
