a11y-android: Refactor UpdateTextPositionForSelection to accept AXPosition.

Previously, `BrowserAccessibilityManagerAndroid::GetSelectionRange`
would unpack AXPosition objects into a node and an offset, and
`UpdateTextPositionForSelection` would then re-create a new AXPosition
to perform its logic (finding the highest leaf node ancestor).

This change avoids these redundant operations by passing the AXPosition
directly to `UpdateTextPositionForSelection`.
