exo: Ensure resize_component_ is HTCAPTION after drag-resize

A lot of window manager code assumes that resize_component_ is HTCAPTION
when in a non-dragging state and will branch off of this condition.

While client-initiated resizes clean up in such a way that this is true,
it turns out that this is not true in the aftermath of server-initiated
resizes, which can and often does happen with Lacros.

There are no known bugs that result from this technically inconsistent
state, so this change is more of a preventative measure.
