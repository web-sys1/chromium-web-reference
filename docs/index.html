Add regression tests for 1170038

In https://crbug.com/1170038#c3, by looking at the crash stacktrace, I
suspected it was possible for the synchronous navigation to about:blank
to be initiated by a document, different from the parent/opener. I also
suspected it could start loading at a random time, instead of
immediately after the initial empty document.

Instead of having:
- The initial empty document
- The empty document, created by the initial synchronous navigation to
  about:blank.
to be perfect twin. We would have two blank document inheriting
properties from two different documents.

That's particulary bad, because the synchronous navigation to
about:blank is the last remaining one that isn't driven by the browser
process. This could potentially block RenderDocument, SwapRfhAtCommit,
OutOfBlinkSandbox, OutOfBlinkOrigin, ...

This patch adds several WPT tests. One of them shows the synchronous
navigation can be initiated from a arbitrary same-origin document. The
other reproducest show
the crash.
