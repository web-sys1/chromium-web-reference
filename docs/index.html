Extends logging of audio output glitches

We have found in native WebRTC logs from Meet in Chrome that logs related to accumulated glitch statistics from actual output streams are sometimes missing even if RTCAudioPlayoutStats shows that glitching does happen. The root cause is most likely that the stream outlives the audio output controller (AOC) which prevents logs from being added when the AOC stops its stream.

This change ensures that the AOC also accumulates glitch information provided to it in OnMoreData callbacks. It ensures that new logs can be added in OutputController::StopStream when the ErrorStatisticsTracker is destructed.

Example log using CPU stress test loading the CPU to 100%:
