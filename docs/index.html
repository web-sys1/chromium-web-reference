gpu: win: Enable DCompPresenter by default

This change switches the default output device on Windows to
SkiaOutputDeviceDCompPresenter, which uses a gl::Presenter-based
DCompPresenter. This means that DirectCompositionSurfaceWin will no
longer be used in the default case.

Instead of wrapping the root swap chain/surface allocation in
DirectCompositionSurfaceWin as a GLSurface to be drawn as the
DirectRenderer "current frame", the root render pass backing is now
allocated in SkiaRenderer itself and rendered as a normal render pass
and scheduled as an overlay.
