[PWA]: Move web-feature use counters to not count for default entries

Start_url, manifest_id, and scope are never empty after parsing.
However, use counters should only be updated if the field has been set
by the developer and not if it was set by default.

Moving UseCounter::Count() for the above mentioned fields to their
specific parse methods. This ensures the counter is only updated if the
field has been set by developer.

Additional cleanup: capture_links has been deprecated and should be
removed from the parser.

Adding tests to verify this behavior.
