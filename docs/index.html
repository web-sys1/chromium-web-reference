Add HasPendingActivity() to MediaElementAudioSourceNode

When an HTMLMediaElement goes away (due to GC), the associated
MediaElementAudioSourceNode may still be running and continue to
access the media element.

This CL fixes this problem by making the node (and the linked media
element) stay alive as long as the context is rendering.

Bug: 1127559
Test: The repro case doesn't crash any more.
