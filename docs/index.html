Fenced frame: Use initiator frame referrer policy for beacons.

Event-level reports are sent out with a referrer header and a referrer
policy. The policy is currently always set to "origin". However, if a
frame wants to have a more restrictive referrer policy, the Fenced
Frames Ads Reporting API does not honor that, and instead will always
send out the origin.

This CL modifies the FFAR API to set the referrer policy to the policy
that is set by the frame that initiates an event-level report. This CL
also ensures that the most information that can be sent out in the
referrer header is the origin; for example, a frame with the
"unsafe-url" referrer policy will still only have beacons be populated
with the initiator frame's origin, not its full url.

Ad components will have their referrer policy set to the referrer policy
of the ad frame root. This matches the existing behavior of setting
the referrer to the origin of the ad frame root, rather than that of
the component ad frame itself.

See "Referrer Policy" section in design document:
https://docs.google.com/document/d/1gKRZ9g_X5HCZbW__GinViwcC8iWEai_kBXsiJCXrt80/edit?usp=sharing
