[Extensions] Add promise support to chrome.storage

Add promise support to the chrome.storage API. Since chrome.storage uses
custom types for the StorageAreas, this required modifying the bespoke
StorageArea C++ class in the renderer. This CL:
- Re-adds the "returns_async" schema definition in storage.json (to
  indicate that the signature supports promises)
- Implements promise handling in the StorageArea class
- Adds targeted unit tests in the renderer for the StorageArea
- Adds end-to-end browser tests exercising the flow with a "real"
  extension.

After this, MV3 extensions can use promises with the storage API, e.g.

await chrome.storage.set({foo: 'bar'});
const {foo} = await chrome.storage.get('foo');
