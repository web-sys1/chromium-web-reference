FileStreamDataPipeGetter waits for all receivers

FileStreamDataPipeGetter is self owned, and each instance waits for all
receivers to disconnect before deleting itself. Previously,
FileStreamDataPipeGetter would only wait for the first receiver to
disconnect.

FileStreamDataPipeGetter now implements the DataPipeGetter interface
correctly, allowing Read() to be called more than once.
