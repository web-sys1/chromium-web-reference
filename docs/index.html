Original change's description:
> [Nearby] Add high-level transfer and connection success/failure metrics
>
> In this CL, we record explicit success/failure metrics for
> * "transfers", which encompasses all aspects of the sharing flow after
>   device discovery;
> * "transfers" sliced by Critical User Journey (CUJ) scenarios, for
>   example, sending to a phone using contact-based sharing (see
>   go/cros-nearby-cujs-sheet);
> * establishing an outgoing connection with a receiving device, which
>   is the first step after discovery (note: this step is also captured
>   in the "transfer" metrics.)
>
> We currently have the metrics "Nearby.Share.Transfer.FinalStatus[...]"
> and "Nearby.Share.Connection.EstablishOutgoingConnectionStatus". These
> metrics log specific result enums that can be used for diagnostics
> and more effective debugging. In this CL, we add explicit
> success/failure metrics to avoid ad hoc determinations during data
> analysis of what constitutes a success or failure.
>
> Manually verified metrics via chrome://histograms.
>
