build: add cros_needs_rewrapper_path_arg gn arg

While transitioning from building Chrome for ChromeOS using ChromeOS's
toolchain to using Chrome's bundled toolchain, we need to account for
the situation where ChromeOS's toolchain requires the path to rewrapper
and its configuration to be specified using command line arguments,
whereas Chrome's bundled toolchain does not recognize these arguments
and will error if they are specified.

This CL adds a new gn arg cros_needs_rewrapper_path_arg which controls
whether we should pass the rewrapper command line arguments when
building with reclient. It defaults to true, which is the correct value
for ChromeOS's toolchain, which is currently what we use to build Chrome
for ChromeOS. When switching to Chrome's bundled toolchain,
cros_needs_rewrapper_path_arg should be set to false. Once the
transition is complete, the argument can be removed.
