BrowsingDataModel: Initial structure + AddBrowsingData function

CL lands the initial structure of the new BrowsingDataModel[1],
which will serve as the replacement for the CookiesTreeModel.

This CL defines the interface for accessing BrowsingDataEntries, and
implements the AddBrowsingData function, which allows for direct
insertion of information into the model. This function will be used
both by consumers using the model to represent specific accessed data
(e.g. Page Info) and by backend fetch methods to load data into the
model.

Some methods required to make sense of comments are introduced as
stubs.

[1] go/simple-browsing-data
