[Read Later] Android should fetch larger favicons

When fetching favicons from Google, Chrome on Android always requested
24dip. The 'dip' in the name of kDipForServerRequests created a false
sense that the requested size would scale with the display density.

Contents of ResourceBundle::InitSharedInstance suggest that the dip->px
scaling factor on Android is always 1x and has nothing to do with the
unit 'dp' used in Android.

Feature LargeFaviconFromGoogle now makes it possible to set the
requested icon size to 128px, which is equivalent to a 32dp drawable on
a xxxhdpi display. In future the Android UI code could set the desired
favicon size at runtime.

Also, adding a DCHECK verifying that requests sent to the favicon
service are supported size values.
