Use persistent hash for policy-to-pref test chunking

All the policies were split into even-sized chunks based on their hash. However, this hash was using FastHash() instead of PersistentHash(), which creates unstable hash values. 
That could cause a policy to end up in chunk X, if that chunk fails, it gets rerun. However, due to the unstable hash, that problematic policy might now be in chunk Y, so chunk X succeeds and CQ let's the CL pass.

This CL fixes that by using persistent hashes, which are stable.
