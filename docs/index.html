UpdateViewportContainerSize() on every commit in pending tree

During commit, inner and outer viewport_container_bounds_delta_ is
copied to 0 from propertytrees of main via the operator= of the property trees. The value of main is always 0 because main does not compute the value. Therefore, the bounds_delta_ need to be calculated in pending later by UpdateViewportContainerSize().

When user scroll, the top_controls_delta is sent to the main and the main use the delta and push the shown ratio of top control to the pending. If the delta + main's topcontrol is 0, the main push 0 to the
pending as shown ratio. PushMainToPending() return false if the active already set the ratio to 0.

In this case, UpdateViewportContainerSize() is not called, so the
pending will have invalid bounds delta of top control. This patch
fix this exception case.
