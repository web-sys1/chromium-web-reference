[Test Scripts] Save raw logs for JUnit local machine runs.

Previously JUnit running on local machine uses

  -Drobolectric.logging=stdout

(if -v) to control the presence of logcat at Robolectric level.

We'd like to save logcat output in TEST_RESULTS_* directories:
* This should happen regardless of whether -v is specified.
* "Non-verbose" (no -v) console output should continue to omit logcat
  lines.

This CL implements the feature via the following:
* Make Robolectric emit logcat output always.
* Make test script save these to

  TEST_RESULTS_*/raw_logs/raw_logs.txt

  for local machine runs.
* If non-verbose, heuristically filter (detect and omit) logcat lines
  from console output.

Details:
* TestRun.RunTests(): Add optional |raw_logs_fh| param.
  * Update LocalDeviceTestRun.RunTests() (param unused).
* LocalMachineJunitTestRun:
  * RunTests(): Take serialized lines generated from
    _RunCommandsAndSerializeOutput(), output to |raw_logs_fh| if
    specified, then omit logcat lines if non-verbose.
    * Using RegExp to heuristically detect logcat lines.
    * Log (in Main) the number of omitted lines, if > 0.
* RunTestsInPlatformMode() (caller):
  * For local machine runs: Create io.StringIO() and pass it as
    |raw_logs_fh| to capture logs, then write to raw_logs.txt on
    completion.
  * Doing the above instead of writing to file directly (which might be
    useful by leaving logs on early termination) for simplicity:
    * As before, TEST_RESULTS_* is only created at the end.
    * Current infra uses contextmanager heavily, and adapting this to
      conditionally (on local machine runs) create raw_logs.txt would be
      unwieldy.
