Deferred Shaping: Handle deferred IFCs like content-visibility:auto

Calls DisplayLockContext::SetRequestedState(kAuto) for deferred IFCs,
and we re-shape for unlocked deferred IFCs.

- SetRequestedState(kAuto) is called as a post-layout task because it
  must not be called during layout.
- If a non 'visible' value is specified as content-visibility to a
  deferred element, we cancel the "deferred" state.
- Reject SetRequestedState(kVisible) while the element's box is
  deferred.
- Skip painting contents of deferred IFCs.

This CL has no behavior changes by default, and Deferred Shaping
feature becomes workable with --enable-blink-features=DeferredShaping.
