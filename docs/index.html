[StyleBuilder] Don't mutate ComputedStyle in SelectorChecker

We currently set some flags on ComputedStyle during matching of the
:-webkit-drag, :focus-within, :hover and :active pseudo-classes.
This is not ideal, since selector-matching on its own is supposed to
be a read-only process.

This CL instead propagates "MatchFlags" all the way up to MatchResult,
which allows the calling code to either handle or ignore the flags.

We seem to be over-marking in some cases (e.g. :has()). This CL aims
to preserve the existing behavior, including over-marking cases.
