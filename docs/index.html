[Android] Add test of recording pre-foreground first nav commit

This CL adds a test of the recently-added recording of the case where
the first navigation commit on startup occurs before
UmaUtils.hasComeToForeground() becomes true.

This CL would ideally also test the recording of the case where
StartupPaintPreview's first paint on startup occurs before
UmaUtils.hasComeToForeground() becomes true, but that turns out to be
challenging to test: it is nontrivial to set things up such that
StartupPaintPreview performs a preview. We leave that as future work :).

This CL is a reland. The first CL incorrectly added an unconditional
assert to assertHistogramsRecorded() that the first nav commit was not
recorded as having occurred pre-foregrounding. This assertion should be
made only if assertHistogramsRecorded() is asserting that the actual
startup metrics were recorded. The fix is between PS1 and PS2 here.
