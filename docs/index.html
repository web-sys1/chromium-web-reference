iwa: Introduce `IsolatedWebAppTrustChecker`

This class is responsible for answering whether the browser trusts the
public keys of the signatures of an Isolated Web App to run it.
We only want to run Isolated Web Apps that are trusted, given the
additional capabilities available to Isolated Web Apps. Trust for an
Isolated Web App can be established in multiple ways:
- An enterprise policy can configure trust. Currently, only the
  IsolatedWebAppInstallForceList policy, which implicitly configures
  trust for all Isolated Web Apps that it is trying to force-install,
  does so.
- If Isolated Web App developer mode is enabled, all apps are trusted.
- [Later]: If an Isolated Web App is signed by a trusted distributor, we
  trust it as well.
- [Later]: If an Isolated Web App is on an allow-list of trusted apps,
  we trust it as well.

The `IsolatedWebAppTrustChecker` is used to check trust as part of the
Isolated Web App parsing and verification process:
1. The Signed Web Bundle's Integrity Block is parsed and the public
   key(s) contained in it that were used to sign it are extracted.
2. The `IsolatedWebAppTrustChecker` is used to check that we trust the
   public key(s).
3. If we trust them, we verify that the file contents match the
   signatures corresponding to these public keys.
4. If the signatures match, we then continue parsing the Signed Web
   Bundle's metadata, etc.
