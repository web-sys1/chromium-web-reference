Rename PublicSets to GlobalFirstPartySets.

The name "PublicSets" is misleading, since the instances actually hold
the command-line-supplied set as well as the public First-Party Sets
data. In reality, the important aspect of this class is that it holds
all of browser-level sets data, and allows querying of it. So, a better
name would be something like GlobalSets or something like that.

This CL changes the name from PublicSets to GlobalFirstPartySets and
renames some variables/fields. It does not rename methods. No behavior
is changing in this CL.
