Centralize canvas recording initialization

Before this CL, the recording initialization and the canvas clip stack
restoration was performed "manually", by each and every functions
triggering a flush or a reset of the canvas. This manual maintenance
was error-prone because it was distributing the recording state
maintenance across the code base. New callers of
`recorder_->finishRecordingAsPicture()` for instance would have to
know to restore the recording and do it correctly.

With this change, the `MemoryManagedPaintRecorder`, which owns the
recording, takes care of keeping that recording initialized and valid.
