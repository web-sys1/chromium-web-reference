Always attempt video decoder init if we don't have fallback.

For codecs such as H264, H265 we may not have software fallback
depending on the platform. As such even if OS metadata suggests
the playback will fail, try it anyways because it certainly will
otherwise.

This is useful since the OS metadata doesn't always appear accurate
on Android and may be too expensive to compute accurately on other
platforms (e.g., requires testing resolutions one at a time).
