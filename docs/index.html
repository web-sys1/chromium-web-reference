Collect and shard tests in wrapper layer of run_wpt_tests.py

This uses the same way to collect and shard tests as in RWT. We then
dynamically build subsuite config file, pass the config file and the
tests to Wptrunner. Wptrunner will not do any sharding and
always run all the tests passed in.

The benefits are:
1) This automatically get all the features for test collection
from RWT, e.g. --smoke, command line auto expansion.
2) This lifts the need to add additional functionalities for subsuite
support in Wptrunner, including run individual test from a subsuite,
sharding for subsuite tests.
