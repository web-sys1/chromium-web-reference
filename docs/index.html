T&S Sentiment: Create service + service factory

CL creates a stub TrustSafetySentimentService which will be used to
collate signals of feature usage and determine if and when to request
HaTS surveys.

The service need only exist when the user is eligible to receive a HaTS
survey. To quickly exclude users who definitely cannot receive any HaTS
survey, a function CanReceiveAnySurvey is added to the existing HaTS
service.

This new function extracts the existing logic from the survey specific
CanShowSurvey, which is actually applicable to all surveys.

The bulk of this CL comprises the the new service factory using this
logic to selectively create the TrustSafetySentimentService.
