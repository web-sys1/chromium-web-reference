Update AV1 encoder wrapper to use an active map

This CL updates the AV1 encoder wrapper to use an active map and a
member bool to quickly enable or disable it for testing. I don't
want to enable it yet and a number of other benchmarks were taken
w/o an active map and I'd like to keep that data comparable for the
time being. I'm planning on doing some testing on a few different
machine configs to see if using an active map is beneficial.
