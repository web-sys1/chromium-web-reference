Workaround GPUExternalTexture color space issue in 1-copy path

Current 1-copy path of GPUExternalTexture uses intermediate cache to
do conversion. It set color space parameter to null when getting canvas
resource from cache. This means the canvas resource provider will be set
to a default color space (for now, it is SRGB). But GPUExternalTexture
internal shader has its own color space conversion logic. But due to issue crbug.com/1407112, chromium failed to execute color space conversion when paint video frame.
This CL tries to workaround crbug.com/1407112 by setting the canvas
resource provider color space to the video paint cache so that no
color space conversion happens during paint video frame. And then
pass video frame source color space and user dst color space to Dawn
to do the conversion

Bug:chromium:1396078, chromium:1395227
