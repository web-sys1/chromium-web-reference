[scroll-animations] Diff Scroll/ViewTimelineName by value

When scroll/view-timeline-name was updated to use a ScopedCSSNameList
(wrapped in Persistent), this diff function wasn't updated, which
resulted in comparisons by pointer. This causes expensive subtree
recalc propagation when calculating a new style for an element
with *-timeline-name, even when the name doesn't change.

Fixed by using base::ValuesEquivalent.

Note that there are no unit tests for the case where we *do* need
a diff, because those cases are covered by WPTs already.
