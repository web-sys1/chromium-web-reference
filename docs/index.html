> Reason for revert: Suspect for breaking blink_wpt_tests
> https://ci.chromium.org/ui/p/chromium/builders/ci/WebKit%20Win10/102100/overview
>
> Original change's description:
> > webauthn: be more careful reporting transports.
> >
> > Prior to this change, Chrome would report the union of the transport
> > used to communicate with an authenticator, and any transports learned
> > from its attestation certificate, in a navigator.credentials.create()
> > response.
> >
> > However, if we don't get transports from the attestation certificate
> > then the used transport might be too narrow. E.g. a USB+NFC
> > authenticator used over USB would be reported as ['usb'], and wouldn't
> > work over NFC in the future.
> >
> > This change only reports transports if we learned of them from either
> > the attestation certificate or from getInfo. We didn't record the latter
> > previously so this change also adds support for collecting transports
> > from getInfo.
