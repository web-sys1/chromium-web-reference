[editing] Add IsValidFor DCHECK in VisiblePosition::Create()

A DCHECK in MostBackwardOrForwardCaretPosition asserts that the position
is valid. But it has been failing multiple times lately, and probably it
will fail again in the future, so this patch makes it print the bad
position when the assert fails.

Also, usually this assert fails when some code creates a position, then
it alters the DOM, potentially invalidating the position, and then it
normalizes it with CreateVisiblePosition(). It's the caller of that
CreateVisiblePosition what we are interested in, the following functions
in the stack trace aren't much relevant. Therefore I'm adding the same
DCHECK into VisiblePosition::Create, so that it fails earlier.
