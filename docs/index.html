Organize and broaden expectations for Pixel_SVGHuge

This test is always somewhat slow, there's no harm in marking it Slow on
all platforms. Some platforms are too slow so they're skipped.

Other than the reorganization, the net changes here are intended to be:
- Upgrade a few old Android configs from Slow to Skip
- Mark all non-skipped configs as Slow

Based on results here:
- https://ci.chromium.org/ui/test/chromium/%3A%2F%2Fchrome%2Ftest%5C%3Atelemetry_gpu_integration_test!flat%3A%3A%23gpu_tests.pixel_integration_test.PixelIntegrationTest.Pixel_SVGHuge?q=V%3Abucket%3Dci+
- https://ci.chromium.org/ui/test/chromium/%3A%2F%2Fchrome%2Ftest%5C%3Atelemetry_gpu_integration_test_android_chrome!flat%3A%3A%23gpu_tests.pixel_integration_test.PixelIntegrationTest.Pixel_SVGHuge?q=V%3Abucket%3Dci+

... in which the flakes were mostly on configs with no Slow expectation,
except for the few Android configs that were already Slow.
