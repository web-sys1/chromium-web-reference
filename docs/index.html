[Fuchsia] Fix FuchsiaAudioRenderer to call Stop() only after Start()

AudioConsumer expects Stop() to be called after Start().
FuchsiaAudioRenderer uses |state_| field to track the current state.
Previously it was also used to track end-of-stream state, but
end-of-stream packet may be processed without the stream being started.
FuchsiaAudioRenderer::StartTicking() was assuming that state_!=kStopped
indicates that the stream is started, but that wasn't always the case.
Added a new boolean field to track end-of-stream, separately from
started/stopped state, which allows to ensure that Stop() is never sent
before Start().
