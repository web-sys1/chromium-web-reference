Make output of compare_build_artifacts.py more accessible

This CL makes the output of compare_build_artifacts.py more accessible
to folks who are not deeply familiar with the output format and only
interact with the files occasionally (e.g. during sheriffing):

This output:

```
Unexpected files with no diffs:

  nacl_test_data/extension_vcache_test/glibc/extension_validation_cache_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/pm_exit_status_test_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/simple_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/sysconf_nprocessors_onln_test_libs/lib64/libppapi_cpp_lib.so
  ppapi_nacl_tests_libs/lib64/glibc_x64/libppapi_cpp_lib.so
  ppapi_nacl_tests_pnacl_newlib_x64.nexe
  test_data/ppapi/tests/extensions/packaged_app/pnacl/ppapi_tests_extensions_packaged_app_newlib_pnacl.pexe
fixed_build_dir /b/s/w/ir/cache/builder/src/out/Debug exists. will try to use orig dir.
Checking nacl_irt_x86_64.nexe difference: (2058 deps)
  irt_x64/obj/native_client/src/untrusted/irt/irt_core_lib/irt_private_pthread.o                                          : different size: 41592 != 41512
```

becomes:

```
Unexpected files with no diffs:
  nacl_test_data/extension_vcache_test/glibc/extension_validation_cache_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/pm_exit_status_test_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/simple_libs/lib64/libppapi_cpp_lib.so
  nacl_test_data/glibc/sysconf_nprocessors_onln_test_libs/lib64/libppapi_cpp_lib.so
  ppapi_nacl_tests_libs/lib64/glibc_x64/libppapi_cpp_lib.so
  ppapi_nacl_tests_pnacl_newlib_x64.nexe
  test_data/ppapi/tests/extensions/packaged_app/pnacl/ppapi_tests_extensions_packaged_app_newlib_pnacl.pexe
fixed_build_dir /b/s/w/ir/cache/builder/src/out/Debug exists. will try to use orig dir.

Differences split by build targets:
Checking nacl_irt_x86_64.nexe difference: (2058 deps)
  irt_x64/obj/native_client/src/untrusted/irt/irt_core_lib/irt_private_pthread.o                                          : different size: 41592 != 41512
```

- The CL removes the newline after "Unexpected files with no diffs:"
  and "Unexpected files with diffs:" to indicate that the following
  lines belong to this section.
- The two big sections - an analysis on the file level and an
  analysis on the build target level are now separated by a newline
  and have section titles:
  - "Differences of files in build directories:"
  - "Differences split by build targets:"
