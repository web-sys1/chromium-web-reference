Pass relaunch window as an argument to AdjustDeadline method

Currently we fetch the relaunch window policy value in
UpgradeDetectorImpl::DoCalculateThresholds() twice - once inside the
method to find out if policy is enabled and next in AdjustDeadline() to
adjust the relaunch deadline fall within the window.
This CL passes the relaunch window as an argument to AdjustDeadline()
so that we can avoid reading policy value twice.
