Android: Disallow android_resources() targets from having manifests

It used to be that adding a manifest to an android_resources() target
did two things:
1) Made it participate in manifest merging
2) Provided the package name for R.java files.

Manifest merging is now done via mergeable_android_manifests and R.java
package names are set via "resources_package".

This removes an unnecessary build dep for write_build_config.py steps.
