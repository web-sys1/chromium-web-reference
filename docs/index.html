> [Media] Make VideoColorSpace a wrapper of gfx::ColorSpace.
>
> This is part is the first step to eventually deprecate VideoColorSpace
> in favor of gfx::ColorSpace. With this change, aside from UNSPECIFIED
> handling and guessing, VideoColorSpace holds minimal functionality.
>
> Note that this makes VideoColorSpace immutable. Callers which did
> in-place modification of the internals (primary, transfer, matrix, or
> range) would need to instead create a new VideoColorSpace object.
>

> so there shouldn't be any user-facing cases where we'd see issues
> due to this new limitation.
>
> An additional change introduced with this is removal of codec-specific
> tests that make use of BT2020 CL, as it is not supported in
> gfx::ColorSpace.
