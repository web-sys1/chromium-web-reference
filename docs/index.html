value_store: refactor tests & ValueStoreChange type

This change:

1. Makes ValueStoreChange a struct instead of a class, replaces its
   methods with direct member accesses, and removes or simplifies most
   of its tests
2. Rewrites the ValueStore test suite into a larger number of single-
   purpose tests, with simpler setup code, and removes some of the
   helpers that were previously needed
3. Replaces most of the custom equality checking code in the ValueStore
   tests with use of operator==, which works on base::Value and
   absl::optional<base::Value> with no additional work
4. Removes the "complex changed keys scenarios" test, which (on closer
   examination) duplicates many of the existing tests.

This significantly reduces the amount of deprecated base::Value API
surface that this component depends on, although there is more work
yet to do - especially migrating the public APIs away from
base::DictionaryValue.
