Support bounds changes for floated ARC++ windows

This CL adds the proper support for moving floated client-controlled
windows.
In short, this makes all bounds change result in sending
SetBoundsWMEvent instead of calling Window::SetBoundsDirect() which
directly changes the bounds without waiting for the ack from clients
(e.g. ARC++).
Otherwise, the window bounds are updated only in Chrome-side whereas
