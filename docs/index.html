Set require_persistent_mode in ModelBrokerAndroid.

By default, require_persistent_mode is set to true since enabling a
feature on non-persistent mode has a higher bar, which requires explicit
approval from the Android System Health Council.

One exception is scam detection. We are waiting for more data from Pixel
to decide if the feature should be restricted to persistent mode. Before
that, we are setting require_persistent_mode to false, but leaving a
feature flag so we can enable it without waiting for a new Chrome
release.
