weblayer: don't crash if onNativeLoaded called multiple times

If loadAsync() is called and loadSync() is called berfore loading
completes, then WebLayerImpl.onNativeLoaded() is called twice. This
breaks assumptions of the code onNativeLoaded() calls, leading to
crashes.

This patch fixes this by adding a member so that only the first
call does something.

This is non-trivial to test, but this is blocking experiments, so
but I will try to follow up with a test.
