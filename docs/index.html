Change idle action to "Do nothing", if idle delay is zero

PowerManagementIdleSettings policy allows setting zero to the idle delay
to disable the idle action, but ChromeOS does not support zero value for
the idle delay. If the idle delay was set to zero, the system
immediately triggers the idle action, which blocks the user from doing
anything.

To simulate disabling the idle action when the idle delay was set to
zero, |PowerManagementIdleSettingsPolicyHandler::ApplyPolicySettings|
function was updated to change the idle action to “Do nothing”, if the
idle delay was set to zero.

Add unit tests for |PowerManagementIdleSettingsPolicyHandler|.
