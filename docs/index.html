Add a resource recycle mechanism for WebGpu import texture api

This import texture API doesn't work efficiently when there are many
videos in one frame. Constantly creating and destroying shared images
hurts performance.

The basic idea is when there is a request to create a WebGPU Image
resource provider, we search the unused cache. If there is a cache key
match, reuse the provider, otherwise create a new one.
When the provider is done and is not used, we move it to the unused
cache if the cache is not full, otherwise we just destroy it.

There is no change to the canvas resource and this CL does not utilize
the canvas resource recycling function.
