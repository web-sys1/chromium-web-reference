[crd host][linux] Unset CRD environment variables

This CL makes the service script unset problematic systemd environment
variables in cleanup. This allows single-process->multi-process
migration to work smoothly. One strange problem I've seen is that if we
unset these variables immediately in cleanup(), something will add them
back. Gemini suggests that it is GNOME or one of GNOME's processes doing
this. Simply adding a one-sec delay seems to pretty much eliminate the
issue.

Also remove the code to restart PipeWire from the dev script. It appears
that PipeWire doesn't need to be restarted as long as the environment
variables are properly cleaned up.
