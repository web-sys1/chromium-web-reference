[MBI] Migrate CanvasResourceDispatcher to use per-ASG CompositorTaskRunner

Before this CL, CanvasResourceDispatcher uses per-thread
CompositorTaskRunner.

After this CL, CanvasResourceDispatcher will use per-ASG (Agent
Scheduling Group) CompositorTaskRunner.

CanvasResourceDispatcher is used by OffscreenCanvas. OffscreenCanvas
provides a functionality to draw the Canvas from Worker
thread. Previously, CanvasResourceDispatcher was accessing the main
threadâ€™s objects from the worker thread by using
`Thread::MainThread()->Scheduler()->CompositorTaskRunner()`. But this
time, we want to use the per-AgentSchedulingGroup compositor task
runner instead. To plumb per-ASG compositor task runner from main
thread to worker thread, this CL added
agent_group_scheduler_compositor_task_runner to WorkerGlobalScope.
