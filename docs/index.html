Files TS: Re-organize the cr.ui decorate

Rename the function decorate() to crInjectTypeAndInit() to be a bit
more explicit.

Remove the static method decorate() of cr.ui elements except for those
that require more arguments to decorate, aka FileGrid and FileTable.

Rename the instance method decorate() to initialize() to be more
descriptive.

Remove legacyDecorate(), all usage has been superseded by
crInjectTypeAndInit()

Add the abstract class `DecorableElement` act as a interface for all
cr.ui elements, except for FileGrid and FileTable.

NOTE: Tree, TreeItem, DirectoryTree and all other TreeItems still use
the old decorate(), but they will be removed in the near future.
