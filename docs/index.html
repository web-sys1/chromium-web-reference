[OnionSoup] Remove content::ResourceDispatcher

ResourceDispatcher was once used to serve as a communication interface
to the ResourceDispatcherHost in the browser process to start a request
from the render process. Along with Mojofication & Onion Soup work,
the ResourceDispatcher doesn't serve any host any longer and it becomes
feasible to remove it from content/ eventually.

As many classes under content/renderer/ depend on the ResourceDispatcher,
this patch makes the following changes in order to remove it:

- Create a class WebResourceRequestSender under
  third_party/blink/public/platform/ which is a thin layer used to send
  the async/sync resource requests in blink. Most of its functions are
  migrated from the original ResourceDispatcher, but it will not
  maintain the combination of a request_id and a mapped request, because
  each WebURLLoaderImpl::Context owns a WebResourceRequestSender.
  (Previously, ResourceDispatcher was owned by sort of fetch context.)

- Add the blink platform unit tests for the WebResourceRequestSender
  migrated from the unit test of ResourceDispatcher.

- Because SyncLoadContext binds with WebResourceRequestSender tightly
  for sending a sync request, this patch moves it to
  third_party/blink/renderer/platform/loader/fetch/url_loader/ together
  with WebResourceRequestSender.

- The interface ResourceDispatcherDelegate is renamed to
  WebResourceRequestSenderDelegate and moved into blink and gets
  implemented in content.

- Remove the ResourceDispatcher from the codebase and the classes that
  depend on it.

- Remove the TestRequestPeer used by the ResourceDispatcher unit test.

- Remove the wrapper class RequestPeerImpl.

So, before this CL, the situation looks like:

                        +--------------+
                        |WebRequestPeer<-------------+
                        +----^---------+             |
                             | <derives>             | <derives>
                             |                       |
                       +-----+---------+      +---------------+
                       |SyncLoadContext|      |RequestPeerImpl|
                       +--------+------+      +-----^---------+
                                |                   | <weak>
+--------------------------+    |                   |
|ResourceDispatcherDelegate|    |               +---+-----+
+-----------^--------------+    |               | Context +---+
            |                   |               +-----+---+   |
            |                   |                     |       |
            |               +---v---------------+     |       | <weak>
            +---------------+ ResourceDispatcher<-----+       |
                            +------+------------+    +--------v--------+
                                   |                 | WebURLLoaderImpl|
                                   | <derives>       +-----------------+
                      +------------v-----------------+
                      |WebMojoURLLoaderClientObserver|
                      +------------------------------+

and after this CL, the situation looks like:

          +--------------+
          |WebRequestPeer|            +----------------+
          +--^---------^-+            |WebURLLoaderImpl|
             |         |              +--------^-------+
   <derives> |         | <derives>             |
+------------+--+  +---+-----+                 |
|SyncLoadContext|  | Context +-----------------+
+-----+---------+  +--+------+     <weak>
      |               |
      |               | <owns>
      |               |
  +---v---------------v----+           +------------------------------+
  |WebResourceRequestSender+----------->WebMojoURLLoaderClientObserver|
  +-------------+----------+ <derives> +------------------------------+
                |
           +----v---------------------------+
           |WebResourceRequestSenderDelegate|
           +--------------------------------+
