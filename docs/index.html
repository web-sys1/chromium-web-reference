Don't load components from unencrypted stateful partition

This was used to support loading preinstalled "offline" demo mode
resources for Chrome OS devices in demo mode, and thus support demo mode
setup without requiring connectivity (or downloading full demo mode
resources component). This behaviour is no longer supported, so loading
preinstalled cros components from a Chrome OS specific path is no longer
needed.
