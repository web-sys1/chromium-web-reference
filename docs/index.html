Fix syntax error and bulletproof cert prov tests

Fix a syntax error in the certificateProvider API tests, and make
similar errors fail the test in the future.

Apparently, by default errors occurring in API event handlers or
callbacks are ignored (only logged). To transform them into actual test
failures, we're using a combination of chrome.test.setExceptionHandler()
with window.addEventListener('error').
