[SVG] Add Chromium-internal cross-API invariant tests for coordinate parity

Add web tests that pin the expected relationships between SVG coordinate
APIs (`getBBox`, `getCTM`, `getScreenCTM`) and the DOM geometry API
(`getBoundingClientRect`) at default zoom. These establish the baseline
parity contract that must hold during the SVG internal-coordinate
migration tracked in crbug.com/469958423.

The tests are intentionally Chromium-internal regression guards rather
than WPTs: assertions encode Blink-specific tolerances and use
deterministic expected matrices that other engines may not yet match.
Each file carries a `TODO(vinaysingh): Move to WPTs once the migration
stabilizes` header comment.

Test plan:
https://docs.google.com/document/d/1t40xYtLkCV_ElOxmY4Tx1GbO8i7sIROH5bZKhsgftB4/edit
