system-web-apps: fix flaky link capture test on linux and lacros

This CL reduce the usage of RunUntilIdle and FindLastActive in SWA link
capture tests.

We suspect these methods are causing flakiness on linux and lacros
builds. They are replaced by WaitForBrowserToClose and
BrowserActivationWaiter.

Note, we need to assume window activations are processed in FIFO order.
In other words, `window1->Show(); window2->Show();` triggers window
activations in the order of Show() method being called. This is a
reasonable assumption.

Also move system_web_app_ui_browsertest to run in interactive_ui_test,
which runs test in serial (browsertests may run in parallel). Since
window activation is global (across all running instances), this helps
ensuring consistent window activations.
