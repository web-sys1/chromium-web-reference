[parkable_image] Add ParkableImageSegmentReader

Currently, the SegmentReader created by DeferredImageDecoder can be
quite long-lived, even though the data inside it is not needed for most
of its lifetime.

However, since the SegmentReader currently needs direct access to the
data inside ParkableImage, this would prevent said ParkableImage from
being written to disk for the entire lifetime of the SegmentReader.

This CL adds a SegmentReader that wraps a ParkableImage (instead of
wrapping the internal data of the ParkableImage), fixing the above
issue.

This is hidden behind a feature flag which is disabled by default, so it
should not cause any change in functionality.
