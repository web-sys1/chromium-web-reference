Don't do form restoration if element *or* form has autocomplete=off

Currently, the form restoration behavior is inconsistent. Input
elements check whether the element itself has `autocomplete=off`,
and won't restore the input contents if so. On the other hand,
`<textarea>` and `<select>` elements only check their associated
`<form>` for whether *it* has `autocomplete=off`.

This CL normalizes these behaviors, so that if either the form or the
element itself has `autocomplete=off`, then the element is not
restored.
