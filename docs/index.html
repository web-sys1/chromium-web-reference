Record MojoResult when writing synthetic response fallback body.

This change adds a UMA histogram to record the MojoResult returned by
MojoDataPipeProducerHandle::WriteData() when writing the fallback body
for synthetic responses. This helps monitor the outcomes of this
operation, especially since a non-OK result currently triggers a CHECK.

To support this in a type-safe way, this change also introduces a new
C++ `enum class` wrapper, `MojoResultForMetrics` in a new header
result_for_metrics.h.
