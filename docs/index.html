Restore early return in Android encoder QueueInput for kTryAgainLater

This restores the early return that was present in
AndroidVideoEncodeAccelerator::QueueInput whenever the Android
MediaCodec API returns kTryAgainLater, prior to the refactor in
71fa704be1067e339d814b3a43375939a1814305.

Without this change, hardware encoding fails immediately whenever the
hardware encoder is slow to initialize (e.g. within ~100ms). Slowing
down the browser process artificially results in hardware encoding
working consistently, while spamming calls results in intermittent
successes.

With this change, the hardware encoder's slow initialization is
handled properly by skipping frame processing. Subsequent encoding
attempts once the hardware encoder has finished initialization
results in successful hardware encoding.

TEST=Manual
1. Open Chromium on a device with an H.264 hardware encoder which
   takes some time to initialize (e.g. ~100 ms).
2. Open a web page which uses the MediaRecorder web API to encode a
   stream using H.264 hardware encoding.
