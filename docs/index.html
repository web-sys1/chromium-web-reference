WebNN: Fix UBSan failure for XNNPACK reshape to scalar test case

When XNNPACK backend reshapes to 0-D scalar where the new shape is
empty, there is an UndefinedBehaviorSanitizer (UBSan) failure, for
example [1]. The previous implementation uses a `WTF::Vector` to
represent the new shape. `WTF::Vector` doesnâ€™t allocate a buffer for an
empty vector. That causes `new_shape.data()` to return nullptr which is
passed to XNNPACK and used as the source address for `memcpy`.

This CL fixes this issue by using a stack-allocated array to represent
the new shape instead. Its address would be valid even when the array is
empty.

[1]: https://ci.chromium.org/ui/p/chromium/builders/ci/linux-ubsan-fyi-rel/1272/test-results?sortby=&groupby=&q=xnnpack
