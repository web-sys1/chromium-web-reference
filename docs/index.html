FaceGaze: Update mouse position and perform actions

This change makes FaceGaze update the mouse location and perform
mouse click actions based on results from the FaceLandmarker API. A
few changes of note:

** Logic changes **
1. We spin up the FaceLandmarker API in the camera stream context
because the camera stream cannot be JSON-ified and sent to
the background page. This actually is a simpler approach because the
FaceLandmarker accepts an HTML video element as input, which can
easily be accessed from the camera stream context.

2. The x coordinates we receive from the FaceLandmarker need to be
reflected across the device's horizontal bounds. This is because the
webcam doesn't act like a mirror and instead renders the user's
forehead in the opposite direction that the user moved.

3. Implements two mouse click actions: left click and right click.
More facial gestures and actions can be easily added in later CLs
and we ensure that the mapping and confidence thresholds for each
gesture can be customized.

** Build changes **
1. We add vision.d.ts to the mediapipe tar file so that we can use
it when building for typescript.

2. In order to use both checked-in and generated/unpacked files for
the TS build, we first need to copy all assets into a preprocess
directory. Otherwise the typescript build won't be able to find the
generated files because it's rooted in the accessibility common source
directory. Once all files are copied to the same directory structure,
we can execute the typescript build as normal.
