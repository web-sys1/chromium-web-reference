[iOS][ForcedSignIn]üè¢ No command to show advanced sync settings

The sync screen may be presented by ForcedSigninCoordinator outside
of the FRE. Since ForcedSigninCoordinator already occupies the
signinCoordinator slot in SceneController, the application command
-[showAdvancedSigninSettingsFromViewController] cannot be used to
present the advanced settings which also requires the signinCoordinator
slot.

A fix is to have ForcedSigninCoordinator using and owning its own
advanced settings coordinator.

The advanced settings in ForcedSigninCoordinator doesn't require any of
the special handling that is done by
-[showAdvancedSigninSettingsFromViewController], hence there is no need
to reuse the command.

Using the command for ForcedSigninCoordinator results in a weird
freeze crash that is reported in the bug report associated with
this CL.

--
Tested

Since interruptions dismissing the forced sign-in screens aren't
expected (but still need to be handled), we hacked the intent code to
dismiss the forced sign-in screens when handling intents. Normally
the intents are delayed until forced sign-in is done (all screens
dismissed).

The only interruption we expect is the no dismiss interruption that is
triggered when tearing down the scene while screens are being
presented.
