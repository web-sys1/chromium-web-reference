Avoid dangling pointers in CreditCardFidoAuthenticator.

This CL addresses a dangling pointer that can occur outside of tests as
follows:
- CreditCardAccessManager owns a CreditCard and a
  CreditCardFidoAuthenticator.
- CreditCardFidoAuthenticator stores a raw pointer to the CreditCard
  when CreditCardFidoAuthenticator::Authenticate is called.
- CreditCardFidoAuthenticator owns a FullCardRequest, which, on
  completion or failure, calls CreditCardFidoAuthenticator::OnFido*.
  This, in turn, calls
  CreditCardAccessManager::OnFIDOAuthenticationComplete, which calls
  CreditCardAccessManager::Reset that deletes the CreditCard, to which
  the CreditCardFidoAuthenticator still has a raw pointer.

This CL resolves this by keeping a copy of the CreditCard instead of
maintaining a pointer.
