Activate ARCVM on policy change remote install

ARC on demand delays activating ARCVM so that ARC may not be activated
when remote installs are triggered.

Currently ARC on demand will only be enabled when the user is a managed account and there are no apps installed on the device (see arc_activation_necessity_checker.cc). This CL adds handling for remote force install on admin policy change.
