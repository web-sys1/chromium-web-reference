Revert "Update last interacted form when JS modifies a field's value"

This reverts commit 05c6be3dfec78f091d537673fbf65605f7a41776.

Reason for revert: Speedometer2 regressions. I will reupload another CL
that tries to achieve the same result with possibly no regressions

Original change's description:
> Update last interacted form when JS modifies a field's value
>
> Autofill is currently working on replacing the observer it uses for
> disappearing elements by a direct blink connection notifying of removals
> and hiding of elements.
>
> Some notification about hidden elements are provided to Autofill during
> style recalc (e.g. changing visibility attribute to 'hidden') and during
> this Autofill is not allowed to query the focusability of a node.
>
> This causes a problem for Autofill: When submission is inferred,
> Autofill tries to extract the form of the element it was tracking, and
> part of this is querying the focusability of fields. Which means that
> if submission is inferred from a notification sent during style recalc,
> Autofill is not allowed to extract a form at submission time.
>
> The alternative is to use the provisionally saved form, which is a form
> that Autofill updates on autofill operations and user edits. In this
> context, form means either an HTML <form> tagged section or the
> collection of all unowned control elements.
>
> This CL tries to make the provisionally saved form approach the state of
> the extracted form at submission time (should we have extracted it)
> by also updating it on JS changes. To achieve that, we modify the signal
> JavaScriptModifiedAutofilledValue to JavaScriptModifiedValue.
>
> Note that this update is only performed if JS modifies the form that is
> already tracked by autofill. Which means that only user edits and
> autofill (also user triggered) have permission to change the form
> tracked by autofill from one form to another, but JS only has permission
> to update the currently tracked form.
>
> The CL also adds logic to default to the provisionally save form
> unconditionally at submission time instead of extraction.
>
> All behavior changes are gated behind a feature flag.
>
