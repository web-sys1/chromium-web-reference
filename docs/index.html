[//gpu] Fix WebGPUMailboxTest tests' SI usage with fallback adapter

When the fallback adapter is used, the SharedImage created for these
tests will be read to and written from via Skia. Thus, RASTER_READ |
RASTER_WRITE is needed. The lack of this specification concretely
surfaces as a test failure on Linux, where OzoneImageBacking can be
used if there is only WebGPU usage but it then rightly complains when
there is an unexpected write by Skia.

This CL just fixes the two tests most recently added; there are many
other tests that will need to be similarly fixed in followups.
