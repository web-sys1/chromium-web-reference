Storage service - make path conversion stricter in FilesystemProxy

When converting from an absolute path to a relative path within
the sandboxed directory, the code currently just assumes the
absolute path is a descendant of the sandboxed data directory
but doesn't verify that. This was discovered because some tests
were setting the storage service directory incorrectly, and the
MakeRelative operation was happily ignoring this fact, causing
downstream operations such as LockFile to fail.

Bug: none
