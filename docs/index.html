[Segmentation] Load launcher_filter_file via importlib.util

Fixes presubmit import breakages introduced in crrev.com/3610783.

When testing/__init__.py was added to the repository root, presubmit
checks executing 'from testing import launcher_filter_file' began
resolving the top-level testing/ package from src/ instead of
components/segmentation_platform/tools/testing/, causing ImportError.

This change updates FetchLauncherFilterFileData in PRESUBMIT.py to use
Python 3's modern dynamic loading (importlib.util) from explicit file
paths. This avoids sys.path manipulation and works around PEP 420
(implicit namespace packages) collisions in sys.modules.
