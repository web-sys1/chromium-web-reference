[feedback] show the WebUI using WebDialogDelegate

The feedback dialog manages its state to show a singleton window.
1) If a Feedback UI has been created, simply show it
2) Otherwise, create a new one.

This behaves the same as the existing app.

If the features::kWebUIFeedback is enabled, feedback requests from
different entry points will be redirected to the new Feedback App.

The <input type="file"> html element does not work on the WebDialog
yet.

A message handler will be added in a followup CL.
