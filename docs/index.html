Use throttled frame interval when sending frame timing details.

cc/metrics/jank_metrics.cc is using the interval in the frame timing
details to compute whether or not there is jank. Even when the caller
has requested to throttle the frame rate, the frame interval attached
still reflects the default vsync interval (typically 60 fps). This
causes the metrics to incorrectly report jank when the frames are
intentionally presented at a slower rate. Although this is the one
immediately noticed, there may be other consumer use cases out there of
similar nature.
