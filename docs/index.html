[ozone] Fixed getting the root transform for child windows.

DesktopWindowTreeHostPlatform::GetRootTransform() used the primary
display for windows that are not yet initialised, which caused problems
for popup windows upon creation: as they always got the transform that
used the scale factor of the primary display, they had to fix their
bounds on mixed DPI setups when their parent window resided on a
secondary display that had scale different to the one of the primary
display.

This CL fixes getting the root transform: if the window has a parent,
the parent's display is used.
