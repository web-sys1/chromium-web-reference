[Editing] Fix insertion point behavior with hidden <br> tag property

When toggling the list, if the <br> tag has its visibility set to
"hidden", the insertion point does not appear before the placeholder
as expected. Instead, the insertion point shifts to either the
previous or next visible position in the document.
To resolve this issue, the visibility of the placeholder is adjusted
to ensure it remains visible, allowing the insertion point to
consistently appear before the placeholder, even when the <br> tag's
visibility is set to "hidden".
