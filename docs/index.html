Fix an invalid pointer access in the overlay code

If the host window for an overlay is closed before the overlay is
closed, the container view is destroyed, but then when the overlay
window is closed, it tried to use the now invalid pointer to reset the
focus behavior, leading to a segfault (luckily).

An earlier changed removed some code to null out the pointer when the
view is destroyed, and this regression wasn't caught by the tests.

This patch restores the line to clear the pointer, and also the
associated observer. And adds a unit test to explicitly test that
destruction order (and I confirmed it reproduced the crash).
