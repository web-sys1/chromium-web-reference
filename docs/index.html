Remove use of apps::mojom::PermissionValue::New().

apps::mojom::PermissionValue::New() has a generic-sounding name but
actually creates a union containing the first declared subtype. In
general, this behavior is non-intuitive and is slated for removal from
the generated Mojo bindings. Here, it is also wasteful since it
allocates storage for a subtype that is then often immediately
discarded.
