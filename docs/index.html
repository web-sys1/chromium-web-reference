[anchor-position] Support containing blocks in multicol

r1035362 crrev.com/c/3823145 supported anchors in multicol,
but not for each containing block inside of the multicol. This
patch extends that, to compute stitched queries one for each
containing block.

When it's not block-fragmented, this is done by propagating
anchors to ancestors. But when block-fragmented, OOF objects
are added to their fragmentainers instead of their containing
blocks.

This patch adds |NGLogicalAnchorQueryForFragmentation| to
support this case. It creates one anchor query for each
containing block that has anchors.

|LayoutFragmentainerDescendants| creates this instance. Then
it chunks OOF nodes by their containing blocks so that the
layout order matches non-block-fragmented case. For the
performance reasons to reduce the number of rebuilding
fragmentainers, this is done only when anchors exist.

As in TODO comments in this CL, not all cases are supported
yet. They will be in following patches, on top of this change.
