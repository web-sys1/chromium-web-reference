Webium: fix the "Received message for unknown endpoint 2" error

This was caused by calling tabsApi_.getTabs() twice, both in
loadTabStripModel_ and registerTabChangeCallbacks_. The associated
receiver for the loadTabStripModel_ result's tabSnapshot.stream would
never be connected, and so messages sent to it would produce this
warning.

Instead, use a single call for both ops. This should also avoid
potential races.
