[Blink] Have WebGLRenderingContextBase check explicitly for passthrough

It currently checks whether the provider that it creates is
single-buffered and then effectively assumes that the provider is
passthrough in that case. Following [1], the *only* single-buffered
provider created for WebGL is the passthrough provider - the other
single-buffered providers are CRPSwapChain and CRPSharedImage with
CONCURRENT_READ_WRITE usage, and neither are created for WebGL [2].
Hence, this code can explicitly check whether the provider is
passthrough.

This opens up a followup that moves ImportResource() to be on the
passthrough provider.
