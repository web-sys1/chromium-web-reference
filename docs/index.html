Add checks for simultaneous active low and limited entropy layers.

This change introduces a new seed rejection reason for when a client has
both an active limited entropy layer and an active low entropy layer
with Google Web visibility. This state is not allowed, as it could lead
to unexpected entropy usage. The `MisconfiguredEntropyResult` is updated
to also report this disallowed state.
