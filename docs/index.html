[FPF] Refactor WKContentRuleListProvider to support arbitrary rule lists

This change refactors the WKContentRuleListProvider from a manager of a
fixed set of hardcoded rule lists to a generic manager capable of
handling any number of arbitrary rule lists.

Previously, the provider managed a specific set of lists with hardcoded
identifiers. The new implementation introduces a RuleListKey for clients
to identify lists and an UpdateRuleList() method to compile, replace, or
remove a list with arbitrary JSON rules. All internal members are now
keyed by RuleListKey, allowing for a flexible number of managed rules.
This approach, along with improved error handling and documentation,
enables the provider to be used by other features that require content
rule manipulation, such as the upcoming fingerprinting protection work.

Existing features have been updated to use the new interface.
