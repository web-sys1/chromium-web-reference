[PA][BRP] Don't allow enabling BackupRefPtr support if PA-E is disabled

It's been too many times we've tried to test CheckedPtr without PA-E,
wondering why things don't work the way they should, or worse, thinking
that everything is working while it doesn't.

The most typical culprit are debug builds, which turn on component
builds, which in turn silently turn off PA-E, which silently makes
CheckedPtr useless outside of Blink partitions.

NOTE: If you only want to test BRP support in PA (ref-counting), you
should comment out this assert. You don't need PA-E for this to work.
