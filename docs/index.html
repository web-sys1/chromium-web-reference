Enforce clean serializations

- Do not send events with nonexistent targets. The event serialization
loop has been moved until after the dirty object loop, so that it's
easy to check whether that event's target was serialized, before passing
the event to the serializer.
- Do not send serializations that have no updates

This also cleans up the last failing tests on the a11y CI bot, which
were failing because RenderFrameHostImpl was received a11y
serializations where details.updates.empty() == true, and triggering
a DCHECK.
