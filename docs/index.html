Deflake virtual/text-antialias/custom-font-data-crash2.html

The test is currently flaky because:
- Due to crbug.com/347460, if we recalculate style after removing a
  style sheet, all existing @font-face rules will be reset. If any of
  these rules contains an invalid url, this will immediately trigger a
  console message for this error.
- The test removes an in-body style sheet right before finishing
- Then the style recalc and the test finishing can race with each other
  and cause flakes.

This patch deflakes the test by forcing a style recalc before test
finishes.
