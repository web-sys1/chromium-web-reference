[Office] More fine-grained UMAs for invalid alternate URLs

Split OfficeDriveOpenErrors::kInvalidAlternateUrl into
- kWaitingForUpload (empty URL because the file is not uploaded yet)
- kEmptyAlternateUrl (empty URL resulting from another error)
- kInvalidAlternateUrl

Checks in `OpenDriveUrl` were redundant for the "upload to drive and
open" flow, since the alternate URL is already checked by the
DriveUploadHandler. Checks are removed from this method.
