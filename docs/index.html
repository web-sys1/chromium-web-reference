Fix threaded_input_handler treatment of scroll snap via scrollbar arrows.

Mirrors the fix for main thread scrolling in:

https://chromium-review.googlesource.com/c/chromium/src/+/2686150

Additional plumbing was required for threaded scrolling since the
source device information was lost by the time we needed to apply the
scroll snap. Ideally, we should treat scroll by page and scroll by line
slightly differently, but the granularity information is not correctly
reported. Left a TODO comment for the remaining work.

Note that this patch not only addresses the snap behavior, but fixes
the DCHECK problem in the bug report.
