Refactor paint under-invalidation checking

- Separate under-invalidation checking code from PaintController
  into PaintUnderInvalidationChecker.

- The end of under-invalidation checking: previously this was
  determined by checking under_invalidation_checking_end_. Now
  for single display item, under-invalidation checking ends after
  checking the next display item; for subsequence, under-invalidation
  checking ends when the subsequence ends.

- For paint chunks, check under-invalidation when the paint chunk
  is complete instead of checking all paint chunks at the end of a
  subsequence.

- Simplify API of PaintController::BeginSubsequence()/EndSubsequence()
