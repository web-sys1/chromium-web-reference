IncognitoReauth: Update the re-auth show logic on restore case

To tackle the case when we need to show a re-auth dialog for incognito
tabs from a restored session, we relied on TabLaunchType.FROM_RESTORE.
However, there can be cases where some tabs can be labelled as
TabLaunchType.FROM_CHROME_UI. Relying on the same also adds coupling
on other client's logic of how TabLaunchType is set for tabs.

This CL simplifies the flow by decoupling from that concept and instead
relying explicitly on the presence of a re-auth pending key we save
during actvity onSaveInstanceState.
