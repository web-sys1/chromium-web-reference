[tracing] Fix race with nested tracing scenario rule

The race happens when the parent scenario is stopping.
The easy fix is to update the CHECK in content::TracingScenario::OnNestedScenarioUpload
to allow `kStopping`.
I find that async operation confusing and not very useful though,
so instead I'm explicitly disabling the nested scenario when
the parent scenario gets a stop trigger.
