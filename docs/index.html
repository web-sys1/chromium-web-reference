TestNavigationObserver: adds NavigationOfInterestDidFinish method

OnDidFinishNavigation can be called repeatedly with the state
different than the one specified at the construction time.
Inside the method it determines if it truly has seen the navigation
of interest. A class that wishes to intercept, say, WebContents*,
cannot access those tests. Thus overriding OnDidFinishNavigation is
not a valid solution.

This change adds a hook method that provides overriding classes
with access to NavigationHandle* at the time TestNavigationObserver
determined that the navigation met all conditions specified at the
construction time.
