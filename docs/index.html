[MirrorSync] Split STREAM and MIRROR syncing status update

The existing OnSyncingStatusUpdate method receives events from both the
streaming and mirroring stack. To ensure all the DriveFsHostObserver's
that are watching these changes do not receive the events from the
MIRROR stack, split the two events (on the drivefs side) into
OnSyncingStatusUpdate which contains the streaming events and
OnMirrorSyncingStatusUpdate which contains the mirroring events.

Note whilst here performed some code cleanup by removing the deprecated
(go/totw/164) MOCK_METHODN macros and replaced with the MOCK_METHOD
equivalents. It seems the std::pair<int64_t, std::string> type was also
causing the preprocessor to interpret the arguments to FetchChangLogImpl
as 2 arguments, so introduced a type alias to work around that
compilation error.

Relevant drivefs side change: http://cl/458131717
