Original change's description:
> [views-ax] Add RootView's ViewAccessibility as a WidgetObserver
>
> This CL makes the RootView's ViewAccessibility object a
> Widget Observer.
>
> This is because we want the accessible data of a view to "react"
> to when the widget closes. There are multiple code paths that lead
> to this happening so instead of adding function calls there to
> manually update the data, we add the RootView's ViewAccessibility
> as a WidgetObserver so we take advantage of the already existing
> system to notify observers of a closing widget.
>
> Previously, we were setting data for when a widget was closed
> at "query" time in GetAccessibleNodeData, but we're moving away from
> this pull system to a push system for the ViewsAX project and these
> changes align with the project.
>
> Additionally, this CL adds a macro that is added to some setters
> in ViewAccessibility so that we no-op when the widget is closed,
> since otherwise the closed widget data may be overwritten by callers
> when the widget has already closed. This would be wrong behavior but
> also could lead to crashes.
>
> Lastly, this CL adds a check in ViewAXPlatformNodeDelegate's
> GetData so that we exit early (while setting the correct closed
> widget state data) if the widget does not exist.
>
> This CL is part of the ViewsAX project:
> https://docs.google.com/document/d/1Ku7HOyDsiZem1yaV6ccZ-tz3lO2XR2NEcm8HjR6d-VY/edit#heading=h.ke1u3utej413
