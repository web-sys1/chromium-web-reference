Use `float` for `tilt_x` and `tilt_y` before UI event creation

The precision of `altitude_angle` and `azimuth_angle` relies on `tilt_x` and `tilt_y` to be a double. Since `tilt_x` and `tilt_y` are rounded once the DOM event is created, we should be using doubles before then to ensure the previous statement. Since OSes usually provide tilts with float precision, we restrict the values to floats.
