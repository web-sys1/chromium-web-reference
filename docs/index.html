Original change's description:
> SVG Text NG: Support baseline-shift property
>
> This CL implements 'baseline-shift' property so that it has the behavior
> same as the legacy SVG <text>.  That is to say,
>  * 'baseline-shift' and 'vertical-align' are independent properties
>   though the specification says 'vertical-align' is a shorthand property
>   of 'baseline-shift'.
>  * SVG <text> doesn't support 'vertical-align'.
>  * Offset computation for 'baseline-shift''s percentage values and
>   'super'/'sub' are slightly different from 'vertical-align'.
>
> Implementation
> * For SVG <text>, we skip 'vertical-align' handling and apply
>   'baseline-shift' instead in |NGInlineLayoutStateStack::
>   ApplyBaselineShift()|.
> * Add |scaling_factor| data member to NGInlineBoxState for the above.
> * Fix a wrong y offset in NGSVGTextLayoutAlgorithm::SetFlags().
>
