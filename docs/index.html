Fix DCHECK in webrtc transport code

These DCHECKs fired when starting a CRD connection on ChromeOS,
and were caused because RtpSenderInterface::SetParameters takes
a semaphore (and this code ran on the IO thread where that is
disallowed).

I fixed this by using `ScopedAllowSyncPrimitivesForWebRtcTransport`
which supresses these DCHECKs.

Fixed: b/264975455
Test: Started CRD connection with DCHECKs enabled
