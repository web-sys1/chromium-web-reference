[BRP] Make BRP-ASan more robust

While testing BRP-ASan on bots, we've discovered two issues with the
current implementation:

- __asan_address_is_poisoned may crash, for example, if the address
  points to the shadow map.
- Other users of the ASAN_POISON_MEMORY_REGION macro interfere with
  our freed allocation detection.

This CL addresses both issues.
