PDM: Consider wallet integration disabled in OTR and guest profiles

SyncService doesn't exist in off-the-record or guest, so wallet import
is effectively disabled, regardless of what the preference holds. It
seems more appropriate that
PersonalDataManager::IsAutofillWalletImportEnabled() returns false in
such cases, which is precisely the behavioral change in this patch.

Note however that some of the autofill flows use the regular PDM
instance even from incognito, and autofill works normally. This patch
doesn't change the behavior for these cases, as the regular PDM
instance's behavior doesn't change (it uses a non-null SyncService
before and after this patch).

The change is done in preparation for kAutofillWalletImportEnabled
moving fully into SyncService, which also means that the preference
cannot be evaluated if SyncService doesn't exist.

Many unit-tests need updating because they now required injectinng a
non-null SyncService, simply because they rely on
IsAutofillWalletImportEnabled() returning true.
