Mark raw_ptr with gsl::Pointer.

This provides compiler warnings when the raw_ptr holds a pointer to an
object in a gsl::Owner (like unique_ptr) that is destroyed before it,
in some cases. For instance:

  raw_ptr<int> ptr = std::make_unique<int>(2).get();

Without gsl::Pointer, this compiles without an issue, but with
gsl::Pointer it generates a warning (error for us) that the ptr
is holding a dangling reference.

The Owner and Pointer attributes are described here:
http://www.open-std.org/jtc1/sc22/wg21/docs/papers/2018/p1179r0.pdf

Notably: Pointers follow copy semantics, and if there is a move from
a Pointer we assume it does not change the source.

This means the compile will warn incorrectly if raw_ptr is
moved-from in a way that avoids dangling as the compiler will assume
the moved-from raw_ptr still holds a dangling reference.

There are no existing (incorrect) errors generated if we keep the move
constructor/assignment, but there _is_ a binary size regression, and a
perf regression, if we remove them. So this keeps them present accepting
the possibility of incorrect errors for the chance to get errors on
clear memory bugs with raw_ptr<T>.
