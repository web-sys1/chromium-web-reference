Convert PageContext to a wrapper and delete PageContextRegistry

Doesn't include the extended accessors from PageContextRegistry that
map PageContext to RenderFrameHost, since they were just convenience
functions for data that can already be calculated from
PerformanceManager, PageNode and WebContents API's. The registry class
mirrored the interface of PerformanceManager, but ProcessContext is a
lighter-weight wrapper so there's no need to complicate it by
duplicating functionality.

This is the last Resource Attribution registry, so delete all the
supporting classes too.
