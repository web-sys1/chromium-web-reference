[Sync] Maintain syncing user prefs for non-syncing users on ChromeOS

On ChromeOS, when the user is signed in without sync consent and
kReplaceSyncPromosWithSignInPromos is enabled, SetSelectedType now
updates both the account-specific preferences and the syncing-user
preferences. This is done to facilitate a possible future migration of
users from kSignedInWithoutSyncConsent to kSyncing.
