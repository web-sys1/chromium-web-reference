Clear shopping item meta when bookmark URL changes

This patch adds a BookmarkModelObserver to the shopping service to
watch for URL changes on nodes that have shopping meta. In this case
the meta is cleared since we can't currently guarantee that the new
URL is also a shopping page.

The subscription will be removed the next time the user is on their
android device as a result of the preexisting validation logic.

This patch also adds the shopping service to both android and ios
service lists as it now has a function.
