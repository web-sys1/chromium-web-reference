Revert "Only read chrome/VERSION in Chromium Windows builds"

This reverts commit 7abdc93319fc59b40f98d2ef3fd37dbc1fa6662a.

Reason for revert: The change breaks compile step on Win32: https://ci.chromium.org/p/chromium/builders/ci/win32-official. Exmaple: https://ci.chromium.org/ui/p/chromium/builders/ci/win32-official/5358/overview


Original change's description:
> Only read chrome/VERSION in Chromium Windows builds
>
> Chromium's build/ directory is used by multiple products - it is
> extracted to its own repo - so any Chromium-specific modifications to it
> can cause problems. Thus, when I added code to
> compute_build_timestamp.py script this caused issues for the Pdfium
> build.
>
> This change modifies compute_build_timestamp.py to take an optional
> parameter to specify the version file to read from. In order to only
> pass this parameter in for Chromium builds I needed to make a modified
> copy of timestamp.gni in the build/config directory and import this from
> the Chromium-Windows specific build/config/win/BUILD.gn.
>
> build/dotfile_settings.gni needed modification because we have an
> allowlist for what GN files are allowed to run exec_script.
>
