media/base: Use uint32_t for all bitrate types

Prior to this CL, the data types of the bitrates stored in Bitrate
and VideoBitrateAllocation (VBA) classes conflicted; Bitrates stored
uint32_ts and VBAs stored ints. The types were being converted
unsafely in many places. After this CL, VideoBitrateAllocation has been
updated to use and store uint32_ts, and the VideoBitrateAllocation's
SetBitrate method requires a uint32_t as input (no longer accepting
signed integer values).

A doc with more details is available and visible externally at
https://docs.google.com/document/d/1Z3zf1FY9MnMnZCzUF6mXKxM3-bNCL_nmA9A_4R1lWhQ/edit?usp=sharing&resourcekey=0-xd_ZzkfwcBkswkOgrGRVFQ
