[Extensions] Create ExtensionInstallWatcher as a BrowserWindowFeature

This change refactors the logic for waiting for an extension to install
and showing a post-install UI from ExtensionInstalledWaiter into a new
class, ExtensionInstallWatcher, which is implemented as a
BrowserWindowFeature (BWF). The design doc can be found here:
https://docs.google.com/document/d/1wbqmFywwEmThIWaKbz7T452QzK2l1VO72ifvIJv1B_U.

Motivation: 1. Fix Race Condition: ExtensionInstalledWaiter currently
manages its own lifetime, leading to a race condition and potential
use-after-free if the browser window is closed shortly after
installation initiation. Using a BWF is the standard way to handle this
issue as it ties the lifetime of the watcher to the browser window.
2.Android Compatibility: This change also replaces direct usage of
`Browser*` with `BrowserWindowInterface`, making the logic compatible
with Android, which does not support `Browser*`.

Implementation:
-   Creates `ExtensionInstallWatcher` as a `BrowserWindowFeature`.
-   Moves the core waiting logic from `ExtensionInstalledWaiter` to `ExtensionInstallWatcher`.
-   Updates call sites to use the new `ExtensionInstallWatcher` via `BrowserWindowFeatures`.
-   Removes the old `ExtensionInstalledWaiter` files.

Tested that installed dialog logic still WAI.

Works on extension installations that trigger a new tab page:
https://drive.google.com/file/d/1QLF7G9wA_P_bJXr8hXostjMx4i9tobtY/view

Also works on extension installations that don't trigger a new tab page:
https://drive.google.com/file/d/1CCRNTESb7zasAlj9RwvotIQJs7luBHNy/view
