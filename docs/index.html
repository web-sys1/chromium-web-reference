Remove PaintLayerPaintingInfo

The only useful field in PaintLayerPaintingInfo was
global_paint_flags, so we can pass GlobalPaintFlags instead of
PaintLayerPaintingInfo.

Then because we have only a few paint flags, the need to distinguish
global and local flags seems not very important. Also the distinction
between the two types of flags is not very clear. For example, both
global flag  kGlobalPaintFlattenCompositingLayers and local flag
kPaintLayerPaintingRenderingClipPathAsMask can be specified during
a paint for a subtree. Thus all the paint flags are combined into a
single PaintFlags type for simplicity. The painters decide whether
to pass a particilar flag to descendants.
