[CrOS HaTS] Rework HaTS/UMA integration

Moving forward, we will allow surveys to opt-in to the sending
of all survey questions to both Lisntr and UMA. This CL provides
the infrastructure for a survey to flag on UMA metrics for their
survey by providing a `histogram_name` parameter to their survey
feature. Since it is important for the UMA metrics to be tied to
the HaTS survey ID, it requires both of those values to be provided
in a way that can be updated out-of-band of normal ChromeOS releases.

Testing this new change involved running ChromeOS on Linux with specific
survey feature parameters to enable the survey for testing purposes. The
histograms produced were confirmed via chrome://histograms. The flags
used are as follows:

--force-happiness-tracking-system
--enable-features=HappinessTrackingSystem:prob/1.0/trigger_id/test/histogram_name/ChromeOS.HaTS.TestSurvey
