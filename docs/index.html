OzoneBacking - gpu workaround for ARM Mali on ChromeOS

ChromeOS kevin board uses ARM Mali gpu drivers. When using it for
producing GLTextures with passthrough command decoder,
SharedImageRepresentationGLOzone is created which in turn creates a
GLImage for each representation. These images use the same internal
NativePixmap for binding to the texture but the driver confuses between
these images/textures and doesn't call the GL commands in expected
order which causes some WebGL2 conformance tests to fail
(https://crbug.com/1327044)

This CL adds a workaround for these drivers on ChromeOS where we wait
on the write fence on the same GL context that allows for the associated
GL operations to be completed on a texture before doing them on the
next texture.

Test: Tested on ChromeOS kevin board using crosfleet running WebGL2 conformance tests
