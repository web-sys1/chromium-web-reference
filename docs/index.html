webauthn: test the Intent extra that GMS Core actually uses.

These test helper functions were creating response intents that used the
deprecated RESPONSE_EXTRA and ERROR_EXTRA keys. But GMS Core always uses
the CREDENTIAL_EXTRA key and sends everything in a single
PublicKeyCredential[1] object. Chromium currently supports both styles,
but we should test the one that is actually used, which is what this
change does.

Also, the old code very awkwardly constructed error responses. Fix all
that up.

[1] https://developers.google.com/android/reference/com/google/android/gms/fido/fido2/api/common/PublicKeyCredential
