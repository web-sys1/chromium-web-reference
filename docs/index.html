[Region Capture][cropTo][#9] Create CropIdManager

Complete the move of crop-ID production into the browser process.

This provides several benefits:
1. Crop-IDs are now produced in a trusted process, making them
   easier to reason about, as we can now assume that a malicious
   application which has hijacked the render process,
   nevertheless can *not* induce collisions.
2. It now becomes possible to verify that a crop-ID handed
   to cropTo() was (i) produced and (ii) is associated with
   the display-surface on which cropTo() is invoked.
   (This verification will be added by the next CL in this chain.)
3. We wish to resolve the Promise returned by produceCropId()
   only when the crop-ID is ready for use by cropTo().
   This CL moves the resolution closer to that point,
   and facilitates moving it even further.
