bindings: Make ASWManager show up as pending activities object

Instead of creating an artificial root group in heap snapshots just
provide the ActiveScriptWrappableManager with the "Pending activities"
name. This way it actually reflects that the ASWManager object retains
the ASW objects.

After this CL, DevTools frontend tests are adjusted to check for the
object instead of the root group.

When switching to the Oilpan library nothing needs to be done then, as
the library can represent the same object hierarchy without the magic
root group.
