[RecentlyClosed][RWM] Only trigger UI update once when bulk remove session entries from storage

Currently, we remove excess entries if the total count exceeds the max of 25 or if they are expired after 6 months. To support bulk removal, I'm adding a new RemoveLeastRecentlyUsedEntries method. This triggers #NotifyEntriesChanged only once at the end, rather than calling #RemoveEntryById in a loop, which triggers #NotifyEntriesChanged on every iteration.
