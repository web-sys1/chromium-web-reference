chromedriver: Support non-BMP SendKeys input

ChromeDriver currently rejects UTF-16 surrogate pairs while flattening
Element Send Keys input, returning an unknown error before the key
conversion path can generate text events. That prevents WebDriver
clients from sending valid non-BMP Unicode characters such as emoji.

Allow valid surrogate pairs through flattening and convert each pair into
a text-only key event with VKEY_UNKNOWN. Keep the existing BMP key,
modifier, NULL, and special-key handling unchanged, and reject malformed
surrogate input in the converter.

Add unit coverage for valid and invalid surrogate input, and update the
Android WebDriver expectation now that the existing WPT surrogate test
passes.
