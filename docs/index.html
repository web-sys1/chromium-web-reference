Mojo JS modules: Generate absolute import URLs for certain cases

This CL undoes a previous change to mojom_js_generator.py from r955225,
and applies a more appropriate fix instead, to address the following two
issues.

a) An incorrect assumption was made that any Mojo JS module that is not
   served under chrome://resources/mojo/ can be referenced by a relative
   URL. This is not the case for Mojo JS modules residing under
   chrome://resources/cr_components/. This is what r955225 addressed,
   but in a different way.

b) When a Mojo JS file in chrome://resources/cr_components/ referred
   to a file in chrome://resources/mojo/ a relative path would be
   produced. Relative paths between JS modules that belong to different
   ts_library() targets are problematic as TS compiler can't resolve
   them correctly. This was an unforseen side-effect of r955225, and
   partially contributed to the transient build errors at
   https://crbug.com/1286288.

With this change an absolute path is produced, which can be correctly
resolved by TS compiler via the use of path mappings.

Verify (a) is fixed as follows:
1) Build chrome/browser/ui/webui/new_tab_page_third_party:mojo_bindings_js__generator
2) Observe that the generated file at gen/mojom-webui/chrome/browser/ui/webui/new_tab_page_third_party/new_tab_page_third_party.mojom-webui.js
   imports chrome://resources/cr_components/most_visited/most_visited.mojom-webui.js (at line 12)

Verify (b) is fixed as follows:
3) Build ui/webui/resources/cr_components/most_visited:mojom_js__generator
4) Observe that the generated file at gen/mojom-webui/ui/webui/resources/cr_components/most_visited/most_visited.mojom-webui.js
   imports 'chrome://resources/mojo/mojo/public/mojom/base/text_direction.mojom-webui.js (at line 12)
