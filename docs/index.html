> [Clank CBCM] Disable UMA controls on pre-MICe FRE when policy disables metrics
>
> This implementation follows go/clank-metricsreportingenabled-design document.
>
> This change waits for Native to check the status of MetricsReportingEnabled then updates UI state accordingly. After this, we don't respond to changes in the policy. (I.E. CBCM as explicitly stated in design doc).
>
> For non-enterprise/policy users we don't need nor wait for Native to have a working interface, and for enterprise/policy users we've already waited for Native to be initialized so this does not affect the FRE initial latency.
