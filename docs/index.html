[CSC] Support sticky permissions

Prior to this CL, the permission for Captured Surface Control was scoped
to the capture-session - unlike all other permissions. This was done
until bespoke UX could be added to better reflect to the user that the
permission was granted and is active.

This CL makes the CSC permission into a normal, sticky permission, but
does so behind a default-disabled flag,
CapturedSurfaceControlStickyPermissions.

Later CLs will add the bespoke UX and then flip
CapturedSurfaceControlStickyPermissions to default-enabled.

(This ordering of the CLs is needed because the bespoke UX refers back
to the permission itself, which needs to appear in PageInfo. It also
supports better parallelization of the work.)
