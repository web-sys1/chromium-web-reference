Adjust stack quality metrics for Android

Since we're mainly investigating the unwinder performance on Android,
changes the stack quality metrics to be more actionable there and stops
collecting them elsewhere:

* Counts frames that are in the Chrome module, instead of frames that
  are in any recognized module, since on Android only the Chrome module
  can be symbolized.
* Records the percentage of frames that are NOT found in the module,
  which is easier to interpret since higher percentiles are farther from
  the ideal.
* Also only collects the count of short stacks on Android, since the
  threshold for a stack to be too short was only measured there and may
  differ on other platforms.
* Moves the stack quality measurements to a helper class so that the
  whole class can be stubbed out on non-Android platforms instead of
  sprinkling BUILDFLAG checks through the code.
