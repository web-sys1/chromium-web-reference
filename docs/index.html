Make PerformanceEntry::entryType() return const AtomicString&

The strings returned by these methods are either global (part of
`performance_entry_names`) or have a lifetime that (at least) follow
that of the entry (`PerformanceEventTiming::entry_type_`), so a
reference to said string can be returned to let the caller take a
reference, if needed, rather than having all the entryType()
implementations do it. This reduces binary footprint (and maybe a bit
of ref-churn).
