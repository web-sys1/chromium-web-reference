[AdTagging] Add protocol support for "ad" adorners on arbitrary nodes

Summary:
This patch introduces the backend and protocol changes required to
support ad-tagging (for "ad" adorners) for arbitrary Node types in
DevTools.

Background:
Historically, the DevTools "ad" adorner was restricted to iframes
because backend ad-tagging only supported iframe elements. The backend
has since been updated to identify and tag arbitrary elements via
`Element::SetIsAdRelated()`. This change bridges that capability to the
frontend.

Implementation Notes:
* The new protocol API accepts a boolean parameter to indicate the new
  ad status.
* While the underlying `Element::SetIsAdRelated()` is currently a
  one-way setting, the boolean parameter was intentionally chosen for
  the protocol API to allow for future flexibility (such as supporting
  the un-tagging of elements).
