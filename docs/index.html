viz::VideoCaptureOverlay: Remove gfx::ColorTransform

Change the function to use skia::ConvertRGBAToOrFromYUVA. In the center
of VideoCaptureOverlay::Sprite::TransformImage, populate an array of
SkColor4fs instead of a separate color and alpha array (this is needed
because the old gfx::ColorTransform doesn't take an SkColor4f).

This change threw up a bunch of UNSAFE_TODO warnings. Fix all of the
ones that this change touched by changing transformed_image_ to a
base::HeapArray instead of just an array, and use base::span to make the
pointer math safe.

Of note is that all of this CPU-side pixel manipulation should be moved
into Skia calls (and ideally done on the GPU).
