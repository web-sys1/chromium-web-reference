[conflict dialog] Update dialog for GAR4 ARIA conformance

Conflict dialog 1) has no title, and 2) uses a <cr-dialog> element
which assumes there is a title, and will focus that title when the
dialog is shown. ARIA announces the empty title because it has the
initial focus as "Heading 2 Heading 2".

Fix this: remove <cr-dialog> #title element and all <dialog> aria-
attributes that make ARIA tools read out irrelevant content.

Instead, add tabindex="0" to the dialog #message element and focus
it when the dialog is shown. On show, ARIA now reads out:

  "Dialog A file named ... exists. Do you want to replace it
   with the one you are moving?"

Update the testDialogShowFocus test to check that the #message has
the initial focus (ARIA announced on show), and check the <dialog>
"aria-*" attributes are removed (block spurious announcements).

Add an aria-modal="true" attribute to the <dialog> element per W3C
ARIA role=dialog best-practice recommendations [1].

[1] https://w3c.github.io/aria-practices/#dialog_roles_states_props
