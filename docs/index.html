[ios] Only create native session restoration cache object when enabled

Mark the native session restoration cache feature as incompatible with
the session restoration optimisation (as optimised code will manage the
native session directly).

Ensure that the native session restoration cache objects are created
only when the feature is enabled and remove the unit test checking the
objects are no-op when the feature is disabled (as they now assert the
feature is enabled when constructed).

Remove code to support iOS 14.
