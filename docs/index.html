[PA] Selectively depend on //base/tracing

This CL gates the usage of `//base/tracing` in `raw_ptr.h` depending on
whether or not the code is being built for Chromium; the related
functionality is not available when `raw_ptr.h` is used in the
standalone PA distribution.

A better candidate might be `BUILDFLAG(ENABLE_BASE_TRACING)` (see also:
https://crrev.com/c/3474386), but that is exposed by a buildflag header
defined in `//base/BUILD.gn`, which we cannot seize for use in
standalone PA (for now). The removal of this may impact `//base` users
who expect traces on `raw_ptr` or `raw_ref`, but hopefully will not
break their builds.
