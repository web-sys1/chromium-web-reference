Rename and Move RFHI::InsidePortal() to FrameTree::IsPortal()

Current naming of RenderFrameHostImpl::InsidePortal() is confusing
considering various cases like fenced frame embedded inside a portal.
Also InsidePortal() API checks for IsPortal() value.

To avoid this confusion, we want to rename it back to IsPortal and
looking at the usecases of InsidePortal, it would be more apt to have
it inside FrameTree to avoid FTN -> RFH usages.

We also double check that a fenced frame when embedded inside a portal
FrameTree::IsPortal() returns false.

BUG=1294795,1179502
