base/android: Split mLibInfo into two objects: mLocal and mRemote

The split will becoming necessary soon when ModernLinker adds one more
async step in initialization: finding the webview address reservation.
It would be natural to store this information in mLocal, then load the
library using it, extract the RELRO info. After that mLocal is compared
with the incoming mRemote to establish sharing.

Making this as a standalone change because it can be done as a pure
refactor without altering behavior.
