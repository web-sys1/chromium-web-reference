Start Isolate in foreground mode for mimehandler processes

In Edge, we've seen a regression in the time to load a PDF document when
the feature V8EfficiencyModeTiering is enabled. I'm not sure how much
code is shared between Chrome and Edge regarding PDF rendering, but I
believe at least the overall architecture matches: there is an extension
renderer process (the "mimehandler") which needs to run some script.
This renderer process is initializing its V8 Isolate in background mode,
which causes a slower overall load. As described at [1], extension
processes start their Isolates in background mode because they might not
have any way to detect visibility. However, the PDF mimehandler process
does in fact get notified when its visibility changes, so it can safely
start up in foreground mode.

In this CL, I propose adding another command-line switch so that the
browser process can indicate that the renderer should start its Isolate
in foreground mode.

[1] https://source.chromium.org/chromium/chromium/src/+/main:third_party/blink/renderer/bindings/core/v8/v8_initializer.cc;drc=7477a270035df963ba48d3f7ea4d8b384bce770c;l=951
