Revert "[Permissions] Relay TabInterface subscription during tab discard"

This reverts commit 5785017ee0aecfa93eeedb181d117b6acc5f320b.

Reason for revert: A failure of PermissionRequestManagerBrowserTest.TestSplitViewDiscardInitializationRace breaks lots of Linux builders.

Failure Link: https://ci.chromium.org/ui/p/chromium/builders/ci/Linux%20Tests/184162/test-results?q=ExactID%3A%3A%2F%2Fchrome%2Ftest%5C%3Abrowser_tests%21gtest%3A%3APermissionRequestManagerBrowserTest%23TestSplitViewDiscardInitializationRace+VHash%3A9615faa4d856567b

Original change's description:
> [Permissions] Relay TabInterface subscription during tab discard
>
> When Memory Saver discards a background tab, the replacement WebContents
> and its PermissionRequestManager are constructed before the
> TabLookupFromWebContents entry is moved to the replacement WebContents:
>
> 1. TabStripModel::DiscardWebContentsAt() first calls
>    delegate()->WillAddWebContents(), which runs
>    TabHelpers::AttachTabHelpers() on the bare replacement WebContents.
>    So all tab helpers are constructed while
>    TabInterface::MaybeGetFromContents() still returns null for it.
>    https://source.chromium.org/chromium/chromium/src/+/main:chrome/browser/ui/tabs/tab_strip_model.cc;l=419;drc=2de67ed6b4cd714fe6728e8d588cf4a13f6f0c45
>
> 2. The TabLookupFromWebContents entry for the new contents is only
>    created at the very end of TabModel::DiscardContents():
>    https://source.chromium.org/chromium/chromium/src/+/main:chrome/browser/ui/tabs/tab_model.cc;l=474;drc=2de67ed6b4cd714fe6728e8d588cf4a13f6f0c45
>
> As a result, the replacement manager's constructor skips tab wiring. We
> therefore subscribe each wired PermissionRequestManager to
> WillDiscardContents. When a discard occurs, the predecessor manager
> catches the pre-swap notification and hands its TabInterface pointer
> directly to its replacement manager via AdoptTabInterfaceIfNeeded(),
> ensuring the replacement manager is fully wired before the swap
> completes.
>
