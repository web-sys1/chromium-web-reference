Fail Deserialization for malformed 1P StorageKeys

A first-party (1P) StorageKey should always serialize to the format
`origin`, specifically excluding the top-level site (since it's
same-site to the origin).

Because this is the case, the deserialization will now fail if it
encounters a serialized string for a 1p StorageKey that includes the
top-level site as that would indicate a malformed input.

Valid:
https://example.com
https://example.com^0https://notexample.com

Malformed:
https://example.com^0https://example.com
