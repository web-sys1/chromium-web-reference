pip-pinch: Add resistance effect to PiP Pinch-to-Resize

This commit adds a new UI effect which we call the 'resistance' for
the Picture-in-Picture (PiP) Pinch-to-Resize feature. With this new
effect, when the PiP window reaches its min or max size during the
pinch, the PiP window can shrink or expand by further 15%. The PiP
window animates back to its min/max size when released.

Previously, when users attempted to expand the PiP window beyond its
maximum size using a pinch gesture, the resizing abruptly stopped with
the window size remaining at its maximum limit. Now when the window size
reaches its maximum limit, it follows a differentiable curve that
converges at 115% of the maximum size. Upon releasing the pinch gesture,
the window automatically returns to its maximum size. This behavior also
applies to when users shrink the window.

Since window bounds cannot actually be changed to one that exceeds its
limits, this commit utilizes `gfx::Transform` to visually change the
window size.

Some of the notable changes in this commit are:

1. `SetTransformDuringResize()` was added to `WindowResizer` which sets
 `gfx::Transform` to the resizer's window.

2. `CalculateTransformForPinch()` was added to `PipWindowResizer` which
 handles scale calculations and returns appropriate `gfx::Transform`.

3. `PipWindowResizer::CalculateBoundsForPinch()` as well as
 `PipWindowResizer::CompleteDrag()` were modified to center the scale
 transform around the pinch center.
