feedback: Call GetFeedbackLogsV3 to get debugd logs when flag is enabled

Fetching logs from debugd for feedback reports is slow. The average time
is about 20 seconds. Sometimes it could exceed the timeout limit of 2
minutes.

One solution is to run logs collecting commands in parallel so to reduce
the client side wait time.

Currently, the GetFeedbackLogsV2 is used to get logs from debug daemon.

Change:

- When EnableGetDebugdLogsInParallel is true, use GetFeedbackLogsV3
  which collects logs in parallel (to be implemented) [1].

- When EnableGetDebugdLogsInParallel is false (default), use
  GetFeedbackLogsV2.

1. Sceenshot: http://screen/BWeQSyoPU6UTxpp.png.

Bug: b:238782324
Test: tested in DUT with the flag enabled and disabled.
 Press Alt+Shift+I to open the feedback app and click "system & app
 info" link to preview the logs.
