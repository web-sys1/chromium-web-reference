[Autofill] Remove deprecated Value types from GetAddressComponents()

This CL is removing the deprecated base::ListValue and
base::DictionaryValue from GetAddressComponents() implementation.

Following the guidelines[1], they have been replaced by
std:vector<base::Value> and base::flat_map<std::string, base::Value>
respectively.

This CL doesn't introduce behavioral changes.

[1] https://source.chromium.org/chromium/chromium/src/+/master:base/values.h;l=84;drc=9d82515060b9b75fa941986f5db7390299669ef1
