Always allow background tracing to end when saving to a local file

This refactors the two ChromeTracingDelegate::IsAllowed functions to call the
same helper to find out if tracing actions are allowed. This prevents the logic
when traces are allowed to begin and end from getting out of sync.

Fixes a regression introduced in
https://chromium-review.googlesource.com/c/chromium/src/+/2860569 where using
the command-line to write traces to a local file would start tracing, but never
write the file because of throttling.  Traces saved to a local file should
always be allowed because they're requested explicitly by the user.

Also adds more browser tests.
