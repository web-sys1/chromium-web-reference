DanglingPtr: Check for dangling raw_ptr.

See docs:
https://docs.google.com/document/d/15qq2wr7Xza5awgJAfXUzO1UZFAG7lXDm4K8hd7vC3GU/edit
https://docs.google.com/document/d/1iw2GtM_f-h5WbHjKf5iVzK_XN5-4AkWwyfYjvPWuhTQ/edit

This patch:
1. Add build flag `enable_dangling_raw_ptr_checks`
2. Make partition_alloc detect dangling raw_ptr.
3. Make content/ handle those by printing stacktraces.

When installed, the handlers prints simultaneously two StackTrace:
- One where the memory was freed.
- One where the (last) raw_ptr stopped referencing the memory region.
and crash.

Getting both locations make it easier to track and fix them. For
instance, one can makes a list like:
https://docs.google.com/spreadsheets/d/1BAfz1uHhMkFe2sK2vyTHsJ733VQUhY9v2M4HxOwaM7E/edit?resourcekey=0-H9SwkJTMbBxQPKUEuIrwow#gid=767599093
