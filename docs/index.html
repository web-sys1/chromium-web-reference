[Bottom Sheet] Observe layout changes and unsuppress on GTS

Treat layout changes similar to tab switches and close the bottom sheet
in response to LayoutState or StartSurfaceState changes. This allows
removal of suppression for the bottom sheet on the TabSwitcher.

It is unusual to have a situation in which the bottom sheet is not
already closed during a layout state transition as by default clicking
the scrim exists the sheet.

This can only happen if
* The sheet is in peek state.
* The sheet has a custom scrim.
* The sheet has a custom lifecycle.

Based on a recent audit of all Sheets this is maybe 3-4 sheets at
most and the persistence across GTS visibility does not appear to be
integral to their function.

This is necessary to unblock showing a bottom sheet
on the grid tab switcher.

This change is behind a default enabled killswitch in the event it
does break an important feature in an overlooked edgecase.
