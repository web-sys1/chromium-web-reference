[ios] Implement Omnibox HTTPS upgrade fallback

This CL implements the initial version of Omnibox HTTPS Upgrades (aka
Typed Navigation Upgrades) on iOS.

This feature automatically upgrades scheme-less navigations to https://.
If the user types "example.com" in the omnibox (without http:// in
front), previously Chrome would load http://example.com. With this
change, it will now attempt to load https://example.com first. If the
load fails, it will fall back to http://example.com.

The upgrade is done by the autocomplete code. This CL introduces the
fallback logic and corresponding tests.

See the design doc at https://docs.google.com/document/d/1YfO5wCR-nC8-hxB_o3Ic--Cxcur3foTJWH2pxp363Dk/edit for details.
