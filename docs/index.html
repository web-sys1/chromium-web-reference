graphite: Specify BGRA explicitly for OffscreenCanvas WebGL

OffscreenCanvas disables the use of IOSurface backed GMB for WebGL
DrawingBuffer, but with Graphite IOSurfaces are used without GMB or
scanout usage since that's the only way to share textures between WebGL
on ANGLE and Graphite on Dawn.

The RGBA vs BGRA distinction matters for Graphite, so we explicitly use
BGRA on the GMB path, but we're still allowing RGBA on the non-GMB
path. The OS compositor doesn't like RGBA IOSurfaces so we should
explicitly use BGRA in the non-GMB path too. This CL does that by moving
the RGBA to BGRA override outside the ShouldUseImageChromium() branch.
