Move presenter inititialization from InitializeUiState to constructor

Currently, computing of the UI inputs and recodring of the opening
page info UI is performed in PageInfo::InitializeUiState. All references
create the instanse of the PageInfo and call InitializeUiState
immediately.

Page info v2 consists of multiple pages, each one of those is a
representation of PageInfoUI. Now PageInfo construtration will
compute UI inputs and recording opening dialog action. This makes the
lifecycle more clear: PageInfo instance is bound to lifetime of the
page info surface, not to the current implementation of the PageInfoUI.
This allows to change between PageInfoUI while keeping shared PageInfo
instance without recomputing values.
