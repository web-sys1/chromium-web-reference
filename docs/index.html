[Gap Decorations]: Suppress Gaps in Row Flex Containers

This CL implements the CSSWG resolution from Issue 11520[1], which
states that gaps overlapping a break, or appearing as the last content
before a break, should be suppressed across all container types.

In this change, when a row in a flex container cannot fit within the
current fragmentainer, the gap that would otherwise appear at the start
of the next fragmentainer is suppressed. This is achieved by computing a
`gap_delta`, which is incorporated into the `item_offset_adjustment` of
the `flex_line`.

Previous tests that expected row gaps to remain untruncated have been
updated to reflect this change.

A subsequent change will implement the same behavior for column flex
containers.

[1]: https://github.com/w3c/csswg-drafts/issues/11520
