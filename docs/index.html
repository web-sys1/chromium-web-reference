[CrOS Settings] Change VPN type should update certs list

When configuration a VPN service, the dropdown lists for "Server CA
certificate" and "User certificate" are supposed to be updated when the
"Provider type" is changed, since different VPN types may have
different requirements for the usage of certificates. For example,
user cert is optional in OpenVPN, but mandatory in L2TP/IPsec-cert. This
patch fixes this behavior, and adds tests to verify it.

The user-observable change in this patch is, before this patch, when
there is no user certificate installed, for an L2TP/IPsec VPN, it will
show "No user certificate" in the "User certificate" dropdown, and the
service is connectable even if this option is selected, which is not
correct. After this change, "None installed" will be shown instead, and
the service will not be able to connect in this case.
