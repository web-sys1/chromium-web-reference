Optimize ui::ConvertColorProviderColorIdToCSSColorId()

- Change it to take a std::string_view instead of a std::string, so
  callers can potentially avoid copying a string.
- Call std::string::reserve() since the resulting length can be
  estimated.
- Preprocess the first character. It's always replacing 'k' with '-'.
- Use std::string::push_back(char c) instead of
  std::string::operator+=(std::string) to avoid having to instantiate a
  std::string for every single character in the input.
- Only call base::ToLowerASCII() when needed.

Along the way, fix some nits:

- Use early returns.
- Use `it` for iterator variable naming instead of `i`.
- Use `c` for character variable naming instead of `i`.
- Use "kFoo" naming for true constants.
