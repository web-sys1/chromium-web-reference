[RemoveLegacy] Use `BidiParagraph` in `Font::DrawBidiText`

This patch switches `Font::DrawBidiText` from `BidiResolver`
to `BidiParagraph`, the BiDi engine in LayoutNG.

`bidi-multi-run-expected.html` is updated as the switch fixes
handling of LRO/RLO within the string. The new result matches
NG[1]/Gecko/ICU.

`bidi-override.html` is added because there are no tests for
the code path of BiDi override set on the canvas. This logic
was added in 2008[2] when drawing text is supported in WebKit,
but it's not defined in the spec[3].
