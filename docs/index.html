[Sync] Fix updating with incorrect server ID if the update is not applied

This CL prevents updating entity's server ID while processing an
incremental update if the update is not applied. This could lead to
incorrect server ID when there are two updates for the same GUID, the
first one is a new entity (with a new ID), and another one is a
tombstone (with an old ID).
