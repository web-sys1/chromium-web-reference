Upload function names for stack samples in traces

This CL adds support for passing function names from unwinder to
profilers. This is temporary until we have a concerete design for
it - crbug.com/1371105.
This is only used by libunwindstack_unwinder which is enabled on
browser main thread tracing profiler. The unwinder already gets
function names from libunwindstack. Here we finally start uploading
function names for frames belonging to modules other than native chrome.
We already have privacy approval for uploading function names -
crrev.com/c/3916218
