manifest: Extend display_override to accept url_patterns

Currently display_override in the web manifest accepts a list of
DisplayMode strings.

This CL extends it to additionally accept objects in the format

  { "display": "unframed", "url_patterns": [ ... ] }

The list of url_patterns can be used to select in which URLs this
display override applies.

manifest_parser.cc permits out-of-scope and out-of-origin URL patterns.
Downstream consumers of these patterns should validate and only apply
the display mode when in-scope.

For the time being only the "unframed" display mode is allowed to have
url_patterns.
