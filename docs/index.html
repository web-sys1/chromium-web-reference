zygote: raise max message length

Since features & switches need to be packed into the zygote's fork
message, the max message length constrains (indirectly) how many
features can be enabled at any one time. As we experiment with more
features and as users enable them by hand we bump up against this limit;
when we hit it chromium stops running, which is bad.

Raise it to 64KiB, and add a static_assert that it never gets raised
too far.
