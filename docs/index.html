media/gpu/chromeos/VDPipeline: Enable specifying decoder task runner

The VideoDecoderPipeline creates a new dedicated thread.
crrev.com/c/3682007 enables to specify the maximum number of decoder
threads. The experiment results (b/231110972#comment17) shows a single
thread is enough for multiple decoders.
This CL removes the code of specifying the maximum decoder threads and
adds the chrome option, --chromeos-decoder-task-runner, of specifying
TaskRunner for the decoder.
If the option value of --chromeos-decoder-task-runner is
* Sequence, then SequencedTaskRunner
* PooledThread, then SingleThreadTaskRunner (one of threads in
  ThreadPool)
* SingleThread, then SingleThreadTaskRunner of
  base::Thread("VDdecThread"), which is unique and only used for video
  decoders.
Note that the current default behavior is SingleThreadTaskRunner of a
dedicated thread newly created per VideoDecoderPipeline.
