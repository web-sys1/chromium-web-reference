Remove call to deprecated GetProfile() from BrowserReportGeneratorAndroid

This code only had the effect of force-loading all profiles from disk.
It had no other effect, since GetProfile() always returns non-OTR
profiles in this case, since System and Guest profile are never stored
in the ProfileAttributesStorage.
This was probably unintended and can be removed.

Calls to GetProfile() are problematic for several reasons:
- they may cause file I/O on the main thread which is a performance
  concern and is disallowed
- they may crash if any other code is loading the same profile at the
  same time, as it is not supported to load a profile synchronously and
  asynchronously at the same time.
