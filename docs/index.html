[CrOS Hotspot] Prevent double triggering enable/disable hotspot

Currently, both hotspot-subpage and hotspot-summary-item have toggle to
enable/disable hotspot. If we enable the toggle in hotspot-subpage,
it will also trigger the enable hotspot in hotspot-summary-item because
once we turn on hotspot in hotspot-subpage, hotspot info will be updated
and will trigger the onHotspotInfoChanged_ to set the
isHotspotToggleOn_. Updating the isHotspotToggleOn_ will trigger the
observer function and call enable hotspot again.
This CL removes the observer and rely on `on-change` function to
enable/disable hotspot.
