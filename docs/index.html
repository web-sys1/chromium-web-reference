[shelf-palm-rejection]: Add flags for testing

This CL incorporates 2 flags into ChromeOS that will allow to shift
different values for the palm rejection constants in the shelf.

Swipe Offset will add a pixel delay amount to the swipe gesture that
drags the in-app hotseat into an extended state.

Touch Area will decrease the touchable area in which the swipe gesture
gets triggered in the hotseat.

Both constants will be adjusted to reduce the accidental triggering of
the hotseat.
