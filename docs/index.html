[SBv5] Add OHTTP support in HashRealTimeService

This CL adds initial support to send real time hash lookup requests
via OHTTP. The request is issued through the GetViaObliviousHttp API
in NetworkContext. The key config is currently empty, it will be set
in a follow-up CL.

Add a feature flag kHashRealTimeOverOhttp to control this feature.
Also introduce a parameter to make the relay endpoint configurable.
The feature flag is disabled by default. The current default relay
endpoint is also empty, since we haven't determined the proxy provider
yet.
