History: More reliably populate visit_duration

A visit's duration should be computed and stored once the visit is
"finished", either because the user navigated away or because they
closed the tab.
Before this CL, the "navigated away" case was handled directly in
HistoryBackend: When a new visit was added, the referrer visit's
duration got updated.
That failed to handle any cases where there is no referrer - e.g. due
to the site's referrer policy, or because the user typed a new URL
into the Omnibox.

This CL fixes the issue by moving the visit-duration computation into
HistoryTabHelper, which anyway had to detect new navigations (and
which already handled the "tab closed" case).
