[feed] Bundle SIGNED_IN_WITHOUT_SYNC and SYNCED into single value

These were handled identically in the server, so they can be a single
value. This CL causes clients to send a new SIGNED_IN value in both
cases. The old values are deprecated - not reserved - because they are
used by the server to handle requests from old clients.
