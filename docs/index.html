ExternalVkImageBacking: Updated to ANGLE_external_objects_flags

This extension now takes the pNext chain the application used to create
the Vulkan image.  That is nullptr in Chromium.

This is used to prevent ANGLE from adding structs to the pNext chain,
which could lead to a different memory requirement for the image.  With
the update to this extension, Chromium would be able to chain structs to
VkImageCreateInfo in the future, and have ANGLE use an identical chain.
