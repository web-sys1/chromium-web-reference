Close self.pipe_out immediately after launching the test server

When launching a child process with a pipe, it is standard for the
parent process to close the child's (writing) half of the pipe. This
ensures that, if the child dies, the parent's (reading) half of the pipe
will observe EOF and be visible through select.

This fixes the bug where, if the test server fails to start,
chrome_test_server_spawner.py just hangs until the timeout.

While not the cause of issue #946475, after this we maybe can remove the
timeout. The timeout is pretty tight and causes a lot of fragility with
cold vpython caches. (I'll leave that to infra folks to decide, as they
own this script.)
