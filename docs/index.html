Start removing taskId

Remove taskId from FileTask and replace it with a FileTaskDescriptor
struct which has the three fields that make up with taskId.

Do not remove taskId from fileManagerPrivate.executeTask() yet. This
will be done in a follow-up to avoid the CL getting too big.

We need to introduce makeTaskID() to the frontend code to construct a
TaskId from a descriptor. This is partially due to the in-between state
until executeTask() and setDefaultTask() are updated, but also because
we store taskIds in local storage and send them to UMA as enum values
as well.
