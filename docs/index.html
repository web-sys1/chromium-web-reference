exo: Refactor wayland_fuzzer setup and API for use in WLCS

This has a few goals:
 - The ServerEnvironment holds the ui_thread, and we need to exercise
   ash/exo APIs on the UI thread, so we provide a function we can call
   to execute code on it.
 - The ServerEnvironment (mistakenly) performs a lot of 1-time setup in
   SetUpOnUiThread. Most of that functionality is for initializing
   globals and so shouldn't be called multiple times. We move the code
   to the ServerEnvironment's constructor to avoid re-running it every
   time SetUp() is called.
