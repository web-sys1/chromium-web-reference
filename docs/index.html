Remove ETSConnectionListener::OnResponseCompletedSuccessfully().

The EmbeddedTestServerConnectionListener method wasn't being used any
more. I believe it was once used to simulate CONNECT proxy behavior in
one or two tests, but there are better options for that now, which those
tests have likely been migrated to using.

Moreover, the test for it is disabled on ChromeOS due to flake, and is
likely flaky everywhere (the test sends an HTTP request and expects the
network stack to keep the socket open, but the request has both
"Connection: close" and "Connection: keep-alive" - Chrome likely obeys
the first one). A bit weird that this issue only got the test disabled
on ChromeOS, but not worth investigating, at this point.
