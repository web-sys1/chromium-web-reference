[Webdata] Overload ScheduleDBTaskWithResult() with callback

Before this CL, WebDataServiceBase defines (only) a function

  Handle ScheduleDBTaskWithResult(
      const base::Location& from_here,
      ReadTask task,
      WebDataServiceConsumer* consumer);

where `consumer` is effectively a cancelable OnceCallback:
- It implements an interface with a single function that's called a
  single time.
- It has a WeakPtrFactory to safely call the function asynchronously.

Therefore, this CL adds a function that expects a OnceCallback:

  WebDataServiceBase ScheduleDBTaskWithResult(
      const base::Location& from_here,
      ReadTask task,
      base::OnceCallback<void(WebDataServiceBase::Handle,
                            std::unique_ptr<WDTypedResult>>);

This should make the control flow at the caller's side more obvious.

As examples, the CL also migrates Autofill's
{Address,Entity}DataManagers to the new function.
