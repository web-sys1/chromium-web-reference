Add GTK ColorMixers to ColorPipeline P1

This CL defines a ColorProvider initialization function for the GTK
ColorMixers. Currently gtk_unittests pass but remain disabled until
theme change support has been added to Color Pipeline.

The initializer in ColorProviderManager has been updated to be a
RepeatingCallbackList of initializer callbacks. This allows
ColorProvider clients to chain additional initializers on-top of the
existing ui initializer. This mechanism can be used for supporting
//c/b/ui mixers later on.

There are also minor updates to CommonTheme that do not change output
color but update semantics to match the ui_mixer.

The GTK initializer is added after GTK setup finishes in the
NativeThemeGTK constructor (which happens during browser startup).
This ensures the GTK mixers are only added to the pipeline once
the GTK theme is ready and setup has completed successfully.

Follow up work will add ThemeChange support to ColorPipeline and
clean up the GTK color definitions.
