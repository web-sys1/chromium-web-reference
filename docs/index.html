Implement PartitionedCookies HTTP Origin Trial

The Origin Trial for HTTP partitioned cookies will use a client hint,
Sec-CH-Partitioned-Cookies to indicate to sites that they are opted
into the Origin-Trial correctly.

A site opts into the Origin Trial by sending the Origin-Trial header
with a valid token and an Accept-CH: Sec-CH-Partitioned-Cookies header.

If Chrome detects the site has successfully opted into the OT, on the
next request it will send Sec-CH-Partitioned-Cookies: ?1 to indicate
to the site it is in the OT.

If Chrome detects the existence of partitioned cookies but it has not
verified that the site is in the OT, it will send the
Sec-CH-Partitioned-Cookies: ?0 header (added in the network service,
because that is where cookies are stored).

When a site opts-out of the OT, we convert their partitioned cookies
to unpartitioned. This may cause some cookies with duplicate
name/value/path's, so we coalesce duplicates using the following
algorithm:

1. If an unpartitioned cookie exists with a name/value/path, select
   this cookie.

2. Select the partitioned cookie with the most recent LastAccessDate.

This algorithm is implemented in
CookieManager::ConvertPartitionedCookiesToUnpartitioned.
