Reland "Prerender2: verify chrome.declarativeNetRequest behaviors"

This is a reland of commit b37e7fc26893ab35c42df9f744a238dc910b4d8a

The test failed on mac because the first network request reaches
to the embedded server even though the URL is registered as a
static rule for blocking. This is because the test starts running
before the static rules are applied. So, the tests are fixed to use
dynamic rules to workaround the setup race issue.

Original change's description:
> Prerender2: verify chrome.declarativeNetRequest behaviors
>
> chrome.declarativeNetRequest API is used to monitor all network
> requests regardless of the request initiator's page state.
> So, we just extend this capability to pre-rendered pages.
>
> This patch verify if rules to block requests work for prerendering,
> or network requests from pre-rendered pages.
>
> Also we should clear the pending navigation information
> in the ActionTracker also on finishing non primary page navigations.
>
> Relevant explainer in the design doc:
> https://docs.google.com/document/d/16Sy2Xq-AcnhwnpC0q232OMSmxTpQYDANRAO-zJ8DKiA/edit#bookmark=id.k18lj9xlkzfl
