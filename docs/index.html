Allow both client-side error and epitaph in context provider test

This is part of an LSC for Fuchsia FTP-028.

In https://fuchsia-review.googlesource.com/c/fuchsia/+/471000,
wrong object type errors are detected client-side in HLCPP and
reported through a callback function on the client
(ZX_ERR_WRONG_TYPE).

This behavior conflicts with the existing test which expects
the error to be caught at a later point - on the server side
and reported via an epitaph on a different channel (context).

This CL updates the test to support BOTH cases to allow
migration to the new behavior. After the migration takes place,
the test will be updated to disallow the old behavior.
