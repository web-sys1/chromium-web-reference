graphite: Do not use implicit MSAA in XRDrawingBuffer

WebGL cannot use implicit MSAA resolve when Skia Graphite is enabled
because Skia is not using ANGLE and thus ANGLE cannot do an implicit
resolve when Skia is sampling the WebGL back buffer. The regular WebGL
DrawingBuffer path was already checking for this and using explicit
resolve, but WebXR has its own XRDrawingBuffer that wasn't checking for
Graphite. This manifested as a D3D11 debug layer error because the color
buffer was a single-sampled D3DImageBacking while the depth-stencil
attachment was multi-sampled.
