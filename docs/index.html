[Focusgroup] Fix Focusgroup parser with ShadowDOM

In this CL, we fix a known issue with the parser of the focusgroup
attribute in ShadowDOMs. The bug was caused by the fact that the nodes
in the shadow tree is parsed *before* the parent node that connects it
to the main tree.

When parsing the focusgroup attribute for a specific element, we expect that the ancestor elements have already parsed their own focusgroup
attribute, so extending a focusgroup when in a ShadowDOM didn't work if
the nearest focusgroup ancestor was declared outside of the ShadowDOM.

We fix this bug by re-running the parser for all nodes within a shadow
tree when we insert the shadow host into the main tree. To avoid running
this every time we insert a shadow host in the main tree, we added a new
bitfield on the ShadowRoot that keeps track of whether one or more node
in the shadow tree has the focusgroup attribute set. We then only re-run
the parser on the shadow trees when we know that there is at least one
focusgroup contained within that tree.

KNOWN ISSUE: This CL doesn't fix the parser when shadow doms are built with nodes that are assigned to <slot> elements. I'll fix this bug in a separate CL. See https://crbug.com/1327752 for more info.
