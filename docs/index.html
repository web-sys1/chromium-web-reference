[FilesTrash] Exclude trashed files from the tote model

Prior to the trash functionality files were permanently deleted, this is
now moving them to the trash. The trash is a hidden folder on the users
cryptohome so when files are "trashed" they're technically still backed
by a real location. To avoid having users confused with this new
behaviour (files still in tote despite being trashed) have them removed
from the holding space model if files are parented at an enabled trash
location.
