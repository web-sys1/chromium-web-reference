chromeos: Consolidate bubble closing event filter logic

Introduce bubble_utils::ShouldCloseBubbleForEvent() to hold the logic
that ignores certain click and tap events. Events in capture mode and
events that hit menus, settings bubbles, and the virtual keyboard are
ignored.

Use the utility in the existing app list event filter as well as the
new event filter for AppListBubble.

This is a utility function, and not a base class, because there are
differences in which events are tracked by the various event filters.
For example, the system tray bubble closes immediately on touch
press. However, the peeking launcher waits for a full tap, so that
the launcher can stay open during swipe gestures on the shelf.

Introduce capture_mode_util::IsCaptureModeActive() to encapsulate
both the check for the feature flag and whether the feature is
active. This avoids including capture_mode_controller.h, which
is large due to transitive mojo includes.
