[fuchsia] Automatically set attribution tag for process logs.

ScopedFxLogger::CreateForProcess() now prepends an attribution tag,
based on the calling process command-line's "program" argument, to any
caller-supplied tags, to ensure useful attribution of log output.

This has two benefits:
1. It avoids relying on the fuchsia.logger.LogSink implementation being
   able to correctly attribute logs to the Component, via the special
   COMPONENT_NAME tag, which is currently broken in some common cases.
2. It avoids relying on the fuchsia.logger.LogSink implementation
   implicitly adding an attribution tag to messages logged with no tags.

and addresses two known issues with COMPONENT_NAME:
- "run-test-component" causes the test Component to be mis-attributed
  as "UNKNOWN".
- WebInstanceHost creates a web_instance.cmx that receives LogSink from
  the calling component, and is therefore mis-attributed.

ScopedFxLogger is also updated to support callers supplying more than
one tag, and to provide a convenience default for the zero-tags case.
