[clang] Fix the pinned clang version used for bootstrapping

Using the version from HEAD~ was elegant but didn't work when run
outside a Git repository or when HEAD~ contained changes to the clang
version.

Just using a fixed version which we can bump every once in a while is
less elegant, but it avoids these problems and is also a little simpler.
