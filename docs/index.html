MacDesktopCaptureV2: Move CGDisplayStream handler to main thread

When capture is initiated by the screen capture API, the
DesktopCaptureDeviceMac is created on the main thread, and the use
of CFRunLoopGetCurrent is fine.

When capture is streaming to ChromeCast via 3-dot > Cast, it is a
worker thread that doesn't use a CFRunLoop. Consequently, the
CGDisplayStream's handler is never called.

The fix here moves the handler to always be on the main thread, and
has it post a task to the device thread when it receives a new
IOSurface.
