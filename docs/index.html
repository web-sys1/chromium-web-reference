Revert "heap: Allow using non-tracable T in HeapVector<T>"

This reverts commit cd8b529b0e2630213d900e5d504fc921a15499ea.

Reason for revert: Potential cause of ASAN failures in crbug.com/1292392

Original change's description:
> heap: Allow using non-tracable T in HeapVector<T>
>
> This CL allows non-tracable T in HeapVector<T> enabling HeapVector<T>
> usages instead of Vector<T> from GarbageCollected objects. The
> benefits are:
> a.  Accounting used memory is more accurate for GC scheduling
>     (initial bug).
> b.  Backings can be compacted, resulting in better memory usage.
> c.  Uses like HeapVector<char> avoid introducing destructors on the
>     host object.
>
> Note that this does not lift any other requirements for T. E.g.,
> kCanClearUnusedSlotsWithMemset is still necessary which prevents
> certain types from being used in HeapVector.
>
