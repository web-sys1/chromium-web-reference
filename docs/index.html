Tweak GetOutermostWebContents usage in PrintIfAllowedByPolicy

The fix in crrev.com/c/4651379 works for LCAC as all URLs are sent for
scanning, but that might not be the case when cloud scanning uses this
code. When that happens, the WebContents used by
ContentAnalysisDelegate::IsEnabled also need to be the outermost one,
otherwise the URL comparison to check if a scan should happen might be
against the PDF reader's URL (aka chrome-extension://<some_id>), or
any other embedded page.

This is fixed by simply moving the GetOutermostWebContents earlier in
PrintIfAllowedByPolicy instead of later in ScanAndPrint.
