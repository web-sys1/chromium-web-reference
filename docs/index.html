[css-nesting] Just use ConsumeDeclarationList for nested grouping rules

In CSSParserImpl::ConsumeRuleListOrNestedDeclarationList, we have some
code which inserts a wrapper rule for any leading declarations found.
However, the observer event flow is not correct when the runtime flag
CSSNestedDeclarations is enabled: it would suggest that anything
parsed by ConsumeDeclarationList is *inside* the wrapper rule
(see CL:5806629 for an example).

To fix this, we can instead use the "native"  CSSNestedDeclarations-
emitting capabilities of ConsumeDeclarationList, but modified to also
(optionally) include the leading declarations, via
the nested_declarations_start_index parameter.

This brings declaration debugging for nested grouping rules up to
the same level as for CSSNestedDeclarations appearing within normal
style rules.
