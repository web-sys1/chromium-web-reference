[webauthn] Use CredMan for conditional GetAssertion requests

This CL implements the skeleton for conditional GetAssertion requests.

It currently sends an empty list of |discoverableCredentials| to
WebAuthnBrowserBridge. The API does not support retrieving passkey
metadata without showing the bottom sheet.

Followup CLs will wire `hasPublicKeyCredentials`,`hasPublicKeyCredentials`,`hasRemoteResults`
and `pendingGetCredentialHandle` to user input focus.
