Use ThreadType::kCompositor for the video compositor thread.

Video frame compositor was running on a Thread pool thread thread with
the default thread priority (i.e. ThreadType::kDefault). That make's it
impossible to prioritize the thread to ensure that it can be scheduled
to submit each video frame on time. Updated RenderThreadImpl to create a
thread outside of the ThreadPool, which allow to set thread type to
kCompositor.
