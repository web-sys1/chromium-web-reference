Remove duplicate list of allowed DevTools metrics

It appears that originally the checks where written
so that each call to UMA has a unique line. Later it
was refactored because the UMA didn't require unique call
lines for reporting anymore but the allow list was kept.

This CL replaces the exact list with a startWith check
enforcing the convention that all DevTools metrics start
with "DevTools.". This allows adding new histograms without
the need to make an extra backend change in the bindings file
similar to RecordPerformanceHistogram and RecordUserMetricsAction
methods. Perhaps, even the startsWith check can be omitted but I
am not sure if there are any negative security implications. It looks
like UmaHistogramExactLinear should handle unknown names gracefully.
