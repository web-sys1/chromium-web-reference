Reland "Offload synthetic response cloning to background thread"

This is a reland of commit c1046a2dd535f6c7571ec4a3456fca1a3e6344d4

This addresses the user-after-free in
`RaceNetworkRequestSimpleBufferManager` by extending the object deletion
via `DeleteSoon()`.

As written in TODO, using `RefCountedThreadSafe` is probably a more
robust solution, but let me keep the diff minimum since we'll soon
replace `RaceNetworkRequestSimpleBufferManager`. We'll update it to use
`RefCountedThreadSafe` after landing crrev.com/c/7459409.

Original change's description:
> Offload synthetic response cloning to background thread
>
> Update ServiceWorkerSyntheticResponseManager to perform data pipe
> cloning on a background thread using base::ThreadPool. Previously, the
> RaceNetworkRequestSimpleBufferManager was a class member, and cloning
> occurred on the current default task runner.
>
> This change introduces static helper methods to handle the cloning
> process independently of the manager's lifecycle. It uses
> base::BindPostTaskToCurrentDefault to ensure the completion callback
> returns to the original sequence safely. Offloading this work prevents
> potential blocking on the main thread during synthetic response
> processing.
>
