Optimize SubstringSetMatcher [patch 5/5, SSE2 GetEdge()]

SSE2-optimize the search for edges when they are not stored inline
(which usually means there are many of them), allowing us to do
four at a time.

The fact that the microbenchmarks below go slightly negative may be a
bit confusing. However, other tests we've done are clearly positive;
e.g. repeated lookups on a different dataset (the perftest does only
a single lookup, using a very long string) show 27% faster search
performance.

Memory usage is unchanged.

SubstringSetMatcher.init_time:      45214 -> 48497 us (-6.8% perf)
SubstringSetMatcher.match_time:       128 ->   135 us (-5.2% perf)
