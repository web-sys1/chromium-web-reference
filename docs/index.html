testing: Remove BrowserWidgetFocusSupplier

This supplier observes browser window activation. It is no longer needed
after https://crrev.com/c/6693572, where the default supplier
(NativeViewWidgetFocusSupplier) becomes capable of observing the
activation of *any* widget, including browser window's widgets.

Also fixes views::WidgetTest::GetAllWidgets() so that it actually
returns all widgets. This was a limitation that would block the removal
of BrowserWidgetFocusSupplier.

Previously, GetAllWidgets() could not be used in ChromeOS browser tests
(it would trigger a CHECK). The utility relies on `AuraTestHelper`,
which is not available in that test environment. Getting a list of all
root windows on ChromeOS in browser tests requires calling
ash::Shell::GetAllRootWindows(), but //ui/views cannot depend on //ash,
otherwise it forms a circular dependency as //ash already depends on
//ui/views.

This CL solves the layering difficultly by injecting
ash::Shell::GetAllRootWindows() as a callback in WidgetTest via a new
static method, WidgetTest::SetRootWindowProvider(). The callback is
registered during browser test set up time at a higher layer
(InProcessBrowserTest::SetUp() in `//chrome`). This makes
`GetAllWidgets()` universally usable.
