Some cleanups and fixes for TransferableSocket.

This CL cleans up and fixes some issues with TransferableSocket.

These were discovered during development of future CLs that
contain the the rest of the socket broker code on Windows.

1. Constructor needed to pass process handle on platforms that didn't
   need to supply one, so this was made optional on those platforms.
2. Constructor was taking a const ref to net::SocketDescriptor when
   the standard in net code is to take just net::SocketDescriptor.
3. On Windows, there was a handle leak in the code that sent the
   socket. This was because the socket was not being correctly
   closed after duplicated.
4. As the constructor took a base::ProcessHandle it was possible
   that the base::Process might have gone out of scope before
   the call was made to duplicate the socket. To avoid this,
   the constructor now takes a fully scoped base::Process.
5. When passing a net::kInvalidHandle, the other side would
   mistakenly get a null handle rather than an invalid handle.
6. Some unneeded methods and fields were removed from the internals
   of the TransferableSocket, that only served a purpose to check
   correct usage.
7. Because some fields were removed, DCHECK-only code was added to
   verify that the TransferableSocket was only passed once across
   the process boundary, and no attempt was made to pass it back
   again, or try and Take the socket before it had been passed.
8. Tests for all of the above were added.
9. Comments were added to the public .h definitions.
