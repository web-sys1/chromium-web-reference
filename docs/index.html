Use MUST_USE_FOREGROUND on the GpuPersistentCache task runner.

The GpuPersistentCache uses a task runner to do background writes to the
database file. The priority on the runner was too low which causes the
writes to be interrupted while holding the database lock. On Windows
this would also set the IO priority to low.

Set MUST_USE_FOREGROUND to ensure that once the task starts running, it
is unlikely to be interrupted.
