Factor out common single-tile draw optimization for BoxPainter

Both DrawTiledBackground() and PaintFastBottomLayer() contain highly
similar code for determining if a background image can be painted as a
single tile (with a simple DrawImage all).

This keeps the (sometimes quirky) logic consistent between the two code-
paths. For the existing (minor) differences in behavior, go with the
more conservative version for the new function.

The no-intrinsic-size case is simplified quite a bit by recognizing that
the tile scale will always be one.
