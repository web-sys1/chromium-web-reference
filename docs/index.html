Add metric for the time spent updating paint following throttling

This patch adds a UMA metric for the time spent updating paint following
a frame becoming throttled:
  Blink.RenderThrottling.PaintUpdateTime

This metric will provide two datapoints: how expensive the paint update
is (value), and how often it occurs (count). This will only measure one
aspect of the full benefit of crbug.com/1325886. For example, this will
not measure compositor-thread costs following clearing of paint.
