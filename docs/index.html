Always replace on iframe synchronous about:blank commit

The synchronous about:blank commit should only happen when the frame is
currently showing the initial empty document. Since all navigations
that happen on the initial empty document should result in replacement,
this CL makes sure all synchronous about:blank commit that happens
at frame creation in HTMLFrameOwnerElement::LoadOrRedirectSubframe()
will use kReplaceCurrentItem. (Previously, the frame load type can be
set to kReloadBypassingCache when the frame is reloaded, setting the
DidCommitParams' should_replace_current_entry to false, and causing a
DwC hit in VerifyDidCommitParams).
