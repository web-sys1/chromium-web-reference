[Affiliations] Parse response body only if request succeeded

Response body was parsed before checking the request's status. If
request had failed but contained non-empty body it would result into
malformed response. This CL fixes this.

Also, response code and errors are recorded for
all requests. Since semantics changed new histogram names are used.
