Add method to translate animation within AnimatedImageView.

The use case is for ChromeOS, where it was recommended to apply very
tiny jitter (not user perceptible) to the whole animation to prevent
any chance of screen burn. Translating the entire view does not work
well in our case since the view is full-screen, and it leaves a black
border where the view got translated by a few pixels. We can instead
scale the animation a tiny bit larger and translate the canvas when
rendering. This has the same effect, but does not move the view's
entire content bounds.

This CL adds a the method to AnimatedImageView to allow this to
happen, and it will be used in the next CL.
