Query a11y tree including ignored nodes

crrev.com/c/2505362 changed the logic for queryAXTree
command to exclude ignored nodes. That was probably done
by accident given that the original intention was
to return ignored nodes as well as documented on the CDP
level: https://chromedevtools.github.io/devtools-protocol/tot/Accessibility/#method-queryAXTree

We discovered the issue while working on the recorder: it's possible
to compute an accessible name for ignored nodes but it's not possible
to find the node later. Therefore, this CL reverts queryAXTree to the
original behaviour.
