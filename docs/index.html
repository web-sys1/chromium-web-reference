[Indigo] Implement prototype page heuristic classification and config loading

This CL implements the prototype for the Indigo page heuristic classification
and the asynchronous loading of its configuration.

- Blink:
  - Implemented DocumentMetadataServer::ClassifyProductDetails to extract
    Product and ProductGroup entities and perform case-insensitive
    word-boundary matching using ICU WordBreakIterator.
  - Optimized lookup by using a single HashMap with ReserveCapacityForSize.
  - Added unit tests in document_metadata_extractor_test.cc covering
    Unicode case folding and punctuation handling.

- Browser:
  - Defined IndigoConfig proto for configuration.
  - Implemented IndigoService to asynchronously load the configuration from
    the component directory.
  - Added --indigo-config-proto command-line switch to override the config
    path for manual testing.
  - Updated IndigoPageActionController to trigger classification on
    same-document (SPA) navigation after a 3-second delay.
  - Added unit tests in indigo_service_unittest.cc and updated
    indigo_page_action_controller_unittest.cc.
