[TabItemPicker] Fix background thumbnail fetch by using translucent theme

Problem: When ChromeItemPickerActivity is launched over
ChromeTabbedActivity, fetching thumbnails for background tabs fails
because the GPU is unable to execute a `CopyFromSurface` operation.

Root Cause: ChromeItemPickerActivity was using an opaque activity theme.
When an opaque activity covers the screen, the Android WindowManager
transitions the underlying activity (ChromeTabbedActivity) to the
STOPPED state. When ChromeTabbedActivity is STOPPED,
CompositorViewHolder formally destroys the CompositorView's surface to
free memory. Without an active GPU surface, any WebContents tied to that
WindowAndroid cannot render frames, causing the thumbnail fetch to fail.

Solution: Changed the theme of ChromeItemPickerActivity in
AndroidManifest.xml to `@style/Theme.Chromium.Activity.FakeTranslucent`.

Because the Activity is now visually translucent to the OS
WindowManager, ChromeTabbedActivity is transitioned to the PAUSED state
instead of STOPPED. In the PAUSED state, CompositorViewHolder
intentionally keeps the compositor surface alive in memory. This
preserves the GPU context and allows background tabs to successfully
fulfill the TabContentManager's thumbnail fetch requests.

Note: The picker UI itself remains completely opaque to the user because
a solid background color is explicitly set on its root view in Java.
