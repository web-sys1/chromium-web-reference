[ios] Create Create MagicStackCollectionViewCell

MVTs use MagicStackModuleContainer, which is a UICollectionViewCell
subclass, and it seems like the Discover UICollectionView has
awareness of it as a cell (see c#14 for logs that lead to this
conclusion. This is the reasoning behind the unexpected behavior seen in
the bug.

To fix this, MagicStackModuleCollectionViewCell is created as the
cell used in the Magic Stack, while MagicStackModuleContainer will be
used by MVTs and as the primary subview for
MagicStackModuleCollectionViewCell. Most of the code is just moving
necessary scoped logic (i.e. long-press context menu) to the cell
class and forwarding calls from the cell to the container. The
only changes are in MagicStackModuleCollectionViewCell's initWithFrame:,
where no longer sets self.contentView's corner radius and clipToBounds
since it does not seem to be necessary for UI consistency.
