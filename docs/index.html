Remove GPU pixel test grace period

Removes the GPU pixel test grace period functionality, as it is
infrequently used, causes issues, and can be largely replaced with
normal test expectations. Issues include but aren't limited to:

1. Unexpectedly breaking builders, including branched builders, if a
   grace period expires
2. Incompatibility with tests that use inexact matching due to the
   inclusion of the `ignore` tag, which affects the trace ID and makes
   Gold think that the image is coming from a different config.

The one group of tests that currently has a grace period (MeetEffects)
instead has Failure expectations added.
