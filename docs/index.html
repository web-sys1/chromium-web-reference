Relax expectation about nested script parse duration

Sometimes, both the outer and nested script parse durations are reported
as 0 ms. This might be related to the timestamp resolution loss
happening in the conversions made by PerformanceTimingForReporting and
WebPerformanceMetricsForReporting. In any case, strict inequality
doesn't seem required here (PageLoadMetricsUpdateDispatcher only bales
out if the nested script parse duration is srictly greater than that of
the outer script).
