[AtMemory] Only add non-empty credit card attributes to metadata

Previously, FetchCreditCardData in AutofillDataProvider added card
number and security code (CVC) entries to the search result's metadata
list without checking if they were empty. This resulted in empty card
number entries or dummy CVC placeholders (•••) being attached as
metadata for cards where CVC or card numbers were not saved. This CL
updates FetchCreditCardData to verify that the obfuscated card number
and CVC are non-empty before adding them to the metadata list, and adds
unit test coverage for empty field omission.
