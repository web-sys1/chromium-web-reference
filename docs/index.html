Allow launcher scrollbar thumb drags outside the 100 pixel "snap" margin

Views scrollbars by default "snap back" to the original drag position
when the mouse moves more than 100 dips outside the thumb. Most views
UI elements are small, so the 100 dips margin feels good.

Web content scrollbars do not have this behavior. The user can drag
the mouse anywhere outside the thumb and the scrollbar continues to
scroll. This also feels natural, perhaps because web content windows
are large.

The productivity launcher is quite large (~1/2 of the display) but is
built with Views. Plumb a boolean into the scrollbar thumb to allow
disabling the "snap back" behavior. This feels more natural for this
larger element.
