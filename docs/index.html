Make GetActiveControlLayoutBounds work with Iframes correctly.

The current implementation of GetActiveControlLayoutBounds fails to take
into account iframes, leading to incorrect results on ChromeOS.  The
basic issue was that the iframe offset from the root window was not
taken into account, so the returned bounds were shifted up and to the
left by the iframe bounds.

Construct 2 new methods which call TransformPointToRootCoordSpace which
is the approach used by GetCompositionCharacterBounds which serves a
similar purpose (see usage here:
https://source.chromium.org/chromium/chromium/src/+/main:content/browser/renderer_host/text_input_manager.cc;drc=905c8b3accbc33a648e6ab44e55a475307b9e7c4;l=350
)
