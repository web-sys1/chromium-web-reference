MPArch: Re-enable SendEngagementLevelToRenderFrameOnFencedFrame test

Mac10.14 Tests builder reported a failure on this test
because AddAutoplayFlags method was not called during the
test. It looks like we need to wait until the interceptor
calls AddAutoplayFlags method because calling the interceptor
(MockAutoplayConfigurationClient) is not synchronous.

So this CL adds base::RunLoop().RunUntilIdle() at the end
of the test like the prerendering test.
