[PartitionAlloc] Don't double-check pointer ownership in free() on mac.

On Apple OSes, free() in libsystem_malloc.dylib first queries the size,
then calls free(). But the shim layer checks that the pointer is owned
as well, leading to checking twice (once for the size query, once in the
shim's free()). This is wasteful, so don't do it.

Note that this is unlikely to be too visible on base_perftests, since
they run without the thread cache, hiding a lot of this overhead.
