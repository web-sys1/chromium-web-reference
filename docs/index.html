[Modular] Modularize payments junit tests

Motivations:
* GooglePay unit tests which will depend on PaymentRequestParamsBuilder
  will not need to depend on the monolithic chrome_junit_tests target
  to use the test supports.
* The new targets will have clearer dependencies.

Summary:
This CL modularizes payment java unit tests into two targets:
* //chrome/browser/payments/android:junit,
* //chrome/browser/payments/android:junit_test_support,

Changes:
* Moved all payment unit tests and supports from
  "chrome/android/junit/src/org/chromium/chrome/browser/payments/" to
  "chrome/browser/payments/android/java/src/org/chromium/chrome/
  browser/payments/" (and its test_support/).
