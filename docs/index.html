Differentiate backend choice for variable COLRv0 fonts

For variable COLRv0 fonts, send them to DirectWrite if it supports
COLRv0 and variations, otherwise choose the FreeType backend, as on Mac
we do not have variations support for COLRv0 from CoreText, through the
SkTypeface CoreText backend.

For now manually tested on Windows and Mac using
https://v-fonts.com/fonts/gimme-constructo-variable as finding out which
parts can be tested using layout tests or WPT needs more investigation
due to GPU accelerated code path being involved in Skia.
