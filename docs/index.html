[CrOS Settings] Remove .m extension from ensure_lazy_loaded.m.js

All Chrome OS Settings pages are using JS imports and not HTML
imports for ensure_lazy_loaded.m.js. The '.m' part should only be used
for auto-generated JS modules, to differentiate from the pre-JS module
file. Once a JS module is checked in the repository and no longer
auto-generated the '.m' part should be removed.

Similar to https://chromium-review.googlesource.com/c/chromium/src/+/3489749
