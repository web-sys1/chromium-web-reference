[Connection-Allowlist] Don't bind MdnsResponder if WebRTC is blocked.

The MdnsResponder interface is used by WebRTC to hide local IPs via
mDNS. WebRTC can already be blocked by the Connection Allowlist header,
at which point we prevent the P2PSocketManager interface from being
bound in the network service. If WebRTC is blocked, we can just give
the MdnsResponder interface the same treatment. This prevents any
resulting mDNS multicast responses from being sent.

There isn't really a good way to write tests for this, because
construction of RTCPeerConnections is already blocked in the
renderer.
