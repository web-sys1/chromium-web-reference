Route CrOS 1P experimental input methods to System PK.

These used to be set up using legacy Extension-based physical
keyboard (PK) code path when Latin System PK was work-in-progress.

Now that System PK has graduated, its flag removed and Latin
Extension-based PK is being dismantled (https://crrev.com/c/3267728),
these experimental input methods got broken and should be migrated
to use System PK instead.

NOTE: The above is only for the physical keyboard. Virtual keyboard
(VK) still goes through the CrOS 1P VK+IME extension for these
experimental input methods.
