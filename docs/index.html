[sync] Leverage bookmark parent GUID for initial sync

BookmarkModelMerger implements the initial sync procedure, where the
local bookmark tree needs to be reconciled with a remote equivalent,
needed primarily when the user enables sync for the first time.

Just like https://crrev.com/c/3327034 did recently for incremental sync
updates, this patch does the analogous for initial sync: it migrates
BookmarkModelMerger to stop relying on |SyncEntity.parent_id| and
instead constructs the remote tree based on the more modern
|BookmarkSpecifics.parent_guid|.

Again, the resulting behavioral difference should be negligible,
because the new field should be available even for legacy data, as per
logic introduced in https://crrev.com/c/3326621.

One minor behavior difference is about UMA, which no longer can detect,
without additional complexity, a rare edge case among corruption states:
bookmark nodes that hang directly from the root node (parent of
permanent folders), without involving a permanent folder. Hence, this
UMA bucket is marked as deprecated, since it is no longer recorded.
