Revert EmulationHandler update functions to early return

When Electron force swap site instances, our debugger class
(a subclass of content::DevToolsAgentHostClient) isn't aware of those
changes unless we hook RenderFrameHostChanged. If we don't do this,
some navigation lifecycle events {loadingFinished, dataReceived}
emitted by the renderer are lost. We disconnect and reconnect the
webcontents to the DevToolsAgentHost to prevent this from happening.

As of https://chromium-review.googlesource.com/c/chromium/src/+/3758294
this results in a DCHECK, since DevToolsAgentHost::DisconnectWebContents
results in a call to UpdateDeviceEmulationState and host_ is nullptr.

To fix this, this CL reverts those state update calls to early returns.
