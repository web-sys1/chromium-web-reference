[SuperSize] Viewer: Fix 3 UI bugs related to keyboard navigation.

The symbol tree view admits keyboard navigation, during which one item
gains "keyboard focused" status shown by a black outline. This CL
provides simple fixes to 3 UI bugs:
1. Item hover highlight can obscure black outline of keyboard focused
   neighbor: Use transparent color instead of solid color for highlight.
2. Collapsing and re-expanding group ends keyboard navigation state if
   the keyboard focused item is a descendant: Move highlight to a
   group if it has a keyboard focused descendant.
3. Cannot return to system tree view via Arrow keys after using Down
   Arrow to navigate to "Metadata": Exclude "Metadata" from items that
   can be reached via Arrow keys from symbol tree view.

Details:
* Fix for (1): Change CSS of .node:hover.
* Fix for (2): Handled in TreeUI._toggleTreeElement(), using
  |groups.contains()| for descendant test.
* Fix for (3): TreeUI CTOR: Use |this._symbolTree| instead of
  |document| to assign |this._liveNodeList| for .node elements.
* Also rename ids and variables for symbol tree view templates, and fix
  misc. comments and orderings.
