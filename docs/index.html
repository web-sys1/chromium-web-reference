[Blink] Fix CanvasResourceProviderSI SCANOUT usage on Windows

SCANOUT usage is not generally supported by D3DImageBacking. However, it
is supported when the swapchain is used to implement concurrent
read/write [1]. On the client side, we should preserve SCANOUT usage in
CanvasResourceProviderSI *only* when the swapchain will actually be used
to implement concurrent read/write (not just when the swapchain is
supported).

[1] https://source.chromium.org/chromium/chromium/src/+/main:gpu/command_buffer/service/shared_image/d3d_image_backing_factory.cc;l=930-942;drc=0c3eb92c416a23d3a55a33a22cc2a65073105063
