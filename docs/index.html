cros_usb_detector: allow HID to pass through

At some point in the past, a decision was made that allowing USB HID
devices was considered dangerous, and so they were blocked from
appearing in the system UIs in ash. However, it seems like the ultimate
arbiter of this decision should be permission_broker, and not to split
policy across planes. Devices that permission_broker has deemed to be
safe, even if HID, should be allowed in the system UI (since someone
could use `vmc usb-attach` to pass the device through anyhow).

There are additional questions regarding if permission_broker itself
can be more clever as to what devices it decides should be allowable,
but that is outside the scope of this CL and will be addressed in
subsequent work.
