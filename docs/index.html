Reintroducing Error Checking in ReadingListDistiller

In a previous CL, ReadingListDistiller's HandleGoogleCachedAMPPage() was
refactored to use the new WebFrame::ExecuteJavaScript(). At the time,
there was no way of checking if the executed arbitrary JavaScript
generated an error, which was included in the original implementation.
With the extension of ExecuteJavaScript() to support callbacks with an
error parameter, this CL restores the HandleGoogleCachedAMPPage() to its
original implementation while maintaining the use of the new WebFrame::
ExecuteJavaScript().
