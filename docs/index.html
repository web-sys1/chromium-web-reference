[Re-Factor Client Hints] (3) Consolidate client hint name lists

Right now there's a map of names in
services/network/public/cpp/client_hints.cc and
third_party/blink/common/client_hints/client_hints.cc that mirror each
other. Let's consolidate them to reduce redundancy. The actual enum
they're using lives in services/network/public, so its client_hints.cc
list name should be the one true one.

This is a part of a series of CLs to re-factor the client hints code:
(1) Label hints with legacy names
(2) Fix the naming of (unlaunched) `lang` client hint
(3) Consolidate client hint name lists
(4) Migrate from array to map indexing for client hint names
(5) Remove (unlaunched) `lang` client hint
