[Clarity] Defer bubble widget destruction on deactivation close

When the anchored message bubble widget is closed on deactivation,
destroying the widget synchronously inside macOS
NativeWidgetMac::OnWindowKeyStatusChanged() causes a Use-After-Free
crash during native post-processing frames.

This CL resolves the crash by deferring the widget's destruction in
PageActionView::OnAnchoredMessageWidgetClose() using PostTask bound to
a WeakPtr and performing instance identity checks. This allows native
stack frames to unwind first, while ensuring that early parent
destruction or quick-succession bubble recreations do not cause UAF
crashes or accidental deletion of newer bubbles.
