ime: Fix crash bug when selecting PK candidates.

The extension based candidates window system uses an index <-> ID
mapping. When the extension wants to show a bunch of candidates, every
candidate must be given a unique ID. This ID is used to identify the
candidate later on when the candidate is selected.

The native mojo PK candidates window system does not use IDs. So when
we select a candidate using mojo, Chrome crashes because it cannot
find the ID of the candidate.

The logic for converting index <-> ID is done in the parent class of
the native mojo PK code, so we will override that behaviour to directly
use the index as the ID instead.
