printing: Log CUPS job state transitions

cupsd produces logs about the progress of print jobs, but there isn't a
way to match these up to Chrome's idea of job status. To help with that,
log failures in the creation of print jobs. For created jobs, log the
sequence of status changes as a job is processed.

Some changes in the job state do affect how the job is being processed,
but don't show up as a change in the actual job-state field, e.g.
changes in job-state-reasons. To track these without logging a message
every time the state is polled, cache the last status message for each
active job and only log when it changes.
