Reduce the size overhead for permission element translations.

This improves binary size by ~96KB on arm32 and ~472KB on arm64. The
original code incurred a large binary size cost due to multiple
redundant map lookups and insertions. But since the contents of the map
are fixed and cannot change at runtime, the entire mapping can be a
constexpr data table backed by FixedFlatMap instead.

The updated generator script also implements a small optimization to
reduce the number of relocations required for embedding strings in
constexpr data tables. There are multiple (language, base message)
mappings for each language: embedding the string directly means each
pair requires a relocation. Instead, the language strings are stored
once in a FixedFlatSet, using the position in the set as an implicit ID,
and the actual translation mappings use the implicit language ID + base
message as the key instead. This reduces the number of relocations by
10x (from 800+ to 81), though this is not free, since the translation
code needs to perform 2 lookups at runtime instead.
