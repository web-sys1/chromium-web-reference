Shimless: remove setupSuite from browser tests

setupSuite() is run once at the beginning of each test suite,
creating objects like the fake Shimless RMA Service. If a
method from such object is replaced by one of the tests, it
will persist between tests and potentially affect other tests
in the suite. To mitigate that, this CL moves the initialization
of those objects to the setup() function, which is run at the
beginning of each individual test.
