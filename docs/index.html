webaudio: Fix exception handling of decodeAudioData

This patch fixes the following two points.

1. DOMArrayBuffer::Transfer may throw a TypeError, but the spec
requires throwing a DOMException of DataCloneError.

2. Possible double-throw.

This is a follow-up of https://crrev.com/c/4048602 , which made
it possible that DOMArrayBuffer::Transfer throws a TypeError.
