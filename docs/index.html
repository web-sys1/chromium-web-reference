[iOS][finch-first-run] Adds iOS variations seed fetcher

This CL implements an almost native Objective-C version of variations
service, in the way that it could execute independent of Chromium
initialization. This way, when the user runs Chrome on iOS the first
time after installation, we could retrieve the seed before Chrome iOS
starts Chromium, and have it ready to be used to establish field trials
when variations service starts.

- IOSChromeFirstRunVariationsSeedFetcher initiates the HTTP(s)
transaction to get the seed.
- IOSChromeFirstRunSeedResponseManager stores the seed returned from the
variations server, so that it is exposed to variations service when it
needs to be used to establish field trials.

Since the code should only execute on first runs, logic related to
updating existing seed is removed.

Follow up CLs for importing the seed response to variations services and
test cases will be created.
