Ensure ServiceWorkerSettings will be used when set

Instantiate the io and background thread clients, as well as the
AwServiceWorkerSettings in the constructor and send references to the
native layer, to ensure that any settings set by the user on the
AwServiceWorkerSettings object will be picked up by native code.

Previously, the user had to set a ServiceWorkerClient before the binding
to native code was performed, which meant that it was possible to set
settings that would get ignored by the implementation.

Adding lock around access to the ServiceWorkerClient, as it may be
accessed from multiple threads at the same time.

Adding test case to instrumentation test to verify that settings are
being respected, even if no ServiceWorkerClient is being set.
