Fix bisect-builds for M90 and before

Change crrev.com/c/3594384 used a new URL to get the revision number
for recent builds when notations like "-g M90" were used in
bisect-builds.py. However a logic error meant that the fallback to the
old method never worked. The error is quite obvious in hindsight:

    if not revision_text:
      response = urllib.urlopen(OMAHA_REVISIONS_URL % revision_text)

Having ensured that revision_text is no longer a string (it's None) we
probably shouldn't use it to create a URL.

I reviewed the change that introduced the bug. I missed the bug.
