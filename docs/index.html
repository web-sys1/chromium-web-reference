[wifi/mac] Refresh connected network properties from hardware in GetProperties()

WiFiServiceImpl::GetProperties() on Windows already queries the connected network live via WlanQueryInterface(wlan_intf_opcode_current_connection),
returning fresh signal_strength, frequency, and bssid without a scan.

WiFiServiceMac::GetProperties() on macOS has no equivalent — it returns
these values from the networks_ cache, which is only updated when
UpdateNetworks() runs an active scan. Between scans the values are stale;
frequency and bssid can also change silently on AP roam.

CWInterface provides equivalent live hardware reads for the connected
network: rssiValue, wlanChannel, and bssid. This CL adds
RefreshConnectedNetworkProperties() to update these fields immediately
before GetProperties() returns, bringing macOS to parity with Windows.

Testing:
  Manually verified with wifi_test --get_properties: SignalStrength updates
  in real time with no airportd scan activity
  (`log stream --process airportd --level debug`).
