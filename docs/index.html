Fetch engaged sites earlier in LookalikeUrlNavigationThrottle

This change attempts to optimize the lookalike throttle by fetching
engaged sites on request start instead of when the response is received.
This should avoid having to defer the navigation while waiting for the
engaged sites data. This logic is behind a base::Feature so we can run
an experiment to determine if this makes a difference.
