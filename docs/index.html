wayland: make window drag controller more resistant to bogus dnd events

If wl_pointer events, eg: wl_pointer.button release, get sent before the
dnd session ends, window drag controller code crashes in some cases,
such as in the linked crbug. The root issue is likely an Exo bug, which
must be fixed, though we can better handle such cases at client side,
making window drag controller more resistant to bogus compositors.
