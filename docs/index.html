InternalPopupMenu: Rely on CSS serializer for serializing font-family

Currently InternalPopupMenu serializes font-family as an array of
strings (font-family names) ; listPicker.js parses that array and
rebuilds a font-family string by quoting each item and joining them
with commas. This is incorrect for generic font families : they
should not be quoted [1].

Additionally, the serialization of each item relies on
FontFamily::Family() which can lead to results different from
standard CSS serialization [2] [3] [4]. In particular, some
generic names like "cursive" are currently parsed internally as
"-webkit-cursive" which is the non-standard value listPicker.js
would receive (see for example popup-menu-appearance-styled.html).

This CL simplifies this by relying on the CSS serializer to serialize
the whole font-family value and then passing it as it to listPicker.js
so it can directly set font-family without further work. This removes
one more place where a custom serialization of font-family is made,
fixing inconsistencies and helping with the goal of fixing [5].

[1] https://www.w3.org/TR/css-fonts-4/#generic-family-value
[2] https://drafts.csswg.org/cssom/#serialize-a-comma-separated-list
[3] https://drafts.csswg.org/cssom/#serialize-a-string
[4] https://drafts.csswg.org/cssom/#serialize-an-identifier
[5] https://bugs.chromium.org/p/chromium/issues/detail?id=1065468
