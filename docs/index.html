[WebAuthn] Better handle discovery with multiple platform authenticators

Currently when a platform authenticator discovery fails it removes
`kInternal` from the list of available transports. However, in some
cases it is now possible for multiple platform authenticators to be
available.

This patch counts the number of platform authenticator discoveries
and only removes the transport if they all fail.
