[AIM] Rewire attachment calls through session instead of coordinator.

The FuseboxCoordinator held a FuseboxAttachmentModelList from before the
session rework, and it is mostly unused. However the other window scoped
objects in omnibox code were observing this for attachment changes. This
unused model list now does not correctly emitted attachment changes, and
these window scoped objects missed out on correctly updating. Most
notably this causes us to not recalculate the tool mode when an
attachment is added. To fix this, the model list in the coordinator was
removed, and the window scoped objects directly listen to updates from
the version inside the session.

The attachment count could similarly go through the session. The adapter
was already being set correctly by the fusebox mediator.
