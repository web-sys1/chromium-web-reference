RfhToIoThreadClientMap: Remove reference to WEBVIEW_PRERENDER2 feature flag

Before this CL, `RfhToIoThreadClientMap::RenderFrameHostChanged` logic
to handle prerender activation was guarded undef rlag.

This CL removes the flag guard, since we are moving away from the
WEBVIEW_PRERENDER2 feature flag.

The change is safe, since the logic is safe to be applied to non-Prerender2
FrameTree swaps.

The author tried to keep the if condition, but it seems difficult to do so:
- We can't rely on `new_rfh->GetLifecycleState()` check, since both RFHs
  is in state `kActive` on Prerender2 activation.
- `WebContentsDelegate::IsPrerender2Supported()` is not reliable, since
  Prerender2 allowance condition may have changed (TOCTOU).
