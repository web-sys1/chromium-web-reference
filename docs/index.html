Bug fix for flickering favicon

Check if a Tab's cached URL equals the NavigationHandle URL before
setting a default favicon for a tab on a page load in the Tab Switcher.
This fixes an issue where the favicon can flicker to the default icon
briefly if the tab needs to create a new renderer (e.g. upon unfreeze).

This notably happens

1) When the selected tab is changed on the grid tab switcher due to
   closing the current tab and the newly selected tab doesn't have a
   renderer.
2) When the selected tab is changes on the Group Tab Bottom Bar and
   the newly selected tab doesn't have a renderer.
