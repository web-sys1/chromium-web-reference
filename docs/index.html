Wait force-installed extension before launch Kiosk sessions

The policy ExtensionInstallForcelist has been supported on the client
side for Kiosk sessions. To provide better experience to Kiosk users, we
want to keep users stay in the splash screen until all force-installed
extensions are loaded successfully.

To achieve this purpose, we need to synchronize the main UI thread and
extension-loading thread. An additional stage 'Initializing extensions'
will be added in to Kiosk launch process.

At the implementation level, the ForceInstalledTracker::Observer can be
used to monitor the status of these extensions and notify the UI. Two
main cases should be considered.
1. The extensions are ready before launching the app window.
2. The extensions are not ready, we need to wait until it finishes
before launching the app window.
