[A11y] Fix errors when content-visibility causes node to be attached

Fix for test SidePanelBookmarksTest.List (browser_tests suite) when it
is run with a11y on.

Problem:
The content-visibility CSS causes a node to be attached to the layout
tree, and we update the children of the node with content-visibility.
However, even though the node has new children, it's cached
children aren't updated, leading to the serializer running into
a bad child.

Fix:
Call ChildrenChanged() when NodeIsAttached() is called with a
content-visibility subtree.

Also add an additional check to AXObjectCacheImpl::CheckTreeIsUpdated(),
so that future similar problems are flagged before they hit the
serializer.
