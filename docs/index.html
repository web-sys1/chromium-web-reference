Enable real-time priority thread for Audio Worklet by default

Until M97, the real-time priority thread for Audio Worklet is only
enabled when it is allowed by the server-side control. Based on
the consensus [1], we will enable the real-time priority thread by
default, and it will be disabled only when the server-side config
specifically deactivates the feature.

[1] https://groups.google.com/a/google.com/g/web-audio-team/c/2b3HdOL-GnA
