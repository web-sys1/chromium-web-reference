Report post-mouseover LCP as an LCP type

It seems like we have an issue with zoom widgets triggered by a
mouseover event over a previous <img>-based LCP image.
Such widgets are loading larger-than-current-LCP images, and since
moving the mouse doesn't count as an interaction, those LCP are
counted.
That goes against the goal of LCP to be a stable metric that doesn't
change based on user actions.

At the same time, it's not currently clear what the impact of this is on
CrUX results. This CL will enable us to get a better understanding of
when this scenario happens, and what is its impact when it does.
