Use SecurityPrincipal in instant process helpers

Replace URL-based instant process assignment functions with
SecurityPrincipal-based equivalents to remove calls to
GetDeprecatedSiteURL() in chrome_content_browser_client.cc.

Rename ShouldUseProcessPerSiteForInstantSiteURL to
ShouldUseProcessPerSiteForSecurityPrincipal and add
ShouldAssignSecurityPrincipalToInstantRenderer for callers
that already have a SecurityPrincipal (post effective URL
translation). The new functions operate directly on
SecurityPrincipal using SchemeIs() and GetHost() rather than
extracting a GURL.

The original ShouldAssignURLToInstantRenderer is preserved
for callers that work with navigation URLs before effective
URL translation.

Add SecurityPrincipal::CreateForTesting factory backed by a
real SiteInfo to support browser tests.
