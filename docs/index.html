Fix callback list check.

(Sync up with missive)

Check fired when CallbackList was notifying subscribers while being
deleted. We only had 2 places where the CallbackList was used, and did
not need its full functionality.
For that reason I am removing CallbackList altogether and replace it
with vector of callbacks, guarded by sequence task runner and not
needing CHECK protection from a colliding use.

Also hardended test of RefCountedClosureList to more closely resemble
its actual use: releasing the test-owned reference before workers
complete and release it.
