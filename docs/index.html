[A11yPerformance] Reduce overhead in state computation

Methods such as AXPlatformNodeWin::MSAAState query many properties of a
node to compute some overall state. This can be an especially hot
codepath when unserializing a large tree.

This change makes a number of small changes to reduce the overhead in
MSAAState and related functions by:

- Introducing AXStates (a helper for operating on a collection of
  ax::mojom::State bits) and using it to make repeated queries of a
  node's state.
- Capturing the platform node's role once for repeated comparisons.
- Capturing the platform node's delegate once and calling directly into
  it repeatedly.

Local traces indicate that this alone chops about 25% off the time spent
in MSAAState.
