Make "libcxx_abi_unstable" not a gn arg

This setting existed as a workaround for a bug that's since been fixed.
Nothing should have to set this flag:

- If you want to build against system lib(std)c++ for some reason, you
  really want to set `use_custom_libcxx = false`

- If you want to build against hermetic libc++, you always want this
  for perf reasons I think.

If this _does_ break a use case of yours, please describe it on the
review of this CL.
