snap-group: Revamp split view divider in tablet and clamshell

This CL revamps the split view divider Z order in tablet mode which
is `kFloatingUIElement` and `kShellWindowId_AlwaysonTopContainer` to
be on the same parent container of the top window. For the parent
container, it will be set to the same parent container of the top window.

The stacking logic now is to always keep the split view divider right
on top of the observed windows, both of which are under the split view
divider.

This CL also centralizes the stacking order update that happens in
the split view to be in `SplitViewDivider`.
