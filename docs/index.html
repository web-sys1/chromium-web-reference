LoginUnlockThroughputRecorder: Wait for browser icon explicitly

Currently we wait for the first browser window to be restored to ensure
that the browser icon is present on the shelf. Due to this, the metric
for the shelf icons readiness can be dominated by the browser restore
performance, especially if the browser is Lacros.

This CL changes LoginUnlockThroughputRecorder to explicitly wait for
browser icon to be listed in the shelf model. The value for
"Ash.LoginSessionRestore.AllShelfIconsLoaded" is expected to be smaller
after this change.
