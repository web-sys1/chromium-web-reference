Restrict mime type for content scripts

The purpose of this change is to improve the browser's security and
reduce mistakes. It aligns content script loading rules with other areas
of the browser, such as loading scripts in HTML pages via script tags,
and service worker scripts (including in MV3 extensions). This will
prevent mistakes and misuses such as loading content scripts from JSON
or image files.

The change applies to content scripts loaded via the extension's
manifest and the following APIs:
- chrome.tabs.executeScript
- chrome.scripting.executeScript
- chrome.userScripts.execute
