Only run tast tests on vmlab for reven

We try to add reven to vmlab tests, but Chromium CI buildbot doesn't
have enough hard disk to generate artifacts for all unittests + tast
tests, if cros_boards_with_qemu_images is set.

qemu run will include the ChromeOS image as part of artifacts causing
very large squashfs even for a small unittest on vmlab, ending us no
space left. example:
https://ci.chromium.org/ui/p/chrome/builders/ci/chromeos-reven-chrome/11670/overview

So stop run tests using legacy virtualization method, and run tast tests
only instead in vmlab (as legacy method is also running tast test only)

We can use try job to verify if it works.
