Remove JavaScriptDialog and BFCache NavigationThrottles

Both of these throttles want to conditionally defer a navigation from
committing if it's happening in the primary frame tree. This means they
need to defer only the "activating" navigation of a prerender or BFCache
navigation. This forces us to run all throttles a second time for just
these two cases.

In this CL we add a new content-internal mechanism for deferring a
navigation commit, NavigationCommitDeferrer, and implement the
functionality from these two throttles in it.

When ready to commit, a NavigationRequest checks with its delegate
(WebContents) whether it's ready to commit the navigation. WebContents
will try to create a NavigationCommitDeferrer. If a commit deferrer is
needed the navigation commit will be deferred and the
NavigationCommitDeferrer will then run the checks, in order, and resume
the NavigationRequest when the checks are passed.

See below doc for more discussion and details:
https://docs.google.com/document/d/1gJhjnZvah9er3iKk_FZcqXtT0M5dar_jWkZNUHXYLRM/edit#heading=h.1e0iap127qb4
