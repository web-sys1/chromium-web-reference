Fix flaky TimeFormatUtilTest.GetViewedItemText

Time should be passed using base::Time, passing it as an int lost
precision and resulted in flaky results. Storing it as an int is fine
but that should be an implementation detail at the storage site,
understanding that we loose precision there as well but being okay
with that.

Tweaked the "Viewed in the past month" test case as it was
effectively testing (now - base::Days(30)) < (now - base::Days(30)).
The only reason this passed most of the time was the loss of
precision making the recomputed value land a second short...
Flakily being == when the microseconds representation of Now() luckily
had enough 0's in its tail to not lose precision...
