Add internal helper class for deserializing PaintOps.

Issues with the current state:
* Code duplication for initialization and finalization steps when
  deserializing.
* Since placement new is used, you need to manually call the destructor
  in fast return conditions. It is easy to forget to do this, and I
  actually did myself when modifying the SkottieOp (hence prompting this
  CL).
* It's inconsistent. PaintOp::DestroyThis() checks if the object is
  trivially destructible and skips the destructor call if so
  (supposedly for performance reasons). Whereas PaintOp::Deserialize()
  methods have been blindly calling the destructor. It's now all
  routed through DestroyThis().

I figured we may as well address this before putting up the pending
SkottieOp CL that does deserialization.
