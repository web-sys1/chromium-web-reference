Make ExtensionUrlOverrideStateTracker destructor virtual.

`ExtensionUrlOverrideStateTrackerImpl` is a derived class that inherits
from `ExtensionUrlOverrideStateTracker` (the base class).

In `ExtensionUrlOverrideRegistryManager`, the member `state_tracker_` is
declared as `std::unique_ptr<ExtensionUrlOverrideStateTracker>`.
However, it is being initialized with
`std::make_unique<ExtensionUrlOverrideStateTrackerImpl>()`.

Without a `virtual` destructor in the base class, when `state_tracker_`
is destroyed, only the base class destructor is invoked. This leads to:
*   The wrong size parameter being passed to sized delete.
*   Undefined Behavior because members unique to `ExtensionUrlOverrideStateTrackerImpl`, such as `std::unique_ptr<RegistrarSynchronizer>`, are not properly destructed.

Making the base class destructor virtual ensures that the derived class
destructor is also called, thus preventing these issues.
