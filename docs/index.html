Replace cr-toolbar with a custom header in browser-switch internals.

This change removes the dependency on cr-toolbar and replaces it with a
custom `<header>` element. The custom header includes the page title
and the "Export to JSON" button. The search bar provided by cr-toolbar
was not intended to be shown on this page, and "show-search" was set to
false, however there appears to be a bug due to the cr-toolbar
defaulting "show search" variable to true. The absence of the
"show-search" boolean property does not set it to false but rather
defers to the default which is true. Since cr-toolbar could not be used
in hide-search mode without either a breaking change or a hacky
workaround, it makes more sense to construct the browser-switch page
using raw divs instead.
