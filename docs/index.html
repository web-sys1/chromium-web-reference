FSA: Allow the getUniqueId() method to raise an exception

Previously, we returned an empty string if the mojo pipe was
disconnected. This is not an ideal behavior. Instead, we'll reject
with an invalid state error. This also opens the door to adding other
logic that may throw exceptions, such as access checks.

Also adds a bunch of TODOs pointing to https://crbug.com/1327821 in
case mojo ever adds something like base::expected *fingers crossed*
