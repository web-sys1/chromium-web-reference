Correctly choose device or account domain name for update notifications

There are two policies that can trigger update notifications:
 - RelaunchNotification account/user policy
 - DeviceMinimumVersion device policy

When RelaunchNotification user policy triggers notifications, we should use account domain manager, which ensures that if the logged-in user is managed by a different domain, we use that domain. For unmanaged accounts, which may have this policy configured via other mechanisms (extension, flags, etc.), we fall back to using a generic message without domain name (IDS_UPDATE_NOTIFICATION_MESSAGE_LEARN_MORE).

When DeviceMinimumVersion device policy is set, the user policy RelaunchNotification is ignored and overridden to kRequired. This CL implements logic that ensures that we display device domain in this case.
