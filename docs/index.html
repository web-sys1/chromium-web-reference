Reland "Support ChromeOS external protocol dialog for Fenced Frame navigations"

This is a reland of commit 56fb5f726ca08146a3bd0095c3e65537d82b776b

The original CL was reverted (crrev.com/c/3508452) because it caused
the test failures of UrlOverridingTest.

The original CL had a bug that the argument order of
ExternalProtocolHandler::RunExternalProtocolDialog() was wrong for
Android. |is_in_fenced_frame_tree| should be added after
|has_user_gesture|. But, the order was swapped incorrectly. So
|has_user_gesture| flag is always false for non-fenced frame
navigations.

This CL fixes the argument order.

Bug: 1303626

Original change's description:
> Support ChromeOS external protocol dialog for Fenced Frame navigations
>
> Currently ChromeOS Chrome doesn’t support external protocol links (eg:
> tel:, intent://, market://) in fenced frames. This is because we use
> AUTO_SUBFRAME PageTransition for all navigations in fenced frames
> (crbug.com/1264936), and apps::ShouldIgnoreNavigation() check in
> arc::RunArcExternalProtocolDialog() returns false for such navigations.
>
> External protocol links are very important in Fenced Frame’s ads use
> cases. So this CL makes them work by passing |is_in_fenced_frame_tree|
> flag from NavigationURLLoaderImpl::PrepareForNonInterceptedRequest() to
> apps::ShouldIgnoreNavigation().
>
>  NavigationURLLoaderImpl::PrepareForNonInterceptedRequest()
>   => ChromeContentBrowserClient::HandleExternalProtocol()
>    => LaunchURL()
>     => ExternalProtocolHandler::LaunchUrl()
>      => OnDefaultProtocolClientWorkerFinished()
>       => RunExternalProtocolDialogWithDelegate()
>        => ExternalProtocolHandler::RunExternalProtocolDialog()
>         => arc::RunArcExternalProtocolDialog()
>          => apps::ShouldIgnoreNavigation()
>
> And apps::ShouldIgnoreNavigation() will return false iff
> |has_user_gesture| is true when |is_in_fenced_frame_tree| is true.
>
> And also this CL do the followings:
> - Removes ShouldIgnoreNavigation()'s argument |allow_client_redirect|
>   which is always true after https://crrev.com/c/1924329.
> - Introduces NavigationHandle::IsInFencedFrameTree() to use in
>   IsNavigateFromLink() to call ShouldIgnoreNavigation().
>
