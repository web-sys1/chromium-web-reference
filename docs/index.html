third_party/jinja2: Set create_system=3 (unix) to ZipInfo for cross platform determinism

When creating `zipfile`, ZipInfo automatically sets
create_system=0 (=Win) or 3 (=Unix).
https://github.com/python/cpython/blob/f878d46e5614f08a9302fcb6fc611ef49e9acf2f/Lib/zipfile/__init__.py#L419-L423.

This makes mojom_bindings_generator.py generating different zipfiles
between Win and Unix.
The only diffs are `Created OS` info. The unzipped contents were identical.

This CL addresses this cross-platform non-determinism by always setting
`create_system=3` inside jinja2.
