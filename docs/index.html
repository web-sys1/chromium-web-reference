RTCDataChannel: Send should throw when the send queue is full

Previously, the implementation would close the data channel when the send
queue was full.
According to https://w3c.github.io/webrtc-pc/#datachannel-send it should
throw a DOMException with OperationError code.

Removed from fast/peerconnection/RTCPeerConnection-datachannel.html the
parts that are already covered by WPT.
