Fix off-by-one frame issue with EarlyFullScreenVideoOptimization

There was an issue with EarlyFullScreenVideoOptimization in which we
resize the swap chain for full screen after the present. This means that
the new size won't latch until the subsequent frame. This is normally
not noticeable except in the first frame case or in the case when a
video does not present (e.g. when the image does not change).

This change fixes this by delaying the actual swap chain presentation to
after TryDisableDesktopPlane. It also moves TryDisableDesktopPlane and
FinishPresentToSwapChain into PresentToSwapChain to simplify
SwapChainPresenter's API.

This also adds tests to ensure that common cases of entering and exiting
full screen mode do not regress.
