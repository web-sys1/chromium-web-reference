Migrate Android CQ to use check_for_flakiness_with_resultdb

This is one of several changes to migrate all CQ builders to use the
ResultDB-based workflow for identifying new tests.

This change migrates all Android CQ builders to the new workflow by
changing the property to check_for_flakiness_with_resultdb

* android_optional_gpu_tests_rel
