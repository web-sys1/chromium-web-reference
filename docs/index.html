Nullify MediaRouter after shutdown and add no-op checks.

There have been hard to repro, but multiple reports across several OS of invalid media_router pointers within the access code cast sink service.

After KeyedService::Shutdown() has been called, we shouldn't be fetching or calling methods on MR. Even if we got a MR pointer back, its internal state may no longer be valid.

Additionally a unittest is added to the ACCSSFactory and some duplicated test code is extracted to a common class. A TODO bug is added to add a browsertest case to properly verify this since these interactions occur at the browser level and cannot be tested on the unittest level.
