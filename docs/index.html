[APS] Install OEM preloaded PWAs through App Preload Service

This CL completes the basic flow for the App Preload Service by taking
app definitions that are downloaded from the remote server and
installing them as PWAs. This implementation is incomplete, and does not
yet handle app icons, lacros, or all error cases. These cases will be
filled out in follow-up CLs.

This also fixes a bug found in the web apps system where the specified
manifest ID in the WebAppInstallInfo would be overwritten unexpectedly.
This appears to have only affected "external" install flows, so I think
it has had limited/no impact before now.
