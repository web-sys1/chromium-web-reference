[iOS]Partially replace isPresentingFirstRunUI by a FRE init stage

This is a subtask of moving FRE to an app state agent
(crbug.com/1178821).

We introduce a new InitStageFirstRun that sits between InitStageSafeMode
and InitStageFinal. The app will remain in InitStageFirstRun while the
FRE screens are being presented. The app will transition out of that
stage when FRE is done.

We still need to keep isPresentingFirstRunUI around during the
initialization of the UI to propagate the information immediately to the
other scenes, which isn't possible with init stages. This issue will be
solve when the FRE is in an agent.

The foreground active level branch in the scene controller will only
be called when transitioned to the final stage out of the FRE.This
code branch takes care of showing sign-in promo, handling external
intents, and showing the startup surface if necessary.

The post-FRE behaviour will be the exact same as the regular startup
behaviour where a part of the UI is handled when the scene is active
in the foreground. This will avoid ad hoc init code to handle post-FRE.

Tested these cases (see crbug.com/1207586 for more details):
  1) FRE with intents
  2) FRE and restoration
  3) FRE and regular startup
  4) FRE and multi windows
