Ensure FontGlobalContext::Get() not to return nullptr

This patch changes |FontGlobalContext::Get()| to use
|ThreadSpecific<T>::operator*()| instead of |IsSet()| to check whether
thread-specific value |FontGlobalContext*| is populated or not.


Before this patch, |FontGlobalContext::TryGet()| is called before
|Get()|, |FontGlobalContext*| isn't populated.

Background:
The crash is caused by following scenario:
  1. ... no call of FontGlobalContext::Get() ...
  2. FontGlobalContext::TryGet() -> allocate TLS slot with nullptr
  3. FontGlobalContext::Get() -> returns content in TLS

|ThreadSpecific<T>::IsSet()| returns true when TLS is allocated and
initialized. When T is a pointer, slot is initialized with nullptr.

|ThreadSpecific<T>::operator*()| returns reference to slot address and
allocate slot with |new (slot_address) T()| if slot isn't allocated.
