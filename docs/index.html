[iOS Testing] Add Release build support to EarlGrey

Motivation: To automate the profiling of UI interactions we want
to leverage the EarlGrey framework to reproduce the interactions
under investigation while recording traces and/or recording
profiles with xctrace. For performance investigations, it
is important to always use release builds.

EarlGrey ordinarily only supports Debug builds because it requires
debug symbols in order to perform whitebox testing. Because
Chromium developer release builds are setup to generate debug
symbols, it is actually possible to use EarlGrey with Chromium
release builds.

This CL fixes an issue where xcodebuild defaults to searching
for the app under test in Debug output directories.
It's necessary to use the CONFIGURATION_BUILD_DIR commandline
override to ensure the intent of the run_egtest.py `--out-dir`
argument is fully respected when it refers to non-Debug builds.
