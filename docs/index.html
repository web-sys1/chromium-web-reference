Make custom HDR ColorSpaces have the CUSTOM_HDR transfer ID

This CL cleans up the is_hdr param of ColorSpace constructors.
Previously, the ColorSpace(SkColorSpace, is_hdr) constructor would
create a ColorSpace with the transfer ID of CUSTOM, even if passing true
for is_hdr. This would make ColorSpace::IsHDR return false, which can
cause problems (e.g. on Windows, which uses the color space to determine
the best buffer format).

The is_hdr parameter is removed from the full constructor; it's HDR
state is implied by its transfer ID.
