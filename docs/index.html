[win] Refactor PermissionBubbleInteractiveUITest::CmdWClosesWindow

This CL removes the NoWidgetsWaiter from CmdWClosesWindow and
replaces it with a RunUntil check.

views::test::WidgetTest::GetAllWidgets() uses tracked NativeWidgets
to work backwards and determine all Widgets. NoWidgetsWaiter would
wait for the reported count to reach 0 only when OnWidgetDestroyed()
events were observed.

However this only works for the NATIVE_WIDGET_OWNS_WIDGET ownership
scheme since the NativeWidget will always outlive the widget. This
will not work for CLIENT_OWNS_WIDGET since OnWidgetDestroyed() will
be observed before the NativeWidget is destroyed (typically async)
and thus the waiter could end up stuck and time out.
