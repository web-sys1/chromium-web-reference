[rebaseline-cl] Fix baselines not written to `platform/linux-chrome/`

Starting in https://crrev.com/c/5731230, `rebaseline-cl` always passes
an explicit `port_name` to `TestBaselineSet.add(...)`. This bypassed
special logic in the implicit `port_name` case to coerce the name to
`chrome`, causing wdspec baselines to be written to `platform/linux/`
instead of `platform/linux-chrome/` [1].

Fix forward by always applying the coercion logic, which is temporary
until the `chrome` port is removed.

[1]: https://crrev.com/c/5757692
