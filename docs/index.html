Add an arena for CSS parser selectors.

During parsing, we allocate parser selectors (on PartitionAlloc),
convert them into their final form (on Oilpan), and then immediately
deallocate them. There's a TODO to rearchitect this to avoid the
intermediate step, but until that, we can at least make the allocation
and deallocation much cheaper.

We make a very simple implementation of the arena pattern, which
allows us to allocate very quickly and deallocate everything
in O(1) at the end. There will probably be a tiny bit higher peak
memory allocation during the milliseconds that the parser runs,
but that should be bearable.

This is really only a win for parsing entire stylesheets, but for
simplicity, we use the same code when parsing e.g. a single selector
for querySelector(). (The next patch contains code to make the use
of the arena conditional on a Finch flag, but it won't really change
this, and it should be temporary.)
