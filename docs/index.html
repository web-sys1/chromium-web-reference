Release gated Skia API change in SkFontArguments::PaletteOverride

In the mentioned bug below, Skia changes the API type for the index of a
color override from an int to a uint16_t type as that is closer to the
font specification and avoids issues in ensuring the value is
positive. That change was gated by a define
SK_DISABLE_SKFONTARGS_CHANGED_TYPE. Remove that define from Chromium
compilation and adapt Blink type usage where needed.
