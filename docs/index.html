[style_var_gen] Fix bug with variable references.

Previously, a variable referring to another variable would only generate
a different RGB value in CSS.

i.e

color_a: $color_b

would generate

--color-a-rgb: var(--color-b-rgb);
--color-a: var(--color-a-rgb);

This drops any opacity value that color_b may have had.

Some tests and validation have also been added for self-references which
caught me out last CL.
