[Service Workers] Fix UAF in ServiceWorkerContextCore

During ServiceWorkerContextCore::RemoveLiveVersion or
OnRunningStateChanged, the code iterates over sync_observer_list_.
An observer's callback could lead to the destruction of the
ServiceWorkerContextWrapper, which owns the ServiceWorkerContextCore
and sync_observer_list_.

This causes a use-after-free when the loop attempts to continue
iteration on the now-deleted sync_observer_list_, leading to a crash.

This was verified via analyzing a crash report with CloudLLDB:

```
(const scoped_refptr<content::ServiceWorkerContextSynchronousObserverList>) sync_observer_list_ = {
  ptr_ = <read memory from 0x41b403beb578 failed (0 of 8 bytes read)>
}
```

```
(content::ServiceWorkerContextWrapper) *wrapper_.wrapped_ptr_ = {
  base::RefCountedThreadSafe<content::ServiceWorkerContextWrapper, content::BrowserThread::DeleteOnUIThread> = {
    base::subtle::RefCountedThreadSafeBase = {
      ref_count_ = {
        ref_count_ = <read memory from 0x5938c9a0aafc3c00 failed (0 of 4 bytes read)>
      }
    }
  }
```

This change fixes the issue by holding a scoped_refptr to the
ServiceWorkerContextWrapper for the duration of the methods. This
ensures that the ServiceWorkerContextCore and its members remain alive
until the observer loop is complete.
