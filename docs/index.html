[views_ax] Migrate SetAccessibleRole/SetAccessibleName to cache system

This CL migrates both the View::SetAccessibleRole and
View::SetAccessibleName functions to use the new accessibility cache
we introduced in ViewAccessibility. This change only modifies the
core implementation of these functions without renaming all callers
to the new ViewAccessibility setters. This is because there are
hundreds of callers across views, and we prefer to separate the
core change from the simple rename that will soon follow.

The SetAccessibleRole and SetAccessibleName functions must be migrated
together because of the strong dependency the name has on the role.
We initially tried to migrate the role by itself, but then it caused
issues when the name was being set on a different AXNodeData member
that had no role.

This CL removes a few tests that are no longer needed as a core
behavior has been removed: we can no longer set an accessible before
setting the role on a view. This aligns the expectations for views
with the existing expectations in AXNodeData and simplifies the code.

We have landed around 20 changes in preparation of this change in
which we fixed thousands of tests failures. There should be no change
in behavior now as the tests are green, but in the off chance that
my change introduced a regression and you are only now reading this
commit, please reach out directly to me on Slack or over email --
regressions can be hard to investigate, but after fixing thousands of
them, they became very trivial to investigate and fix to me. Please
reach out instead of reverting as many, many CLs are about to be based on top of this one.
