[Extensions] Use browser context unique id in extension utils

Previously, extensions util generated a "unique id" given a browser
context and stored them in a `std::map<content::BrowserContext*, int>`. This was problematic because if the BrowserContext gets
destroyed, another BrowserContext could be placed in the same address
and get the "unique id" of the destroyed BrowserContext.

Instead, we can use the BrowserContext::UniqueId that guarantees a
unique id. We maintain the map generated by extension utils because
the types are different (int vs string) and because it's also used by
the renderer.
