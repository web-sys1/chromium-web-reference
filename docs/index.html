personalization: enforce policy controlled wallpaper for managed guest
sessions.

Policy controlled wallpaper for managed guest sessions is updated on
login screen where it is not possible to determine the types for these
users. Thus the wallpaper info is persisted in local state instead of
the ephemeral map. This creates a bug where managed guest sessions are
able to update their wallpaper because we fail to lookup the correct
wallpaper type.

This CL adds a fix by modifying |SetPolicyWallpaper| to take in an additional param |user_type| where it is correctly determined from |WallpaperControllerClient|.
