[attributon_reporting] Add <img> attributionsrc processing for Register-Source

This CL implements a new surface for Attribution Reporting as documented at:
https://github.com/WICG/conversion-measurement-api/blob/main/EVENT.md#registering-attribution-sources

This API allows an image element to set an attributionsrc=url attribute
which performs a request whose response is allowed to configure
an attribution source.

This only implements the img API for views, which will be extended to
clicks. For this reason, the infrastructure is meant to process and
response to these responses in a keep-alive fashion.

Note: This change adds a fair amount of repetition to processing/parsing
the new AttributionSourceData struct above blink::Impression.

This change reflects the desired end-state, and code which operates on
blink::Impression will be either removed or migrated to the new
pending_receiver<> design.
