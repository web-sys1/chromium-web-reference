r&r benchmark: Use layer's raster scale instead of {1, 1}.

==============
PERF SHERIFFS:
Note that this changes rasterize_and_record benchmark. Any changes
in the metrics should be assigned to vmpstr@chromium.org.
Do not revert as a result of regressions.
==============

This patch ensures that the rasterize and record benchmark uses the
raster scale as computed by the picture layer impl, instead of hard
coding {1, 1}. This, in turn, ensures fair comparison for things like
zoom-for-dsf which trade recorded layer size for raster scale. IOW,
before this patch it would look like a regression if we reduced raster
scale but increased the recoded layer size. Now it should be a wash.
