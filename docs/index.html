Reset composition when text_input::reset is called.

Currently, on text_input::reset, InputMethod::OnCaretBoundsChanged
is called.
However, it is not the method to reset the input method,
and as the result, wayland client and server has inconsistent
state and it causes weird behavior.
With this CL, it invokes CancelComposition to reset IME state.
