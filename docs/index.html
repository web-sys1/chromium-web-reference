[iOS] Re-enable testPlusAddressFallback egTest

FakePlusAddressService is used instead of an instance of the
PlusAddressService for the egTests.

In a nutshell, when a plus address is created, PlusAddressService informs this to a backend server that saves this plus address for this particular user and domain.
On get calls, this plus address is fetched from the backend by the PlusAddressService.
In the egTest, we want to make sure that there is an existing plus address saved to test out the manual fill behavior.
Using FakePlusAddressService seemed to be a simpler and better solution.
