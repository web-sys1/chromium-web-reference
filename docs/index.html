Prevent excessive scroll[Horizontal|Vertical]PositionChanged events

In some settings sliders (see bug), we get the following sequence of events:
1. OnIntAttributeChanged attribute=kScrollX old value=<empty> new value=0
2. OnIntAttributeChanged attribute=kScrollX old value=0 new value=<empty>

Currently, we only fire an event for 1, but not 2.

This change makes it consistent by firing neither 1 nor 2.
