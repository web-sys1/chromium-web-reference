Do not normalize text range in atomic text field

The CL introduces the behavior if a text range is inside an atomic
text field, we skip normalization. Previously, we only skipped
normalization when AXTree::Selection is present. However, ATs such as
Windows Text Cursor Indicator relies on degenerate range as a cursor
and display it if it is in an editable text field (i.e.
ReadOnly=false). Previously, we would normalize such degenerate range
at the end of text field to the next position, resulting in the
degenerate range to be no longer in text field, thus with attribute
ReadOnly=true, so Text Cursor Indicator no longer displays any cursor.

Ax-Relnotes: Windows TextCursor Indicator now shows up at the end of
text field.
