[@scope] Make CheckInStyleScope look at activations directly

CheckInStyleScope needs to do two things: check if the subject
actually is "in scope" [1], and produce the correct proximity.
We don't need to look at the ancestor chain to answer this question,
because we already did that ancestor traversal in order to calculate
activations. In other words, the list of activations represents the
scopes where the subject element is "in scope". The proximity is just
the closest activation.

There should be no behavior change, this is just simplification.

[1] https://drafts.csswg.org/css-cascade-6/#in-scope
