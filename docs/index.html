Adjust folder icon and animation for RTL

To generate a folder icon within the apps grid, FolderImage class
uses the top icons that will show as a preview for the folder.
These are the 4 first icons on the folder, and are laid out in a grid in
the order in which they appear.
When using an RTL configuration, FolderImage::GetTopIconsBounds does not
take into consideration that apps are displayed in a reverse order. As a
result, the preview looks the same in RTL as well as in LTR.
This CL swaps the calculated bounds for the left and right icon when the
UI is in RLT configuration.

This CL also uses mirrored coordinates for the open/close folder
animation. FolderImage::GetTopIconsBounds is used to know the starting
bounds and TopIconAnimation::GetFirstPageItemViewsBounds fetches the
ending position of the animation. Both methods need to use mirrored
coordinates when RTL is configured.
