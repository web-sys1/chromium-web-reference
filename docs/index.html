Support CTAP2.2 hmac-secret-mc extension in make credential request

Some authenticators (e.g., security keys) do not support the PRF
extension but can still derive symmetric secret via the hmac-secret
extension. To support PRF-on-Create in those authenticators, this
CL adds support for CTAP2.2 hmac-secret-mc extension in make credential
request.

Since the hmac-secret-mc extension follows the behavior of hmac-secret
extension in get assertion request, this CL copies and reuses the logic
for the hmac-secret extension to implement the behavior of hmac-secret-mc
extension.

This CL also adds a flag kWebAuthnHmacSecretMcExtension, which controls
whether the prf_input request is converted into the hmac-secret-mc
extension request. The flag is enabled by default.
