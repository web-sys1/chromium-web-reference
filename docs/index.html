[ios] Remove SyncSetupService error APIs in favor of SyncService

Even before this patch, and starting with https://crrev.com/c/4239517,
syncer::SyncService in components/sync is capable of reporting the
precise error that the UI should display. This logic was previously in
SyncSetupService, which now became a thin layer on top.

In this patch, the API in SyncSetupService is removed entirely and all
callers migrated.
