Register VM token to dispatcher only when camera permission is on

Since we are going to replace camera socket with mojo service manager to
bootstrap the camera connection between plug-in VM and
CameraHalDispatcher, we can't control access to the camera based on
mount/unmount camera socket. Therefore, a new mechanism is needed. This
CL makes VM permission service only registers VM token to
CameraHalDispatcher only when the camera permission is On. When the
camera request is coming, CameraHalDispatcher will check the VM token
first, if token is not registered, it will reject the request.

Bug: b:304685896
Test: Verify CameraHalDispatcher only get token when permission is on.
