Reland "[Fixit] Use variable pixel offset for initial position"

Reason for reland: fix issue with TabListContainerViewBinder failing.
The issue appears to be with optimizing the RecyclerView spy causing

> Original change's description:
> > [Fixit] Use variable pixel offset for initial position
> >
> > Prior to this CL we used fixed index offsets for tablist recycler view
> > initial position. This was static and had errors if the span count was
> > dynamic (e.g. landscape vs portrait). It also failed to account for
> > visible row count on the screen so we were using a 1 row offset with
> > the assumption 3 rows of 2 tabs were visible. This could break in cases
> > where 4 rows were visible for example.
> >
> > The basic solution is to just use a dynamic span count. However, this is
> > not robust to screen orientation, and size. Instead this CL uses pixel
> > based offsets that attempt to center the currently selected tab in the
> > RecyclerView. This is much more robust to changes in screen size,
> > orientation, etc. However, it is more complex to implement as it
> > requires calculating both recyclerview and item size and computing the
> > correct pixel position.
> >
> > This CL is tested and should work reliably for both the Tab Switcher
> > and Tab Groups Dialog. Please file bugs or update the attached bug
> > if this breaks under some edgecases.
