Use priority-based TaskRunner in URLLoader

This CL updates the URLLoader to use the new priority-based task
runners.

This change is guarded by the "url_loader" feature parameter of the
`NetworkServiceTaskScheduler` feature.

This is part of a larger effort to migrate the network stack to use the
new NetworkServiceTaskScheduler, instead of the generic default task
runner.
