Reland "Do not clip gutters created by scrollbar-gutter"

This is a reland of 87556ad01e74492343e1f906433257ae0976eacf

The motivation for the original change was that events
happening on the additional gutter created by the "both"
keyword were being ignored.

The reverted commit included a unit test that used overlay
scrollbars and issued scrolling events. However, this
combination turned out to cause a DCHECK failure on Mac with
ASAN enabled.

This CL replaces that unit test with a similar one that uses
classic scrollbars, which is suitable for testing the fixed
functionality while avoiding that Mac/ASAN failure.

Original change's description:
> Do not clip gutters created by scrollbar-gutter
>
> The gutters created by the scrollbar-gutter property don't occlude
> the content underneath them. Therefore, to prevent missing content
> and ignored events, those gutters should not be clipped out by
> LayoutBox::OverflowClipRect.
>
> In order to do this, ExcludeScrollbars and ComputeScrollbarsInternal
> are optionally able to exclude those gutters from their results.
>
> Web and unit tests are included.
>
