[bedrock] Move contextual task features to BWF::Init()

This CL moves contextual task features from
BWF::InitPostWindowConstruction() to BWF::Init().

This is necessary as there are parts of BrowserWindow
initialization that rely on contextual task features,
resulting in a circular dependency chain that makes
startup brittle.

The BrowserWindow dependency in contextual task features
was previously fixed and this can now be initialized
before the BrowserWindow construction.
