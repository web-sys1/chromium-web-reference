Combine text blob pixel tests into one suite

Use ::testing::Combine to generate all permutations over different
testing strategies for how the text blob is drawn, the canvas matrix
type, and LCD text state. This generates a superset of what was
previously being tested and gives the instantiated tests more
descriptive names since the parameter set is a lot larger now.

The older tests are mapped as follows:
P/OopTextBlobPixelTest.DrawTextBlob/0 = P/OopTextBlobPixelTest.Config/Direct_IdentityCTM_NoLCD
P/OopTextBlobPixelTest.DrawTextBlob/1 = P/OopTextBlobPixelTest.Config/Direct_IdentityCTM_LCD
 - And in addition, DrawTextBlobOps are now tested under scale and complex CTMs x (no-lcd, lcd).

P/OopRecordShaderPixelTest.ShaderWithTextScaled/0 = P/OopTextBlobPixelTest.Config/RecordShader_ScaledCTM_NoLCD
P/OopRecordShaderPixelTest.ShaderWithTextScaled/1 = P/OopTextBlobPixelTest.Config/RecordShader_ScaledCTM_LCD
 - And in addition, RecordShaders containing text are tested with identity and complex CTMs.

P/OopRecordFilterPixelTest.FilterWithTextScaled/0 = P/OopTextBlobPixelTest.Config/RecordFilter_ScaledCTM_NoLCD
P/OopRecordFilterPixelTest.FilterWithTextScaled/1 = P/OopTextBlobPixelTest.Config/RecordFilter_ScaledCTM_LCD
P/OopRecordFilterPixelTest.FilterWithTextAndComplexCTM/0 = P/OopTextBlobPixelTest.Config/RecordFilter_ComplexCTM_NoLCD
P/OopRecordFilterPixelTest.FilterWIthTextAndComplexCTM/1 = P/oopTextBlobPixelTest.Config/RecordFilter_ComplexCTM_LCD
 - And in addition, RecordFilters containing text are tested with identity CTMs.

Because of increased CTM testing against text blobs, I noticed that
the RecordShader tests had been using NN sampling if the record shader
needed to be post-transformed by the CTM. This resulted in flaky tests
that had high rgb differences because of popping between background and
glyph colors.

I updated the test code to just always use bilerp sampling when the text
blob is embedded in a RecordShader or RecordFilter so that the magnitude
of these flaky pixel differences is kept smaller. However, the increased
testing coverage also required expanding how the fuzzy comparator was
configured in order to keep the bots green.

I was able to split the rules into OS and android version, since
android lollipop (nexus 5) was the worst offender,
android marshmallow (nexus 5x) was next, and then Mac and Windows having
only very slight differences (Linux had none). I confirmed in all cases
that these differences are not perceptible.
