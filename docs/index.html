[omnibox][composebox] Toggle voice search and smart compose features.

* Updates the default value of `kAutoSubmitVoiceSearchQuery` to true for
the WebUI Omnibox and Contextual Tasks. This ensures voice queries are
submitted automatically without additional user input.
* Updates `ShowSmartCompose` to false for the WebUI Omnibox. This lowers redundancy within Finch config, where this feature is overrode to false.
* Removes voice search "input" checks from `ContextualTasksBrowserTest.Composebox`. We do this instead of using a `loadtimedata` override, since this is testing part of the system that the tests for `<cr-composebox>` should (and already do) test [1]. Leaving the rest of the test as-is allows us to keep verifying ToT behavior.

b:481341619 tracks the eventual removal of this parameter and its
associated legacy code.

Followup CLs address reverting the mitigations made for voice search
e.g. suggest and text transferring to the Omnibox.
