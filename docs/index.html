autocorrect: ensure ac override honors pref set

Currently we override the autocorrect_enabled value sent to the IME
service when certain experimental conditions are not met. This override
has two issues in its logic;

  1. It is not fully guarded by the relevant autocorrect experiment
     flag.
  2. It does not take any explicit user preference set by the user for
     autocorrect into account.

This CL fixes the issue by adding in the required flag guard, and
the logic to honor the autocorrect preference explicitly set by the user.
