Add token and byte counts to PumpTokenizer traces

It is often helpful in exploring parser behavior to know how many
bytes are queued, how many were parsed, and how many tokens were
parsed in a single pump session. This CL adds those metrics to
the trace event for HTMLDocumentParser::PumpTokenizer.

To implement this, I had to add a "next" pointer to the
SegmentedString class, so that HTMLInputStream can walk the
entire input stream to get a size. This walk is only done when
tracing is enabled, to avoid any performance penalty.
