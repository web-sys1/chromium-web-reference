SmartCard: Fix crash when tab closes during getStatusChange()

Due to wrong order of members, the callback (passed to platform
implementation) was destroyed before the pipe was closed when closing
the tab before getStatusChange() returned. This caused the browser to
crash, as it is an error to destroy the callback before the pipe is
closed.

Added also simple test for this.
