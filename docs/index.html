[Android][IncognitoReauth] Refactor the logic of canAuthenticate method

The current use of canAuthenticate returns true only if biometrics are
enrolled. It doesn't take into account if screenlock might be enabled
for authentication instead. This was done so to support the password
manager's requirement.

However, with new clients like Incognito reauth, the requirement for
only relying on biometrics is not true in that case. Hence to support
that, this CL:

a) Renames the canAuthenticate method in BiometricAuthenticatorBridge
to canAuthenticateWithBiometrics and adds a new method canAuthenticateWithBiometricsOrScreenlock to consider the general
case stated above.

b) Updates the BiometricAuthenticatorAndroid#canAuthenticate to check
on the requester and invoke canAuthenticateWithBiometricsOrScreenlock
for reauth page and canAuthenticateWithBiometric otherwise.
