IndexedDB: Fix OOM crash in DisallowInactiveClient by limiting client-side remotes

This CL fixes an OOM crash occurring in IndexedDB's
DisallowInactiveClient mechanism when rapid successive calls create
unbounded mojo::Remote objects on the client side.

The crash was happening in BindNewPipeAndPassReceiver() during partition
allocator operations when DisallowInactiveClient was called repeatedly
for the same reason on the same connection, leading to memory pressure
before server-side deduplication could take effect.

The fix adds client-side deduplication by limiting
client_keep_active_remotes_ to at most one remote per reason per
connection. This aligns with the server-side behavior (added in CL
6575645) which already deduplicates receivers by connection_id.

This fix preserves document lifecycle protection in majority of
scenarios, mixed-reason calls may skip some handle creation, but
  core protection mechanisms remain intact.
