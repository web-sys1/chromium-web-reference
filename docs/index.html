shortcuts: Move accelerators map to AshAcceleratorConfiguration

AshAcceleratorConfiguration is responsible for owning and maintaining
Ash accelerator mappings including deprecated, debug, developer
accelerators.
This CL moves the accelerator mappings from AcceleratorControllerImpl.

This CL does not change any existing functionality of accelerators, it
is only moving the acclerator mappings.

KeyboardCode::KEY_DELETE has a conflict with existing linux input
event code macros that weren't caught until ash/ included
accelerators.mojom in accelerator_info.mojom. Renaming to DELETE_KEY to avoid the conflict.See linux/input-event-codes.h
