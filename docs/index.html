Warn before muting system audio

When screen-capture is requested by an application or an extension,
users are presented with a media picker, allowing them to choose
which surface to share, and whether to share audio.

If the user chooses to share audio as well, it is possible for
an application to to suppress local audio playback, meaning the
audio is captured into a MediaStreamTrack, but is no longer played
out over the user's local speakers. If an application intends to
use this, it is important to warn users, especially if they are
using a screen reader.

We therefore display a warning, that if the user chooses to
share audio, then local audio playback will be muted.

A feature flag, kWarnUserOfSystemWideLocalAudioSuppression,
acts as a killswitch for all new logic introduced to this effect.
