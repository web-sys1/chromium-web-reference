[conflict dialog] Use CONFLICT_DIALOG_FOLDER_MESSAGE text

When the file name in conflict is a directory (aka a folder), then
the dialog message text should show 'A folder named ...' [1].

[1] go/strpic/8a40aaef74c7870aa9289244903e4dcddb0beebf

Update the dialog show API to accept an optional |directory| param
and add a test case: testDialogShowDirectoryMessageText.

Change remaining test cases to check for 'A file named ...' if the
test uses the show API |directory| param default: false.

Change the test case comments about initial focus. The dialog must
have the focus when shown; never its child UI elements (buttons or
the checkbox). The user must explicitly select, or click the child
UI elements per the conflict dialog design spec.
