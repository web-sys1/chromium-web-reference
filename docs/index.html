Check IsDocumentOnLoadCompletedInMainFrame for FileSystemAccess request

Currently FileSystemAccessPermissionRequestManager class which was
introduced by https://crrev.com/c/1687563 queues the requests when the
|main_frame_has_fully_loaded_| flag is false. But this flag is false
only when the WebContents is newly created (eg: opened as a new tab).
So the requests are not queued when the page was navigated from another
page. This flag should be set to false when
WebContentsObserver::DidFinishNavigation() is called.

But according to the comment of DocumentOnLoadCompletedInMainFrame()
added by https://crrev.com/c/1827428, we should check
WebContents::IsDocumentOnLoadCompletedInMainFrame() instead of
listening to WebContentsObserver::DocumentOnLoadCompletedInMainFrame().
So this CL changes this class to check
WebContents::IsDocumentOnLoadCompletedInMainFrame().
