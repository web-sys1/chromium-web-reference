ozone/wayland: fix race in wayland buffer fuzzer

Given the fuzzer and the test wayland server run on different threads,
the objects on the test server side can only be accessed after the
server has processed the requests and is paused (the same what
we do in unittests).

Thus, to avoid usage of unitinitalized values, explicitly sync
(pause, resume, paus) after operations that require the server
to process the requests.
