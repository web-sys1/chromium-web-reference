OMTPrefetch: Separate out AwPrefetchManager's mutable members

A preparation CL of crrev.com/c/7544249.

Current `AwPrefetchManager`'s mutable members which are read/written via
public API call are these four: `all_prefetches_map_`,
`last_prefetch_key_`, `ttl_in_sec_`, `max_prefetches_`. (*)

This CL introduces `AwPrefetchManagerData`, which decouples these values
from `AwPrefetchManager`. All `AwPrefetchManagerData` accesses should be
managed via limited getter/setter, which creates a clear boundary and
we can easily add some enforcements that is closed within
`AwPrefetchManagerData`.

More specifically: after PrePrefetch, these can be accessed from both
UI/non-UI thread [1]. crrev.com/c/7544249 will introduces a lock
enforcement to `AwPrefetchManagerData`'s interface. `AwPrefetchManager`
simply call `AwPrefetchManagerData`'s interface in a thread-safe manner.

Currently no behavior change should be included.

(*) TODO: mentioned / add code comment about `java_obj_`
