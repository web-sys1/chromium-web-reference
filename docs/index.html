ozone/drm: [Testing] Add a DrmGpuDisplayManager unittest

This CL adds a unittest for DrmGpuDisplayManager with some preliminary
test cases:
  1) Support boundary for number of DRM devices.
  2) Support boundary for number of connectors per DRM device.
  3) The successful identification of different displays within a single
     DRM device and the retrieval of named device for modesetting
  4) The successful identification of different displays across multiple
     DRM devices and the retrieval of their respective DRM device for
     modesetting.
  5) That displays' origins persist between configurations as long as
     displays were not moved around or added/subtracted. This mostly
     affects software mirror mode.
