Apply FPS enterprise policy enable control at profile level.

Change the location of the FirstPartySetsEnabled enterprise policy from
local state to profile, since policies that affect the behavior of the
web platform are prohibited from applying at the device level.

This change also involves splitting the existing FPS enabled bit used in both browser side and network side into a machine level param (to allow
embedders to disable FPS entirely) and a profile level param which is
plumbed to network context via FirstPartySetsAccessDelegate.
