Choose policy validation algorithm based on response

When reading policy cache from Google Update location, the policy
proto might be signed with SHA256 instead of SHA1. Choosing algorithm
based on type in PolicyFetchResponses instead of hard coded SHA1.

Note that we now limited the behavior to CBCM policies which is the
only policy type that use google update cache. Other types will follow
once we have more testing.
