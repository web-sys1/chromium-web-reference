Correct the default value of selfBrowserSurface

The new media-picker order (tabs/windows/screens) is being rolled out
gradually. It is intended to nudge users away from sharing screens,
and towards sharing tabs, which prevents oversharing and significantly
improves privacy. More on this in the following articles:
* https://developer.chrome.com/docs/web-platform/screen-sharing-controls/
* https://developer.chrome.com/blog/avoiding-oversharing-when-screen-sharing/

However, an unintended side-effect of offering tabs first, is that it
also increases the rate with which users choose the current tab.
With very few exceptions, the current tab is the worst thing to share.
In video conferencing applications, this produces the hall-of-mirrors
effect, and worse - audio feedback loops leading to "howling".
Apps that *are* interested in self-capture,
use {selfBrowserSurface: "include"} and/or {preferCurrentTab: true}
to signal that to the browser.

We should therefore ensure that,
if neither {selfBrowserSurface: "include"} nor {preferCurrentTab: true}
were specified, the current tab should be excluded from the list
of tabs. This change will be rolled out gradually alongside the gradual
rollout of the new media-picker order.
