Path service: add method to override path with a maybe-invalid value

It's possible for some of the path to be relative or even invalid in
real life, however, those cases were not covered in test because path
service override ensure the validity of the path set.

This CL adds methods only for testing to override the path with a value
that might be invalid.

See the associated bug for more information.
