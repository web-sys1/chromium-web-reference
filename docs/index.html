Fix PiP crash from duplicate child dialog destruction notifications

This change fixes a Picture-in-Picture (PiP) window crash that occurs
during the destruction of a child dialog widget. The crash happens due
to a CHECK failure in
base::ScopedMultiSourceObservation::RemoveObservation.

The theory is that a race condition during widget teardown can cause
OnWidgetDestroying to be called twice for the same dialog. The first
call is triggered "manually" when the parent widget is notified of the
child's removal. A second call can still be dispatched from the child's
own destruction notification.
