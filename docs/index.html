PlzDedicatedWorker: fix crash in DevTools with Worklet

This CL fixes the crash in DevTools when Worklet is used by adding
ContextType enum and use it in
DevToolsRendererChannel::ChildWorkerCreated().

Previously, I assumed DevToolsRendererChannel::ChildWorkerCreated()
is used only for dedicated workers but it is actually used for worklets
too. When PlzDedicatedWorker is enabled, WorkerDevToolsAgentHost is
created in the browser process and the agent host is re-used in
DevToolsRendererChannel::ChildWorkerCreated().
However, for worklets, a new agent host should be created in
DevToolsRendererChannel. When PlzDedicatedWorker is enabled and the site
uses a worklet, `agent_host` is null and it makes the site crash.

This CL adds a new enum to DevToolsRendererChannel, `context_type`, to
tell DevToolsRendererChannel::ChildWorkerCreated() is called from
dedicated workers or worklets. When it's called from worklets, a new
agent host is always created.
