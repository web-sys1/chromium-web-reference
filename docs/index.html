Fix RenderFrameImpl::createChildFrame trace event.

Currently RenderFrameImpl::createChildFrame trace event doesn't cover
the entirety of the function with the same name as it wants to emit the
new routing id, which becomes available only after the completion of the
sync call to the browser.

Use a pair of TRACE_EVENT_BEGIN/END macros rather than a TRACE_EVENT to
cover the entirety of the function and emit the new routing id at the
end of the function.
