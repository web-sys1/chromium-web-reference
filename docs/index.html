Prerender2Fallback: Add debug metrics for prefetch ahead of prerender at matching

This CL adds debug metrics to understand the state of prefetch when a
prerender navigation fails and falls back to a regular navigation.
The metrics are recorded when `PrerenderURLLoaderThrottle` aborts a
prerender navigation.

The new metrics include:

- Whether a prefetch ahead of prerender existed.
- `PrefetchStatus`
- The servable state and matcher action
- The size of the prefetch queue and the index of the candidate in the
  queue.

These metrics will help diagnose issues with the prefetch/prerender
integration and improve its reliability.
