IDB: SQLite store - refactor migration code and add error handling

* Bubble up errors during migration.
* Move the BackingStore migration routine into sqlite::BackingStoreImpl.
* Migrate all databases before moving blob files for any one database.
  This way, an error in any database will abort the entire migration.
* Move all files at the end of migration, but ignore errors, including
  if the file was missing. This is a behavioral change from the
  migration code that exists currently, however it preserves existing
  behavior in the sense that LevelDB stores may be (and frequently are)
  missing blob files, and Chromium doesn't do anything about that except
  fail to read the missing contents. The migrated DBs will carry on with
  that behavior.
