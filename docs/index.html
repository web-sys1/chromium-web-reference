[tracing] Migrate async track events to more appropriate alternatives in media/cast

This CL changes async events on custom tracks (TRACE_EVENT_BEGIN/_END)
to more appropriate alternatives: either on-thread events connected with
flows, on synchronous events. In this case, the _END event was actually
missing outside the if condition. This CL fixed the missing _END.

There's a cost to using async events:
- async events take a lot of vertical space in tracing UI
- async events looses context of where the start and end happen
- custom track are tricky to use, and  Track::FromPointer is problematic because it can alias when pointers are reused.
https://chromium.googlesource.com/chromium/src.git/+/HEAD/docs/trace_events.md#asynchronous-events

As a general rule of thumb, synchronous events are more appropriate when
a slice is well nested in a function.
