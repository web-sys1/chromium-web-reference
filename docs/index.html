Fix <position> serialization for ellipse() and circle()

According to the spec[1], the <position> should be serialized preserving
keywords. These rules happen to match our canonical representation, so
just drop any pre-processing.

The computed value is also overly complicated, and we're dependent on
the serialization code to end up correct (0% vs. 0px). Adjust this too,
and always produce offsets from top-left. It should be possible to
simplify this in the future.

This matches Gecko and WebKit.

Guarded by "CSSShapeEllipseCirclePositionSerialization".

[1] https://drafts.csswg.org/css-values-4/#position-serialization
