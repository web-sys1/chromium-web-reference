ozone + wayland: Route color space through all of the overlay types

Ultimately, for HDR support in lacross, we need an accurate description
of the surface's color space from the renderer. Since delegated
compositing tries to upgrade quads into overlays, which roughly map into
surfaces, we need to copy that color space data all the way down from
viz to OzoneWayland.

This change just routes the data and makes it available to the window. A
follow-up CL (likely crrev.com/c/3430225) will use it with
zcr_color_management to send data over the wire to wayland.
