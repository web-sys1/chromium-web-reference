[a11y] Implement standardized kCollapse and kExpand for treeitem role

This CL set out to fix a bug where UIA::Expand and UIA::Collapse
were only sending click actions rather than actually expanding or
collapsing an element.

Since it's an ARIA best practice that `treeitems`
should respond to a right arrow event to expand (if collapsed)
and a left arrow event to collapse (if expanded), this CL changed
the corresponding UIA APIs to now send a `kCollapse` or `kExpand`
actions over to the renderer.
https://www.w3.org/WAI/ARIA/apg/patterns/treeview/examples/treeview-1b/

However, these were not implemented yet in the renderer side so
this CL adds implementation for these by simulating and sending
the appropriate keydown keyboard event.
