Fix <label for=..> inference as a feature.

This is crrev.com/c/3720655, but behind a feature flag. This way, the
legacy code cannot be cleaned up yet, making the CL significantly
smaller.

By extending the AssignedLabelSource histogram, we will be able to
measure the impact and whether more labels are inferred this way.

This logic exists separately in Javascript on iOS [1]. But as discussed
offline, due to the extra effort to accommodate the finch flag in
Javascript, this will be rolled out without iOS first.
