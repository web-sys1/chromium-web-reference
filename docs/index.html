Speculative fix for wheel-scroll-latching-on-scrollbar

This patch aims at addressing some of the source of test flakes
observed for this test.  We defer the start of the test until the
compositor is ready, and wait for loading to complete before measuring
positions. Though the end value of the scroll should precisely match
being integer valued, we occasionally see an error of less than a pixel.

At the time of the patch, this was the top scoring flake for
Blink>Scroll.
