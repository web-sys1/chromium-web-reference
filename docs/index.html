Have png_codec use tryAllocPixels to handle oom.

DecodeInfoCallback collects the png info and also attempts to allocate
space for the resulting image. This allocation may fail on Windows.
Instead of calling allocN32Pixels call tryAllocN32Pixels and handle any
error by calling png_error to abort loading the png. This allows the png
to simply not load instead of crashing the process.
