Original change's description:
> [mac] Fix child widget rendered behind its parent in fullscreen
>
> A child widget might be rendered behind its parent in fullscreen. When
> this happens, the child widget would receive the mouse event when
> clicking on its parent. Given the inconsistency between rendering and
> event routing this is likely caused by a bug in the macOS window server.
>
> It seems that this can be fixed by explicitly calling StackAbove() to
> place the child on top of its parent. I had an unsuccessful attempt
> to add a regression test because IsWindowStackedAbove() is not returning
> the correct result.
>
