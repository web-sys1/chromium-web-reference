Fallback to TabState if FlatBuffer limit is 0

As we transition from TabState to FlatBuffer backed
CriticalPersistedTabData we need to fallback to TabState
in the event that the CriticalPersistedTabData file doesn't
exist. This is typically signaled by the response from the
file read being null, however, we're seeing in the wild that
the response can be non-null and the limit is 0 in some
scenarios. This CL accounts for that.
