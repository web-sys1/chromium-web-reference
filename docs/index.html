[NTP][Panorama] Remove arrow key navigation for categories and themes

Currently, cr-grid does not support its arrow key navigation with grid
items that are of varying spans. It does a calculation of what to select
when switching rows by moving forward or back the same number of items
as there are columns, not taking into account how many columns each item
is taking up.

Categories, chrome colors, and themes do not need the arrow key
navigation, since they are not within any kind of grouping that would
make the user expect to move with arrow keys (such as a radio group). An
attribute was added to cr-grid to allow disabling the arrow key
navigation.

I thought about just using a div with display of grid. However, cr-grid
nicely does the Enter and space key handling for us already. This way we
don't have to put a copy of that handling in each ts file.

Also, padding was moved slightly for chrome colors between the header
and the content. It didn't match themes.
