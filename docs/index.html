Remember if window is on current virtual desktop.

Use the result of IsWindowOnCurrentVirtualDesktop in native window
occlusion tracker to avoid calls to IsWindowOnCurrentVirtualDesktop
in BrowserView::IsOnCurrentWorkspace. Calling the COM method on the
UI thread can be very expensive and cause jankiness.

NativeWindowOcclusionTrackerWin by its nature has to be very up to
date on whether Chrome windows are on the current virtual desktop or
not, and it does this off the UI thread.
