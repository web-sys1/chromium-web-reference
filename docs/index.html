personalization: preview mode a11y timing issue

While in fullscreen preview mode, ignore updates to currentSelected
wallpaper; the received attribution is for the old (non-preview)
wallpaper. This will cause the app to remain in a "loading" state
underneath the fullscreen preview. Then exit fullscreen first and
confirm or cancel preview second. Upon confirming or canceling, another
onWallpaperChanged call is received, ending the "loading" state and
displaying the correct wallpaper in the app. This makes the timing
consistent between local and online images, and causes a11y text
announcing current wallpaper to be read out consistently.
