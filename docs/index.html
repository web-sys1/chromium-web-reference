Init accessibility background tasks from BrowserMainLoop.

BrowserAccessibilityStateImpl was kicking off some background tasks on
construction (collecting histograms, monitoring state changes), which
was causing problems for tests that need BrowserAccessibilityState
but don't need any of those background tasks (e.g. this revert -
http://crrev.com/c/2777473).

Solve this by requiring an explicit call to kick off these background
tasks. Now they're triggered by content::BrowserMainLoop,
so they won't be triggered by tests by default.

I also added a new test just to ensure that the code that kicks off
background tasks gets some test coverage.
