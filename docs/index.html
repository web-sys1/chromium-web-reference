Create line_layout_item.cc with non-trivial logic from header

line_layout_item.h is included from so many places. So modifying one of
the headers included by it is slow to recompile.

Therefore, this patch removes most of these included headers, and moves
the code that needs them into a new .cc file.

For example, when modifying position_with_affinity.h, this reduces the
number of units that need to be recompiled from 810 to 191. Or from 758
to 37 when modifying layout_object_inlines.h
