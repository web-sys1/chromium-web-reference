Strip Authorization header on cross-origin redirect in KeepAliveURLLoader

KeepAliveURLLoader::EndReceiveRedirect() passes removed_headers={} when
following a cross-origin redirect, so the Authorization header is never
stripped. This violates Step 13 of the Fetch spec's HTTP-redirect-fetch.
Normal (non-keepalive) fetches correctly strip it renderer-side in
resource_loader.cc, but keepalive fetches bypass the renderer for
redirect handling entirely when the page is unloaded.

Add a url::IsSameOriginWith() check before updating the request URL,
and pass Authorization in removed_headers when the redirect is
cross-origin.
