[white-space] Fix `EditingStyle`

This patch fixes `EditingStyle` when the `white-space`
property becomes a shorthand.

This class computes styles for copy&paste clipboard operations,
or when elements are duplicated by inserting paragraphs. The
`white-space` property is special that:
* It needs to be copied even when the elements to copy don't
  include it, because it affects how space characters are
  represented.
* When inserting a tabulation character, wrap it with a span
  with the `white-space` property to preserve it.

This patch fixes following tests when `CSSWhtieSpaceShorthand`
runtime flag is enabled:
```
editing/pasteboard/dragstart-contains-default-content.html
editing/pasteboard/onpaste-text-html.html
fast/events/ondrop-text-html.html
external/wpt/clipboard-apis/async-unsanitized-html-formats-write-read.tentative.https.html
WebFrameTest.SmartClipData
WebFrameTest.SmartClipDataWithPinchZoom
```

Note, changing the order of properties in
`kStaticEditingProperties` affects the order of properties in
expected style strings, so they are adjusted, but the set of
CSS properties are not changed.
