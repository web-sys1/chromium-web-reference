DevTools: respond with error to messages with unknown session id

In some scenarios with nested OOPIFs it is possible that parent OOPIFs
session is detached and client will never see Target.detachedFromTarget
for the child session (because its parent session has already been detached).
In scenarios like this it helps if the browser returns a protocol error
instead of silently swallowing the command.
