Reland "Ash: Close dialog on PrintPreviewDone & restore lifecycle test coverage"

This reverts commit 39abee05dce665fe5a959fb206ec431690073ec6.

Reason for revert: Reland with the original issue fixed. More
specifically, the production fix in the original CL was still correct
all along; the failure came from the newly added test setup, which never
created a real selection but still passed has_selection = true, hitting
a renderer DCHECK on Linux Chromium OS Asan LSan Tests. The lifecycle
fix itself is unchanged in this reland, and the test coverage is the
same aside from correcting that invalid selection assumption.
