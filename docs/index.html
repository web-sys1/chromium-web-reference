Add kHighEfficiencyModeEnabled migration code to an enum pref

This CL introduces kHighEfficiencyModeState, a replacement pref for
kHighEfficiencyModeEnabled that can represent more than 2 modes. This is
to prepare for an eventual new mode of operation for Memory Saver.

The pref isn't actually migrated in this patch. The migration function
will start being called in the same CL as the settings UI and
enterprise policy are switched to the enum pref.
