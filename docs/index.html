cros_healthd: Revisit fake Event API

This change is related to crrev/c/4151907, which introduced
a base::flat_map that holds the Mojo connections as values.
Apparently the RemoteSet that holds the connections
is neither movable, nor copyable, so in order to hold it in a
flat_map it was wrapped in a std::unique_ptr. This change removes
the std::unique_ptr and replaces the flat_map with a std::map.
A std::map provides a faster insert and access (O(log n)), than the
flat_map (O(n)) and only has a negative effect on the memory
footprint if it contains a lot of elements. We only put
five elements in the map, so this is arguably better than the
std::unique_ptr hack.
