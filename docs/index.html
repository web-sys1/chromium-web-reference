Remove dependency from WebAppRegistrar to OsIntegrationManager.

As per the docs in WebAppSyncBridge,

// While WebAppRegistrar is a read-only model, WebAppSyncBridge is a
// controller for that model.

As a model, WebAppRegistrar should not reach into controllers
such as OsIntegrationManager. Instead, it notifies observers
of interesting events, in this case via
AppRegistrarObserver::OnProfileMarkedForPermanentDeletion(), and the
logic that profile deletion implies uninstallation of OS hooks is
thus encapsulated in OsIntegrationManager.

This also removes a mutual dependency, which makes the code
incrementally more modular.
