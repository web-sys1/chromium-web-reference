Refactor GL bindings for Skia

There are few different cases when we need to wrap GL function before
passing it to skia to add addtinional functionality:
* Some calls (glClear, glDraw*) can be dropped for browser tests that
  don't care about pixels.
* Some slow calls are wrapped in ProgressReporter
* On Mac some calls need to be wrapped in glFlush()

All of those can exists in combinations.

Additionally, we'd like to add check that those functions are called on
valid GL context (not part of this CL), which increase number of
combinations.

This CL tries to change recursive approach of wrapping code to more
declarative one for easier extension and improved readability.
