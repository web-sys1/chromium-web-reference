[Bookmarks Bar] Update TopToolbarOverlayMediator to account for Bookmarks Bar

With this CL we add a temporary workaround to offset the Toolbar layer
by the Bookmarks Bar height in order to get a correct positioning. The
Toolbar was written to always be the bottom of the Top Controls, but
this is incorrect now that we have the Bookmarks Bar present. A full fix
of this will require the TopControlsRefactor to be implemented so we can
control offsets via the TopControlsStacker, and ideally the snapshot
bounds update work to be completed as well. To unblock the Bookmarks
Bar, we add a Supplier to the TopToolbarOverlayMediator so it can get
the current bookmarks bar height so that the Toolbar can adjust its
offset.
