ime: Don't send OnActivate to extension if system PK is enabled.

System PK only forwards events to the IME service except one case:
OnActivate, where it sends it to the extension as well.

This was done as a precaution since OnActivate is a special event that
is sometimes used to wake the extension from sleep.

However, sending OnActivate can cause the extension to run indefinitely
while system PK is active (e.g. the extension might install a NaCl
module, which keeps it alive forever).

Therefore, don't send OnActivate to the extension when forwarding
events to IME service. Theoretically there should be no side effects.
