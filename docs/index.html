> [//gpu] Have ArcSharedImageInterface properly bounce ops to GPU thread
>
> ArcSharedImageInterface runs on the IO thread, but it needs to make
> SharedContextState current on the GPU thread. This CL has it do so:
>
> * It takes in a task runner for the GPU thread
> * It bounces all operations to the GPU thread
> * It maintains an atomic_bool that allows it to signal errors on the
>   GPU thread and then pass these errors to the client on the next
>   operation initiated by the client (on the client's thread)
