[PolicyLogs] Logger Now Deletes Old Logs

To delete unneeded logs in in the list for memory purposes, a deletion
task is posted for logs in the PolicyLogger.

When a new log is added, the logger will check if a deletion task is
already scheduled. If not, a delayed task will be posted. Otherwise, no
task will be posted. When the delayed callback runs, the logs older
than 30 minutes will be deleted. After that, if there are still logs
in the list, another deletion task will be rescheduled.

Note: a log can live for up tot 59 minutes under this logic.
