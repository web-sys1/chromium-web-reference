[omnibox] Don't run open tab provider for empty input.

Previously, the open tab provider was still being ran for empty inputs
in keyword mode, after the keyword was stripped.  This resulted in the
"No Results Found" message appearing erroneously when the user would
type a search query in @tabs mode, then delete their query but stay in
@tabs search mode (be in @tabs mode, but have no additional input).

This CL solves the issue by adding a check and early return for empty
searches in keyword mode.
