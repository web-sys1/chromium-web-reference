Revert "Pass empty path when writing bookmarks to clipboard on macOS."

This reverts commit f7d0665db1d166b55c7541a04ba406afe0028f6d.

Reason for revert: produces duplicates for drag and drop
operations in the bookmark manager (crbug.com/510757566).

Original change's description:
> Pass empty path when writing bookmarks to clipboard on macOS.
>
> Writing `profile_path_` in the `BookmarkNodeData::WriteToClipboard`
> function of bookmark_node_data_mac.mm is unnecessary and exposing the
> user's local path to the clipboard. It is recommended to modify it to
> write an empty path to align with non-macOS platforms.
>
> Drag and Drop operations do require `profile_path_`. During drag and
> drop, Chromium needs to determine whether the drag source originates
> from the same Profile: if it is from the same Profile, a Move operation
> is typically performed; if it is from a different Profile or the path is
> empty, a Copy operation is performed. The relevant logic is in the
> `BookmarkUIOperationsHelper::DropBookmarks` function.
>
> Copy/Cut/Paste operations that call `BookmarkNodeData::WriteToClipboard`
> do not require `profile_path_`. By design, the Paste operation
> `BookmarkUIOperationsHelper::PasteFromClipboard` always enforces copy
> semantics and never move semantics, so the paste logic does not care
> about the Profile path of the data source.
>
> Non-macOS platforms (Windows/Linux/ChromeOS/Android) explicitly write an
> empty base::FilePath() in the implementation of
> BookmarkNodeData::WriteToClipboard in bookmark_node_data.cc. Thus, macOS
> should write an empty path to align with other platforms.
>
