[ozone/wayland] Enabled tests that needed XKB support.

Initially it was assumed that XKB is not set up in the configuration
used on the test bots.  However, it turned out that the real issue is
related to threading.  The keymap is supposed to be set up at the stage
of initialisation, and when the standalone Weston is used,
XkbKeyboardLayoutEngine::SetKeymap() is called early, before entering
the test body.  However, When the bundled Weston is used, that call is
delayed and happens *after* the test body (and the test fails because it
tries to use XKB before it has got the state).

This patch fixes the issue by adding a closure that the keyboard layout
engine calls back when it is ready to accept calls that require the
keymap.  Tests that require XKB may now wait until it is ready.
