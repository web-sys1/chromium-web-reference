Create FirstPartySetsAccessServiceFactory for per-profile policy.

This change is a result of the FirstPartySets Overrides Policy changing from being on the local_state to per_profile.

This CL:
- Registers the First-Party Sets Overrides policy as a per-profile
  policy and updates the policy_templates description and tests.
- Updates FPSHandler interface + implementation to expose a
  GetCustomizationForPolicy method which will be called by
  FirstPartySetsAccessServices (one per profile).
  - Once the lists of First-Party Sets is initialized, the
    customizations are computed using ComputeEnterpriseCustomizations,
    and the customizations are sent back to these services via callback.
- Re-enables browser tests that were temporarily disabled.
