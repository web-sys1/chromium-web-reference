cros: Make FocusChangeObserver checked observer

This converts `FocusChangeObserver` into checked observers. With
this, dangling pointers of `focus_observers_` in `FocusController` would
be errored out more verbosely.

There is also a dangling pointer in browser test
`DefaultEnableFullRestoreAndGhostWindow`. The pointer is lost here
because `exo::WMHelper` is released before the dtor of
`CrostiniUnsupportedActionNotifier`(here for more context [1]). More
specifically, `TearDownOnMainThread()` is called before dtor of
`CrostiniUnsupportedActionNotifier`. The fix is to make the test fixture
a `ash::ShellObserver` so than when `OnShellDestroying()`, it releases
`exo::WMHelper`, and the focus observer can get unregistered.

[1] https://source.chromium.org/chromium/chromium/src/+/main:chrome/browser/ash/crostini/crostini_unsupported_action_notifier.cc;l=210-212
