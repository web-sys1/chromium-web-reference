Update PipeWire stack

Introduce PipeWire 1.4 as a replacement for current PipeWire we have in
version 0.3.56. The new version is API/ABI compatible, but we keep the
old version as an intermediate step to avoid breaking PipeWire tests in
WebRTC.

Add Wireplumber as a new PipeWire session manager, because the original
PipeWire Media Session serves as a reference implementation and is not
suitable to be used in the long term. All the Linux distributions use
Wireplumber and it's the only session manager we can use currently to
extend tests for PipeWire camera support. The old PipeWire Media Session
is again kept as an intermediate step so we don't break current tests.

Also add a newer version of Glib as a CIPD package, because Wireplumber
is using Glib API that is not available in the Glib version available on
the CI.

All the new packages are not pulled into chromium/src, they are pulled
in as generated CIPD packages through DEPS (used only in WebRTC).
