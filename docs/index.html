Don't load non-stored types from AddressAutofillTable

When writing an address to the database, it is persisted as a list of
(type, value) pairs, for every types in
GetDatabaseStoredTypesOfAutofillProfile().

When reading a profile, an address is constructed by calling
SetRawInfo() for every type found in the database, independently of
what GetDatabaseStoredTypesOfAutofillProfile() is.
Since GetDatabaseStoredTypesOfAutofillProfile() is not explicitly
checked, the code might call loading code downstream with non-stored
types. This can in turn lead to CHECK() failures, crashing Chrome on
startup [1]. This is possible e.g. when the database was tampered with
by external means, or when a stored type becomes non-stored.

This CL checks that if the loaded types are stored types and discards
the others.

[1] https://source.chromium.org/chromium/chromium/src/+/main:components/autofill/core/browser/profile_token_quality.cc;l=299;drc=bf39f1210258bfe98ab635146d3417f537518842
