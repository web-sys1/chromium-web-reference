Sync presubmit: Warn if proto files were changed but not proto_visitors.h

Many changes to Sync's proto files (notably, adding a new field) should
come with a corresponding update to proto_visitors.h. This CL adds a
presubmit check for this.
It's a warning (rather than an error) since it's a simple check which
will produce false positives - it only checks if matching files were
modified; it doesn't check the contents at all.
