Handle case when user accidentally creates non-Parallels PvmDefault

"vmc start" by default creates Termina instance when VM with given name
does not exist. Users often do "vmc start PvmDefault" and then get
confused when Parallels installer fails on them. Correct this by
alerting users when diagnostic page locates PvmDefault but in a wrong
place.
