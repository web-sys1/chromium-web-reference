[GPU] Change EGLImageBacking to use ScopedEGLImage

EGLImageBacking currently uses NativeImageBuffer to indirectly create
and manipulate its EGLImage. This CL changes that usage to
ScopedEGLImage, which is directly suited for this use case.
Specifically:

- A ScopedEGLImage ivar now creates and holds the EGLImage, with its
  creation params being specified to be identical to what
  NativeImageBufferEGL::Create() previously did
- The code of NativeImageBufferEGL::BindToTexture() is now inlined
  into the callsite
- ScopedEGLImage destroys the EGLImage in its own destructor, as
  NativeImageBufferEGL does. One difference is that
  NativeImageBufferEGL caches the EGLDisplay that it was passed on
  construction (which was
  gl::GLSurfaceEGL::GetGLDisplayEGL()->GetDisplay())
  and uses that in the destructor, while ScopedEGLImage
  calls gl::GLSurfaceEGL::GetGLDisplayEGL()->GetDisplay() again.
  eglDestroyImageKHR() will fail if the EGLDisplay passed to it is
  not the same one with which the image was created. Fortunately,
  gl::GLSurfaceEGL::GetGLDisplayEGL() is set on the first call and
  never subsequently changed, and GLDisplayEGL::GetDisplay() returns
  a value that is set in GLDisplayEGL::InitializeDisplay() and never
  subsequently changed.

This CL also deletes native_image_buffer.*, which now has no usage.
