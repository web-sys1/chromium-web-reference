Increase the number of log instances to be copied for test failures.

CopyLog(), which copies the CAS outputs for integration tests, can be
called multiple times during a test execution. It is called once
for each Uninstall() call, then it is called once more during
TearDown(). Then, this is done for each test run and retry in case
of failure.

More updater.log instances are needed to be copied. Note that in
case of test success, the updater log is only saved one time.
