Original change's description:
> [Autofill] Added execution control to captured-site tests.
> 
> This CL gives simple control over the execution of captured-site-test
> recipes: the recipe can be paused, unpaused, stepped through, and
> inspected.
> 
> The commands are read from the file specified in the
> --command_file switch, which typically should be a FIFO special file
> (named pipe). (That is, the file is re-opened and -read once the
> end is reached.)
> 
> Typical usage is like this:
> 
> $ mkfifo $PIPE
> $ .../captured_sites_interactive_tests --command_file=$PIPE ... &
> $ echo "next 4" >$PIPE      # execute 4 actions
> $ echo "show -4" >$PIPE     # show last 4 actions
> $ echo "run" >$PIPE         # resume execution
