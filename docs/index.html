Add a stub ScreenshotContentProvider.

This change introduces a new ContentProvider,
`ScreenshotContentProvider`, and its implementation,
`ScreenshotContentProviderImpl`.

The provider is declared in the AndroidManifest and added to the build.
Currently, all methods in `ScreenshotContentProviderImpl` are stubs,
returning null, zero, or throwing an exception. Unit tests are included
to verify the stub behavior. The provider is set up using
`SplitCompatContentProvider`.
