Update animation flags when calculating animation updates

Currently, we apply the pending animation update first, then inspect
the resulting Animations and set flags on ComputedStyle accordingly.
This is not compatible with the planned solution for animated container
queries, because the animation update will be postponed until after
layout has finished.

Hence this CL changes the calculation of those flags to use the
information on the CSSAnimationUpdate, before it's actually applied.
