[Fast Pair] Extend timeout for same device found

If the same device is discovered by the scanner, extend the notification
timeout if it is the device for which the notification is already
showing for. Adds logic to the Mediator to check if the device found by
the FastPairScanners for initial or subsequent pairing or
RetroactiveDeviceFound for retroactive pairing matches the stored device
for the device corresponding to the notification currently shown to the
user. If so, triggers showing the notification via the UiBroker. The
FastPairNotificationController checks if there is already a timer
running, and if so, this means that the notification is already shown
(since the timer is stopped when the notification is closed) and we
keep track of which device is for which notification in the Mediator to
enforce first come first service notifications. If the timer is running,
reset the timer to extend it.

TESTED=Verified on DUT that if the device is found again by the
scanners and is already being shown via a notification, the
timeout is extended. Made sure the timeout didn't happen after the
initial 12 seconds using JBL 125TWs.
