[//gpu] Set RenderAttachment in ExternalVkImage Dawn SI rep if needed

The ExternalVkImage Dawn rep passes the layout of the backing's Skia
backend texture in the ExternalImageDescriptor that is used to create a
Dawn texture wrapping the underlying memory [1]. This layout
can be VK_IMAGE_LAYOUT_COLOR_ATTACHMENT_OPTIMAL. For a Vulkan-backed
Dawn texture to be usable with this layout, it must have
RenderAttachment usage (see [2] for examples of test failures that occur
without the RenderAttachment usage set). This CL adds that usage to the
texture's internal usages in this case when using client-specified
internal usages.

Note: This is clearly a workaround. I don't know at this time what the
proper long-term solution here is.
