Fix computed style for highlights under HighlightInheritance

We were passing the originating element style in
Element::EnsureOwnComputedStyle() instead of the parent highlight style.

This patch fixes that under HighlightInheritance,
so we pass the parent highlight style.

This patch modifies highlight-pseudos-visited-computed-001.html test
so it doesn't depend on currentcolor. We already have a different test
to check that highlight-pseudos-currentcolor-visited-computed-001.html.

This fix makes us fail some tests, because getComputedStyle()
for currentcolor is not returning the expected value for highlight pseudos.
This will be fixed as part of crbug.com/1295264.
