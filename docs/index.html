Move SetDisplay() into SetBoundsOrigin()

(This commit is to address a comment from crrev/c/3670321.)
Currently when toast_surface_set_position() is called, we trigger
SetDisplay() then SetBoundsOrigin(). Considering the |origin| passed
into SetBoundsOrigin() is only valid on that specific display, it's
better to SetDisplay() inside SetBoundsOrigin(). It also aligns with the
behavior of SetBounds(), where we update the display and bounds together.
