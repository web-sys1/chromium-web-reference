multipaste: Consolidate update events to the virtual keyboard

This CL combines the virtual keyboard "clipboard history changed" and
"clipboard item updated" events into a single event that always causes
the entire clipboard history state to be re-queried. There are at most
five clipboard history items at any time, so this should not have
performance implications; the purpose of this change is just logical
simplification.
