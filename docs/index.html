Fix race condition with GPU channel establishment

In Edge we found a class of issues where renderer processes would run
out of memory with Gpu::EstablishGpuChannelSync() on the stack.

Inspecting some of these crashes, the main thread task queue was running out of memory due to accumulating tasks. This is caused by EstablishGpuChannelSync blocking on the event for GPU channel establishment making no progress on the main thread tasks.

The renderer IO thread that can signal the gpu channel established event, in viz::Gpu::EstablishRequest::OnEstablishedGpuChannel executes on a EstablishRequest object that doesnt have the establish_event_ set on it.

The root cause of this is that the EstablishRequest is posted to the IO thread before setting the SetWaitableEvent on the event and there is no lock preventing the request from executing right away on the IO thread before SetWaitableEvent is called on the main thread.
