[Remoting] Remove unnecessary GTK dependency

This was causing GTK to be a dependent library of many test targets
since it gets pulled in via //ui/display:test_support. Sanitizer builds
are linked without -Wl,--as-needed, so this caused a hard GTK3
dependency, which caused GTK4 loading in //ui/gtk to fail.
