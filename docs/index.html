[A11y] Remove range control values from the name on Android

On Android, the kValue string attribute for range controls (e.g.,
sliders, progress bars) is currently prepended to the element's
accessible name. This pollutes the name with state information.

The Android text property is meant to represent the element's primary,
stable name (its label), while the value is a dynamic state.

Textual values (from aria-valuetext, e.g., "Medium") are moved from the
name to the state description. Numerical values (from aria-valuenow) are
removed from the name and we now correctly rely on the native RangeInfo
to handle them -- the removal from the text property prevents redundant
announcements.
