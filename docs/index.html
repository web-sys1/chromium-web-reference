Enable anchor constraints enforcement on user added roots.

Due to the potential for breakage, there is a temporary enterprise
policy to opt-out, as well as a temporary feature flag kill switch.

Enforcing the anchor constraints matches what the platform verifiers
were doing on Mac and Windows before we switched to the builtin
verifier, so the potential for breakage there should be fairly low.

On Linux and ChromeOS we've been using the builtin verifier for a long
time without enforcing these constraints, so there is a somewhat higher
chance of breakage there.

The policy is mapped to local_state as it controls a singleton that
controls all interactions with the trust store. It is not necessary for
this policy to be usable during the ChromeOS login screen as no NSS
user added certificates are available at that point.
