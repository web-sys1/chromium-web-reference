[editing] Avoid DCHECK failure in ApplyStyleCommand::RemoveInlineStyle

It used to have this assert:

  DCHECK(node == end.AnchorNode() || !node->contains(end.AnchorNode()))

which would fail when crossing an ignored content boundary like

  ^<select multiple><option>|</option></select>

where

  EditingIgnoresContent(*body) == false;
  EditingIgnoresContent(*select) == true;
  EditingIgnoresContent(*option) == false;

This patch removes the assert and just stops iterating if the end node
is inside the ignored content.

Bug: 1239729
