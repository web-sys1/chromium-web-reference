Skip PaintChunksToCcLayer::UpdateLayerProperties on fully cached repaint

This patch skips PaintChunksToCcLayer::UpdateLayerProperties when the
paint chunks are all moved from a subsequence. This is valid because
the data on PaintChunk is only set through paint, and it must be
unchanged if the PaintChunk was moved from a subsequence.

The callsite in PaintArtifactCompositor::Update is more expensive but is
not trivial to remove because we would need to check both that the
chunks were cached, and that the property tree states are unchanged.

represents 0.1% of all non-idle renderer CPU time on Android:
  https://uma.googleplex.com/p/chrome/callstacks?sid=376c740cefc4e6f1bfd6afece419b29a
And 0.08% on Windows:
  https://uma.googleplex.com/p/chrome/callstacks?sid=b3f9af4945a64cb1216ff1fee5afcc7c
