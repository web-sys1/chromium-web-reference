android: Use configuration for display size on S

Adapted from
https://chromium-review.googlesource.com/c/chromium/src/+/3206114
by ctzsm@

This is originally driven by the deprecation of Display#getRealSize and
Display#getRealMetrics but should also be fixing a few issues as well,
bug may introduce potential issues as well.

Changes on S+:
Use ComponentCallbacks#onConfigurationChanged on a window Context to
listen to "window" configuration changes (in addition to
onDisplayChanged).
Use size from WindowMetrics#getMaximumWindowMetrics for display size,
and Resources#getDisplayMetrics to get DisplayMetrics. The latter fixes
crbug.com/1251681.
The rest of the info is still computed from Display obtained from
Context#getDisplay, which hopefully is consistent with the info above.

Potential problems:
It's not clear all the information obtained from separate sources are
consistent, though simple testing on a Pixel 6 shows they are.

Bug: 1241562, 1251681
