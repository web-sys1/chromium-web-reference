dPWA: Web App Settings - hide page until data is available

Initially hide the contents of the Web App Settings page, and only
reveal it after the data for the app is available. This avoids
flashes during layout:

* permissions items, which unhide their shadow root and cause the
  layout to change (in the future, file handling controls will
  behave this way as well)
* other rows, which don't affect layout but show text and toggles
  only after data is available

This also allows us to remove code from Window Mode and Run On Os
Login which hides the contents of the row until data is ready so
as to avoid showing an animating toggle control. Since the whole
page is hidden when data is unavailable, we don't need this code
to be implemented on an individual item basis. In the future,
should optional display be required, these elements should follow
the precedent set by pin_to_shelf_item.js.

Also fix a missing </div> tag in app.html.
