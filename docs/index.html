Defer cross-document view-transition until existing transition finishes

This is a naive first implementation of allowing two-phase
view-transitions.

The cross-document view-transition will stall until an existing
view-transition that is already animating will finish.

The final version of this will likely be tweaked:
- only allow animations that are about to be finished in <N millis.
- Allow regular animations and not just view-transitions
- Allow deferring commits also when there is no cross-document view
  transitions.
- Limit to animations that started as a result of the nav

But this initial version allows us to check whether this experience
makes sense in the first place and to reason about what heuristics we
should pursue.

See explainer: https://github.com/w3c/csswg-drafts/blob/main/css-view-transitions-2/two-phase-transition-explainer.md
