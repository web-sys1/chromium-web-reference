fixup! MPArch: Ensure NavigationPredictor work only on the primary mainframe

The original CL ensured that non-primary pages doesn't create
NavigationPredictor where links cannot be clicked via the test
case and DCHECK with content::Page::IsPrimary method.

However, IsPrimary can return a false on bfcache, which can work
on primary when navigating back/forward user behavior. But, the
original CL only tested it using the prerendering simulation
because the bfcache test simulation is not ready to test yet.

So, this CL changes the DCHECK from content::Page::IsPrimary
to IsPrerendering until bfcache test is added.
