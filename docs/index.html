mobile_config.gni: Stop unsetting target_platform on host toolchains

Remove the part of https://crrev.com/c/6306066 that set
`target_platform` to an empty string when the host toolchain was being used.

The idea of preventing people from writing wrong GN code is good, but
there are not a lot of places that need to check this variable at the
moment and, most importantly, checking this value on the host toolchain
is the only way to know that this is a build targeting tvOS. This is
exactly what we need to do in V8's gni/v8.gni so that we always set
`v8_enable_lite_mode` on all toolchains.
