[Office upload] Find what caused ODFS open flow ACCESS_DENIED error

If a file fails to open from ODFS with an ACCESS_DENIED error, this
could be due to reauthentication to OneDrive being required or another
access error. If reauthentication is required then a special error
notification is shown, otherwise a generic access error notification
is shown.

To determine if reauthentication is required, get the reauthentication
required state by doing a special getCustomActions requests of the root
directory (which gets ODFS metadata).

Add two OneDriveTest tests:
- Expect open to fail due to reauthentication to OneDrive being
required.
- Expect open to fail due to some other access error.

Use FakeProvidedFileSystemOneDrive in the tests and update it to mock GetActions to fail for a non-root entry with a given error, in
this case ACCESS_DENIED, so the open fails with ACCESS_DENIED.
