PlzDedicatedWorker: exclude blob url workers from UseCounter

This CL adds a new UseCounter, counts the number of dedicated workers
that will not be controlled by any service workers after
PlzDedicatedWorkerand the dedicated worker is NOT a blob URL worker.
It's ok to exclude blob URL workers from the UseCounter because a blob
URL worker inherits the URL of its parent document and use it for scope
matching. If we allow blob URL workers to be recorded, it's always
recorded.

Also adds the UseCounter to the UKM feature to analyze sites that
records the UseCounter.
