Make ComputeNGCaretPosition() to handle upstream position after ZWS

This patch changes |ComputeNGCaretPosition()| to return position before
ZWS for upstream position after ZWS to have different lines when line
breaking after/before ZWS for having valid selection affinity.

In the test[1], we expect to have upstream position, but we get
downstream position. Because |InSameLine()| returns false for
upstream and downstream after ZWS position.
