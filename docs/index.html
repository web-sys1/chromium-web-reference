[initial-letter] Change NGExclusionSpace:: InitialLetterClearance() API

This patch changes `NGExclusionSpace:: InitialLetterClearance()` to
take `EClear` and get rid of taking `EClear` and `EFloat` to
make clear about what initial letter clearance used and preparation
of implementing initial letter layout[1].

API after this patch:
  * InitialLetterClearance(EClear) -> LayoutUnit
  * InitialLetterClearance(EFloat) -> LayoutUnit

API before this patch
  * InitialLetterClearance(EClear, EFloat) -> LayoutUnit
  * InitialLetterClearance() -> LayoutUnit

Note: This patch is a follow-up of the CL[2] which introcued API
`InitialLetterClearance()`.


Usage example:

In FindLayoutOpportunityForFloat()
...
  const LayoutUnit clearance_offset =
      std::max({exclusion_space.ClearanceOffset(clear_type),
                exclusion_space.InitialLetterClearanceOffset(float_type)});

  AdjustToClearance(clearance_offset, &adjusted_origin_point);

  return exclusion_space.FindLayoutOpportunity(
      adjusted_origin_point, unpositioned_float.available_size.inline_size,
      inline_size + fragment_margins.InlineSum() /* minimum_inline_size */)
```

```
In NGBlockLayoutAlgorithm::FinishLayout()
```
  // If the current layout is a new formatting context, we need to encapsulate
  // all of our floats.
  if (ConstraintSpace().IsNewFormattingContext()) {
    intrinsic_block_size_ = std::max(
        {intrinsic_block_size_, ExclusionSpace().ClearanceOffset(EClear::kBoth),
         ExclusionSpace().InitialLetterClearanceOffset(EClear::kBoth)});
  }
```

In NGBlockLayoutAlgorithm::LayoutNewFormattingContext()
...
  if (!IsResumingLayout(child_break_token)) {
    // The origin offset is where we should start looking for layout
    // opportunities. It needs to be adjusted by the child's clearance.
    AdjustToClearance(
        std::max(ExclusionSpace().ClearanceOffset(child_style.Clear(Style())),
                 ExclusionSpace().InitialLetterClearanceOffset(EClear::kBoth)),
        &origin_offset);
  }
```
