Implement event-trigger-data matching during attribution

https://github.com/WICG/conversion-measurement-api/blob/main/EVENT.md#optional-attribution-filters

The trigger data, priority, and dedup key associated with the
event-level report are determined by the first matching event trigger
data entry. If none match, defaults of 0, 0, and null are used.

Note that it is safe to remove
AttributionTrigger::EventLevelResult::kNoMatchingEventTriggers and reuse
its integer value because that enum has never been deployed and
therefore no metric with that value has been recorded.
