Justify RAW_PTR_EXCLUSION for TranslateMediaSubtypeToPixelFormat

The C++ reference member in TranslateMediaSubtypeToPixelFormat only ever
points to constant global values which are statically allocated and
never freed. Thus it cannot ever become a dangling pointer, so we don't
need to replace it with a raw_ref. Since this struct is defined locally
inside the function, it is very unlikely it will ever point to any other
kind of memory.
