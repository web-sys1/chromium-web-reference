Fix dangling pointer warnings in CdmAdapter tests

CdmAdapterTestBase::cdm_helper_ references an object that is owned by
CdmAdapterTestBase::cdm_. Proper ordering of these two member variables
would normally keep cdm_helper_ from being a dangling ptr. That ordering
is now fixed in this CL, meaning any subclass that doesn't modify cdm_'s
lifetime should get the correct member destruction order by default and
avoid dangling pointers.

The CdmAdapterTestWithClearKeyCdm subclass does modify the default
destruction order though, in order to satisfy requirements with its
helper_ member. Adding a ClearCdm() method allows subclasses to clear
cdm_ (and related CdmAdapterTestBase state) and maintain the needed
ordering with their own subclass state.
