Fix manual testing importer.

Originally, the manual testing importer only supported importing
AutofillProfiles from a JSON file. crrev.com/c/4401107 added support
for credit cards. The intention was that importing should fail if
neither a "profiles" nor a "credit-cards" key specified in the JSON.
However, due to a bug both are currently required:
- DataModelsFromJSON<T>(json_array) receives the "profiles" or
  "credit-cards" array extracted from the JSON. It returns an optional
  vector of Ts. nullopt means that parsing failed.
  If no key was specified and json_array is null, the intention was to
  return an empty vector, indicating that parsing trivially succeeded.
- Later, in SetData(), it is verified that parsing of profiles and
  credit cards did not fail using CHECK(...profiles.has_value() && ...
  credit_cards.has_value()).

The bug is caused by the "return an empty vector":
absl::optional<std::vector<T>> DataModelsFromJSON(
   const base::Value::List* const json_array, ...) {
  if (!json_array) {
    return {};
  }
  ...
}
Here, "return {}" doesn't return an empty vector, but a default
initialized absl::optional<>. It has value absl::nullopt.
As a result, both a "profiles" and a "credit-cards" key is required.

This CL fixes the problem by explicitly returning a vector.
