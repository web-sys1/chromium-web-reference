[PartitionAlloc] Report syscall count and duration.

On non official builds, count how many memory management system calls
are made, and report their total duration. The aim of this is to
diagnose cases where PartitionAlloc is making too many syscalls.

For instance, on Linux, running Speedometer 2.0 makes >100k system
calls, which is a lot, particularly on operating systems with slower
memory management calls than Linux. This is important for PartitionAlloc
beyond its malloc() use case.

As the instrumentation is per-PartitionRoot, we do not instrument
PageAllocator, but the points where calls to PageAllocator (or
AddressPoolManager) are made.
