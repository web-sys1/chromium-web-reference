Make container query body propagation more robust

We don't require an <html> root element for direction, writing-mode, or
overflow, but we do for background and other propagated properties.
There is no plan to propagate the former ones from container-query
dependent body styles due to circularity, but don't check the type of
the root element anyway. This also removes unnecessary updates if we
have <html> elements in the document which are not root elements.

Removed left-over font-size declaration in test.
