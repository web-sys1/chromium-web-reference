[iOS] Implement sign-in promo with bookmark account storage

This patch adds `signInOnly` property in `SigninPromoViewMediator`.
When the property is YES, the mediator is in charge to show
IdentityChooserCoordinator if the user tapped in "Chooser an account"
button.
And the mediator is in charge to trigger the sign-in only flow once
the user has selected their account or once the user taps on
"Continue as xx" button in the sign-in promo.

The sign-in promo needs to be kept on the screen as long as the
sign-in flow is in progress.
