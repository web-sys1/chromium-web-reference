[base] Don't use fwrite() to log to stderr

On Windows, we see crashes where fwrite() to stderr calls malloc()
internally (from the CRT), and then crashes due to OOM. To prevent that
from happening, switch to write() directly for these calls.

It's unclear when fwrite() would allocate, but on other platforms it may
happen the first time a given FD is written to using buffered IO.

See linked bugs for example stack traces.
