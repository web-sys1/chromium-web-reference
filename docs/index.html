Extensions: Allow extension updates to be downloaded in the background.

This allows the update engine to choose the best download strategy for
an update, potentially allowing for resumption of interrupted downloads
or for deferral to OS download services to conduct the update.

It also simplifies the code, which is useful as it will soon be
refactored.

If I recall correctly, we didn't allow background downloads at the time
we migrated code to the new extension updater because we didn't want to
rewrite the tests to not use URLInterceptors. This CL also doesn't
change the tests, but instead forces the use of the URLFetcher stack
in tests via the extension updater's update_client configurator.
