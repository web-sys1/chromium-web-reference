LacrosTablet: Get tablet state on TableState initialization

This CL fixes the crash happening when Lacros tries to create
NonClientFrameView too early while tablet mode is enabled.

The crash is caused by early call of DisplayObserver that is introduced
by crrev.com/c/3881753.
This CL is for not to drop the tablet state given before TabletState
initialization, but we remove this observer call inside AddObserver
to resolve crash issues.
Instead, we cache tablet state value and let TabletState get the
cached value.
