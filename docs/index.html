MPArch: MediaEngagementContentsObserver::ReadyToCommitNavigation uses the outermost mainframe's URL for sub frames

ReadyToCommitNavigation method has used WebContents::GetLastCommittedURL
for subframes. This CL makes ReadyToCommitNavigation method do nothing
in prerendering. And, it makes subframes including fenced frames use the
outermost's GetLastCommittedURL instead of WebContents's
GetLastCommittedURL so that fenced frames avoid using their own
mainframe's URL.

The added tests check that a fenced frame or prerendering sends an
engagement level to the render frame to make sure the behavior by
checking if SendEngagementLevelToFrame method calls AddAutoplayFlags.

TESTS: MediaEngagementContentsObserverPrerenderBrowserTest.
       DoNotSendEngagementLevelToRenderFrameInPrerendering,
       MediaEngagementContentsObserverFencedFrameBrowserTest.
       DoNotSendEngagementLevelToRenderFrameOnFencedFrame
