Fix EditingAlgorithm::CaretMaxOffset to take first-letter into account

When EditingAlgorithm<Traversal>::CaretMaxOffset is called on a text
node, it is supposed to return the offset into the text node of the
last position where the caret could go in the node. However, this does
not hold when some or all of the text in the node is part of a
::first-letter pseudoelement. In that case, the return value was an
offset into the non-first-letter part of the node. This change fixes
that.
