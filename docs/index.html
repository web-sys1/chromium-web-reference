usb: Ensure isochronous packet count is not overwritten

When initializing a new Transfer the internal usbdevfs_urb struct is
zeroed out however the custom new operator which allocates space for
the usbdevfs_iso_packet_desc array has already initialized the
number_of_packets field.

This change moves zeroing the struct to the custom new operator so
that it happens before setting this field.
