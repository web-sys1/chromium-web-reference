ime: Anchor multiword window based on caret position not composition.

Previously, the multiword assistive window was anchored to the
composition position rather than the caret position. This was due to a
misunderstanding of the UX specs.

Multiword assistive windows should follow the caret position, with some
horizontal offset based on the "confirmed length" of the top suggestion.
It shift leftwards so that border between the "confirmed" part and the
"prediction" part aligns with the caret.

See http://b/233264555#comment3 for before/after images.
