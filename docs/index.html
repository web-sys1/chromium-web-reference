Ash: Migrate ScreenTimeControllerTest off RunUntilIdle

This changelist is part of an ongoing series to modernize Ash tests by
migrating them away from RunUntilIdle() and toward targeted
synchronization with APIs such as RunUntil(), TestFuture, RunLoop
callbacks, observer waiters, or exact mock-time advancement.

ScreenTimeControllerTest now waits for the lock-screen transition only
when the active-session tests expect screen time to lock the device.
Negative checks keep reading the session state directly instead of using
a broad idle drain that also hides unrelated queued work.
