Reorganize the hash tables in NthIndexCache.

Right now, NthIndexCache has two hash tables; one for nth-child
and one for nth-of-type. Both are keyed by the parent node,
but the one for nth-of-type points to another hash table that
is keyed further by tag name.

In the near future, we'll be adding a third type (for complex
nth-of-child), so this starts becoming unwieldy. Thus, we change
to single table, with a key that is the parent node plus optional
tag name in a simple struct.

Unfortunately, due to Oilpan, we need to wrap this key in a
Member<>, which means we need to do some trait gymnastics to
be able to compare it by value instead of by pointer. Apart from
that, this is a fairly straightforward cleanup that regularizes
the querying paths between the two cases a fair bit.
