[bookmarks] Write account bookmarks to disk (dedicated JSON file)

The new logic is guarded behind feature
syncer::kEnableBookmarkFoldersForAccountStorage, disabled by default.

A second independent instance of BookmarkStorage, owned by the very
same BookmarkModel, is introduced in this patch to take care of
persisting account bookmarks to JSON (dedicated file).

The only difference between the two BookmarkStorage instances, besides
the output file path, is the set of permanent nodes that they use as
input when encoding the JSON file. To distinguish this behavior, a new
enum is introduced, as similar existing enums have slightly different
semantics.
