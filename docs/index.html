[Categorical search] Ranker API and organization improvements

Right now we have a Rank method on Rankers, but we really need two
things: a method to rank results, and a method to rank categories. These
should be able to operate in two modes:
1. Immutably returning a vector of scores for results
2. Mutably changing the score values on result or category objects

These are respectively used in these use-cases:
1. A ranker composed of other rankers (eg. a mixture of experts) wants
   to query the ranking of its constituents.
2. That same ranker then needs to decide and set a score for a result.

It might might make sense to split these into two separate interfaces
in future, but I don't think there's a good reason to yet.

Unrelatedly, I've moved some files around in this CL to make clear the
distinction between rankers, ie. things implementing the Ranker
interface, and the data structures/algorithms that back them. The
latter has been moved to search/util/, our very own //base.
