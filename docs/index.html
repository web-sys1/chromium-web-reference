Prerender: clean up the cancellation logic and remove exceptions

As mentioned in I7114929915fee7d03c74e891498a293caec615b4, I'd like
to make prerender report cancellation reasons without relying on
exception logic. See changes in `RecordPrerenderFinalStatus`.
So this CL does the following things:
1. Replace RecordFinalStatus with RecordFailedFinalStatus and
   RecordActivation.
2. For those nested cancellation reasons, pack the reasons into
   PrerenderCancellationReason and report them to devtools.
   This allows us to remove exceptions and patches for devtools.
3. Move the activation logic into dedicated methods to simplify
   the metrics recording, including UMA, UKM and devtools.
