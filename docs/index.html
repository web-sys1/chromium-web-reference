AX UI: Migrates GetNodeFromTree to AXTreeManager

This method is just a wrapper around AXTreeManager::GetNode
and so does not need to be specialized in subclasses
except on Chrome OS for now.

Any checks that GetNodeFromTree needs to perform
can be covered by GetNode which it wraps.
Since GetNode is virtual, there is
no need for GetNodeFromTree to worry
about any validation or special
cases that GetNode should handle in subclasses.

This patch completes the work to make
AXTreeManager non-abstract.
