Separate trusted vault constants used by standalone implementation

`trusted_vault_server_constants.h` is used by e.g. Android, which
doesn't communicate with the trusted vault backend directly at all.

To avoid leaking dependencies required for the standalone implementation
(see https://crrev.com/c/6508960), this change separates constants
required across all platforms from the ones only required for the
standalone implementation.
