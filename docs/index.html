Fix flake in unit tests that enable PA's thread cache

PartitionRoot::EnableThreadCacheIfSupported CHECKs that the thread
cache is not already enabled. When unit tests call this on
the global Allocator(), it can cause flaky failures depending on
the behavior of previous tests, which may have already enabled
the thread cache. These unit tests then pass on retry, when run
individually.

This CL fixes this flakiness by checking whether the thread cache
is already enabled before trying to enable it.
