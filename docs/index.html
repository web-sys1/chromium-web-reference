Optimize MutableCSSPropertyValueSet::Set*Property().

Several changes, with an eye towards improving the speed of presentation
style attributes:

 - Make a static bitset to check for logical properties, instead of
   looking up the property by ID and asking it through virtual member
   functions.
 - Demand that presentation style attributes are always longhands,
   to skip shorthand expansion.
 - When checking the insertion point (for overwriting existing
   properties), work with pointers instead of indexes when we can,
   which means we don't need arithmetic to convert back and forth.
 - For SetProperty(), make the CSSPropertyName form call the
   CSSPropertyID form instead of the other way round, which saves
   constructing (and destroying) a CSSPropertyName when we don't need it.
 - Make SetLonghandProperty() take in CSSPropertyValue by value instead
   of by reference, since we want to move it into the vector anyway.
 - Make a faster form of SetLonghandProperty() that can be used when
   we know we have a property ID; again, saves some construction time
   on CSSPropertyName (although the compiler unfortunately isn't
   clever enough to remove the destructor completely), and also checking
   for custom property code.
