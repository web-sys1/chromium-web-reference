[shared storage] Enable E2E execution of addModule and runOperation

Add browser side components that receives requests from the renderer
(i.e. renderer --> SharedStorageDocumentServiceImpl), and
sends the request to the worklet environment (i.e.
SharedStorageWorkletHost --> SharedStorageWorkletServiceImpl). Each
worklet will be using a dedicated renderer thread on the same renderer
process that the worklet's owner document belongs to.

A SharedStorageWorkletHost's lifetime is now bound to the document
lifetime, and it lives inside a SharedStorageWorkletHostManager which
lives inside the StoragePartition. The reason for having the "manager"
layer is: for a upcoming change, we will make the worklet to outlive the
document i.e. no longer bound to the document lifetime, so this
structure is more future-proof.

For mojom::SharedStorageDocumentService, implement the addModule and
runOperation method. For mojom::SharedStorageWorkletServiceClient,
implement its ConsoleLog method; the rest of the methods are
implemented by always responding with a "not implemented" error.

Tests are done by running operations and expecting the console.log
messages.
