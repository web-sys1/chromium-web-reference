[Trusted Types] Add warning if script execution fails silently.

Currently, script execution may be silently blocked by Trusted Types
under very specific conditions, namely if a <script> was modified
directly, but no default policy was installed. This is spec compliant,
but arguably surprising. In this case, we will now issue a console
warning.
