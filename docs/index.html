Hide Underlying page from Accessibility when in WebXR/VR

When we begin a Cardboard VR session as part of WebXR, the underlying
page is hidden. However, Accessibility tools still attempt to cycle
through this content. This can lead to confusion of users using these
tools. This fixes the issue by setting the "Content" element (which the
tab controls and actual Web Content are parented to) as Unimportant to
accessibility while we are in a VR session.
