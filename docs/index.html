Textarea: Improve typing performance by reducing `value` creation

Every typing in a textarea built the `value` string four times.
  1. UpdateValue() call in SubtreeHasChanged()
  2. IsInnerEditorValueEmpty() call in UpdatePlaceholderVisibility()
  3. InnerEditorValue() call in SubtreeHasChanged()
  4. CachedTextInputInfo::EnsureCached()

We can use Value() for 2, 3 and 4.

- Add TRACE_EVENT0() to HTMLTextAreaElement::SubtreeHasChanged(), which
  can be slow.

- Use ComputeValue() in TextControlElement::InnerEditorValue().
  ComputeValue() is faster than the existing code with StringBuilder.

This CL reduces 5.9ms of typing lag in a textarea with 23k paragraphs.
