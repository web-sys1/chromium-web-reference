[rust] Cargo crate build script emulation improvements.

This improves our emulation of cargo in three ways which have proven to
be necessary when attempting to build bindgen with gn:

1) Better error handling. If the build script fails, the whole build
should (obviously) fail.
2) Run the build script within the directory of the build script itself.
This is documented cargo behavior and some scripts rely on it.
This is also passed to the script as CARGO_MANIFEST_DIR, which we also
now pass.
3) Properly rebase the OUT_DIR; previously it was a source-relative path
which was incomprehensible to the build script because it started with
//.

This also solves conflicts in the case that a single BUILD.gn has
more than one cargo_crate(..) template present. For example, bindgen
builds both a library and an executable. In this case, their flags
output filename would conflict prior to this change.
