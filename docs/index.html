Move KeepDeviceAliveForTransfer to UserMediaClient

Instead of using the MediaStreamDispatcherHost connections created
ad-hoc in each Platform Source class, use the per-frame instance in
UserMediaProcessor used for almost all other MSDH calls, via
UserMediaClient.

This means:
- UMC can continue its role of knowing which requests need to be queued (gUM/gDM, stop, applyconstraints) or can continue anyway
- we can rely on the ordering guarantees of the single MSDH remote, eg that sending KeepDeviceAliveForTransfer then Stop will be delivered in that order
