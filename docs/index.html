[omnibox] Fixes crash caused by looking up visibility state for group ID

This crash is caused when the OmniboxEditModel tries to look up the
visibility state for a group ID that does not have any value for the
server provided group ID. Since the group ID is present on the match
without a server provided group ID, this is probably caused by the
Journeys dedicated row. This Cl changes the implementation of
AutocompleteResult::IsSuggestionGroupHidden() to check whether an
original group ID is present or returns false otherwise.
