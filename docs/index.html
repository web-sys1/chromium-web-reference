Use existing Ethernet settings when applying Ethernet policy

Use "ethernet_any" shill service when applying Ethernet settings.
This leads to the following effective changes in behavior:
- when re-applying policy, a manually configured IP address is not lost
- when removing Ethernet policy, the network resets to default state.

This revealed that previously, an outdated GUID could get stuck in an
Ethernet adapter's UIData (presumably because policy could provide
multiple Ethernet NetworkConfigurations with different GUIDs) which led
to DCHECKs in the UI code, so fix up Ethernet services' UIData GUIDs.
