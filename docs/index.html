Properly print memory leaks

Web tests can fail due to detected memory leaks as described in
crbug/1164553 The test output includes these extra lines:

```
Content-Type: text/plain
#EOF
#EOF
```

It was assumed it's printed when the test times out but actually
it happens when a memory leak is detected.

This CL adds the StartStateDump before the leak message is added to
make sure that it gets dumped to stdout after the test is over.
The message that is printed looks like:
`#LEAK - renderer pid 377541 (...leak data..)`
