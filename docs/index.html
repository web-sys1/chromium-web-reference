> Original change's description:
> > Store DocumentCreated events as IdentifiableSurface metrics
> >
> > The UKM DocumentCreated event is needed to tie together metrics
> > recorded in different subframes of the same page. However, on stable
> > the DocumentCreated event is down-sampled. This is a huge problem for
> > the Identifiability Study, see go/privacy-budget:dropped-samples for
> > more details.
> >
> > To address this, we implement DocumentCreated events as identifiability
> > metrics (i.e. identifiable surfaces) corresponding to the
> > DocumentCreated metrics currently recorded by UKM. These surfaces are
> > of type 0 and are always be recorded when the study is enabled and
> > at the sample place where we record a UKM DocumentCreated event.
