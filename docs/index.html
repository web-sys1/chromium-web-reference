[FedCM] Add multi IDP desktop UI

Now that multiple IDP information is plumbed to
AccountSelectionBubbleView, this CL uses the information to display
multiple IDP accounts at once.

Two non-trivial changes in this CL worth noting:
1. Fixes a cref issue and avoids copying Account and IDP data by adding
a new method to AccountSelectionBubbleViewInterface.
2. To prevent UAFs for IDP image fetches, uses a similar approach as the
one which is used for account image fetches, i.e. using a wrapper class
to ensure the fetch callback only runs when the View still exists.
