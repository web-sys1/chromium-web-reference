[MPArch] Remove FTN id from InputRouter

RenderWidgetHost's InputRouter stores a FrameTreeNodeId and emits this
id in the trace. It's not currently used, so remove it as a part of
our effort to reduce the usage of FTN ids.

In the future if additional data will be needed in a trace, RWH id would
be the best id.
