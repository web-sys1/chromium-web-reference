Allow WebUIImpl creation without RenderFrameHost

Currently when we create WebUIImpl objects when picking a RFH for a
navigation, the constructor requires an RFH object. This means
creation of WebUIImpl depends on the existence of an RFH for a
navigation, which won't work well if the navigation's RFH is created
only after OnResponseStarted due to speculative RFH creation ban when
a pending commit RFH exists, as WebUIImpl creation is needed to get to
OnResponseStarted on WebUI navigations.

This CL makes WebUIImpl constructor not take a RenderFrameHost, and
creates a new SetRenderFrameHost function to be called sometime after
construction when a RenderFrameHost has been picked for the
navigation, allowing WebUIImpl creation without RFH creation.

This CL also factors out the WebUI creation code in
RenderFrameHostManager::GetFrameHostForNavigation() into a separate
CreateWebUIForNavigationIfNeeded() function. The function creates a
WebUI potentially before a speculative RenderFrameHost has been
created for the navigation.

See doc: https://docs.google.com/document/d/1ke5ZlCgQC5O0vt7FopVnOKW9asz-ZRFLl3XRpE8tLk8/edit#
