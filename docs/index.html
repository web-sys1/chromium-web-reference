[Android Bulk Tab Edit] Fix flaky tests

After reproducing locally it seems like the click() from the Espresso
library is flaking. It happens rarely ~20% of the time. The UI & Views
are present in every post test screenshot and the elements are in an
enabled, displayed state.

Work around this by calling the equivalent "perform" action on the
View/MenuItem.

More investigation details:

The ActionView flake is rarer and seems to be the result of click()
not triggering a click handler. I suspect the click is either too short
or pressed before the view is fully ready.

The MenuItem flake occurs often and the Espresso click() claims throws
an error stating the view is < 90% visible. I suspect the menu takes
a bit of time to display or is rendered slightly off the screen on
occasion.

BlankUiTestActivity disables animations, but this doesn't seem
reliable on Android 12 (S) so that might explain most of the flakes?
See https://crbug.com/1225707.
