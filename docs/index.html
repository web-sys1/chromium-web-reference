[ios] Simplify DataUrlDownloadTask

As DataUrlDownloadTask has access to the full decoded data at once,
use base::WriteFile(...) in a background task to save the it to a
file instead of using URLFetcherResponseWriter whose API is more
complex.

Move the data: URL decoding to the background task (no need to block
the UI while decoding the URL since we may already use a background
task to write the data to disk).

Add unit tests that check IO failures are correctly reported as such
and to check that downloading to disk also work.
