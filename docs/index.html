[headless] Improved headless window size handling on Macs

https://crrev.com/c/4295930 introduced expected desktop window size tracking to work around window size clamping in headless mode. This worked well except for the cases when desktop frame size was retrieved directly from the platform window which is the case when the desktop window geometry cashed in renderer is updated for window.outerWidth/Height.

This CL removes expected desktop window size tracking and instead overrides NSWindow's constrainFrameRect() method thus removing constrains for headless window size.
