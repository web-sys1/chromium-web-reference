bind-gen: Fix the blink component of IDL typedefs

When generating a typedef to an union whose member is a typedef,
respects the component of the member typedef.

Example:
    using V8UnionUUIDOrUnsignedLong = V8UnionStringOrUnsignedLong;
will be generated in modules/ despite that both 'DOMString' and
'unsigned long' are defined in core/, because
    typedef DOMString UUID;
is defined in modules/.
