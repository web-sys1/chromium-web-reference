Update TabGridBottomToolbar to account for floatingButton case correctly

Right now the toolbar is dealing with the floatingButton as its the
default case. Specifically in pointInside and intrinsicContentSize
check for other conditions (mode/page) or orientation and if any of
these conditions are not met it falls back to consider that the
floatingButton is available and returns value based on that.

Given that the floatingButton is a special case it shouldn't be dealt
with as the default case. Instead it should be checked independently
and if not found fallback to the super view methods regardless of the
mode or the case (unless there is another special case for that mode).
