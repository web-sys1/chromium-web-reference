[WebRTC] Migrate WebRtcTimer to PostDelayedTaskAt().

MetronomeSource allows listeners to align their wakeups on metronome
ticks.

This CL refactors WebRtcTimer to align its tasks on metronome ticks
using PostDelayedTaskAt() instead, which it can do using
MetronomeSource::GetTimeSnappedToNextMetronomeTick().

This takes us one step closer from deleting the complexity of
MetronomeSource's ticking logic. When/if we replace MetronomeSource with
a static function, WebRtcTimer logic could also be greatly reduced by
not having to handle "starting" and "stopping" using the metronome, but
I leave that to a separate cleanup CL. This CL changes no behavior.
