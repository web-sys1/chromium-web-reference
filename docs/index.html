split view: Add window snap animation observer to overview

When you end overview in tablet split view, the window snap animation
will be added to OverviewController::delayed_animations_ so that
OverviewController::OnEndingAnimationComplete will not be called until
the window snap animation is complete. This fixes at least two problems
when the snapping window is the only overview window:
1. You can see the wallpaper suddenly unblur at the onset of the window
snap animation (see Issue 1257234).
2. Occlusion tracking is unpaused at the onset of the window snap
animation. This may impact performance. However, there is no overview
animation smoothness metric recorded to any histogram in this case,
because such metrics do not include the overview window being snapped.

Fixed: 1257234
