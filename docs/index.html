brightness: Fix brightness restoration issue if reboot twice w/ external
connected

go/display-issue-external-connected-analysis

Issue:
 * Internal display brightness is incorrectly set to 0% when booting
   with an external monitor connected and the laptop lid closed after
   two consecutive reboots.

Root Cause:
 * When booting with the lid closed, the system incorrectly saves the
   internal display's brightness as 0% to preferences during login. This
   happens because the OnActiveUserSessionChanged() function retrieves
   the current brightness (which is 0% due to the lid being closed) and
   saves it to LocalState. On subsequent reboots, this saved 0%
   brightness is restored, resulting in a blank internal display.

Solution:
* Add a lid state check before saving brightness preferences on the
  login screen to ensure that the system does not save a 0% brightness
  level if the lid is closed.
