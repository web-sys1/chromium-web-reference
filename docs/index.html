IDB: SQLite - Fix crash when enumerating corrupt DBs

A corrupt DB that was attempted to be reopened (post recovery/deletion)
would hit a null std::optional dereference inside
DatabaseConnection::Init. Fix it by (a) not reopening to-be-deleted
databases when enumerating, and (b) handling the failure mode where the
sql::MetaTable is not present in the database file.
