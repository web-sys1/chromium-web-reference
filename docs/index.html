[base/android] Log long-running UI thread Binder calls

Binder has a hidden @SystemApi that can be used to intercept Binder calls. Android utilizes this with the command "adb shell am trace-ipc
start|stop", and in SysUI to detect jank from UI thread Binder calls.
This suggests that it would be beneficial in Chromium too. The CL is a
continuation of @lizeb's draft CL to log slow (> 2ms) Binder calls.

Binder.setProxyTransactListener() has both @hide and @SystemApi and
android.os.Binder$ProxyTransactListener has the @hide property. Therefore, double/meta reflection and a proxy is utilized to bypass these restrictions.
