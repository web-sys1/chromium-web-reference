[MBI] Migrate ScrollAnimatorMac to use per-ASG CompositorTaskRunner

Before this CL, ScrollAnimatorMac used per-thread
CompositorTaskRunner.

After this CL, ScrollAnimatorMac will use per-ASG (Agent Scheduling
Group) CompositorTaskRunner.

[scroll_animator_mac.mm (ScrollAnimatorMac)]

This is a Objective-C++ code which uses per-thread
CompositorTaskRunner. ScrollableArea can be used to obtain per-ASG
CompositorTaskRunner.

[ScrollableArea]

ScrollableArea was already using per-ASG CompositorTaskRunner. But
this time, ScrollAnimatorMac requires the per-ASG CompositorTaskRunner
at the very beginning of initializing the page. So this CL added a
per-ASG CompositorTaskRunner argument to the constructor of
ScrollableArea.

[Page]

When the page object instantiates, the AgentGroupScheduler object is
provided to the page object afterwards. This timing was too late for
ScrollAnimatorMac. So this CL changed it. Also, the previous code
initialized the PageScheduler outside of the constructor. But I
thought it is better to initialize PageScheduler inside the
constructor. So this CL changed it as well.
