media: Fix thread safety in D3D12 VEA Shared Image processing

D3D12 VEA previously accessed `CommandBufferHelper::GetSharedImageStub()`
from a background thread during initialization to create an interop fence.
This could lead to a race condition if the GPU main thread concurrently
destroyed the command buffer.

Additionally, during per-frame encoding on the GPU main thread, the
encoder did not verify that the shared image stub was still valid before
dereferencing it, which could lead to a crash.

This change resolves these issues by:
1. Creating the interop fence directly on the GPU main thread during
   initialization and returning the constructed fence to the encoder.
2. Adding a null check to `GenerateResourceFromSharedImageVideoFrame` to
   gracefully handle mid-flight command buffer destruction.
