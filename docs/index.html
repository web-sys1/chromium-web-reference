[sync] Improve accuracy for GetTypesWithPendingDownloadForInitialSync()

Datatypes that already downloaded the initial data should not be listed
as pending-initial-sync, even while the datatype manager is CONFIGURING.

To achieve so, this patch defers the update of `downloaded_types_`
until the configuration cycle finishes, and the types are known to have
been downloaded successfully.

This change is otherwise considered harmless because
`downloaded_types_` isn't actually consumed during a configuration
cycle, so it shouldn't make a difference whether it gets updated in
the beginning (optimistically) or upon success. At least under the
assumption that the Sync engine reports success cases reliably, which
appears to be the case.
