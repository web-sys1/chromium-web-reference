Update CRD security key forwarding to use Mojo interfaces

This CL removes the deprecated IPC MACROs for security forwarding and
replaces them with well defined Mojo interfaces.  For this flow,
the CRD host exposes an IPC channel which a separate binary
(remote_security_key) connects to when the local SSH_AGENT needs to
send a request to the remote SK.

One notable change is that the IPC server no longer sends 'ready' or
'invalid session' messages to the IPC client.  This seemed overly
complicated since we can rely on the IPC connected event to indicate'
when the channel is ready and the server can close the channel to
signal that there was an invalid state (wrong session for instance).
The client doesn't need to know why the connection is invalid so
this seemed like a reasonable change.
