[vulkan] Set default plane count to 1.

There are a few inconsistencies around the current implementation of
OutputPresenterX11.

1. OutputPresenterX11 creates the GMB handle with planes [1], and gets
   FDs from the image to share memory.
2. When ExternalVkImageFactory creates a shared image from GMB, it
   asserts that plane count is exactly 1 [2]; after that assertion, it
   takes the FD from the GMB.
3. However, OutputPresenterX11 sets the tiling mode of the Vulkan image
   to VK_IMAGE_TILING_OPTIMAL [3], which causes the plane count to be
   equal to 0.  This causes the assertion mentioned above to fail, but
   even if there was no assertion, the shared image would be unable to
   take an FD from the Vulkan image.

Also it is worth noting that VulkanImage::GetMemoryFd() doesn't take
plane count into account [4].

Here the default plane count is set to 1, which resolves the
inconsistency explained above.
