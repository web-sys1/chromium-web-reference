url: percent-encode more characters in path URL fragments

In //url parlance, "path URLs" are URLs without a "special scheme" such
as http and https. Currently, we are canonicalizing them quite loosely,
allowing even un-encoded spaces to be in its fragment. This has
contributed to interop problems such as [1].

The solution is to percent-encode fragments consistently. In this
commit, we encode path URL fragments the same way as HTTP and HTTPS URLs
already are. More precisely, we encode those characters in the "fragment
percent-encode set" [2] designated by the WHATWG URL Standard, which
includes:

        U+0000 through U+001F, U+0020 SPACE, " < > `, U+007F DEL

Among existing other implementations, there is no great controversy over
what characters to encode in the fragment part. Safari, Go, and Node.js
all encode those characters, while Java and Ruby fail to parse the URL
if any of those characters is present. As an exception, Firefox and curl
only encode U+0020, U+007F, and the C0 controls.

[1]: https://bugs.webkit.org/show_bug.cgi?id=225340
[2]: https://url.spec.whatwg.org/#fragment-percent-encode-set
