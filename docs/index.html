Add LiveCollectionIterator for live Set/Map iteration in Highlight API

Extract the live-iteration cursor logic from Highlight::IterationSource
into a reusable LiveCollectionIterator<EntryType> template, which
provides this logic over a HeapLinkedHashSet<Member<EntryType>> for
collections that follow JS Set/Map semantics.

Apply it to both Highlight (Set-like) and HighlightRegistry (Map-like)
iteration, replacing the snapshot-based approach in HighlightRegistry
with true live iteration matching JS Map semantics, consistent with the
existing Highlight (Set) behavior:

  - Paused iterators see items added after the current position.
  - Exhausted iterators stay permanently done.
  - clear() resets the cursor, items added before the next advance
    are visible.
  - delete() of the last-returned item moves the cursor to its
    predecessor.

This fixes the WPT tests for HighlightRegistry iteration with
modifications.
