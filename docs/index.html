Add tests to ensure that client reports are not sent in incognito mode

Downloads can sometimes trigger the Client Safe Browsing Report by
calling into SendSerializedDownloadReport in SafeBrowsingService, which
calls into ReportThreatDetails in PingManager. PingManager has recently
been refactored into a keyed service that returns null for incognito
mode. All consumers are already correctly never calling into the
PingManager in incognito mode, but a couple (download_danger_prompt.cc
and download_item_model.cc) did not have automated testing coverage for
this case. This CL adds that missing coverage.
