Fix loading non-system cursors on Windows on browser_tests

The resource module from which cursors are loaded is different in the
case of browser_tests. The previous code was failing to load some
cursors in some cases, but this went unnoticed as there are very few
tests that deal with the correct loading of cursors. It was discovered
as some CLs that tried to DCHECK that cursors were always correctly
loaded made some tests flaky: https://crrev.com/c/2417893 and
https://crrev.com/c/2691088.
