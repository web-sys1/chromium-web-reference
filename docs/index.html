Enable MetricsServiceAsyncCollection by default

Since now `metrics_service.cc` posts to `base::ThreadPool` by default,
I had to adjust some unit tests. I.e., we are not allowed to use
`base::TestSimpleTaskRunner` in the tests anymore, we have to use
a `base::test::TaskEnvironment`, which is better and more accurate.
For example, the former did not respect delays properly: if your code
posted a task "A" to run in 5 seconds, and a task "B" to run
immediately, when you did
`base::TestSimpleTaskRunner::RunPendingTasks()`, it would just run
all tasks regardless of their delay, i.e., it would run task A first
and then task B. At least, that's what I understood from
https://source.chromium.org/chromium/chromium/src/+/main:base/test/test_simple_task_runner.h

I also rewrote the `MetricsServiceTestWithFeatures.SplitRotation` test
to be a lot more descriptive in what it's doing, and also more
accurate in terms of timing.
