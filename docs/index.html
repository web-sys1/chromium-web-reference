[PartitionAlloc] Don't use std::bitset::test() in fast paths.

std::bitset::test() checks that the access is in-bounds, contrary to
operator[]. This call is used on fast paths on 32 bit architectures,
both on Android for a PA_CHECK(), and in BackupRefPtr.

It seems that we've been saved by the "sufficiently smart compiler"
though, as with the current settings it seems to realize that
out-of-bounds accesses cannot happen. This commit moves to operator[] to
avoid relying on that, and adds a static_assert() to ensure that OOB
accesses really cannot happen.
