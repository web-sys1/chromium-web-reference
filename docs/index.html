Delete TraceLog::Mode

Addresses an old TODO and removes the enum, which has a single value
these days. There is a chance this causes behavior changes in [1].
As far as I can tell, the "category enabled byte" has one bit per data
source, and the category is considered enabled if any of those is
set [2]. [1] is fishy because it only looks at the last bit. After this
CL, the code considers all bits, potentially fixing a bug. For
reference, [1] was introduced in [3].
