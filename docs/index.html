Validate studies before checking their filters.

This came up while creating a fuzz test for CreateTrialsFromSeed, which was producing invalid min_version filters.

In a debug build this crashes (due to DCHECKs), in a non-debug build it would do the comparison with an invalid version. My understanding of the code is that an invalid version will always be bigger than a valid one. This means an invalid max_version filter will always pass.

The comments on CompareToWildcardString also state that IsValidWildcard must be true before using it.
