[js-code-coverage] Enable coverage for shared and service workers

DevToolsAgentHost's extend base::RefCounted, when they are spawned for
non-worker types the places they are spawned grab a reference which
ensures the host is kept alive for the duration of the test.

However when a worker type is spawned the callee doesn't get a reference
to the host, this means after attaching the client to the host all
references are lost and the host gets destroyed. This leaves a race
condition where the host gets created, attached and destroyed THEN the
DevtoolsListener tries to start code coverage, however, the host has
been destroyed.

To avoid this race, let's grab a scoped_refptr to the DevToolsAgentHost
to ensure it doesn't get destroyed immediately. Along with this, let's
extend the DevToolsAgentCoverageObserver to log out the destroyed
lifecycle event.

There are a number of tests which have been disabled due to this
functionality, this test also enables all these tests.
