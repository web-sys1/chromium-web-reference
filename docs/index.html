Check the outline property when invalidating continuations

r830902 crrev.com/c/2535904 started to:
* Set |needs_full_invalidation| in |NGBlockNode::
  CopyFragmentDataToLayoutBox|
* |SetNeedsPaintPhaseDescendantOutlines| in
  |PaintInvalidator::UpdatePaintingLayer|
for all non-inline |LayoutBlock| that belong to continuation
chains. This was done on purpose, assuming finding the inline
ancestor and checking its `outline` property is more expensive
than setting these flags.

This patch does more precise check for the invalidation, by
extracging the logic from |NGBoxFragmentPainter|.

Doing this has positive effects on rasterize_and_record_micro:
https://pinpoint-dot-chromeperf.appspot.com/job/1739ab00d20000

while no noticeable regressions in bink_perf.{layout,paint}:
https://pinpoint-dot-chromeperf.appspot.com/job/139588b3520000
https://pinpoint-dot-chromeperf.appspot.com/job/105e3d30d20000
