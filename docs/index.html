Reland "media/gpu/VP8VaapiVED: Specify a loop filter level by rate controller"

This relands commit 94f4d69fa755b37caf26e9e87815cad9a58c175a.

The original CL causes a GPU process crash in checked_cast. It may be
the default implementation is called for some reason. This fixes the
issue by removing the default implementation.

Original change's description:
> media/gpu/VP8VaapiVED: Specify a loop filter level by rate controller
>
> Originally, we set the highest loop filter level, 63, all the time. It
> causes inefficiency in compression; an encoder driver produces the
> large chunk even for identical frames. We need to rather specify a
> smaller loop filter level [1].
> This CL specifies a loop filter level computed by a software rate
> controller to each frame.
>
> [1] Chapter 6. https://static.googleusercontent.com/media/research.google.com/ja//pubs/archive/37073.pdf
>
