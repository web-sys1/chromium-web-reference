Prerender: add a metric for multiple speculation rules

This CL adds a new metric Prerender.Experimental.
CancellationPercentageByExcessiveMemoryUsage.SpeculationRule that
records the percentage of destroyed prerenders by the excessive memory
usage (kMemoryLimitExceeded).

The metric will be used in a Finch experiment. In the experiment, the
value of `acceptable_percent_of_system_memory` will be a variable and
we try to find a "good" value of it.
The definition of the "good" value is:
- No regression in an OOM killer: can be measured by
Stability.Android.SystemExitReason.*
- More prerenders can start & less prerenders are canceled: can be
measured by Prerender.Experimental.
CancellationPercentageByExcessiveMemoryUsage.SpeculationRule.

Here are example scenarios:
- all prerenders start and they are not cancelled by
kMemoryLimitExceeded => the value of Prerender.Experimental.
CancellationPercentageByExcessiveMemoryUsage.SpeculationRule is 0 (%).
- half of prerenders are cancelled by kMemoryLimitExceeded => the value
of the metric is 50 (%).
- all prerenders are cancelled by kMemoryLimitExceeded => the value
of the metric is 100 (%).

If 0 (%) is dominant in the experiment, we decrease the value of
`acceptable_percent_of_system_memory`.
If 1~ (%) appears in the experiment, we increase or adjust the value
as long as there is no regression in an OOM killer.
