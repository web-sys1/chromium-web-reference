[Pcct-SideSheet] Perform always custom close animation

A logic performing the custom closing animation (side or bottom) was
triggered only when a client app specifies the exit animation
resource, the intention being that the animation will be overridden
only when present, while just letting the system handle the default
animation if not specified by the client app.

This works for bottom sheet where the default animation was
slide-down, but not for side sheet which has 2 options which should
match the opening animation.

This CL simplifies the logic so the custom animation will always be
performed regardless of the presence of the exit animation resource.
Both bottom/side sheet will take care of the closing animation
without delegating it to the system.
