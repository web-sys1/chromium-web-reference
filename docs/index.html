Extract LayerWithExternalTexture from ui::Layer

This introduces a new `LayerWithExternalTexture` base class that
inherits from `ui::Layer` to encapsulate the logic for handling external
textures and transferable resources. `LayerSolidColor` and
`LayerTextured` now inherit from this new class instead of directly from
`ui::Layer`.

This refactoring removes `cc::TextureLayerClient` from the base `Layer`
and pulls down the management of `texture_layer_` and
`transfer_release_callback_`. This slims down the base `Layer` class and
ensures that only layer types capable of hosting external textures carry
the associated state and overhead.
