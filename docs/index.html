Use base::span instead of std::vector in MP2TS code

base::span is preferable to std::vector in situations where you do not
require a growable array (ie, basically anywhere std::vector is passed
by const&). This is due to the fact that base::span can be trivially
constructed from several different array types (c-style array,
std::array, std::vector), without requiring a dynamic allocation. As can
be seen in this CL, there are cases where an array constant is used as
an argument which unnecessarily causes calls to `malloc` and `free` if a
std::vector is used as the argument type.

Further reading here: https://abseil.io/tips/93 (refers to absl::span,
but same idea).
