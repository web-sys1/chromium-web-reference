[WebAuthn] Delay the creation of UV keys on Windows

Windows Hello performs a user verification when a UV key is created,
which isn't ideal at the time of device registration with the enclave
authenticator service.

This change creates a special EnclaveManager state for Windows systems
that support UV, in which UV key creation is deferred. The creation
happens when the user attempts a WebAuthn operation with the enclave.
At that point they will have have to authenticate to Windows Hello,
allowing the key to be created, and the enclave will accept the new
UV public key for future use.
