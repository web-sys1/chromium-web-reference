configure_siso.py: Handle empty rbe_instance var.

It's important to handle the case where the rbe_instance var is empty,
because that's how the user can currently disable remote execution for
Siso builds.

The script did not update the .sisoenv file, which could lead to a
previously configured RBE instance not being removed from it.

With this fix it now correctly generates an empty .sisoenv in that case.
