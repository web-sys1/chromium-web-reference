views: Speculative fix for crashes in TouchSelectionMenuViews

If a views client calls TouchSelectionMenuRunnerViews::ShowMenu() twice
in a row, the first menu isn't closed. I think this is the source of
shutdown CHECK failures in TouchSelectionMenuViews::WindowClosing().

This may also be the source of other crashes in TouchSelectionMenuViews,
since the first open menu might outlive its `client_`.

Explicitly close any open menu when calling ShowMenu().

Bug: 1146270
Test: added to views_unittests
