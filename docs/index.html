[PA] Fix a race between LeakyInstance::GetSlowPath() and ::Replace()

GetSlowPath() already has code that checks if someone else wrote a new
instance, and updates it only if not. However, if Replace() gets in
between, its effect will be overwritten by GetSlowPath(). Most likely
the intention is to let the consciously replaced value prevail over the
default constructed value. An easy fix is to make Replace() use lock.

Reported by glazunov@
