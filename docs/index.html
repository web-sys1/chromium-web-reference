Ash: Fix flaky test LogoutButtonTrayTest.ButtonPressed

This test is flaky because it signs the user out and then continues to
perform actions that require an active user session. This creates an
invalid state and a race condition, causing the test to fail
intermittently.

The fix is to re-login the user after the first sign-out action. This
restores a valid session, allowing the rest of the test to execute
reliably without changing its original structure.

Additionally, the test is updated to explicitly make the logout button
visible via the kShowLogoutButtonInTray pref before clicking it, fixing
a latent bug.
