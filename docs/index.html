Add null check for mTabModelOrchestratorSupplier before closing expired instances.

In MultiInstanceManagerApi31.removeInvalidInstanceData, closeWindows can be called for expired instances early in the activity lifecycle before mTabModelOrchestratorSupplier.get() is initialized, leading to a NullPointerException. This adds a null check to match the pattern used for inactive instances.
