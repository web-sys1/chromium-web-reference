Improve file search scoring

Two changes related to the linked bug.

1. Do proper query matching to generate the relevance score for drive
   files. This brings them in line with local files and helps with
   intra-category ranking.

2. Add a proper access time-based penalty to file search scores. This
   is a gaussian penalty that will multiply a score by ~0.6 if it hasn't
   been opened for a month.

   This replaces the worse version of sort-by-access that we had
   previously and should help keep bad file matches out of best match.
