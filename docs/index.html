Move the transform and opacity properties into the SVG group.

These are not SVG properties, but are frequently used with SVG,
and since the SVG group is pretty small now, it doesn't hurt as much
to put “unrelated” things in there. This seems to win back the
MotionMark performance we lost by changing RareData groups to
Misc groups and then some (the change lost 0.4% overall, this wins
2.0% overall).

The basic problem with the change was that MotionMark has some tests
that love to mutate _only_ transform and opacity, and after the change,
transform and opacity were moved from misc to misc->misc4 and misc->misc6
recently, which meant we were allocating more top-level groups
in this case.

MotionMark (M1 Pinpoint, LTO but no PGO, significant results at
99% CI only):

  Suits                      [ +0.9%,  +3.2%]
  Multiply                   [ +2.9%,  +3.6%]
  Leaves                     [+10.3%, +11.0%]

  motionmark_ramp_composite  [ +1.5%,  +2.6%]

Speedometer3 is neutral.
