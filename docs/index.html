[TreesInViz] Set anchor_position_scroll_data_id to -1 in viz.

Currently if anchor_position_scroll_data_id is -1 in client side,
we don't wire the optional field. On viz side, when the optional
field is nullptr, we fail to set it to -1, thus causing data
discrepancy, leading to a out-of-bound vector access.
