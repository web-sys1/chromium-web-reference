[views_ax] Introduce UpdateFocusableState to ViewAccessibility

This CL introduces UpdateFocusableState to ViewAccessibility.
This is so that we proactively keep the accessibility state on
ViewAccessibility updated, rather than computing it on demand.

As part of this change we needed to do some things:
1) Make sure to recursively update the Focusable state when
a view is added to the hierarchy, since the focusable state depends
on `IsDrawn` which depends in turn on `parent_`.

2) Make sure the submenu in the delegate unittest has a `RootView`
so that the `IsDrawn` state is accurate.

3) Lastly, we needed to add a call to `OnNativeWidgetDestroying` in
the teardown for the `WidgetSetAspectRatioTest` because we are
creating a `ViewAccessibility` object now here (due to the changes to
view.cc) which on Linux as a result we add the Widget to some
observer lists.
The function that is in charge of clearing these lists however,
is never called in this test suite because we use a MockNativeWindow
rather than a NativeWindow. So we make sure this clean up happens
manually.

There is no behavior change.
