Add GLImplementationParts to GPUInfo.

The current software_rendering is an ambiguous concept because
SwiftShader sometimes is considered software rendering but other
times not (for example, in testing, we use SwiftShader but try
to go down hardware accelerated paths).

So getting rid of |software_rendering| in GPUInfo to avoid
confusion and misuse in the future. Instead, pass back
the actual GLImplementationParts.

In a followup, we should get rid of the |gl_implementation| and
|angle_implementation| string from GPUInfo.
