[memories][persist-clusters] Fix edge case with cont params bounds.

Currently, we consider history exhuasted when the continuation params
reach either 'now' or `begin_time_limit_`.

This is usually accurate. Except when we reach a clustered visit, we
'rewind' the continuation params 1 query (i.e. 1 day) so that the last
query can be repeated a 2nd time. Otherwise, the clustered visits of
that day would be omitted.

If there is a clustered visit today, this can cause us rewinding the
continuation params to [yesterday, today, which we incorrectly interpret
as having exhausted history.

This CL conditions the exhaustion detection to only consider
`begin_time_limit_` when iterating recent 1st, and only consider 'now'
when iterating oldest 1st.
