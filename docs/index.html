[white-space] Disables a use of `white-space` in `editing_style`

This patch disables a piece of code in the `StyleChange`
constructor in `editing_style.cc`.

As part of making the `white-space` property to a shorthand,
this code needs a change, but the desired change isn't obvious
because no tests fail without this.

From the git log, this was added in 2006[1] with the
`pasting-tabs.html` test. Then it was refactored in 2009[2]
and then moved from `ApplyStyleCommand` in 2011[3].

In ToT, the `kWhiteSpace` code in
`ConflictsWithInlineStyleOfElement`[4] is relevant to pass
this test. It looks like this code is a left over.

If regressions are seen, a runtime flag is added to enable it
back, and we will know how to rewrite this code when the
`white-space` becomes a shorthand.
