[GridNG] Move last_indefinite_indices to NGGridLayoutTrackCollection

1. Previously, the `last_indefinite_indices` vector, which is used to
   determine whether a track span contains an indefinite size track, was
   stored exclusively in `NGGridSizingTrackCollection` and whenever we
   created a subgrid collection those indices would be dropped.

   Moving the vector to the `NGGridLayoutTrackCollection` base class and
   allowing `CreateSubgridCollection` to copy these indices.

2. Removing a check in `ContributionSizeForGridItem` where the available
   inline size of an orthogonal grid item should only be indefinite when
   we perform layout to compute its contribution size for column sizing.

   This was previously being checked because we never computed the rows
   of a grid before its columns, but we started failing the check since
   we now translate the axis in nested subgrids methods (see CL:4387496)
   and correctly copy the indefinite track indices.
