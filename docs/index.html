Switch PipelineReporter to use Global TraceIDs

We've been seeing traces where PipelineReporters are nested within each other.
The reports are generated when the PipelineReporter is deleted, and was using a pointer as an identifier.

However they represent timestamps that can pre-date their existence.

We seem to be running into hash conflicts when generating perfetto::Track. This change switches to using base::GetNextGlobalTraceId to get guaranteed unique Ids.
