[Android] Don't override locale if language split not available

This CL stops the app locale override if the language split for the
override locale is not available.

Before this CL if the language split was not available the value of
java.util.Locale.getDefault would still be overridden so parts of the UI
would be expecting the override locale but the default OS locale would
be used. (For example some layouts use the text direction of the Locale
returned by Locale.getDefault to change spacing and these could be
have been different).
