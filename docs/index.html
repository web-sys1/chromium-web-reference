Deduplicate code for skipping entries for back forward navigation

NavigationControllerImpl currently duplicates the skipping logic for
the target index for CanGoBack/GoBack and CanGoForward/GoForward.

We introduce helper methods to determine the target index.

Furthermore, `GoBack` currently handles the case of it being called
when it's not possible to go back, due to tests depending on this
behaviour. There's currently only one test which does this, so we
update the test and now DwoC that `GoBack` is called correctly.
