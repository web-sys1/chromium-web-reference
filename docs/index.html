[LayoutNG] Handle "new" columns during nested frag of OOF elements

For simplicity, when an OOF element fragments beyond the last
existing fragmentainer in a nested fragmentation context, instead
of adding a new column to hold that element, we add that element
to the last existing fragmentainer with an additional inline offset
to mimic its column offset.

Paint isn't working for this yet, so unit tests were added to test
that layout is performed correctly. One such unit test is hitting a
DCHECK, which will also be addressed in a follow-up change.
