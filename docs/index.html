breadcrumbs: add desktop entry point

This change adds an entry point to breadcrumbs for desktop. This does
not enable breadcrumbs logging - that will be done via gradual
rollout of the kLogBreadcrumbs feature.

As part of adding the entry point, this change creates a
BreadcrumbManagerKeyedServiceFactory for desktop. It exists to create
a BreadcrumbManagerKeyedService for a given BrowserContext
(essentially, a profile). The keyed service, in turn, exists to attach
a BreadcrumbManager (which stores breadcrumb events) to its associated
BrowserContext.

A parallel implementation of the keyed-service factory exists for iOS
at ios/chrome/browser/crash_report/breadcrumbs/
breadcrumb_manager_keyed_service_factory.h. Parallel implementations
are needed because iOS does not have BrowserContexts or profiles; it
has BrowserStates (of which there are only two, normal and incognito).
