[Signin] Sync disabled flow should always signout for ABORT_SYNC

This CL fixes a bug in DiceTurnSyncOnHelper that it did not respect the
phrasing in the sync-disabled dialog. The button that results in
ABORT_SYNC action has a "Cancel sign in" label (to the contrary of the
other button labelled "Sign in anyway") which means the account should
get signed-out, independent of SigninAbortedMode.
