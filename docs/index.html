History Sync: Clean up redirect-related unit tests

In a redirect chain, all entries *except the first one* should have a
redirect-type marker (CLIENT or SERVER). Some of HistorySyncBridge's
unit tests were setting these markers in "wrong" ways.
This doesn't really affect the tests, but just makes them slightly
unrealistic and potentially confusing/misleading, so this CL cleans
that up.
