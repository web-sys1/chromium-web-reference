Defer begin main frame requests during early navigation stages

Android has a browser side compositor which can schedule begin main
frame updates between navigation start and the start of the navigation
request to the network. Local traces showed this could delay the
navigation request. This change defers main frame updates for the
browser compositor until the request has been sent.

The CompositorImpl is taken from the current Android window. The
compositor may change, but we will just let the lock timeout in this
case.

The base::Feature name for this change is generic since I may put some
other related optimizations behind the same feature.
