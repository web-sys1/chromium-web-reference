[PDF Ink Signatures]: Use cr-grid in ink color selector

Allows users to navigate the grid using only left/right arrow keys,
and removes a bunch of bespoke keyboard handling code that is already
implemented in cr-grid.

Originally the PDF code was using custom logic where arrow keys would
only navigate within a single row or column, but this is not obvious
to screen readers because this is labelled as a radiogroup with radio
options, which should be fully navigable using up/down or left/right
arrow keys.

Since the PDF ink color selector wants to actually select items when
the arrow keys navigate, not just focus them, have cr-grid fire a
cr-grid-focus-changed CustomEvent when a new item is focused with the
arrow keys, so that the color selector can use this event to select
the newly focused element. Add a test for the new event.
