Fix keyboard accessibility focus issues in Tab Grid Dialog

This CL addresses two keyboard accessibility issues in the Tab Group UI
dialog when using an external keyboard:
1. Focus on non-interactive elements: Removed `focusable` and
   `focusableInTouchMode` attributes from `dialog_container_view` and
   the toolbar's `main_content` so they no longer incorrectly receive
   focus.
2. Focus not trapped: Implemented focus trapping for the dialog by
   suppressing the descendant focusability of background views
   (`FOCUS_BLOCK_DESCENDANTS`) when the dialog is shown, and restoring
   it when hidden. This prevents users from navigating to elements
   behind the scrim.

Additionally, this CL sets the initial focus to the back button in the
toolbar when the dialog animation completes, ensuring a clear starting
point for keyboard navigation within the trapped dialog.
