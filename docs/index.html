[mparch] Use the correct UKM source id for MParch features in navigation

This CL is a part of MParch tasks and adds tests to check MPArch
features use correct UKM source id in navigation.

Fenced frames should use the primary page's UKM source id as iframes.
Otherwise prerendered pages and portals should use their own UKM source
id. Prerendered pages and portals works properly now, so the change in
this CL limits to fenced frames.

For fenced frame navigation, RenderFrameHost::GetPageUkmSourceId and
NavigationHandle::GetNextPageUkmSourceId eventually have the same value
(i.e. the id for the primary main frame) by this CL.
