[Actor] Dismiss overlay snackbar on tab switch

In ActorOverlayCoordinator, if the current tab switches, and if the
snackbar is visible, it should be dismissed since it's out of context
when the overlay is already hidden or changing.

To achieve this, a dedicated SnackbarController was added to the
coordinator to target only the snackbars created by the Actor overlay,
avoiding interference with other snackbars in the system. The mediator
now triggers this dismissal callback whenever the current tab changes.

Additionally, added a UMA identifier UMA_ACTOR for the snackbar.
