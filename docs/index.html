Prerender: Fix DCHECK failure on back navigation after activation

This CL fixes a DCHECK failure on back navigation from a prerendered
page.

The failure happens like this:
Activating the prerendered page creates a speculative RenderFrameHost
like regular navigation and decides whether to swap BrowsingInstance.
Actually activation always swaps BrowsingInstance, so the speculative
RenderFrameHost is cleared during activation. On the other hand, the
swap decision unexpectedly still remains after activation, and that
causes the DCHECK failure when the back navigation starts.

To fix the failure, this CL clears the swap decision in
RenderFrameHostManager::ActivatePrerender() along with the speculative
RenderFrameHost.

Notes:
- This failure happened only when the BFCache for cross-site is
  enabled (the BFCache for same-site is disabled).
- Same-origin prerender activation allows the previous page to be cached
  even when the BFCache for same-site (same-origin) is disabled. This is
  because prerender activation always swaps BrowsingInstance and it
  makes the previous page cacheacble unlike regular same-origin
  navigation.
