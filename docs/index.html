[base] Fix build when `use_allocator_shim=false`

The iOS WebView is built with the following gn args:

    ios_partition_alloc_enabled = false
    use_allocator_shim = false

This removes poisson_allocation_sampler.cc from the list of files
built in //base causing linker failure due to the uses of the
PoissonAllocationSampler.

Excluding //base/allocator/dispatcher/dispatcher.cc fixes the
linker errors.
