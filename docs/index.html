Fixed camel case of fetchPriority IDL attribute

The priority-hints spec originally launched with an all-lowercase IDL
attribute for fetchpriority. It was recently corrected to match the TAG
design princilples of camel case IDL attributes.

This updates the IDL attributes and relevant tests to use fetchPriority
and leaves the content attribute lower-cased (to match the latest spec).

Spec change: https://github.com/WICG/priority-hints/pull/71
