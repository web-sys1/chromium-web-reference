[loader] Add weakptr wrapper for CodeCacheHost

Add a mojo::Remote-owning wrapper for CodeCacheHost which allows
creating a WeakPtr to itself.

This WeakPtr can then be held by CodeCacheLoader, so that it can safely
access the CodeCacheHost with a way of handling the case when the
mojo::Remote is reset (the WeakPtr will be reset).

A side effect is that the interface passed to the code cache functions
is less mockable, so we have to introduce a more proper mojo message
pipe and corresponding task environment to some of the tests.
