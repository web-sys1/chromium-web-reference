Original change's description:
> [X11] Fix x11::Connection::HasNextEvent() for XCB-queued events
>
> HasNextEvent() was only checking if an event is present in our own event
> queue. With this CL, it also checks XCB's event queue and moves the
> event into our own queue if one is available. This fixes certain race
> conditions or deadlocks that may arise when the event dispatcher
> (X11EventWatcherFdWatch) sees the X11 socket is not readable and there's
> no event queued. Note that this issue only occurs on ChromeOS, not
> Linux, since Linux uses X11EventWatcherGlib which calls ReadResponses()
> before polling.
