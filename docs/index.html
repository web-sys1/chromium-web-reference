[ios] Switch to mach absolute time to align with mac

This updates TimeTicks in both base/time (and partition alloc's
version of said machinery) to use mach absolute time as is done on
mac if the correct build flags are set. To enable this behavior
you can set the following gn args to true
enable_mach_absolute_time_ticks, and
partition_alloc_enable_mach_absolute_time_ticks.

Alternatively, you can set use_blink = true and this will imply
setting both of the previously mentioned gn args to to true.

Additionally, this change also enables thread ticks on iOS and
this is not gated on a buildflag.

While there may be broader clock-related improvements coming
(see crrev.com/c/4225015), this CL should ensure consistency on
iOS in the short term and if there is something better in the
long term, then the approach can be migrated along with mac
(though dtapuska reminded me that absolute time and
CLOCK_UPTIME_RAW are equivalent, so these plans seem aligned).

credit: thanks to rohitrao@ for suggesting the buildflags approach
and to dtapuska@ for suggesting assertions to ensure that use_blink
makes sense wrt to these new flags.
