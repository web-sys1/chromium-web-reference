[media] Fallback to REC709 for GBR for non-444 subsampling in H264

GBR color space should only be used with 4:4:4 YUV planar formats.
For other formats, fallback to REC709 color space in H264 decoder.

This was causing issues when picking stream color space over container
ones with VideoToolbox on macs. This fixes issue.

Verified locally by visiting [1] that colors are renderer correctly
similar to Safari. Also verified with libx264_yuv420p video that
it takes the fallback path and works with patch and is broken before.

[1] https://ambitious-melon-wildebeest.glitch.me/
