Revert "Broaden CECPQ2 prefix for M93."

This reverts commit 5c790e57f89b11bad782018a929d61a7a4dc6c9f.

Reason for revert: We're going to go back down to zero with CECPQ2 for a while while we contact vendors whose products have been found to cause problems.

Original change's description:
> Broaden CECPQ2 prefix for M93.
>
> The prefix was already “a” and making that any smaller results in a 100%
> rollout, which is faster than desired. Thus switch to including all
> hosts beginning with one of a list of initial characters. For M93,
> that'll be a and g. The 'g' is so that Google hosts are included which
> will exercise the larger server reply. (But not much because connections
> to Google are generally QUIC, which doesn't have CECQP2.)
