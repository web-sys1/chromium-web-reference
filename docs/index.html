ozone: Send CanComposeInline to Wayland server

Inline composition is an IME feature for certain languages (e.g. CJK)
which displays the uncommitted composition inside the input field as it
is being typed.

For input fields that do not support inline composition (e.g. PDF input
fields), Lacros did not communicate this information to Exo and Exo
assumed it was always supported. The result is that users of CJK IME
cannot see what they are typing until they have committed it.

In a previous commit, we extended the Wayland text input protocol to
make it possible for Lacros to communicate this information to Exo. In
this commit, we update Lacros to start using the new API. When Ash is
made aware that inline composition is not supported, it displays the
composition in an external IME dialog.
