[M113][iOS] Fix crash when adding to reading list from incognito

The new snackbar shown when an item is added to reading list shows the
primary user's email if the sync is on. We retrieve this from the
IdentityManager, but the latter is null if created from incognito
browser state.

As for bookmarks, reading list items of the original primary account is
shown in incognito mode. Therefore it seems natural to show the
account's email in incognito mode as well when adding a reading list
entry.

This CL ensures that the IdentityManager used is retrieved using the
original browser state, and adds an EGTest for adding an item in incognito mode.
