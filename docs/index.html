Fix bucket max in base::UmaHistogramPercentage

The macro UMA_HISTOGRAM_PERCENTAGE passes a bucket max value of 101
https://source.chromium.org/chromium/chromium/src/+/master:base/metrics/histogram_macros.h;l=130
https://source.chromium.org/chromium/chromium/src/+/master:base/metrics/histogram_macros_internal.h;l=146
However the supposedly equivalent function base::UmaHistogramPercentage
passes a bucket max value of 100 here
https://source.chromium.org/chromium/chromium/src/+/master:base/metrics/histogram_functions.cc;l=42

This patch fixes this discrepancy in the histogram function,and rename
all existing usages of base::UmaHistogramPercentage to
base::UmaHistogramPercentageObsoleteDoNotUse.
