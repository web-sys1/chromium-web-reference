Android: Track target_cpu="x86" by default

Builds are actually slightly faster with target_cpu="x86":

target_cpu="x86":
base_java_sig: 129.7s
default (arm):
base_java_sig: 132.1s

Since build and build+install are often benchmarks that are compared,
they should use a common set of GN args. This means defaulting to
target_cpu="x86" as a common baseline.

Most devs are making extensive use of emulators due to the number of
APIs that Chrome on Android supports, so it is more worthwhile to track
and reduce this build time than the default target_cpu.
