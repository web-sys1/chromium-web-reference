overview: Use Unminimize() when split view isnâ€™t activated

This CL changes to call Unminimize() immediately when the minimized window is selected, if the split view is NOT activated.
Previously (crrev.com/c/2327364), we used Show() instead of Unminimize() as SplitViewController can handle auto snapping cases and calling Unminimize() for async windows (e.g. ARC windows) wrongly bypasses the auto snapping.
But calling Show() causes the animation issue if the split view is NOT activated, so in this CL we change to use Unminimize() for the case.
