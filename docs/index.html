[Autofill Assistant] Change `std::map<>` to `base::flat_map<>`

Main differences between the two are:
- In base::flat_map iterators are invalidated when adding/removing
  elements
- Insertion and removal is O(n) instead of O(logn)

I've checked ussages of all variabled changed and didn't find instances
of iterators being used after mutating the map. The only place I wasn't
sure is in
chrome/browser/android/autofill_assistant/interaction_handler_android.h
where a callback could change the map. Even if we don't have one right
now it would be easy to introduce a bug there if changed, so I left it
as a std::map.

Where possible, I constructed new flat_maps in a single operation by
pushing into a vector first, avoiding a O(n^2) operation.
