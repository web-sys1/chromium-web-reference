dbus: Move concierge generated proto files to vm_concierge

Generated system_api proto files should live in a directory that matches
the structure of the cros_system_api/dbus dir in third_party/. Not doing
so means that proto definitions can not be included in other proto
files, since the generated code for the other proto will (by default)
try to #include the wrong path.

We fix this for the concierge proto. This could also be fixed for other
protos but we leave that until it is needed.
