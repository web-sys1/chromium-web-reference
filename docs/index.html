T&S Sentiment: Add trigger action input for Privacy Settings

Extended the TrustSafetySentimentService to include logic for
Privacy Settings triggers:
  - Spending time on settings after interacting with privacy settings
  - Running Safety Check

To support the first trigger, a SettingWatcher nested class is added
to the sentiment service to watch a WebContents to confirm a user has
stayed on settings.

The set of Product Specific Data for the Privacy Settings feature area
is also included, this consists of 1 bit indicating whether the user is
being surveyed because they ran safety check, and 1 bit indicating
whether the user has any non-default core privacy settings.
