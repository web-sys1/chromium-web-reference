[chromevox]: directly control vertical and horizontal spacing for multi-line braille

This change makes it minimally viable to test image to braille work on a
multiline display. More work to come[1].

Specifically:
- PanStrategy now introduces spacing into braille output for non-word
wrapped braille
- spacing is fixed to 2 dot rows horizontally and 1 dot column
  vertically.
- word wrapped spacing to come in a follow up

1. See go/cros-image-to-braille for more details.
2. depends on https://chromium-review.googlesource.com/c/chromiumos/overlays/chromiumos-overlay/+/5225591
which removes spacing in the underlying driver

Test: manual (on device)
