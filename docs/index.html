Updates GetOrComputeInnerText which like the value attribute should add line breaks between block elements

In preparation for computing the value of a content editable, and possibly other fields, on the browser
instead of in Blink for performance purposes, this patch adds the ability to compute inner text taking
into account line breaks that might be introduced by layout.
Similar to Blink's definition of "textContent" vs. "innerText", "innerText" should contain
any such line breaks, whilst "textContent" should not.

A followup patch renames `AXNode::GetInnerText" to `GetTextContent" for consistency with
this added functionality.
