[omnibox] Change OpenMatch disposition for tab switch action, for metric

This fixes the broken Omnibox.SuggestionUsed.SelectedTabMatch selected
state. It was not being captured due to using the match/action opening
disposition instead of SWITCH_TO_TAB as required specially for this
action and histogram. We were faithfully plumbing the disposition based
on whether user held keyboard modifiers when selecting the action (and
this is useful for other actions like pedals) but an early override
is needed for metrics in this case.
