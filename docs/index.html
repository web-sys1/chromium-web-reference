[block-in-inline] Make NGFragmentItem::BlockInInline() to return LayoutObject

This patch changes |NGFragmentItem::BlockInInline()| to return
|LayoutObject| instead of |LayoutBlock| because the first child of
block-in-inline can be |LayoutImage|, to avoid invalid type cast
|To<LayoutBlock>()|.

Example:
# HTML Document
  <div><span><img style="display:block"></span></div>

# Layout tree
  LayoutNGBlockFlow P id="root"
    +--LayoutInline SPAN
     |  +--LayoutNGBlockFlow (anonymous)  # block-in-inline
     |  |  +--LayoutImage IMG id="target" # first child of block-in-inline
     +--LayoutText #text ""
