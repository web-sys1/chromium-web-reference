[Chromecast] Increase unstable timestamp detection threshold on ATV

Temporary raise the threshold for unstable timestamp detection to 1ms,
on ATV devices. It may take long time for the platform to return a
stable timestamp, which blocks the playback since there's no valid
renering delay returned.

According to CDD [C-1-1]
https://source.android.com/compatibility/android-cdd#5_6_audio_latency
The accuracy for AudioTrack.getTimestamp() is +/- 2ms.

Bug: internal b/180409124
Test: playback on device
