Ensure that EyeDropper is not created when the feature flag is disabled.

Before this CL we were relying on define guards to ensure that
the eye dropper is not created on not supported platforms[1].

This approach created an issue on ChromeOS, where we were attempting
to create an EyeDropper without having the feature flag enabled.

This change adds feature flag checks to ensure that the EyeDropper
is not created when it is not supported.

[1]: https://source.chromium.org/chromium/chromium/src/+/main:chrome/browser/ui/views/eye_dropper/eye_dropper.cc
