ime: [1/2] Use InputMethod instead of InputChannel.

Right now, there are three types of input method engines: "rule-based",
"decoder" and "system". All three share the same mojo interface
InputChannel.

However, "decoder" is very different from "rule-based"/"system", which
makes InputChannel a very confusing interface.

After this CL, "decoder" will be created via ConnectToImeEngine,
and "rule-based" and "system" will be created via
ConnectToInputMethodEngine. The former will use InputChannel and the
latter will use a new interface called InputMethod. Move all the
non-"decoder" methods to the new interface.
