Add password auth view

This CL adds implementation for `password_auth_view`, which is a
subclass of `factor_auth_view` in the `auth_panel` code. The state
for `password_auth_view` lives in the `auth_factor_store`, which handles
storing the state and processing state transitions. `password_auth_view`
itself only takes care of UI logic.

Future CL would cover implementing the timers for the password field, as
well as integration with `auth_panel` and a new feature flag to hide the
entirety of the `auth_panel` code behind for toggling between new and
old implementations.
