ui: Migrate BASE_FEATURE macros to 2-argument version

This CL migrates three-argument BASE_FEATURE macros in //ui to the
new two-argument version where the C++ identifier and the string feature
name match.

This automated refactoring helps ensure that the C++ identifier and the
string feature name always match, reducing potential errors.

The following features could not be migrated automatically and may
require manual intervention:

   * kAccessibilityPdfOcrForSelectToSpeak vs "kAccessibilityPdfOcrForSelectToSpeak"
       * Name has "k" prefix.

   * kEnableAccessibilityAriaVirtualContent vs "AccessibilityAriaVirtualContent"
       * Identifier has "Enable" prefix.

   * kEnableAccessibilityLanguageDetection vs "AccessibilityLanguageDetection"
       * Identifier has "Enable" prefix.

   * kAccessibilityOnScreenMode vs "AccessibilityOnScreenAXMode"
       * Name has "AX" infix.

   * kAccessibilityCaptionsOnBrailleDisplay vs "CaptionsOnBrailleDisplay"
       * Identifier has "Accessibility" prefix.

   * kEnableSingleCancelTouch vs "EnableSingleTouchCancel"
       * Case and word order mismatch.

   * kSettingsShowsPerKeyboardSettings vs "InputMethodIntegratedSettings"
       * Different names.

   * kUIDebugTools vs "ui-debug-tools"
       * Case and kebab-case mismatch.

   * kEnablePixelCanvasRecording vs "enable-pixel-canvas-recording"
       * Case and kebab-case mismatch.
