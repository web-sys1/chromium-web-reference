History Sync: Handle HISTORY data in LoopbackServer

The LoopbackServer notifies its observers of "history commits". Before
this CL, that happened only for history commits via the SESSIONS data
type; this CL adds the logic to send similar notifications for actual
HISTORY commits.
In practice, this functionality is only used in some SESSIONS
integration tests so far, so this has no actual impact. In the future,
HISTORY integration tests will likely exercise it though.
