[ios] Change RetrieveCanonicalUrl() to use a base::OnceCallback<>

Change RetrieveCanonicalUrl() to take the completion as a callback
instead of a block because internally it invokes a function that
takes a callback. This makes the implementation of the function
safer (as there is no risk to capture a variable) and simpler (as
it can use `base::OnceCallback<>::Then` to chain the callbacks).

Introduce an helper function `AddToReadingListUsingCanonicalUrl`
that fetch the canonical URL for the visible page, and add it to
the reading list safely once the fetch is done (this deduplicate
code between `OverflowMenuMediator` and `PopupMenuMediator`).

Fix an invocation of `activity_services::RetrieveCanonicalUrl` in
`ActivityServiceCoordinator` that passed a block capturing a C++
pointer (to `web::WebState`) which is unsafe. The fix uses
`base::WeakPtr<>` to capture a weak pointer to the object and
an helper method (to avoid complex code to dereference the weak
pointers).
