Reland "Unify initialisation of memory system"

This is a reland of commit 2292e533ba1fc7597bf068c4004e79eac1a71e3c

The error was introduced by using HeapProfilerController
unconditionally, although it's not included for iOS without
AllocatorShim. The reland ensures that on iOS the HeapProfilerController
is only used if the AllocatorShim is enabled.

A second error was the inclusion of memory_system as a component, which
led to duplicate definition of symbols in component builds.

Original change's description:
> Unify initialisation of memory system
>
> Initialisation of memory components like GWP-ASan was performed in each
> delegate separately. This CL unifies the initialisation and provides a
> common implementation.
