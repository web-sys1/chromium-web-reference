Linux OSCrypt: Add GetRawEncryptionKey() and SetRawEncryptionKey()

See the attached bug for details on why this CL adds these functions.

Adding these functions requires us to cache the crypto::SymmetricKeys
themselves rather than the passwords used to derive the SymmetricKeys.
This should be slightly more performant anyway as we don't need to
re-derive the SymmetricKeys every time we want to encrypt or decrypt.

This CL adds the function to os_crypt_posix.cc as well to avoid the
unnecessary #if blocks everywhere, even though os_crypt_posix.cc
only uses a hardcoded password.

This CL modifies the test-only functions as well to make testing a
little more clearly.

Finally, this CL stops compiling os_crypt_linux.cc for lacros, and
starts compiling os_crypt_posix.cc like it did for older chromeos
ash. Encryption doesn't work and isn't necessary on Chrome OS so
there's no need for lacros to compile it.
