[Code Health] Lock and guard NTP alphas and translation Y during Omnibox navigation.

When a user navigates away from the NTP using the Omnibox, the focus
change animations are disabled (setUrlFocusAnimationsDisabled(true)).
Because focus-clearing transitions are bypassed: 1. The vertical
translation Y is never reset to 0. 2. The fake search box and logo
alphas (which faded to transparent 0.f during focus) are never restored.
3. Subsequent focus-clearing updates from the browser toolbar overwrite
and clear forced alpha states, causing NTP elements to prematurely
disappear while loading.

This CL resolves the issue by:
- Reordering assignments inside setUrlFocusAnimationsDisabled to force-restore layout translation Y to 0f, search box alpha to 1.f, and logo alpha to 1.f before locking animations.
- Guarding setSearchBoxAlpha and setSearchProviderLogoAlpha to block and ignore subsequent external alpha updates when focus animations are disabled.

Detailed analysis:
https://docs.google.com/document/d/1YpNt-sZXo2058MwkyxFIHSBM1zSYqQ-VcAATnC6puNU/edit?usp=sharing&resourcekey=0-V2PPs8Arvr3UhXnWYcGiSQ
