desktop-pwas: don't become default file handler

This change makes PWA file-handler registration no longer set the PWA
as default handler, and no longer customize the file-type name for
default-handled file types.

Currently, PWAs are set as the default handler for handled file types
if there is no existing default. However, it was found that this
could be disruptive if Windows had already cached an 'implicit'
default handler, which it does when the user opens a file type with no
default handler in any app (the app becomes a soft default until a
real default is set). In an effort to not overwrite users' existing
state, this change stops setting the default handler in PWA installs
altogether.

This change also includes a related change to stop customizing the
file-type name for types that a PWA handles by default. Currently,
this is set to the PWA name, e.g., "TXT File" becomes "My PWA". This
is similarly disruptive while not providing much value. A planned
change to implement custom file-type names will allow PWAs to set this
field to something meaningful. In the meantime, this change stops
customizing it during PWA install.
