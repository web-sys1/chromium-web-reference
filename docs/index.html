Refactor media-device salt utilities

This CL:
* Makes all utilities in //content/browser take a
  MediaDeviceSaltAndOrigin parameter instead of separate salt and origin
  parameters. This simplifies callsites, which no longer have to unpack
  a MediaDeviceSaltAndOrigin. Some of the utilities have a small rename
  to avoid clashes with the utilities in //content/public.
* The interface of the utilities in //content/public remains unchanged
  because  MediaDeviceSaltAndOrigin is a //content/browser type.
  However, GetMediaDeviceIDForHMAC() is now callable from any thread.
* Moves utilities which were static methods from MediaStreamManager to
  media_devices_util.h|cc, which is a better place for these utilities.
* Moves unit tests for the moved utilities to a new unit test for
  media_devices_util. Some simpler tests in the existing browser test
  for emdia_devices_util have been moved to the unit test.
* Removes some dead code in MediaStreamManager.
* Turns MediaDeviceSaltAndOrigin into a class. MDSAO was originally
  conceived as a struct to make it simpler, but the compiler requires an
  out-of-line copy constructor, which eliminates the potential benefit
  of aggregate initialization. As a class, MDSAO:
      * No longer has a default constructor, to avoid silently
        initializing empty salts and origins.
      * Has an explicit static method to return an empty MDSAO.
      * Its constructor has default arguments for all fields, except
        salt and origin, which are the two most basic fields that should
        normally be explicitly initialized.
      * Has getters to access fields. Prior to this CL, fields were only
        set to initialize them since the constructor wasn't always
        convenient. Also added setters, although they are used only by
        one test.
      * Has move constructor and assignment operator.
