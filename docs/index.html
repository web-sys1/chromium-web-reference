Dictation: Fix "type" not being omitted

This change fixes a Dictation issue where "type <phrase>" was being
committed to text fields. For example, if I said "type testing 123",
Dictation would commit "type testing 123" to the text field. The
desired behavior is to omit the "type" prefix. Note: we already do
this for supported commands, so this change just extends our support
to all text.
