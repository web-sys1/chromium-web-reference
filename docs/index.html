This reverts commit 10e7cbd066e077af23749b62524c79287539e5da.

Reason for revert: Crashes observed likely related to this change. See https://crbug.com/1362872.

Original change's description:
> wayland: make wl_output/Display id differences explicit
>
> WaylandOutput's id and WaylandScreen's Display::id are used mostly
> interchangeably throughout Ozone/Wayland code, despite their slight
> differences, eg: 32 vs 64-bit, sentinel value, etc which is OK, though
> may lead to some readability difficulty and even to some bug-prone
> assumptions. In order to improve the situation, this CL does:
>
> - Use a type alias for WaylandOutput::Id;
> - Improve its docs, mentioning explicitly where it comes from and how it
>   relates to PlatformScreen's display::Display::id property;
> - Refactor WaylandWindow::GetPreferredEnteredOutput API to return
>   optional<WaylandOutput::Id> (rationale: output "null" id is zero,
>   while display::Display's null id is -1) and updates the call-sites.
>
> No functional change should be introduced with this CL.
