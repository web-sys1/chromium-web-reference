chrome: make BrowserList deal with Browser deletion

When running unload handlers BrowserList was binding
a vector<Browser*> to a callback. This is not safe
because a Browser may be deleted before the callback
runs. The fix is to use a vector<WeakPtr<Browser>>
and ensure the WeakPtr<Browser> is valid before using it.
