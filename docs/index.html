Original change's description:
> [soft navs] Move Paint area calculation into Context
>
> Previously, we would only use a single "dom modified bit" stored inside each Node as a mechanism to decide if a Paint for a specific Node was due to a soft navigation.  The problem is that this bit is "sticky" and persistent forever, and so each new interaction builds upon all past interactions.
>
> Now, each Context maintains its own list of Nodes, which only it modified directly. This was, we can more accurately attribute paints to specific Interactions.
>
> This is more expensive to manage, and so this patch also introduces a performance optimization to try and trade off the regression: we now only check paint criteria once per animation frame, after paint finishes, rather than once per observed paint.  We also try to check all the inexpensive criteria first.
>
