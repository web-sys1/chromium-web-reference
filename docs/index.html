[Payment][Desktop] Make selected-method logging consistent with Android

Motivation:
The logging of the selected method is at different moments for desktop
and Android. While desktop logs it in OnPaymentResponseAvailable(),
Android logs it in a much earlier time - invokePaymentApp(). This
inconsistency was deliberately made by CL[1] when "Google payment
method name was not available on Desktop at the time of clicking on
Pay" back then. Now that it's no longer true, we should revert the
change to make them consistent.

Change:
On desktop, moves the logging of selected method from
PaymentRequest::OnPaymentResponseAvailable() to PaymentRequest::Pay().

[1] https://crrev.com/c/1596829
