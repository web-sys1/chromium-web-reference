[remoting] Unset some systemd environment variables for Wayland.

Some systemd environment variables are known to cause problems for the
Wayland session. If these variables somehow get imported into systemd,
they typically remain there until the system is rebooted.

This CL unsets GDK_BACKEND and SSH_CONNECTION from the systemd user
environment before starting a Wayland session.
