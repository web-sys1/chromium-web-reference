Use system colors for painting scrollbar corner in forced colors mode

The CL fixes the painting of the scrollbar corner part in forced colors
mode (e.g., high-contrast mode on Windows) by utilizing the system
colors instead of custom colors used for painting light/dark scrollbars.

List of changes:
    - Use GetControlColor() instead of direct color initialization in
NativeThemeAura. kColorScrollbarCorner color was added to the color
piepline in the CL [1].
    - Paint the scrollbar corner part with SystemThemeColor::kWindow if
forced colors mode is enabled.
    - Update web test expectations.

[1] https://chromium-review.googlesource.com/c/chromium/src/+/3966310
