Fix "Effective" default value in augmented ONC

If ONC policy forces or recommends a default value that is taken from
then ONC signature, also set "Effective" augmented field to this value.

This makes the UI recognize values as policy-recommended /
policy-forced when they only have an ONC default value.

Side effect:
Having done this, I also had to set a default for IPConfig type,
because otherwise I was running into a DCHECK in cros_network_config.
