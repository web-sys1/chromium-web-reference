[iOS 27] Fix Tab Grid Misalignment

In iOS 27, the Tab Grid is misaligned when a user navigates from a tab
in either Incognito or regular mode to a tab in the opposite browsing
mode.

It is unclear what has changed internally in iOS 27. However, there is
an autolayout pass being executed during the grid-to-tab animation. The
TabGridVC calculates its intended position relative to the position of
other UI components. So, when the autolayout pass is run and the other
UI elements are out of position, it causes the TabGridVC to miscalculate
its position, resulting in its misalignment.

This CL resolves the misalignment by disabling the TabGridVC's
constraints while the tab-to-grid and grid-to-tab animations are running
and reenabling the constraints once the animation completes. This
solution has been tested on older versions of iOS successfully. The CL
implements the solution by:
  1. Adding activate/deactivate constraints methods to GridContainerVC
  2. Introducing two new commands in the TabGridCommands protocol.
  3. Updating TabGridCoordinator to implement these new commands.
  4. Calling these activation/deactivation commands at the beginning and end of the grid-to-tab and tab-to-grid animations.
