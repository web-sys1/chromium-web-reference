[Autofill] When refill triggered, disregard query_id comparison for the new sectioning

Query ids are used for communication between the renderer and browser process in AutofillAgent::FillOrPreviewForm().
In this CL, we avoid the query_id comparison for the new, parameterized sectioning algorithm. This is because in some cases when the form dynamically changes an incorrect element is identified as the matching, corresponding element to the null or unfocusable AutofillAgent::element_ when AutofillAgent::ReplaceElementIfNowInvalid() is called. As a result, the refill might be unsuccessful because of different sections.
