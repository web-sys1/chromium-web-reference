[PreconnectManager] Eliminate max_inflight_preresolves feature

After experimentation, we have determined that the current default
value for the maximum number of in-flight preresolves is optimal. This
CL eliminates the ability to experiment with this value via Finch,
partially reverting
https://chromium-review.googlesource.com/c/chromium/src/+/2316324. We
leave in place the ability to experiment with the maximum number of
in-flight prefetches, as we have not done any experimentation on that
front. This CL also eliminates the experiment from
fieldtrial_testing_config.json.

This CL has no behavioral impact on production, as it's not changing the
current default value. It's just code cleanup.
