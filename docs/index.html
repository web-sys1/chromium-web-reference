Partially Revert "Revise Windows App Icons"

This is a partial revert of 85d74a4f9b7e4bf535cec0e5270460c7ceaa7722.

The issue is that while "transparent" is a permitted background color
visual elements [1], it doesn't appear to be permitted for classic
apps.

The observed behavior will be as follows:
Win 8       - Directly uses the background color
              Will go back to the previous gray
Win 10 1511 - Does not appear to use tiles for classic apps
              Visual Elements is ignored
Win 10 21H2 - Appears to ignore the background color
              and uses the transparent PNG.

[1] https://docs.microsoft.com/en-us/uwp/schemas/appxpackage/appxmanifestschema/element-visualelements
