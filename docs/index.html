[ios] Fix accessors for "unrealized" WebStates

Implement GetTitle(), GetVisibleURL() and GetLastCommittedURL()
for "unrealized" WebStates by reading them from the serialized
data.

Ensure that the serialized user data is attached to "unrealized"
WebStates as it contains the `tab_id` from TabIdTabHelper which
is used to map snapshots to tabs (this data should probably be
stored directly in the WebState as it would simplify new session
serialization format implementation).

Update the serialized user data when returning the serialized
state for an "unrealized" WebState as it may have been modified
by the client code (as it is read-write even for "unrealized"
WebStates).
