[rustmojo] Implement and migrate data pipes to new FFI

This CL implements the functions for reading from and creating data
pipes, then migrates the existing system bindings to use the new
constructs. There's more work to do there to make things ergonomic and
implement additional data pipe functionality, but this covers the
alread-written code.

One difference from the previous API is that we now take in an array
of uninitialized bytes, which saves the effort of default-initializing
them but also requires the user to `unsafe`ly mark them as initialized
after calling the function. This is a bummer but is unavoidable if we
don't want to impose unnecessary overhead on users. One of the future
changes we could do is adding safe equivalents that use already-
initialized bytes.
