Get rid of forced layouts when a11y is first turned on

First, defer processing of AXMode changes until the layout is clean anyway.

Also, handle all AXMode changes in a consistent way, reusing some recently written code to mark the entire document dirty and reserialize everything. After all, arbitrary changes between AXModes
could lead to arbitrary changes in the AX tree.  This could occur more often now that we'll be adding a forms mode.

This also improves the MarkDocumentDirty() method to make it does its work at a safe time, and that no irrelevant nodes are marked dirty between when the document becomes dirty and when serialization occurs.

After this, there are still a few places where layout is forced, notably, when there is an action to perform.
