Minor fixes to StackSamplerImplTest

Fix some minor issues with StackSamplerImplTest_CopyStack*. In
particular:

 * TestStackCopier was setting stack_top to a place inside the stack
   instead of a place inside the copy of the stack in stack_buffer
 * TestStackCopier was not correctly copying out the stack contents
 * TestStackCopier was not rewriting pointers; in particular, it left
   the RegisterContextStackPointer pointing to the "real" stack instead
   of the correct location in the copied stack.
 * TestUnwinder was getting the stack size directly from the
   constructor, unlike a real unwinder which needs to use the stack_top
   and the RegisterContextStackPointer to figure out the size of the
   stack.
 * TestUnwinder had a number of features which were not used, making the
   code harder than necessary to follow.
