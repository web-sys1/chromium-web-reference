[v8][wasm] Extend expiration of some histograms

- V8.WasmMemoryAllocationResult
- V8.WasmModuleCodeSizeBaselineMiB
- V8.WasmModuleCodeSizeTopTierMiB
These histograms are still needed to understand resource consumption of
WebAssembly memory and code.

- V8.WasmModuleNumberOfCodeGCsTriggered
This shows if the GC keeps being efficient, and will be more interesting
as we implement more dynamic compilation and tiering strategies.

- V8.WasmModulesPerEngine
- V8.WasmModulesPerIsolate
These histograms guide some optimization decisions in the engine. We
currently mostly optimize for the single-module use case.

- V8.WasmMemoryProtectionKeysSupport
This is the main indication of PKU adoption, and was only recently
added.

- V8.WasmCatchCount
- V8.WasmReThrowCount
- V8.WasmThrowCount
- V8.WasmTimeBetweenCatchMilliseconds
- V8.WasmTimeBetweenRethrowsMilliseconds
- V8.WasmTimeBetweenThrowsMilliseconds
These are useful to understand the general usage of exceptions, and to
make optimization decisions.
