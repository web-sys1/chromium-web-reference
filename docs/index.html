Original change's description:
> Updater: Rename ChromiumUpdater_Test_V2.app.
>
> This moves ChromiumUpdater_Test_V2.app to
> ChromiumUpdaterTestAppV2/ChromiumUpdater.app. This is important because
> the executable name within the application matches the name of the .app,
> but the app has assumptions about the executable's name compiled
> in. (E.g. for setting up launchd services or other branded elements.)
>
> Additionally, this functionality is helpful to build the secure variant
> of the updater described in issue 1218109 and the design doc at
> https://bit.ly/chromium-updater.
>
> Context: when compiling the updater, we build multiple variants of the
> app bundle that vary by one or two .cc files (e.g. changing the program
> version or presence of certain functionality). On Mac, ideally these
> variants can have identical app names (and app bundle structure), but
> this isn't easy to achieve because GN targets must produce uniquely-
> named output. Therefore, this CL introduces the ability to create app
> bundles in subdirectories to avoid collision while retaining identical
> bundle names.
>
