CSP: Implement back-and-forth type conversions mojo vs mojo-blink

Parsed Content Security Policies are stored in some mojo types and
need to be synced back and forth between the browser process and
blink. While at the moment we only need sending a subset of Content
Security Policies from blink to the browser for performing
navigational checks, for incorporating CSPs in the policy container we
will need a full sync in both directions. Unfortunately, (some of) the
mojo methods performing the sync go through the content/renderer
layer, hence we need to implement type conversions between

- network::mojom::<CSP types>
- Web<CSP types>
- network::mojom::blink::<CSP types>
