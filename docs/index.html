Drop dispatched invalidations from cache

As discussed in crrev.com/c/5490112, it is theoretically possible that
an invalidation gets delivered to a handler twice, for instance by
updating the interested topics twice with the same topics.

This CL removes this possibility by erasing all successfully dispatched
invalidations from the cache.
