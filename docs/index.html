PrefModelAssociator: Improve pref type checking

If an incoming synced pref value doesn't match the registered type of
the pref, it should be ignored.

Before this CL: PrefModelAssociator checked the pref type by comparing
to the type of the pref's value in the user store. If the pref didn't
have a value in the user store, the type check was skipped. (And if
the type actually didn't match, a NOTREACHED would get hit in PrefService.)

After this CL: PrefModelAssociator checks the actual type of the pref
registration, independent of the presence of any value.

This improves the accuracy of the type check, and it unlocks a
cleanup (also done by this CL): PrefModelAssociator doesn't need a
reference to the user pref store anymore (it now only interacts with
prefs via the PrefService).
