webauthn: use the new Crypter construction with recent QR codes.

In a caBLEv2 connection, the phone sends the first encrypted message
after the Noise handshake. Therefore, if the client expects the new
Crypter construction, the phone must use it for that message. Chromium
accepts either construction and we're waiting for old versions to die
out before switching the default for Android to send.

However, iOS is moving towards the new construction and probably won't
auto-detecte. Thus this change makes it so that an Android phone will
use the new construction if triggered by a QR code from a client that
advertised two (or more) registered tunnel server domains. This will
cause the new construction to be used with modern clients, such as
future iOS versions.
