media/gpu/v4l2vd: Reallocate OUTPUT buffers if needed on resolution change

Some V4L2 stateless decoder kernel drivers override the buffer size we
request. This likely results in a tiny buffer size due to the initial
minimal frame size. Attempting to use the same buffers after a dynamic
resolution change would fail if the next coded frame does not fit in
the old buffer.

To fix this, after the new resolution is applied, check if the returned
buffer size in `sizeimage` is larger than the buffers we currently hold.
If so, return the old buffers and request new ones.
