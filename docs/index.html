Preloading: Add eagerness filtering mechanism to PreloadingDecider

Currently, `PreloadingDecider::IsSuitableCandidate()` filters eagerness
based only on PreloadingPredictor. In the meanwhile, we plan to add new
mouse hover heuristics for "eager" eagerness, which means that pointer
hover events will be called twice on the same link for "eager" and
for "moderate". To distinguish them, this CL adds mechanism to filter
preloading candidates by eagerness. This is a preliminary change for
future implementations.
