Reland [Messages] Use webcontents focus/lostFocus to control navigation scope

On start surface and tab switcher, #wasShown can be called by the
webcontents of selected tabs, which leads to activate navigation scope
by mistake.

Use onWebContentsFocused/LostFocused to determine if the current
webContents is user interactable or fully visible.

Reland: The PortalTests failed because for unknown reason,
the web contents is used before initialization.
Make #getViewAndroidDelegate return null if it is invoked before
initialization.
See details: https://crbug.com/1278223#c19
