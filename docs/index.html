Fix FullRestoreAppLaunchHandlerArcAppBrowserTest.NotHideArcWindow.

In the test case, two windows are created for 2 tasks:
"org.chromium.arc.200"
"org.chromium.arc.300"

The first window is destroyed before creating the second window.
However, occasionally the second window is pointed to the same address
as the first window after destroyed. This could affect the test result,
because "VerifyObserver" checks the result based on the window address.

Modify the test case to defer the destroying for the first window to
prevent creating the same address for the two windows.
