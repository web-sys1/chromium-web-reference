MacPWAs: Pass browser bundle path to app shim at launch

Add a mechanism through which Chrome can explicitly tell an app shim
that it launches the bundle path for Chrome. This increases the
robustness of loading the correct bundle in testing (at the expense
of no longer having the exact same code path being used for app shims
launched by Chrome and not launched by Chrome).

In particular, update app_mode::FindBundleById and rename it to
FindChromeBundle. Update the function to have the behavior of:
- Use the command line specified bundle path as a first preference.
- Always prefer a running Chrome bundle path to a non-running Chrome
  bundle path.
- If there exist multiple running Chrome instances, select the most
  recently launched instance (this is a vanishingly rare circumstance).
