Sync: Remove special-casing for commit-only types in the processor

ClientTagBasedModelTypeProcessor contains logic which, on startup,
verifies that any persisted sync metadata is in a consistent state.
One of the conditions is that if any metadata exists, then
initial_sync_done should also be true.
However, there used to be an exception for commit-only types:
Historically (before M72, crrev.com/c/1136307), these didn't persist
initial_sync_done, so it can legitimately happen that there's
persisted metadata without initial_sync_done.

As of M111 (crrev.com/c/4161698), such invalid metadata gets cleaned
up (removed from disk), *except* for commit-only types.
This CL removes that exception, so that commit-only types get the same
cleanup logic as all other types.

Note that such invalid metadata was never actually *used* (since at
least M72), so in that sense, the actual behavior isn't changing.
What's changing is just that we're now better at actually cleaning up
the bad data, and as a side-effect, avoid hitting a DCHECK (see
crbug.com/1413944)
