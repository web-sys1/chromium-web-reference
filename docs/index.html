lobster: use width and height defined by the inner content

Before this revert (crrev/c/5992679), the enablement of Lobster disables
ShouldAutoResizeHost. Without ShouldAutoResizeHost, the initial width and height must be set from C++.

Unfornately, Lobster and Editor WebUI share the same web content bundle (mako). When Mako's ShouldAutoResizeHost is disabled, Orca UI dimension has to be set from C++ too. This requires the enablement of OrcaResizingSupport flag currently to make it work.

Unfortunately, resizing support for Orca is still ongoing and therefore full enablement of this flag isn't fully polished.

To decouple Lobster UI from OrcaResizingSupport, we apply the WebUI sizing strategy used by Orca without OrcaResizingSupport.

I.e., setting the webui dimension from the web inner content (typescript) and propagating the setting using `WebUIBubbleDialogView::ResizeDueToAutoResize`.
