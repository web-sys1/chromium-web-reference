lacros/color: preload common colorspaces

The Chrome Color Management protocol currently needs a roundtrip between
the client and server the first time a colorspace is requested for use.
This means for certain content the first frame will be in a wrong or
default colorspace before switching over to the correct one. This can
cause flashing.

This CL mitigates the issue some by preloading the colorspaces commonly
seen on the web (youtube, google meets, hdr content) so they are ready
the first time they are requested.
