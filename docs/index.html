Handle attributionsrc trigger registration in data host manager

This change propagates triggers registered via attributionsrc from
the AttributionDataHostManagerImpl to the AttributionManagerImpl,
effectively completing the pipeline for the new API surface.

Since the AttributionTrigger class lacked a suite of test matchers,
this CL also introduces a new way of using those: a single config
struct per API type that contains a single matcher field per API type
field. These matchers default to matching anything, but can be
initialized via designated initializers with arbitrary matchers of the
correct type. In future CLs, we will replace the existing MATCHER_P
definitions in attribution_test_utils.h with analogous structures for
CommonSourceInfo, StorableSource, StoredSource, and AttributionReport.
