Clean up unused initial_interceptors in NavigationURLLoader::Create

NavigationURLLoader::Create used to accept an initial_interceptors
parameter, which was conditionally populated with the interceptor from
WebBundle (previously BundledExchanges) navigation in
NavigationRequest::OnStartChecksComplete.

However, the "Navigation to WebBundle" support was removed in
crrev.com/c/4196674 (commit 22a27b1455cf0), which deleted the logic
populating this vector, leaving behind the unused local variable in
NavigationRequest and the parameter in NavigationURLLoader::Create.

This CL cleans up the dead code by:
1. Removing the unused `interceptor` vector in
   NavigationRequest::OnStartChecksComplete.
2. Removing the `initial_interceptors` parameter from
   NavigationURLLoader::Create.
3. Passing an empty vector directly to NavigationURLLoaderImpl inside
   NavigationURLLoader::Create (as NavigationURLLoaderImpl still
   supports injecting interceptors for unit tests).
