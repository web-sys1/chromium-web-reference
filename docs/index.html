Hide empty header divider to make vertical padding symmetric

When BasicListMenu is created without a header (e.g., the Incognito
indicator context menu), menu_header_bottom_hairline starts as
View.INVISIBLE. Because View.INVISIBLE views still occupy vertical space
in a LinearLayout, the 1dp divider remained measured right above
menu_list. This caused the visible top gap above the hover highlight to
be 1dp taller (5dp) than the bottom gap (4dp).

Before: https://screenshot.googleplex.com/8s8aVUHb9hkLj6x

After: https://screenshot.googleplex.com/4ra8Wxokd3LDmvk

This change sets the hairline divider to View.GONE whenever the menu
header precondition is false (mHeaderModelList is empty).
