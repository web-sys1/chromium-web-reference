AbortSignal: explicit abort algorithm lifetime in modules/credentials

This is a follow-up to crrev.com/c/3931192 which added
AbortSignal::AlgorithmHandle for explicit abort algorithm lifetime
management.

There are several abort algorithms in credentials_container.cc, each
of which is tied to an operation. These operations typically have an
associated promise that is resolved or rejected depending on the state
of the signal and outcome of the operation. This change ties the
lifetime of the abort algorithm to that of the corresponding operation
by plumbing through a ScopedAbortState object.
