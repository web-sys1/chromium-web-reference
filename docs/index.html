Prevent double ZIP code classification

FormFieldParser::ParseSingleFieldForms() triggers local heuristics for
merchant promo codes, IBANs and standalone ZIP code fields. It is called
from FormStructure::ParseFieldTypesWithPatterns() in the
ShouldRunHeuristics() and the ShouldRunHeuristicsForSingleFieldForms()
branch. This CL fixes a bug in the former branch.

Inside the ShouldRunHeuristics() branch, all the various parsers are
called, including the address field parser. Since
ParseSingleFieldForms() is called as well, ZIP code fields are
classified twice, which leads to two ZIP code classifications in the
FieldCandidatesMap. Since the best heuristic type is chosen [1] by
adding up the scores for every type, ZIP codes are boosted by a factor
of 2. This makes ZIP codes the highest scoring type [2].

This CL fixes this issue by removing the ParseSingleFieldForms() call
in the ShouldRunHeuristics() branch and just triggering merchant
promo code and IBAN heuristics manually.
