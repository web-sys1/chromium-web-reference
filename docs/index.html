Make the PaintOp base classes uncopyable.

The PaintOp and PaintOpWithFlags base classes were copyable, although
that operation was not valid. The following code for instance was
compiling, but was leaving `base_op` in an invalid state, corrupting
memory along the way:

  SetMatrixOp matrix_op(SkM44::Scale(1, 2));
  PaintOp base_op = matrix_op;
  if (matrix_op == base_op) ...

That's because the PaintOp copy operator uses the `type` field to
implement it's own virtual-table-like dispatch, which means that
the SetMatrixOp's copy function as invoked and was writing the
matrix data outside of the `base_op` object's memory range.

This would have been caught with better test coverage in
paint_op_buffer_unittest.cc (`ElementsAre` copies it's input as
PaintOp objects, to match ASSERT_THAT's lhs), but the unit tests
were only using ops having no associated data.
