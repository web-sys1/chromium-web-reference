Fix request batching to span multiple preload chunks

The preloads in any given preload were already batched but the
background parsser splits chunks of tokens on any css or script tags
so the batching was a bit more limited than desired.

This change adds a scope lock for the batching across all of the
available chunks and better aligns with batching everything discovered
during the current task while still freeing up the batches when the
parser yields.

The batching itself is still behind an experiment flag to measure the impact.
