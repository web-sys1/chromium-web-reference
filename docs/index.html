IDB: use BigBuffer to pass values over mojo.

With the LevelDB backend, this was not necessary due to the large value
wrapping optimization/workaround/hack. Since this is not applied for the
SQLite store, it becomes necessary, as values can be arbitrarily large,
and won't necessarily fit inlined in mojo messages.

This change technically affects both LevelDB and SQLite backends,
although there is effectively no difference in the LevelDB case since
values should never be large enough to be stored in shared memory.
