extensions: Preinstall Docs Offline via new method on Win/Mac/Linux

This is a followup to https://crrev.com/c/7899711 which introduced a
hard-coded list of external extensions in order to support Android.
To align the install behavior on all platforms, switch to the new
approach on Win/Mac/Linux.

Add tests for preinstalled_apps::Provider::SetPrefs() to check that
the extension will be installed. Update the build file so that
preinstalled_apps_unittest.cc is also not compiled on Chrome OS, just
like preinstalled_apps.h/cc. This makes the test less ifdef-y.

Manually tested on Linux:
1. Run of Chrome with new user data dir, don't sign in (so extension
   won't sync). Extension is installed
2. Restart; extension is still there
3. Add new profile; extension gets installed
4. Uninstall extension in profile 1; extension gone from profile 1,
   still present in profile 2
5. Restart, extension still gone from profile 1
