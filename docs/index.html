[iOS] Limit length of filename for images passed to the Photos service

This CL limits the length of the suggested filename for images uploaded
using the Photos service s.t. if the suggested filename is longer than
100 characters, nil will be passed to the Photos service instead.

This fixes a bug where suggested names with length greater than 255
characters would lead to API errors.
