Fix invalid falsy AutoCapitalizeType returned from IME private API.

The API specs [1] declare that AutoCapitalizeType enum is always truthy.
The same goes for the public API [2], albeit the enum is slightly
different. For the latter, only add a verbose essay to mention a similar
issue exists, but keep the status quo due to it being a public API.

Indeed, the same issue had existed for the private API as well. It got
"fixed" with the introduction of explicit "off" and the invalid-value
code path deemed NOTREACHED [3], but came back later when NOTREACHED
was made reachable again [4].

Also, add verbose essays to clarify the difference between Blink's NONE
(which corresponds to "none", but means and is a synonym of "off") vs.
IME public/private API's NONE (which corresponds to falsy empty string
thus an invalid value as per API specs, and means "not specified").
