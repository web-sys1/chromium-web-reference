[contextual_tasks] Clear task ID on navigation

Currently when we navigate a tab that was showing an AIM page to
a regular non-AIM URL, we don't clear the
ContextualSearchWebContentsHelper and the tab association from
the ContextualTasksService. This results in the tab thinking that
it is still associated with the old task and doesn't reset the
underline. This CL fixes that.
