Linux GPU sandbox: fix filtering of fcntl()

The GPU process's BPF policy was accidentally allowing all calls to
fcntl(), when it was supposed to be allowing only
fnctl(fd, F_ADD_SEALS, F_SEAL_SEAL | F_SEAL_SHRINK | F_SEAL_GROW), in
addition to the baseline policy's allowances.

We can't use C++ branches on syscall argument values, only on syscall
numbers.

This may cause unexpected regressions if some GPU drivers are using
fcntl() calls that weren't supposed to be allowed.
