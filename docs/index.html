Move where ToolbarPhone updates its snapshot state.

When the hardware draw approach is used with RenderNodes, an Android
animation optimization avoids calling most protected/overridable draw
methods for parent views, and goes directly to child views. This
resulted in the old location never being triggered for about half of
the captures.

This CL moves it to somewhere that is guaranteed to be hit for both
software and hardware draw approaches.
