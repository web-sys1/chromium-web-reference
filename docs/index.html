Stop collecting device idle state from the CCaaS extension

This change updates our component extension's background service worker
to stop collecting and reporting the device idle state now that we are
graduating this piece outside the extension as is covered by
crrev.com/c/4257601.

Since the component extension does not collect or report any data
anymore, we should be able to remove the background service worker and
subsequently deprecate the component extension. I will follow through
on this over separate CLs since these resources are coupled with grit
today.
