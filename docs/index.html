Change EventPath caching of size 1 to not use hash sets

Looking at the UMA results, there is a regression in the ARMs. This is
likely due to the additional time searching in the cache hash maps to
find the correct node. For building an EventPath, even the smallest
operation can be costly as it is already very fast.

Since one of the biggest use cases for this feature is for a sequence of
mouse movements that targets the same node, only storing one node's
event path should already show improvements and cover many cases. Further,
that would avoid using the HeapHashMap.
