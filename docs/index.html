[UPM] Introduce HasAccountForRequest check into the password check bridge

The purpose is to move the signed in check from the SafetyCheckMediator to the ChromeNativePasswordCheckController. The reason is that this check is only valid for no-UPM case. This CL is the first CL, which introduces the HasAccountForRequest in the PasswordCheckBridge. In the follow up CLs this check will be called from the ChromeNativePasswordCheckController and removed from the SafetyCheckMediator.
