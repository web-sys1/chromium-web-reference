Extension Safety Hub Refactor

The Safety Hub currently has two separate handlers to direct traffic
between the extensions backend and the main Safety Hub backend.
As the Safety Hub has expanded this has resulted in the need for
duplicated code.

This CL removes the safety_check_extensions_handler class and folds its
functionality into the main Safety hub Handler class. With this CL all
Safety Hub front ends that require data about extensions will get their
data from one location.
