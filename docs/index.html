[network] Decouple P2P/WebRTC from network_cpp component

This CL creates a separate network_cpp_p2p component for P2P-specific
code that is only built when is_p2p_enabled=true.

Previously, P2P/WebRTC code (p2p_param_traits, p2p_socket_type) was
part of the core network_cpp component, which meant all consumers of
the network service pulled in WebRTC dependencies even when P2P
functionality was not needed.
