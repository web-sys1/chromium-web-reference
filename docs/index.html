Mark logical inset properties on boxes as layout dependent

After crrev.com/c/4583533, when getComputedStyle() gets an inset
property value from an out-of-flow box, it needs to check the box's
layout result. For sanity, it added a CHECK to assert that the box's
layout is already done.

This caused a crash because logical inset properties aren't currently
makred as layout-dependent, so we are not updating layout when getting
their values. This patch fixes it by marking them layout-dependent,
and implementing all the IsLayoutDependent() overrides in the same way
as the physical properties.
