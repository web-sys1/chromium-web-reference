[remoting] Don't send legacy video-layout messages for multi-stream.

This CL removes a call to VideoStream::SetObserver(this) when creating
the per-monitor video streams. This observer would listen for
size-change events on the video-stream. Instead, if an individual
monitor changes size, this should be handled by
OnDesktopDisplayChanged(), which will send out an extended layout
message.

This change should guarantee that no further legacy video-layout
messages are sent after multi-stream is enabled (because the old
video-stream is deleted and the new ones are created without setting
any observer).
