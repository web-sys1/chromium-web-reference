[Autofill] Observe the caret position and selection

This CL adds a new mojo event CaretMovedInFormField() that is fired
whenever the caret position and/or selection in a form control or
contenteditable changes. The code is behind a new Finch flag.

The event is throttled and the listener is only registered for the
time a textarea or contenteditable is focused. A future optimization
may be to only register it while an Autofill popup is visible.

For consistency with existing Autofill events, it takes a FormData
and a FormFieldData. All of events will be refactored to avoid
redundant form extractions and reduce the payload.

To make this work, this CL adds an interface for adding and removing
listeners to WebNode: WebNode::AddEventListener() adds a listener
and returns a RAII object that removes the listener.
