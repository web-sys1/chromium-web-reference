[Fuchsia][CFv2 Script] Fix astro devices time out after paving.

The pave command exits after a reboot, but the device still needs
time to start up before it can run tests.
Set `repository.server.listen` to "[::]:0" in our E2E script so that
the repository is started on an available port. Previously on our
hosts that host multiple devices, there would be multiple tests
trying to run a server at port 8083 simultaneously.
