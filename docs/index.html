[Clarity] Prevent anchored message drawer from growing bubble

A previous commit capped the text width of drawer items, but we want
to be more strict than that; when the drawer expands, the overall
width of the bubble should not change at all.  To achieve that, set
the preferred width of these text fields to 0, so that they'll grow
only to the space the layout manager allows for them, but not grow
the bubble beyond what it would be otherwise.  This leaves the top
row of the anchored message dictating bubble width.
