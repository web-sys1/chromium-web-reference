[FilesSemantic] Update common.css to semantic colors, remove unused CSS

Some of the remaining CSS has not been converted to semantic colors as
part of the dark mode project. This ensures common.css is semantic color
ompliant by performing the following:

 - Remove menuitem-button class definitions, these were removed from
   main.html [1] but their CSS was not cleaned up.
 - Move the menu selection into the overridden files-ng rule (will clean
   these files-ng mentions up when dealing with menu.css). This rules
   was affecting the color when a menu item was active but not hovered.
 - Update the slider track background color.
 - Removed styles that were used for the Drive welcome banners which
   have since been deprecated in favour of the new framework.
 - Remove all usages of .files-ng in common.css

[1] https://codereview.chromium.org/443333003
