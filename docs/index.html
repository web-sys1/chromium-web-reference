float: Fix some issues with tablet multitask menu not closing

It does not close when the associated window becomes invisible or
changes bounds. Clamshell multitask menu has this behavior, so move
that common behavior to `MultitaskMenuView`. It also doesn't close
when clicking a button while it is still sliding in. That is because
`AnimateFadeOut` early returns if there is an existing animation.
Modify the early return to only do something when the target opacity
is 0 (i.e. `AnimateFadeOut` has been called already)
