Add unit tests for ScopedPrinterInfo

Introduce tests to cover the expected behavior for the existing
ScopedPrinterInfo class.  Demonstrate that it causes unexpected behavior
if one of the intended keys contains a semicolon.

The semicolon behavior is of concern since some calls which are desired
to be used in new keys can contain a semicolon in the string provided by
some system calls.  Extra semicolons can result in too many keys in the
processing within ScopedPrinterInfo, which presents the danger of having
a buffer overflow on the static printer keys.
