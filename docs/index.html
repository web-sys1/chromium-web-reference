[Product Messaging] Enable multiple simultaneous messages

This allows for a lower-priority message to continue to exist while a
higher-priority message shows, and for certain priority levels to ignore
each other (superseded notifications still go out, but those are already
ignorable if the caller desires).

Also, a number of values, methods, and concepts have been renamed for
clarity.

This CL should not significantly affect any current behavior.
