FLEDGE: Fix timeout handling in JSON conversions in SellerWorklet

In one case we were not detecting the timeout at all, and in
another failing to handle it as a failure. Detect them and
fail cleanly.  This requires some adjustment to when real-time
metrics are filtered for latency to account these later conversion
steps, since they may take non-trivial amount of time, and may
therefore affect which bit ought to be set.
