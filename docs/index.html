Update cargo_crate build to correctly pick up dev-deps for unit tests

Before this change, dev-dependcies were not getting correctly detected
in the auto-generated build targets for a rust library when setting
build_native_rust_unit_tests to `true`. Moving the unit test target
above the main target resolves the issue of Error: `dev_deps` was
never used before it went out of scope.
