[Start] Warmup spare renderer when showing the Start surface.

In this CL, we try to reduce PageLoad metrics regression by warming up a
spare render when the Start surface is showing. When a new Tab is opened
from the Start surface, it will be able to reuse the renderer. The
changes are behind Finch parameter:
1. WARM_UP_RENDERER: false.
2. SPARE_RENDERER_DELAY_MS: 1000 ms.

More details see: go/m91_start_pageload_regression.
