*Scan: Temporarily disable *Scan on try-bots

*Scan + safepoints cause builds with enabled coverage to time out due to
huge contention around counters.

In code instrumented with coverage, every source line (roughly) has a
corresponding global counter that gets incremented when the line is
executed. In a single test there can be >10 threads participating in
scanning. The scanning code is very dense, so there is a huge contention
around counters. This leads to cache-line ping-pongs -> observable
slow-down -> timeouts. There is a related discussion:
https://lists.llvm.org/pipermail/llvm-dev/2014-April/072155.html.

My hypothesis why we start seeing timeouts now. A fairly trivial or
stylish CL changes a PA function used in the scanning routine: a new
line is added, a new forwarding single-line function is introduced, a
new DCHECK, etc... All of this can increase/reduce the number of
counters, which has effect on contention.

This CL temporarily disables *Scan on try-bots.
