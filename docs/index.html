Extensions: Cache feature availability for specified keys on Manifest.

Cache the subset of available and specified manifest keys on the
Manifest object. This makes availability lookups quicker (just a
DictionaryValue key/path lookup) instead of the current cost of
iterating over all parent paths and checking their availability. Expose
the availabile DictionaryValue on the Manifest object as well and use it
for auto-generated manifest parsing clients.

The caveat is that this requires a one-time cost of iterating over all
specified keys in the manifest and checking their availabililty ((this
should be offset by the faster availabililty lookups) and the memory
usage of the Manifest object also increases a bit.
