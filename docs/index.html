Reland "[headless] Fix focus change propagation when there is no NSWindow"

This is a reland of commit 32a2893ed74db2e11b9f860ea15c56f45cf8c877

This CL invokes the OnFirstResponderChanged callback asynchronously.
Focus/Activation notification notifications are nested so synchronous
invocation the original CL used did not always result in the correct
state.

Original change's description:
> [headless] Fix focus change propagation when there is no NSWindow
>
> Headless Shell on Macs runs with only NSView's and no associated
> NSWindow's, as a result, on focus request the first responder change
> does not get reflected back to the widget host causing it to misreport
> the first responder state.
>
> This CL adds a synchronous call to OnFirstResponderChanged() from the
> RenderWidgetHostViewMac::Focus() when there is no NSWindow making sure
> that the first responder change propagates.
>
