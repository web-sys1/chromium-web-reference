Deduplicate prefetch activation beacons during prerender activation.

When both prefetch and prerender activation beacons are enabled, a
prefetched page upgraded to a prerender page would send duplicate
beacons upon activation. This happens because the prerender
activation is processed as a cross-document navigation that
synchronously calls OnResponseStarted with the cached prefetch
headers, triggering a prefetch activation beacon registration, while
PrerenderHost::Activate separately fires a prerender activation
beacon.

This CL resolves the duplication by preventing prefetch activation
beacon registration during prerender activation navigations when the
prerender activation beacon feature is active.
