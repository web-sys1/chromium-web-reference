Don't cache observer iterator in WebContentsObserver.

If a class that listens to a WebContentsObserver triggers another
WebContentsObserver event (eg. by changing the focus of the WebContents)
then any observers after that observer in the observerlist won't
receive the first event at all.

Fix this by not caching the iterator for the observers so it doesn't
get reset when a new event is triggered.
