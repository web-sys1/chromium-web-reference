[Live Caption] Move IsSigninProfile check out of CaptionController.

Move the IsSigninProfile check from CaptionController::Init to be higher
up in the call hierarchy.

Original discussion thread:
https://chromium-review.googlesource.com/c/chromium/src/+/2810057/4/chrome/browser/accessibility/caption_controller.cc#81

Rationale:
Live Caption does not run on the signin profile, and we do not want
CaptionController::Init to be called for the signin profile at all, so
it makes sense to move the check upstream.

This change simultaneously solves the problem where unintentional UMA
histogram data contributions are being made by the signin profile for
the histogram Accessibility.LiveCaption.FeatureEnabled (at the beginning
of CaptionController::Init).
