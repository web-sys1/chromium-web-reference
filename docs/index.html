Revert "Use more conservative direct raster image scale."

This reverts commit edb5c68141f43a8726bb2e375ce87fd96ae4af9b.

Reason for revert: Suspected source of failing blink_web_tests (virtual/gpu-rasterization/images/color-profile-border-fade.html) on the Win7 Test builder.

Original change's description:
> Use more conservative direct raster image scale.
>
> Previously PictureLayerImpl was downscaling directly composited images
> only when the scale on the screen is below 1/4. This resulted in
> excessive memory usage, particularly for large images that are rendered
> at scale below 1/2. Updated PictureLayerImpl to downscale the image when
> the initial scale is below 1/2. The logic in ShouldAdjustRasterScale()
> is kept as it was before to reduce raster frequency: when scale is
> reduced the layer is rasterized only when then new value is below
> 1/4 of the previous scale.
>
> Also removed Fuchsia-specific workaround (see crbug.com/1161327).
>
