protobuf: Add proto_library for duration.proto

This change introduces a proper proto_library target for duration.proto
in protobuf, and updates sharing_log_proto to depend on it instead of
using a manual input hack. The hack worked only because the checked-in
generated version of duration (duration.pb.h) was in the protobuf
include directory by accident.

Additionally, it adds `proto_in_dir = "src"` to Well Known Type targets
to ensure the paths are correctly resolved relative to the source root,
as protoc detects WKTs by their paths.

This is a reland of crrev.com/c/7793236, see crrev.com/c/7808250. One of
the two root causes of the revert was fixed in crrev.com/c/7807025, the
second hasn't been pinpointed yet. Relanding these patches one by one
and waiting 24h in between to see which patch causes the issue.
