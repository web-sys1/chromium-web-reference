Create Local fork of GLES2 renderer for Chrome

The Chrome WebXR implementation uses GL_TEXTURE_EXTERNAL_OES rather than
GL_TEXTURE_2D for its textures due to using Shared Buffers to transfer
the textures across processes when it is available. Unfortunately, this
requires a shader change which is not currently supported in Cardboard.

This change forks the Cardboard GLES2 renderer to have a shader that
can handle the EXTERNAL_OES textures.
