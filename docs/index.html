[iOS][Omnibox] Don't dismiss keyboard when Refine button is used.

Prevents keyboard from being dismissed as if the user scrolled the
popup when the Refine (aka Append) button is used.

There are two fixes in this CL:
1. On iOS 16+, use the new shouldDismissKeyboardOnScroll modifier. This
avoids 5 levels of existing plumbing to dismiss keyboard.
2. On iOS 15 and below, there's no great fix because of SwiftUI doesn't
have a built-in way to detect scrolling, and various combinations of
gesture recongizers always conflict with either swipe-to-delete or tap
recognizers. One solid fix would be to rebuild system swipe-to-delete
from scratch, but I doubt SwiftUI is robust enough to do that either.
Since the only problem with the current long press + tap gesture
recognizer combo is that it sometimes detects a scroll on append tap,
I add a workaround that blocks the onScroll event for a small time when
the tap happens.
