Original change's description:
> Calendar model layer improvements
>
> The main improvement is to only redistribute the events whenever there's
> a timezone change. See more details in this doc:
> https://docs.google.com/document/d/1p_djqUKV1r3kY5HvtcKaAXwBc0Uw_ey2Z6Mhn9g8hC8/edit
>
> Also:
> - Removed several unused the methods and unused instances.
> - fixed the bug in the `GetFetchStartEndTimes` method. This fixed the
> issue that in some timezones the last day of a month couldn't get events
> back since it is not in the fetching time range.
> - changed `FetchEvents(const std::set<base::Time>& months)` to
> `FetchEvents(base::Time start_of_month)`, since each fetch is binded
> with its corresponding month view.
