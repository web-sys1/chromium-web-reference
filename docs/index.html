[APS] Update to new endpoint and new proto files

Preloads is switching to a renamed endpoint, which changes the URL and
the proto message names. This new endpoint uses shared protos for device
and user context parameters, which will live in the shared Almanac API
folder.

In the long term, we may refactor DeviceInfoManager to use these protos
directly, but for now, this just references the new protos with no major
structural changes.
