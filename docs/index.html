Guard SnapControllerLacros from dereference null DesktopWindowTreeHostLacros

... during window tear down.

On the window tear down routine, Lacros calls out to
SnapControllerLacros::CommitSnap(None) as an attempt to
remove any left-over snapping preview window.

Problem is that, by the time this method is called, the
respective DesktopWindowTreeHostLacros instance can already be
deleted/nulled out, and SnapControllerLacros dereferences
unconditionally results in a crash.
This CL protects against such crashes.

BUG=1384340
