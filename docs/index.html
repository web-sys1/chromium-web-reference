ZeroHz screenshare: ensure frame arrival after drop.

Under zero hertz tab capture, a discarded frame ending a
sequence of frames which happened to contain important information
can be seen as a capture freeze. Avoid this by ensuring the
drop information gets known to WebRTC, so that it can adapt
by issuing refresh frame requests.

This change depends on a new WebRTC API function added in https://webrtc-review.googlesource.com/c/src/+/265871.
