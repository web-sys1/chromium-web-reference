background_fetch: Migrate DOMException to `RejectWith-`

When throwing an exception `e`, it's important to make a stack trace
available as `e.stack` as long as possible.  However, DOMException's
constructor doesn't create this `stack` property, so the direct use of
DOMException's constructor is highly discouraged. So this patch migrates
`DOMException` constructor to `ScriptPromiseResolver::RejectWith-`.
