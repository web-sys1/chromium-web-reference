Split Observer::OnFormSubmitted into a before and after event

This CL attempts to fix a race condition in a bunch of
AutocompleteTests. AutofillManager::OnFormSubmitted submitted an
`Observer::OnFormSubmitted` notification before actually handling the
form submission. This means that the TestAutofillManagerWaiter was
informed about the submission before the AutocompleteHistoryManager.

I am not sure if this fixes the problem because
AutocompleteHistoryManager::OnWillSubmitFormWithFields is still informed
on a synchronous code path, but it makes the behavior more explicit.
