Original change's description:
> [RSearches] Cancel Bar height animation on close
>
> When the panel is closed there's a bug causing it to show again
> and be stuck open when an animation is running. This is because
> the animation sets the panel height and that implicitly sets whether
> it's showing or not.
>
> The fix is to cancel animation when the Bar is destroyed.
>
> An additional safeguard is added to make sure we never get stuck in the
> IDLE state with the panel showing.
>
> Also updates the test for the Internal State Controller to Robolectric
> and adds test cases for the safeguard.
>
