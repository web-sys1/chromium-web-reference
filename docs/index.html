Lacros: Fix crash while setting Sync account

Fix CHECK crash in Lacros while setting the Primary Sync account for
Lacros's Main Profile.

This CHECK was triggered because non-branded builds did not allow the
Sync account to be set on non-Ash platforms.

Fix this by allowing non-branded Lacros to set the Primary Sync account
- just like non-branded Ash.
This is done by force setting `prefs::kSigninAllowed` to `true` on
Lacros since it doesn't offer account inconsistency.

Note: The user won't be able to use Sync or other Google services
without a valid set of API keys baked into the build, despite having
set a Primary Sync account. This behaviour is the same as non-branded
Ash.
