Mark control character text nodes for paint invalidation.

A text nodes that gets a LayoutText object that only generates control
characters may still need paint invalidation. FragmentData entries need
to be updated correctly during pre-paint. A re-layout may cause the
start fragmentainer ID to change. Text nodes that actually contain text
will do ClearNeedsLayoutWithFullPaintInvalidation() in
NGInlineLayoutAlgorithm::CreateLine(), but nodes that only generate
newlines would have to make do with a simple conditional
ClearNeedsLayout(), which doesn't mark for paint invalidation.
