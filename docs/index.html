Reland "[a11y] Support atomic live regions on Clank"

This is a reland of commit 5300ea5528a8e0bb8e705eae3d539fc30488947e

Once submitted, a WebContentAccessibilityEventsTest started failing: reportValidityInvalidField(). After further investigation, this is because the test emits a validation message, which have a default role of Alert. Since Alerts are by default atomic live regions, the field trial test config was causing the test to fail. To rectify this, I have set the live region improvement flag explicitly and added a test case to ensure both enabled and disabled states have adequate coverage.

Original change's description:
> [a11y] Support atomic live regions on Clank
>
> At present, Clank does not support atomic live regions. Instead, when a
> change is made to an atomic live region, only the changes nodes are
> announced rather than the entire atomic subtree.
>
> Behind a flag disabled by default, this CL changes this behavior to
> support atomic live regions and alerts (which are atomic live regions by
> default).
>
