Skip splash screen timer when retrying Kiosk launch due to lost network

When a Chrome app Kiosk attempts to launch, it will wait for 10s on the
splash screen. If network is lost during this period, it will wait
another 10s for the network to come back. If network is back during the
second 10s period, we will restart Kiosk launch flow but not resetting
the splash screen timer, as a result it will be stuck at splash screen
forever.

By making Kiosk launcher skip splash screen timer whenever a retry
happens it will launch the Chrome app immediately upon retry, fixing
the issue mentioned above.
