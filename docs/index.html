[devtools] Report issues for invalid SameParty attribute usage

The SameParty attribute can only be used when SameSite=Strict is not
present and the cookie is also marked as Secure. This CL adds an
issue that is filed when SameParty is held wrong.

No CanonicalCookie can be created from a cookie line with wrong
SameParty attribute usage. This means DevTools instrumentation is
also not able to create an AffectedCookie since it relies on
a successful parse of the cookie. To solve this, we add the
raw cookie line to the issue and mark `AffectedCookie` as
optional.
