media: Support Dolby Vision query for EME

This CL adds support for Dolby Vision codec/profile query for
protected content.

Manually tested by following the steps below:
0. Set `enable_platform_dolby_vision = true` in gn args.
1. Hack MediaFoundationService to fake DV support.
2. Hack KeySystemConfigSelector::IsSupportedContentType() to ignore
   the `is_supported_media_type_cb_.Run()` which is currently not
   supported.
3. Build and start chrome, and in JS console, run
```
navigator.requestMediaKeySystemAccess("com.widevine.alpha", [{
  initDataTypes: ["cenc"],
  distinctiveIdentifier: "required",
  persistentState: "required",
  sessionTypes: ["temporary"],
  videoCapabilities: [{
    robustness: "HW_SECURE_ALL",
    contentType: "video/mp4; codecs=dvhe.05.07",
  }],
}]);
```
4. Verify that profile 4/5/8 are supported, and other profiles
   (e.g. 9) are not supported.

A chrome/browser/media/encrypted_media_supported_types_browsertest.cc
should be added but not added in this CL due to the need for a hack
in step 2 above. We'll add the test when we fix that hack.
