Make PaintPropertyNode classes polymorphic

Benefits are:
1. The sparse vector in ObjectPaintProperties becomes more memory
   efficient by avoiding the extra 4*n or 8*n bytes for the identifier
   field of std::variant.
2. It simplifies code to handle PaintPropertyNodes (e.g. in
   PaintPropertyTreePrinter).
3. It makes it easier to convert to Oilpan.

Drawbacks are:
1. vtable pointer in each node. This will be compensated by the memory
   reduction in the sparse vector, and in some cases the pointer may
   fill the gap of alignment without actually increasing memory usage.
2. Cost of calling the virtual functions (AddChanged() and the
   destructor).

Pinpoint job [1] shows no regression of motionmark with this CL. (The
small progressions of leaves and suites are from crrev.com/c/5318197.)

[1] https://pinpoint-dot-chromeperf.appspot.com/job/140996ec9e0000
