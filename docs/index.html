viz: Validate node IDs in TransformTreeUpdate

On the server side (LayerContextImpl), we receive
untrusted mojo input. If any node ID in
nodes_affected_by_outer_viewport_bounds_delta or
nodes_affected_by_safe_area_bottom is out of
bounds, it eventually causes a CHECK failure and
crashes the viz process when attempting to access the
transform tree nodes.

This CL adds server-side validation to reject any
invalid transform node IDs in TransformTreeUpdate and
returns base::unexpected.
