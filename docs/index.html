Roll Sysroots

This is needed to fix the GTK4 build.

In instrumented builds, some code that's part of llvm gets statically
linked which depends on libc.  This means our current hack of setting
symbol versions in header files won't work any more, so this patch
switches to a different hack of rewriting the default symbol versions
in the libc binary.
