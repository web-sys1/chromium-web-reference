Only overlay video without SurfaceControl

With low latency canvas, not all overlay candidates are video anymore.
But without SurfaceControl (ie using OverlayProcessorAndroid),
overlay should only be attempted on SurfaceView-like video, where viz
just does hole punching and does not directly control the video
buffers. In particular it should not attempt to overlay low latency
canvas which will fail.

Modify the bool condition a bit to account specifically for video and
SurfaceView, which no longer assumes all candidates are video.
