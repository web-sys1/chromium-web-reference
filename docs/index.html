Reland "Do not use event listeners for anchor element interaction preloading."

This is a reland of commit df0eb6bc00e90d55beeb400ce9a414c4a909accf

The only change since original CL is that the browsertests simulate a
mousedown event rather than both a mousedown and mouseup (full click).
Generating a full click was triggering navigation which was making the
tests flaky due to there being two UKM sources.

Original change's description:
> Do not use event listeners for anchor element interaction preloading.
>
> Using event listeners causes two issues:
>
> 1) It causes FirstInputDelay calculations to be wrong because all link
>    clicks are not included in FirstInputDelay calculations, as if the
>    page inserted the event listener.
> 2) Many tests assume that there are no event listeners installed on
>    an empty document.
>
