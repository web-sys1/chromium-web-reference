Sync CTBMTP: Only log "metadata was cleared" if it was actually cleared

ClientTagBasedModelTypeProcessor::CheckForInvalidPersistedMetadata()
verifies the passed-in metadata is valid and consistent, and if not,
clears it and returns false.
However, there was one case where it returned false even if nothing was
actually cleared, namely when `pending_clear_metadata_` was true. In
practice, that happens on every browser startup if sync is disabled,
causing some log spam.

This CL fixes this by only returning "false" if data was actually
cleared.

While we're here, this also replaces some DCHECKs with CHECKs, per
the latest guidance:
https://chromium.googlesource.com/chromium/src/+/main/styleguide/c++/c++.md#check_dcheck_notreached_noreturn_and-notreached
