Resolve ex and ch based on initial font for media queries

We used to have hard-coded values for ex and ch units in media query
lengths. Base these values on font metrics from the initial font
instead.

Did quite a bit of refactoring and simplification to avoid reading font
metrics for ex/ch when they were not queried in the MediaValuesDynamic
case.
