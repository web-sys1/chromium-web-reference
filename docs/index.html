[block-in-inline] Make NextWordPosition() to work with block in inline

This patch changes |CreateInlineContentsFromBlockFlow()| to find
first and last layout objects in block flow by iterating from first
child to last child or block-in-inline instead of scanning from
first child and scanning from last child because block-in-inline is
boundary of inline contents.

This patch makes following tests to pass:
  * editing/selection/extend/extend_selection_07_ltr_forward_word.html
  * editing/selection/extend/extend_selection_07_ltr_right_word.html
