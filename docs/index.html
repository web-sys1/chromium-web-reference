ozone/wayland: do not commit buffers until the surface is configured

That's already been done for the dmabuf path, but WL EGL path
was missed. Thus, add some hooks to avoid submitting buffers until
the surface is configured. Otherwise, wayland protocol error
"surface has never been configured" might occur.

Listening for surface configuration through WaylandWindowObserver
is not suitable for GlSurfaceWayland as it CAN be created
after xdg_surface is ack configured and observers are notified.
Thus, it's also possible to check if the surface has been configured
through WaylandWindow now.
