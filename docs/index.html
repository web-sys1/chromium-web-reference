Fix BrowserMinidumpTest for symbol_level=0

Originally symbol_level=0 meant that we had no symbols, which made
minidump tests irrelevant. That was changed a couple of years ago, but
some of the assumptions remain. This means that these tests would fail
if symbol_level=0:

    vpython tools/perf/run_tests BrowserMinidumpTest --passthrough

The fix is simply to adjust the dependencies so that the required tools
get built for symbol_level=0.

This unlocks the possibility of using symbol_level=0 on the CQ, which
can improve bot performance.
