[Cronet] Deprecate Jelly Bean support (API level 16-18)

These changes are in a single CL to remove the possibility of generating
temporary Cronet releases with conflicting build instruction and source
code.

Now that Build.VERSION_CODES.KITKAT is the minimum API level supported,
Build.VERSION.SDK_INT >= Build.VERSION_CODES.KITKAT checks can be safely
removed as they are now redundant.

Also, update the various build files and tools to specify the new minimum
sdk.
