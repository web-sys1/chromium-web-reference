Reason for revert: Broke downstream compile, see crbug.com/1305433

Original change's description:
> Have the message primary action callback decide if it should be dismissed.
>
> Before this CL, whenever the primary action button on the message UI is
> clicked, the message UI will be dismissed immediately after the primary
> action callback is run.
>
> This CL changes the primary action callbacks to return a boolean
> indicating if the message UI should be dismissed immediately or not,
> where returning true means that the message UI will be dismissed
> immediately (which is the current behavior), and returning false means
> that the message UI will remain visible.
>
> This behavior of remaining visible after the primary button is clicked
> is needed for the Translate message UI. See Bug for more details.
>
