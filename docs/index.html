Rename method in TableViewFaviconDataSource to mention *page* URL

No behavior is changed.
An important distinction for favicons is page URL (e.g. google.com) vs
icon URL (e.g. google.com/favicon.ico). Having a generic protocol name
only opens up the possibility that caller and callee use different
concepts, causing the lookup to fail. Rename to mention page URL, which
seems to be the rule.
