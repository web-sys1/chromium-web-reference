chrome/linux: support CHROME_EXTRA_FLAGS* env vars

...to set additional command line switches and arguments to chrome

As of now, each distribution of Chrome/Chromium implements its own
mechanism to support "configurable" user's command line switches, such
as custom wrapper scripts [1], custom launchers to support a similar
mechanism for flatpak'ed Chrome/Chromium [2], amongst others.

This aims to provide a standard system-wide mechanism to set these
additional arguments to chrome via a set of environment variables which,
in *nix world, is a well-known and widely used mechanism for use-cases
like this.

- CHROME_EXTRA_FLAGS is appended regardless the release channel;
- While CHROME_EXTRA_FLAGS_{CHANNEL} is used for chrome binary coming
  from a specific channel. i.e: STABLE, BETA and DEV.
- The value is assumed to be a plain string containing one or more
  arguments/flags separated by whitespaces. Whereas, arguments
  containing whitespaces in their value can be quoted using ' and ".

[1] https://aur.archlinux.org/cgit/aur.git/tree/google-chrome-stable.sh?h=google-chrome#n7
[2] https://github.com/refi64/cobalt#flags
