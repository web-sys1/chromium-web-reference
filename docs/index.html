Delete ArcDemoModePreferenceHandler class

This code's purpose was to restart mini-ARCVM when Demo Mode setup
finished, so that ARC would pick up the newly-downloaded Android Apps
image for Demo Mode, thus allowing the apps to install on the first
session after setup.

Given that mini-ARCVM is no longer started during OOBE (as per
crrev/c/3200677), this behavior is no longer necessary. The demo apps
image will be present by the time the first Demo session logs in, when
when ARCVM first boots up.

session after Demo Setup flow
