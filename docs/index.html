Make ScopedMuteHookedSamplesForTesting safe in tests that don't start the
sampler

It should be safe to create and then delete a ScopedMuteHookedSamplesForTesting
object in tests that don't actually initialize the
PoissonAllocationSampler. (For example if
ScopedMuteHookedSamplesForTesting is created in the constructor, but the
test body calls GTEST_SKIP.)

However it updates the allocator hooks, which isn't safe unless
g_sampled_address_set is initialized from the PoissonAllocationSampler
constructor. To fix this always call PoissonAllocationSampler::Get()
when creating a ScopedMuteHookedSamplesForTesting. This will only
affect test code.
