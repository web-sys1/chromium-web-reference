FSA: Remove disabling of BFCache on use of any local file

BFCache is currently disabled when using the File System Access API to
interact with _any_ local files in any way. This was added back in
https://crrev.com/c/1837813 due to its use of
RenderFrameHost::IsCurrent, which at the time returned true
for iframes

However, the behavior of RenderFrameHost::IsCurrent was changed in
https://crrev.com/c/2060374 such that it's no longer necessary to
disable BFCache in this specific case.
See https://crbug.com/1259861 for more details.

Note that BFCache is still disabled when permission to a file is
requested from the File System Access API.
This behavior was added in https://crrev.com/c/3217485 and requires
further work to address
