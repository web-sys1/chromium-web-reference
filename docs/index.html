Some tweaks for microphone mute notification

*  Change the notification content depending on whether audio input
   is mute due to a hardware switch
*  If the audio input is *not* muted by a hardware switch, add a button
   to unmute input
*  Make the notification pinned so it doesn't get dismissed
*  If microphone gets muted while an app is already using the
   microphone, lower the notification priority so it doesn't get shown
   (the user should already see a toast informing them that the
   microphone is muted)
*  Do not show notification when the number of active audio input
   clients decreases (but do update the notification content, as the
   active app name may change)
*  Update strings to the newest version provided by UX writers.
