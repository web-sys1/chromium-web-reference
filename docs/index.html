[gpu] Plumb context type for non WebGL contexts

Plumb the viz::command_buffer_metrics::ContextType correctly for non
WebGL blink contexts such as those used for blink worker threads via
SharedGpuContext, webcodecs readback and video track recorder. Add new
enum variants for these since they don't map to the existing variants.

Also, rename the existing enum variants to be more consistent.

OBSELETE_HISTOGRAM[GPU.ContextLost.RenderWorker]=Replaced by GPU.ContextLost.RendererRasterWorker
