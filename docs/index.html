[CrOS WebAPKs] Don't generate WebAPKs for apps with no manifest URL

Manifest URL storage has only been added in the last few milestones, so
apps which haven't been used recently may not have a manifest URL saved.
However, the WebAPK Minter requires a manifest URL, and sending a
request without it fails.

This CL detects this case in WebAPKInstallTask and exits early,
preventing local work and a network request. Ideally we'd cancel the
process even earlier, in WebApkManager, but that class does not have
access to WebAppRegistrar, and the manifest URL is not available through
App Service.
