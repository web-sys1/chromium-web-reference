[Uno-d] Add strings for extensions requesting Chrome Sign in

This CL adds strings that will be used in a dialog that is shown in the
following conditions:
- The extension requests an auth token through Chrome.Identity API
- The user is signed in to the Web only. In Dice profiles, this is only
  possible if explicit browser sign in feature is enabled.

This dialog will inform the user that the extension wants them to sign
in to Chrome to be able to use their account.
