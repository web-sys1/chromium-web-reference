[Widget] Fix very verbose and redundant a11y descriptions.

This change replaces the 'Quick action search widget icon search or
type the url search or type the url. Double tap to activate.'
announcement with 'Search or type the url. Double tap to activate'.

The way the original mechanism worked was:
- Announce the IconView (irrelevant)
- Announce the TextView content description (relevant)
- Announce the TextView hint (irrelevant)

The change makes IconView and TextView both irrelevant for
accessibility, and attaches relevant content description to the
parent view.
