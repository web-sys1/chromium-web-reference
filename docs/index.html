Throw TypeError for non-syntax errors in <script type="webbundle">

Before this, all errors occurred in parsing a <script type="webbundle">
rule were reported as SyntaxError. After this, errors except for JSON
parse error will be reported as TypeError.

This aligns with the spec:
https://wicg.github.io/webpackage/subresource-loading.html#parsing
