[media] Update format discrepancy in DCompTextureWrapper

The VideoFrame is created with PIXEL_FORMAT_BGRA whereas the
SharedImage backing it is created with BGRA8888 which corresponds to
PIXEL_FORMAT_ARGB as pixel formats have opposite byte order.

This change updates VideoFrame to be created with PIXEL_FORMAT_ARGB
so that the formats match. This is part of larger refactor to
eliminate VideoPixelFormat param from WrapSharedImage as it should
match format used to create underlying SharedImage.
