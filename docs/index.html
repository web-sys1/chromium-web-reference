Discard logs after failed upload if metrics server URLs not specified

This is in preparation of moving metrics server URLs to src-internal.

Before this CL, if metrics server URLs are empty, Chrome will keep
trying to re-send the same log (but will never succeed, obviously).
This causes an accumulation of logs on disk (until a certain point,
where they are trimmed).

This CL changes the behaviour so that after the first attempt to send
the log, if there are no metrics server URLs, discard the log.

We still "attempt" to send the log once (even though we know it will
fail) so as to keep the uploading codepath consistent, regardless of
whether metrics server URLs exist or not. Otherwise, test coverage
becomes trickier.

This doesn't affect Android/WebView since logs are reported through
a different mechanism.

This CL also adds a "reason" string for when we discard logs (used
in chrome://metrics-internals). I can pull this out in another CL
but I thought it was small enough to include in this same CL.
