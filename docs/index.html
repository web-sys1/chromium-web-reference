Trigger FCP when opacity is changed

We're seeing cases where FCP is not being properly reported when it's
caused by an opacity change whose paint is optimized away using
`will-change: opacity`. This CL fixes that by tying those FCP reports
to our existing mechanism to report ignored LCP entries in those cases.
