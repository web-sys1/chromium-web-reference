[VCManager] Modify triggering logic for extensions

Background:
The current triggering logic starts tracking an app as a VC app
after its first capture of a media device. It removes this app
from its internal list once the app's webcontents get destroyed.

For extensions this becomes an issue as their webcontents never
gets destroyed.

So once an extension captures a media device and triggers the VC
UI tray, the tray never goes away. The current permissions info
logic also does not show the correct permissions for extensions.

This change:
For extensions only, we consider the VideoConferenceWebApp as
being tracked by the client only if the extension is presently
capturing a media device.
