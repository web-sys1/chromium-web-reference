Let getElementById receive MaybeAtomicStrings for fast response.

Implements a new MaybeAtomicString class derived from AtomicString with
the same functionality as its parent, but which holds a null string if
the queried string wasn't already on the AtomicStringTable. With this,
getElementById (and potentially other v8 callbacks) can give a fast
response if the queried name is not in the string table without creating
a new AtomicString, thus avoiding an instance of refcount trashing.
