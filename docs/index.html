Add StorageKey parameter to CreateFileSystemURLLoaderFactory

To ensure that the browser can access the web-exposed (third-party)
StorageKey from the renderer, StorageKey was added to the FactoryParams
and populated by the RenderFrameHostImpl. This StorageKey will be
used by the browser for storage partitioning. To ensure that
WorkerScriptFetcher passed its browsertests, a StorageKey value had
to be piped in from its callers as well. Finally, IPC validation was
added to FileSystemURL, modifying the class to set invalid any URLs
whose parsed origin does not match its StorageKey origin.

It is important to note that this is the first a series of CLs, and
later CLs will address whether navigations in filesystem: URLs are
allowed and will correct the download files to use 3p StorageKeys.
