[DC] Serialize the origin on the C++ side for digital credential flows

origin.Serialize() in C++ isn't equivalent to origin.ToString()

The former omit the port if it is the default[1], while the latter
doesn't.

This CL moves the serialization to the C++ side since moving it to the
Java side caused the port to be added which caused some flow.
