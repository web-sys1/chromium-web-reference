Remove "UIProxyConfigService for..." from network_event_log

The "UIProxyConfigService for user: using proxy of network: ..."
message was added to devs and users when a network is using a
shared proxy.

This turns out to be a very common occurrence (about 50% of all
feedback reports).

The function that logs the message gets called from several
locations causing many lines (sometimes hundreds) of log spam.

Since we are aware of the situation and have no plans to address it,
we should demote the log to use VLOG instead.
