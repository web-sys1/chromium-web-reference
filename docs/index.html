[SPCV3] Support isCrossOrigin for Payment Credential assertion

Before the change, in the SPC assertion flow, ClientDataJson's
"isCrossOrigin" was always false. After the change, the field equals to
whether the assertion's caller is in a frame whose origin is in a
cross-origin relationship with the main frame.

Changes:
* The Android and Desktop's internalAuthenticators became initiated
  with the caller frame instead of the main frame.
* SecurePaymentConfirmationAppFactory::OnAppIcon() no longer requires
  the caller frame to be a mainframe to create the SPC payment app.
* PaymentAppServiceBridge::CreateInternalAuthenticator() will use the
  caller's frame instead of its mainframe to creates
  InternalAuthenticatorAndroid.
