PrintPreviewUi::TestDelegate methods get default implementation

PrintPreviewUi::TestDelegate has used pure virtual methods for the
delegate calls used to notify of certain events.  Switch these to
have a default implementation.  Changing this pattern aligns this
TestDelegate with how printing uses Observers (which this is very
similar to), and allows for it to be extended for other test scenarios
without having to also modify all derived classes.
