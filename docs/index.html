fileSystemProvider: refactor operation dispatching

Break out Event dispatch functionality out of the Operation class into a
separate EventDispatcher class (replaces Operation's callbacks),
encapsulating EventRouter and Lacros-related details. The dispatcher is
owned by ProvidedFileSystem and keeps a reference to the RequestManager
associated with this ProvidedFileSystem, simplifying request cleanup if
Lacros delivery fails.

Operation unit tests no longer require a custom callback for every test:
the tests calls now go through a stub dispatcher that's already
implemented in test utils.
