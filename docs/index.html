MPArch: remove container_host_->GetFrameTreeNodeId() from SWMainResourceLoader

This CL removes `container_host_->GetFrameTreeNodeId()` from
ServiceWorkerMainResourceLoader::StartRequest() because the frame tree
node id may be stale after the navigation commit.

Instead, plumb `frame_tree_node_id` from SWMainResourceLoaderInterceptor
via SWControlleeRequestHandler.

`GetFrameTreeNodeId()` in SWContainerHost is now only called from
StoragePartitionImpl so this CL also adds
`base::PassKey<StoragePartitionImpl>` to make sure it.
