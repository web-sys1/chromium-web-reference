Removed unused PriceTrackingInputDelegate

Currently price tracking backend is queried upfront before invoking
segmentation and the information is passed through InputContext.
The original design was to get this information behind an InputDelegate
instead through segmentation. This approach led to plumbing issues as
the delegate doesn't have a view of the whole world and difficult to
pass more and more things to it through a conduit such as UserData
attached through WebContents. Hence the decision was to get this
information in Java upfront and deprecate the InputDelegate. This CL removes this stale code path.
