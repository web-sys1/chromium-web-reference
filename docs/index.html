Make ad-hoc “very high” priority properties into actual numbers.

We have a number of priorities that are manually checked before
the “regular” high-priority properties are. There's no need to have
separate code for this with manual tests, so instead, move (back!)
to a system of 0–1–2 instead of just low–high.

We also have “cascade-affecting” that are effectively priority 3,
but we need separate tests between them and the other high-priority
tests, so they need to keep being hard-coded for now.
