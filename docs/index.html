blink: Unify scrollbar thumb length calculation

Let scrollbar layer impl objects calculate the thumb length. By doing
this, it can appropriately calculate the thumb length for the current
zoom level. This is possible because
LayerTreeImpl::UpdateScrollbarGeometries() uses
current_page_scale_factor() to calculate the clip layer length for
scrollbar layers.

To do this, the following changes are made:
- ScrollbarLayerImplBase::ThumbLength() calculates the thumb length for all scrollbar layer types.
- NinePatchThumbScrollbarLayer and PaintedScrollbarLayer keep minimum_thumb_length_ and thumb_thickness_ instead of thumb_size_.
- cc:Scrollbar and its implementations provide the minimum thumb length for scrollbar layers.

Also update test expectations because now the tests run with the real
thumb length values instead of random values chosen by the test authors.
