ozone/wayland: do not update buffer scale until new frame size

Fixes flickering when browser is shown for the first time.

TL;DR:
There is a bug that results in a buffer scale set earlier than
new frames with new buffers for a new surface size are sent.

All pending buffer scales are stored during WaylandWindow::SetBounds.
Then, the scales are retrieved during WaylandWindow::CommitOverlay
and applied to WaylandSurface. However, if a new scale is stored,
WaylandWindow::CommitOverlay still sends a frame for old size,
it doesn't retrieve scale from pending scales, but rather uses
window_scale(). That results in setting window's scale factor
as the current buffer scale factor, which is incorrect.

To fix the issue, only apply buffer scales if a new frame
for a new window size is sent, and a corresponding buffer is
about to be attached.
