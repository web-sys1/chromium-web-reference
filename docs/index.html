[FLEDGE] Reuse SellerWorklets.

A new per-frame AuctionWorkletManager loads and tracks SellerWorklets,
and allows them to be reused between auctions.

A followup CL will do the same for BidderWorklets.

Reusing SellerWorklet between auctions has the following limitations:

1) They must run in the same frame (required by DevTools).
2) They must have the same seller script.
3) They must have the same trusted scoring URL.
4) Both auctions must be alive at once.

We plan to relax 4 in a future CL, by keeping idle worklets around
for a short time, when resources are available. We can investigate
relaxing 1) and 3) if needed. 2) doesn't make much sense to relax,
though we could potentially make different worklets share trusted
scoring signals fetches for a shared URL, if it looks a reasonable
avenue for exploration down the line.
