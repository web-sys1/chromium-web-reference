[iOS] Ensure data is destructed with chrome in CBD

My best guess regarding 1429513 is that the BrowsingDataCounterWrapper
are destructed after the preferences themselves are deleted. This is
due to the fact that Objective-C does not have any guarantee regarding
destruction of its object. To avoid this issue, we destruct each
BrowsingDataCounterWrapper when the manager is disconnected, instead
of waiting until the manager is destructed.

While doing it, I realised that all other values had the same
potential issue. For the sake of the safety, I delete all values in
this disconnect function too.

Finally, I rename `_observer` to `_browsingDataRemoverObserver`,
because the name was ambiguous given the number of observers.
