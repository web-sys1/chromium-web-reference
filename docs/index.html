[indexeddb] Add histograms to diagnose transaction leak [2/2].

We have a memory leak (crbug.com/381086791) which seems due to having a
very large number of IndexedDB transactions. Recently, we found hints
that these transactions hit the inactivity timeout
(Transaction::TimeoutFired). This CL adds a bunch of histograms to allow
testing theories we have about what could be causing this.

Histograms added in this CL:
- IndexedDB.TransactionTimeout.MojoDisconnected
