Allow drag and drop of image from browser to CrOS FilesApp

The DataTransfer.File object which is received by FilesApp will not have
a corresponding Entry in a filesystem since it is an image being dragged
from a webpage.

When we notice that we don't have any 'fs/sources' URLs, and the call
to item.webkitAsEntry() fails, we can write the File to the destination
directory rather than using the Entry copy/move actions currently used
for drag-drop.
