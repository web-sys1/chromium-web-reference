Fix and test declarative link capturing origin trial for web apps

The origin trial for web app declarative link capturing was not
implemented correctly. The
WebAppLinkCapturingEnabled(const FeatureContext*) overload needed
to be called to be affected by the presence of an origin trial
token in the document.

Additionally our manifest update process was not disabling the
link capturing after the token was disabled because capture_links
updating was gated behind the browser side feature flag.

This CL fixes both issues and adds a test case.
