[Cronet] Remove base dependency for HttpEngineNativeProvider

The HttpEngineNativeProvider shouldn't depend on chromium base
classes. This provider only does translation between HttpEngine
and CronetEngine and thus, has no rlship with chromium base classes.
Removing this dependency rather than trying to package the base class
with the provider avoids unnecessary dependency bloat.
