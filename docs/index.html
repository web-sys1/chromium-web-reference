[Read Anything] Test the Read Anything WebUI in a browsertest.

The webui test architecture was not designed to handle untrusted
tests. This CL replaces the webui test read_anything_app_test.ts with
a new browsertest, read_anything_app_browsertest.cc. It uses EvalJs
to inject the scripts written in read_anything_app_test.ts. The
trade-off is that this wonâ€™t check for untrusted sources being added.

Alternatives explored:
1. Migrate test to chrome-untrusted://. This would require upgrading
the infrastructure which was explored in
https://chromium-review.googlesource.com/c/chromium/src/+/4292217/
but which would need further design to land.
2. Test in chrome:// with iframe of the chrome-untrusted:// UI. This
requires handling communication across the iframe. Future debugging
might be tricky.
3. Serve webui as trusted just in test build. This means a diverging
implementation and test.
