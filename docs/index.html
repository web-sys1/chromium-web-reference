[contextual tasks] Fix subsequent queries opening in side panel.

Previously, submitted tokens on queries that were already fulfilled
and passed on to co-browsing (including a new handle with the submitted
context) were not being cleared. This caused all subsequent queries
to think that the current tab was in context when in reality that
query had already been fulfilled (and therefore open in the side panel
even if the current tab wasn't in context anymore).

Now, clear the submitted tokens of the original session handle after
fulfillment in cobrowsing.
