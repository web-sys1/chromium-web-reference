[iOS] Reliably clear kSelectedTypesPerAccount pref of removed accounts

When an account is removed from the device (not just signed out from
Chrome!), corresponding per-account settings should get cleared.

This is a follow-up to crrev.com/c/4720751, which implemented the main
part of this, but missed one edge case: An account first being signed
out from Chrome, then separately removed from the device.

This CL fixes that edge case by moving the
ClearAccountSettingsPrefsOfRemovedAccounts() calls to more-reliable
places: It now gets called whenever the list of accounts on the
device changes (whatever the state of Chrome sign-in may be at that
point), plus once on browser startup.
