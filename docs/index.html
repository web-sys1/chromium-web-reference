Reland "Add DumpWithoutCrashing code for beforeunload investigation [2]"

This reverts commit 844eb967a308966c5dec234b749c9c3265825d17.

Reason for reland: The test failures on Linux MSan Tests were fixed.

In the previous CL, The frame_tree_ field was used after it was deleted
in NavigationControllerImpl::DiscardPendingEntry(). This CL fixes it by
checking can_be_in_navigate_to_pending_entry_ flag first.


This relanding CL also changed the feature flags. Initially there are
the following 3 feature flags independently.

- kRunBeforeUnloadClosureOnStackInvestigation
- kRunBeforeUnloadClosureOnStack
- kAvoidUnnecessaryBeforeUnloadCheckSync

To follow the review comment, this CL merged these 3 features into one:
kAvoidUnnecessaryBeforeUnloadCheckSync. And I added the following
running modes instead.

- kDumpWithoutCrashing
- kWithSendBeforeUnload
- kWithoutSendBeforeUnload
