[Signin][Android] Fix restore metric & use bool for "recorded" status

In https://crrev.com/c/5191994, the magic value of the enum value
RestoreStatus.RESTORE_STATUS_RECORDED has been changed. Since it was
saved to prefs, and used to prevent additional histogram recording.

It has broken the restore metrics Android.RestoreResult, since devices
having old prefs and with the status already recorded consider that
the restore status has not been recorded yet, and records the status
histogram again.

This CL uses a new value for the sign-in timed out metric since the old one is polluted, and introduces a boolean pref used to indicate whether
the restore status has been recorded, to limit similar accident in the future.
