Improve cache clearing on accessibility Android, and send text change events

With this CL we update the cache clearing mechanism for the Android
accessibility code, and update our treatment of events to send
a text change event to AT.

With a previous CL: crrev.com/c/2988814
we modified the clearing of a text node from a view when its content
changes. We added the logic that whenever a text node is cleared from
the cache, we must also clear all of its parents up to the first
non-leaf parent, to account for things like a <div> or <p> with only
text inside. This added many extra calls to IsLeaf and resulted in a
performance hit.

With this CL we apply a more general rule that whenever the name of
a node changes, we clear the node from the cache. This fixes the bug
that the above CL also fixed, but does it without superfluous calls to
IsLeaf.

We also include a new event in this case. The Android API has an event
type: CONTENT_CHANGED_TYPE_TEXT which we have not been sending in the
past (added in API 19). We now send the event on name changes and leave
it to AT to respond as desired. The event is sent during a name change,
but only on nodes that act as a TextView on Android (e.g. staticText).
