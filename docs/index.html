arc: Send mounting event to ARC even when volume has watcher attached

Currently, to ARC, we are not sending mounting events for removable
media that have file system watchers already attached.
Unfortunately, this can result in absence of removable media in ARC R,
since ARC R can internally unmount storage volumes when vold is reset
and/or system services are restarted. To restore the unmounted volumes,
ARC R requests Chrome to send mounting events for all existing volumes,
but since such volumes have file system watchers already attached, the
requested mounting events are never sent.
To resolve this issue, basically we should either
(1) just send mounting events even when volumes have watchers, or
(2) make ARC R send a request to detach file system watchers before
    requesting mounting events for existing volumes
Here, (2) is the safer approach since it can prevent Chrome from
accidentally sending multiple mounting events without requests from ARC.
However, fortunately, both ARC P and R are capable of handling such
multiple mounting events by skipping them. Thus the simpler approach
of (1) can be implemented.
