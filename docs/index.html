ozone: Allow specifying a color space for NativePixmap GL binding.

This CL makes GLOzone::ImportNativePixmap() take a gfx::ColorSpace that
is then set on the underlying GLImageNativePixmap.

This is a follow-up to CL:3662321: in that CL, I didn't plumb the color
space for the SharedImageBackingOzone path. This is important because as
of CL:3700122, we use this backing on ash-chrome, and the video decoder
must be able to tell the GL driver how to convert from YUV to RGB.
