[Identifiability Study] Move font active sampling to renderer

Active sampling of fonts performed in the browser requires
instantiating a skia font manager, which can allocate ~1MB of memory,
increasing the memory footprint of the browser process. This causes
several memory regressions (https://crbug.com/1344656).

This moves the logic to the renderer instead, where the skia font
manager is instantiated anyway. The CL also implements some logic for
doing it only in one renderer, avoiding unnecessary work.

Bug: 1344656,1330244
