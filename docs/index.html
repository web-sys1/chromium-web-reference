Reland "chromeos/ash/components/specialized_features: Upgrade SHA-1 to SHA-256"

This is a reland of commit 5eceb2c0ddece23b893921f41b6e9fde2988ad25

Original change's description:
> chromeos/ash/components/specialized_features: Upgrade SHA-1 to SHA-256
>
> SHA-1 is cryptographically broken and deprecated. This change upgrades
> the secret key hash check in FeatureAccessChecker to use SHA-256.
>
> The struct member 'sha1_hashed_key_value' has been renamed to
> 'sha256_hashed_key_value' to enforce this change at compile time for any
> consumers.
>
> SHA-1 is currently being used to check whether a FeatureAccessConfig's
> secret key is valid. This is done by taking a hash of the SecretKey's
> flag value, and is used to gate access to feature usage. Since this key
> is only used to determine whether an account has permission to use a
> feature, and it seems that these keys do not have an external source, it
> should be safe to migrate to using SHA-256 instead. Explicitly setting
> the hashed value seems to only be used under the unittests as well, so
> usage of SHA-1 here has been migrated to use SHA-256.
>
