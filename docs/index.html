Add initial stubs for selecting GTK version at runtime

Since we only ship a single Chrome binary, we can only depend on one
of Gtk3 or Gtk4.  However, we would like to use Gtk4 when it's
available, so this CL adds the initial infrastructure for loading
GTK at runtime.  When this is complete, we will be able to detect
which version of GTK the system supports, and load it dynamically.

* Set up build targets
* Add initial signature files (to be expanded later)
* Add gtk_types.h to provide types only available in specific
  GTK versions.
* Add gtk_compat to dlopen() the libraries and to provide
  replacement functions for GTK functions whose interface has changed
  across versions, but whose (symbol) names have not.
* Add LogNoop to ignore unloaded symbols.
