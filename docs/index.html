remoting: Implement PipeWireAudioInjector

This CL implements the audio packet injection logic for the Linux
PipeWire host. It enables the microphone redirection feature by
bridging incoming audio packets from the client to a virtual PipeWire
audio source.

Key changes:
- Implement Core::InjectAudioPacket() to buffer incoming RAW audio
  data in a deque. Includes a 1 second buffer cap to prevent memory
  bloat.
- Implement Core::HandleStreamProcess() to dequeue PipeWire buffers,
  fill them with available audio data, and pad with silence (zeros)
  if the internal buffer is empty.
- Use base::span and UNSAFE_BUFFERS for memory-safe access to raw
  PipeWire data buffers.
- Ensure thread safety between the Desktop thread (injection) and the
  PipeWire thread (processing) using ScopedThreadLoopLock.
