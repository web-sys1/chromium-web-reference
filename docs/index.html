[BSC] Monitor primary account changes to decide if session is bound

The session is assumed to be bound until proven otherwise. This is to
mitigate the risk of missing adding SIDTS cookie in requests on startup.
We can deterministically know that the session is bound when:
- Sign in keys are loaded
- Tokens are loaded
This measn we slow down all users on startup even if they do not have a
bound session. This can be fixed in various ways and will be looked at
in future CLs.

If at any point the session becomes unbound (e.g. User signs out, Cookie
indicates the user is signed out), renderers are updated, any ongoing
fetch is cancelled, all blocked requests are resumed.
