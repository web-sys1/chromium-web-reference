Reduce usage of AutoEnrollmentTypeChecker::IsFREEnabled

FRE and ZTE state determination were replaced by Unified State
Determination in M131.

This CL replaces the obsolete AutoEnrollmentTypeChecker::IsFREEnabled
with the more appropriate
AutoEnrollmentTypeChecker::FREStateKeysSupported.

This CL also includes Unified State Determination in
AutoEnrollmentTypeChecker::IsEnabled. This is relevant for
AutoEnrollmentCheckScreen::MaybeSkip.

And finally, this fixes
DeviceCloudPolicyManagerAshTest.EnrolledDeviceNoStateKeysGenerated to
actually test what happens without state keys (it tested something else
before).
