[ios] Fix issue with PostRestoreSignIn promo after user signs in

This fixes an issue where the user opens Chrome long enough for the
PostRestoreSginIn promo to display, but then switches to another Google
app and signs in there which gets a new device token resulting in Chrome
also being signed-in, then switching back to Chrome and pressing
"continue" on the alert to sign-in. In this case, we can return early
without continuing on to the sign-in screen because the user is signed
in.
