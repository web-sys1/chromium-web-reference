[PA] Store PartitionRefCount in SubsequentPageMetadata if available

For single-slot spans, ref-count is no longer stored in the slot (even
though the space for it is still reserved). Instead, it is stored in the
subsequent page metadata. Updated single-slot span layout:

  |<---------------------- slot size ------------------------>
  |[cookie]|...data...|[cookie]|[refcnt_placeholder]|[unused]|

This CL allows us to avoid moving ref-count in ReallocFlags, because
the stored position is not affected by raw size. If ReallocFlags needs
larger memory than current slot size, ReallocFlag allocates a new slot
from larger bucket, and frees the current slot.

This CL doesn't affect purge behavior, because it keeps the system pages
at the end of the slot empty and thus decommittable for single-slot
spans.
