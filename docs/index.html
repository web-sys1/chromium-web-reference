gpu: Keep DC layer overlay access in SkiaOutputDeviceGL

Make SkiaOutputDeviceGL own the ScopedOverlayAccess and keep it until
SwapBuffers completes.  This has the same effect as the previous code
which passed along the scoped access in the DC layer params release
callback so it removes the need for that callback.  This approach is
better since the GLSurface is a ref counted object which could be kept
alive in general after the SkiaOutputDeviceGL is gone, which would also
make the scoped access outlive the shared image representation causing
the CHECK failure in the bug.
