Cert provisioning: Debugging details for failed processes

A certificate provisioning process can fail for multiple reasons (basically all
occurrences of `UpdateState(CertProvisioningWorkerState::kFailed);` in
`chrome/browser/ash/cert_provisioning/cert_provisioning_worker.cc`).

When it fails, the UI on chrome://settings/certificates currently only displays
"Failure" and no further information about why the process failed.

This CL adds a debug information string to the UI explaining what led to the
failure.
