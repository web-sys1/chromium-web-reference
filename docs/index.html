magic_boost: check need to orca opt-in when showing disclaimer upon clicking mahi chips

When HMR was previously declined and then its settings toggle is
switched on, the disclaimer UI will be shown again. However, when the UI
shows up again and users decline / approve, HMW consent is affected
regardless of whether HMW consent was previously given or not. This
isn't ideal, this accidentially turn off HMW because of declined
consent.

This CL adds a logic to check if HMW consent has been provided, and if
yes, only opt in HMR.

This work precedes the CL to enable magic boost revamp by default.
