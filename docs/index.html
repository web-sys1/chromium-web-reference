Always access is_evicted_from_back_forward_cache_ on outermost frame.

EvictFromBackForwardCacheWithFlattenedAndTreeReasons checks
is_evicted_from_back_forward_cache_ on the current frame (which could
be any frame) but sets it on the outermost frame. This can lead to
multiple tasks being posted to evict the same frame.

It's unclear if this causes any practical problems but it is confusing.

Also document this constraint and replace a loop with call to GetOutermostMainFrame.
