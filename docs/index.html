[PA] Enable use_partition_alloc on iOS

In preparation to enable PartitionAlloc-Everywhere on iOS, we have to
compile in the PartitionAlloc support to begin with. To pull this off,
we have to switch iOS to use the 32-bit "GigaCage", despite being a
64-bit OS, because "Extended Virtual Addressing" entitlement is needed
to utilize larger address space.

Since use_partition_alloc is now true, it might be tempting to just
remove it and hard-code PartitionAlloc to be always included. However,
it's set to false on Cronet bots to minimize the size of their library.
