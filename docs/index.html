Revert "[reader mode] Disable heuristic triggering in testing config"

This reverts commit d5cfd15079cdaec194d718ef4e2f9446dba5bb4c.

Reason for revert: We ran local stress test experiments (see
crbug.com/453640531 for details). The results show that increase in
peak memory is likely largely due to short-lived memory allocations
that accumulate, before Garbage Collector kicks in. So if there is
more memory pressure (where things matter) then there won't be much
incremental problem. Readability-readerable.js heuristic should be
Okay to use, and we should accept the peak memory increase in the
tests.
