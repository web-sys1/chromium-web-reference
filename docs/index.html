Remove tts_handler.

The existing tts_handler is not recommended, according to the a11y team
we should use live region technique instead. crrev.com/c/3040585 paved
the ground for using live region, but it was restricted in suggestion
window and cannot make announcement when suggestion window is dismissed.

In this CL, we
 - Created a dedicated a11y window for making all a11y announcement
related to assistive features, this view is always open so that we can
make announcement even when all assistive windows are dismissed.
 - We put the a11y window outside screen and make it 0 size so it will
not be noticed by users.
 - Removed tts_handler, all a11y announcement go to a11y window now
 - Removed the live region in suggestion window view since it's no
longer needed
 - Unified all announcement strings in api to be UTF16 strings.
 - Updated the test accordingly, removed some tests for ChromeVox off
situation because this logic is not handled by us now.
