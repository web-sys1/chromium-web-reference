[PA] Simplify `use_backup_ref_ptr`

Through various previously landed CLs, the condition for
`use_backup_ref_ptr` is now identical to the underlying arg that
controls in-PA support.

*   `use_partition_alloc` being false automatically disables all support
    for MiraclePtr,
*   `use_partition_alloc` defaults to false in NaCl build environments,
    and
*   a prior assertion in `allocator.gni` enforces the convention that
    Chromium only uses a live implementation of MiraclePtr when PA-E
    is enabled.

This CL removes `use_backup_ref_ptr` and `use_mte_checked_ptr`,
substituting the `enable_...support` args in their stead. This CL
makes no attempt to deduplicate the `BUILDFLAG(...)`s.

The above is true of MTECheckedPtr (`use_mte_checked_ptr`).
