Deflake largest contentful paint browser test

The test was flaky because the test generates three largest contentful
paints, and they should be sent from renderer to browser, but we always
do not receive the last one before we start to check the metrics in the
test. We need to add the largest contentful paint count to make sure we
have received all of them in the browser side before we start check the
value of metrics in the test.

The page load metric waiter only checks the desired metric but it do not
wait for the number of metrics we want. So I add the number of largest
contentful paint check to make sure we have received three LCPs in this
test.
