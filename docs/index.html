Refactor PrintBackendServiceManager templates

The templates for saved callbacks had a built-in assumption that the
result would be a variety of mojo::StructPtr<T>.  This is no longer the
case for forthcoming wrappers around print pipeline calls, which can
return a more simple mojom::ResultCode.  Refactor the templates to
accommodate that upcoming usage.
