Omnibox: Do not use chrome://blank in tests

chrome://blank/ is not a resolvable URL, and it ends up with an error
page. This makes the CallbackHelper.waitForCallback always time out,
and rely on whether or not TabObserver.onLoadStopped was called to
proceed or fail. This leads to longer test time, and makes it more
prone to failure in case onLoadStopped was not invoked in time.

This CL deletes chrome://blank (this is used in tests only) and
replaces it with chrome://version which is loaded always fine.

Another URL that suffers from the same problem as chrome://blank is
chrome://about which redirects to chrome://chrome-urls. The redirection
keeps TabObserver.onPageLoadFinished from detecting the loading
completion, and causes timeout as well. Replaced this with
chrome://gpu that does not involve redirection in a disabled flaky
test (LocationBarTest#testOmniboxSearchEngineLogo_siteToSite)
to avoid potential failure due to timeout.
