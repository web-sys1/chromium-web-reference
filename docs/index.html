HWR: Add a kHandwritingRecognitionEnabled feature, disabled by default.

Now the feature must be enabled in browser, not just controlled by OT
system in renderer.
The expected/intended behaviour is that the JS API would only be enabled
in blink when this flag is enabled, so the early return in
CreateHandwritingRecognitionService is never hit. However, if the
renderer is compromised it may allow calls there, so we simply don't
respond.
