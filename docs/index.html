[views-ax] Migrating a11y properties to view constructor in //ash

This CL migrates setting of the a11y properties from
View::GetAccessibleNodeData to the view constructor. This ensures that
the a11y properties are updated in the accessibility cache as soon as
the view is constructed. These changes are trivial moves: these
properties never change, so we don't need to add any logic to keep the
cache up to date.

The migration is applied to following directories:
 1. ash/accelerators/*
 2. ash/app_list/*
 3. ash/auth/*
 4. ash/capture_mode/*
 5. ash/clipboard/*
 6. ash/in_session_auth/*

This CL is part of the ViewsAX project:
https://docs.google.com/document/d/1Ku7HOyDsiZem1yaV6ccZ-tz3lO2XR2NEcm8HjR6d-VY/edit#heading=h.ke1u3utej413
