Support container queries with isolated animation updates

Previous CLs implemented isolated animation updates, but it did not
work for container queries, since the second animation update pass
would only include a second round of style and layout-tree building.

This CL extends that to also do a second round of layout, if necessary:

 - When container queries are not in use: we handle the second round
   in Document::UpdateStyle, and ignore layout.
 - When container queries *are* in use: we handle the second round
   in LocalFrameView:UpdateStyleAndLayoutInternal.

Currently "container-queries-are-in-use" is a Document-global flag.
Hopefully this can be narrowed down in the future.

Note that:

 - This CL currently adds *all* Elements participating in a style
   recalc to a HashMap on DocumentAnimations. Obviously this is not
   shippable as-is. It will be reduced later.
 - For each of the calls to LFV::UpdateStyleAndLayoutInternal in
   ::UpdateStyleAndLayout, we are now doing it twice. We already
   have the behavior of triggering animation updates for each
   and every style resolution, so this preserves that behavior as
   much as possible. I don't know if we really want to trigger animation
   updates as a result of auto-sizing, but since we currently do that,
   this CL does that as well.
