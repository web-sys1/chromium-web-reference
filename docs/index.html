Update fetch_handler_type in database if it is updated.

Currently, the fetch_handler_type won't be updated once it is written
to the database until the service worker script is updated.
ServiceWorkerVersion instances created after that will refer the old
fetch_handler_type even if the latest Chromium support the new
fetch_handler_type.

To solve this situation, this CL updates the fetch_handler_type
stored in the database when fetch_handler_type notified from the
started service worker is different from what is stored in
the ServiceWorkerVersion field.

fetch_handler_type was implemented as a replacement of
fetch_handler_existence, and it has not been updated once it is set.
To follow the behavior, we do not update fetch_handler_type in memory
and make only the database updated.

Even with this implementation, fetch_handler_type in memory would
eventually be updated if all references to ServiceWorkerVersion are
released.
