[lacros] Allowed the MRU list to be empty when finding the insertion...

...index.

OverviewGrid::FindInsertionIndex() requires the list of MRU windows to
be non-empty, and hits NONREACHED when that is not the case.  That is
probably not correct because in certain circumstances (e. g., when
creating a new desk and immediately adding some windows) it is empty.

This patch fixes the issue by returning early if the list is empty.
