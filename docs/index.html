[Media Router] Fix message authorization bypass

This CL addresses a vulnerability where a malicious receiver could
bypass consent by spoofing transportId and reserved namespaces.

Changes:
- Filter out reserved namespaces in CastSession::From.
- Reject sessions with transportId == "receiver-0".
- Return Result::kFailed in SendAppMessage for reserved namespaces.

This CL is prepared with gemini-cli
