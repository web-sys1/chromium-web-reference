clang: Add arm and arm64 builtins and crtbegin/crtend files.

For use in lacros builds with `lacros_use_chromium_toolchain = true`.

It's a bit weird that we use builtins and crtbegin/crtend files
from the compiler instead of from the sysroot, but that's how
things are set up at the moment.

The arm and arm64 compiler-rt libs are built with chromium's
sysroot instead of the cros one, which is also a bit weird.
But it's easy to do (after
https://chromium-review.googlesource.com/c/chromium/src/+/3546238)
and shouldn't make a difference in practice.

The 32-bit arm libraries are built without -march=armv7a for now.

Together with
https://chromium-review.googlesource.com/c/chromium/src/+/3703623,
lacros device arm builds with `lacros_use_chromium_toolchain = true`
can now compile base_unittests (but I haven't tried running them yet).

Also add the profile libraries, for local PGO experiments.
