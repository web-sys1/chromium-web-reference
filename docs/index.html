Ozone: Honor Orientation when emitting x/y radius from touches.

Per the Linux MT Protocol, e.g.
https://www.kernel.org/doc/html/v4.16/input/multi-touch-protocol.html

When a simplified ABS_MT_ORIENTATION between [0,1] is available, honor
that X>Y when ABS_MT_ORIENTATION=1. Until now, we always assumed that
TOUCH_MAJOR=X in output.

This change does this by:
1. Adding orientation to the type of data held in InProgressTouchEvdev.

2. Renaming touch_major_scale and touch_minor scale to x/y scale
respectively. Also adds precalculated rotated_* scales for when we
have 'rotated' the screen.

3. Refactoring the calculation of x/y radius from touch_major/minor
into a method.

4. Adding unit tests for this behavior.
