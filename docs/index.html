[FilesSwa] Disable background inspection via SWA window

The SWA variant of Files app does not have a background page and thus
when the chrome.fileManagerPrivate.openInspector private API function
attempts to open one up, it crashes out.

Disable even invoking the fileManagerPrivate API from JS but also
validate that the requesting Extension actually is associated with a
background page. This function can still be called via Devtools so
avoid the unintentional crash from 2 angles.
