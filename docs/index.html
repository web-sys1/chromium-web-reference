[iOS] Allow OverflowMenuDestinationList to set its own height

Before, the list used GeometryReader to calculate its own width.
However, GeometryReader also takes up all allotted space, making it
difficult for the list to set its own height. This Cl changes the
structure a bit.

Now, DestinationList no longer uses a GeometryReader to calculate its
allotted width. Instead, the width is passed into it from the parent.
(The width is not actually used directly in layout, just for calculating
spacing and sizes of sub-elements). Then the list can size its own
height directly.

(Ideally SwiftUI would offer some sort of Single Axis Geometry Reader
to be used in this situation, but it doesn't exist.)
