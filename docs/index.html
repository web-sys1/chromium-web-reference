[webauthn] Polish QR code first flow

An existing feature under the WebAuthenticationPasskeysUI flag changes the passkey flow to jump straight to the QR code for a handful of cases.
This CL tweaks the feature. Chrome will jump to the QR code for:
* A passkey request, which means:
  * Get assertion with empty allow-list, OR
  * Make credential with resident key NOT discouraged
  AND
* There are no paired phones AND
* We are certain there are no usable matching platform credentials

Additionally, chrome will no longer jump to the internal authenticator
for resident key discouraged make credential requests on mac. Instead,
the mechanism selection screen will be shown for consistency with
Windows.

This CL will be followed up by an update to
WinWebAuthnApiAuthenticator::GetCredentialInformationForRequest to
convey that "unknown" platform credential status on Windows <= 10.
