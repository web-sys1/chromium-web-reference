Fixing Combobox Arrow button to show role as button in accessibility tree

Prob Description:
The arrow control is reporting as a combobox, which is not aligned with W3's ARIA spec(link below).
[https://w3c.github.io/aria/#combobox]


In Windows the UIA tree structure convention states that the Arrow control should be a button in Combobox control(link below).
[https://learn.microsoft.com/en-us/windows/win32/winauto/uiauto-supportcomboboxcontroltype#typical-tree-structure]


Solution:
In order to fix the issue we need to change the role of Arrow button in EditableCombobox from kComboBoxMenuButton to kButton.
