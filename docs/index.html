fido: support the CTAP 2.1 makeCredUvNotRqd authenticator option

CTAP 2.0 authenticators require a PIN to make any credential if PIN or
UV have been set up. Therefore, Chrome will fall back to creating
credentials over U2F if the WebAuthn request comes with uv=discouraged
but the authenticator has a PIN or UV configured.

CTAP 2.1 adds an optional authenticatorGetInfo response option,
makeCredUvNotRqd. If an authenticator sets this option, non-discoverable
credentials can be created on it without UV, as long as the alwaysUv
option isn't set as well. This change parses the option from the
authenticatorGetInfo response, and ensures credentials are created over
CTAP2 if non-discoverable credential creation without UV was requested
by the caller and is permitted by the authenticator.
