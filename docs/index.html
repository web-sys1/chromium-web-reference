[Shortcuts] Use subset of icon sizes for shortcut creation

This CL introduces a performance improvement in the image manipulation
code for the Create Shortcut flow by only using icon sizes necessary
for the shortcut creation process instead of using all sizes, and
removing unsued enum sizes if any.

On Windows, this information is obtained from the icon sizes used for
the Chrome product icons.

On Linux, this is defined by go/shortcuts-not-apps-linux.

On Mac, this is obtained from kDesiredIconSizesForShortcut from the
web_applications/ shortcuts code.
