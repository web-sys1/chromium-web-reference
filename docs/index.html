Cache damage_rect in ResolvedFrameData

Move the computation for the "full" CompositorFrame damage_rect
including per quad damage_rects into ResolvedFrameData to avoid an extra
iteration through the quads. Also cache the computed damage so it's only
computed once per frame.

There is some minor refactoring in SurfaceAggregator to support this by
using ResolvedPassData instead of Surface in more places.
