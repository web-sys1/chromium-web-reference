> device selection: Do not special case multi-node changes
>
> Merge the logic of HandleHotPlugDevice and HandleNonHotplugNodesChange:
> When multiple audio devices are plugged, treat as if they are plugged
> one by one. The device selection algorithm is expected to handle
> scenarios like this well. This would also be more robust on reboots,

> SwitchToPreviousActiveDeviceIfAvailable cannot work reliably.

>     user priority.

>     selected when plugged.
