[filesapp] DriveDialogController should check the dialog is open

Closing the device lid: the DriveDialogController.onMountedCompleted_()
is called and calls close() on its |dialog_| member.

However, if the dialog is not open, a console error appears [1]. To fix
this, check if the dialog is open before calling |dialog_|.close().

[1] Files App Dialogs, and their cr.ui.Dialog base class, throw if hide
is called when they are already hidden.
