Microoptimizations in property parsing.

A few microoptimizations that together help ~4% (10â€“15 ns) on slow-path
property parsing:

 - A little strategic inlining of small constructors.
 - More efficient testing of whether a numeric literal is a 0..255
   integer.
 - Make some bitfields in CSSValue into regular fields; the compiler
   generally seems to think that if there are holes, it must do its
   utmost to keep the data in them as-is.
 - Send CSSParserTokenRange by value if we're going to make a copy of it
   anyway (it's two pointers).
