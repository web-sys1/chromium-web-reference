Add ash_ts_library.

See design doc (go/cros-jellybean-build-tooling) for more information.

This CL adds the first iteration of ash_ts_library which acts as a thin
wrapper over ts_library allowing for the use of a slightly more
developer friendly import syntax.

As a proof of concept this CL opts color_internals and the files app
into the new tooling. This CL in itself does not functionally effect
any code and acts as a refactor for both of these apps.

NOTE: ash_ts_library:
  * Does not produce code that optimize_web_ui can inject yet.
  * Does not allow for a neat import for lit directives.

A follow up will add these remaining bits of functionality.
