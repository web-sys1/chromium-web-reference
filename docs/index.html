[carousel] Keep ::column pseudo elements across layouts when possible.

Rather than nuking all ::column pseudo-elements when entering multicol
layout, and regenerating them when finishing multicol layout, re-use the
ones we have from the previous layout. If more ::column pseudo-elements
than that are needed, create them. If there were more ::column
pseudo-elements from the previous than we currently need, remove the
superfluous ones when finishing multicol layout.

This will make ::column::scroll-marker pseudo-elements that need to be
re-attached due to :target-current changes work correctly.

Add some tests to check that relaying out keeps the ::column pseudos
(and their scroll markers) intact. These tests also pass without this
CL. column-scroll-marker-014.html, however, only passes with this CL,
since it needs to re-attach ::column::scroll-marker pseudo-elements.
