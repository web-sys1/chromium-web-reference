Compute kEditedToSimilarValue observations case-insensitively

As claimed by the documentation [1], kEditedToSimilarValue profile token
quality observations do case-insensitive string comparisons. This is
not how it's implemented right now.

This CL fixes the implementation by calling IsWithinLevenshteinDistance
on the lower case strings.
