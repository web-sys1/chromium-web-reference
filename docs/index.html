[views_ax] Temporarily add override_role_ in SetName

We introduced a regression in CL:5309768. It's technically still
possible that a views author will do something like this:

```
view_ax->OverrideRole(my_role);
view_ax->OverrideLabelledBy(other_view, kRelatedElement);
```

In there, OverrideLabelledBy would call SetName, which would set the
name on an AXNodeData with an unknown role since the role was set via
OverrideRole.

Until OverrideRole is removed, we need this.
