[attribution_reporting] Parse attributionsrc on <a> tags

This change integrates attributionsrc parsing for <a> tags. When
an anchor element with the attribute is clicked, it registers an
impression using a similar mechanism to the img API.

See: https://github.com/WICG/conversion-measurement-api/blob/main/EVENT.md#registering-attribution-sources

We add a new token type identifying an AttributionSrc client within the
renderer. This token is routed on blink::Impression and alongside
an AttributionDataHost so that the data host can be tied to a specific
navigation in the browser process. This is required in order to
only use browser process known origins.

This is done rather than the alternative of routing pending_receivers
directly on blink::Impression, which would require forcing every
navigation struct / context menu params struct which embeds an impression
to be move only. This also gives better flexibility for supporting
view event fallbacks in the case of failed navigations.
