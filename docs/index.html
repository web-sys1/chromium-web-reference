CCA: set permission to +r while deploying

rsync by default use source file permission masked by target file system
umask while transferring files, and since workstation defaults to have
file not readable by others, this cause problem that the deployed file
are not readable by Chrome.

Set --chmod=+r to rsync to ensure that the deployed file can be read by
Chrome. Set --perms so existing files (before this patch) will have
their permission fixed.

Bug: None
Test: manually `chmod go-r views/main.html` on host and check that cca.py deploy works
Test: manually `chmod go-r /etc/camera/cca/views/main.html` on DUT and check that cca.py deploy works
