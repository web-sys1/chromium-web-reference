Remove hack in AmbientAnimationPlayer.

The lottie::Animation API only supports looping between 2 timestamps.
It does not support looping from [A, B), then from [C, D). In order
to do that, you need to stop the animation after the first cycle is
complete (which can lead to discontinuity in the animation's
smoothness and is also hard to account for in other parts of the code),
and then restart it again with a new playback config (loop from [C, D)).

This adds an API to do so, and removes the hack from
AmbientAnimationPlayer.
