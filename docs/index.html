guest_os: file_manager: Tests control mock guests from JS instead of C++

The tests are written in Javascript, so it's nice to have all the logic
in one place (and necessary when we start testing dynamic
addition/removal) instead of browsertest_base adding things which are
then verified by the JS.
So we add register/unregister messages and update the tests to set up
their own mocks. As a bonus, this might fix the rare flakes in these
tests where the expected mocks aren't found.
