Optimize CSSVariableData for a single backing string.

In most cases, we'll have a single backing string for any given
CSS variable, and placing that in a Vector (with slop and all)
is wasteful. We can store the number out-of-line, and then have
a path where we just store a single String inline in the class
if we don't need more.

Style perftest PartitionAlloc usage (measured through the same
hooks as last patch):

  Encyclopedia    6410 →  6408 kB (  -2 kB)
  Extension       6342 →  6342 kB
  News            3959 →  3932 kB ( -27 kB)
