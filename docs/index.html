Clean up old code paths for touch selection handle styling.

Remove flag checks for touch selection handle styling related to the
launched ChromeOS "TouchTextEditingRedesign" feature (e.g. dynamic
colour, padding, opacity) and remove old code paths.

This should not change user visible behaviour on ChromeOS, but it will
change the appearance of the touch selection handles for other platforms
since "TouchTextEditingRedesign" is currently only enabled by default
for ChromeOS.

(Chatted with Chrome UX, they are ok with these style tweaks and don't
believe it needs to be flag guarded since it's a minor change.)
