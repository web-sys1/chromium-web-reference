[WebInstall] Add and hook up 1 parameter API signature

Update API signatures on the blink side to reflect the newly agreed upon
3-signature structure (1).
- Add a 1 parameter signature that takes only an install_url
- Switch the order of the existing 2 parameter signature so install_url is always first. (Also switch the order in all other browser/test references for consistency)
- Make manifest_id optional in the InstallOptions mojo struct to
support the 1 parameter, install_url only version. (Update plumbing in
WebInstallFromUrlCommand to reflect this)
