app_data: Kiosk PWA app can be auto-launched offline

Update |LoadFromCache| function to be able to launch a PWA app offline
after it has been successfully auto-launched.

We have two flows for loading an app icon:
  1) Load the icon from |icon_url_|. It’s used for loading the icon when
  a user opens the App menu in the loading screen and the app wasn’t
  installed earlier. Then we update |LastIconUrl|.

  2) Load the icon from |WebAppInstallInfo| during the app installation.
  In this case the icon was not loaded from |icon_url_| and we do not
  update |LastIconUrl|.

If the app was successfully auto-launched and installed, but never
manually launched, |LastIconUrl| will be empty. But the app info is
actually cached and the app can be launched offline. Because of this,
we need to check for updated |icon_url_| only if the app has not been
installed earlier.
