blink: introduce GC_SAFE_FIELD to ignore tagged forbidden fields

With the introduction of the new blink GC clang plugin feature, some
fields/compositions were tagged because the included forbidden fields.
However, in some cases, using those fields is safe, and having a way
to mark them as such is necessary.

This macro is only a wrapper over the already existing
GC_PLUGIN_IGNORE, and is only meant to make it clearer why the field
is being tagged.
