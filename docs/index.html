Use WritePixels to draw canvas hibernation image

The canvas hibernation image is only drawn once and is cached. This can
lead to an apparent memory leak (not a real one) when there is no memory
pressure because the cache limit is quite high.

WritePixels already writes bitmap image without caching, and OOPR-Canvas
already uses it. This just updates non-OOPR to use WritePixels as well
which both simplifies code and reduces memory.
