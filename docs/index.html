Be more specific about what crate version to download

Cargo is opinionated about what version of a crate it expects to use
as a dependency. If we don't have that correct version, show an error
to the user.

And when we don't have any version, instead of having the user
download the latest version matching the epoch, point them to the
exact version that Cargo wants to use.

This avoids problems of having a different version than what matches
complex dependency rules of different crates, as Cargo attempts to
combine version requirements together, and will sometimes choose an
older version of a crate in order to meet the requirements. And we
should do the same.
