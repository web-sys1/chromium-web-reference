media: Remove VD-SFC code.

This CL is a controlled revert of CL:2641088 which introduced code to
use the VD-SFC capability of Intel devices to scale protected content.

Since then, we have landed CL:3089495 which switches to using VE-SFC for
this purpose. Therefore, the VD-SFC path is no longer needed.

By "controlled revert," I mean reverting only the parts of CL:2641088
that are no longer useful.

Additionally, I sprinkled a couple of minor aesthetic fixes in
vaapi_video_decoder.cc:

- I replaced |output_pic_size| with |decoder_pic_size| when calculating
the |decode_to_output_scale_factor|: numerically, they're the same at
that point, but semantically, it makes more sense to use
|decoder_pic_size|.

- Passed |decoder_pic_size| instead of decoder_->GetPicSize() to
vaapi_wrapper_->CreateContext().

Test: video.PlayDRM.cencv1_h264_ctr on volteer.
