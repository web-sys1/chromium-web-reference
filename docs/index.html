CCA: Add custom ESLint rule to check for parameter comments.

Add a custom ESLint plugin and implement a rule that force comments for
function parameters to look like /* var= */.

Note that go/tsstyle#comments-when-calling-a-function states that "The
exact formatting and placement of the comment is not prescribed.", but
we still want a consistent style inside CCA codebase.
