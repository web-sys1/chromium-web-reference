[crossOriginIsolated] Fix isolated ServiceWorkers process assignment.

Currently, ServiceWorkers are never considered cross-origin isolated and
do not share a process with their parent page if they are. While safe,
this has significant performance impact, because a new process needs to
be spawned. Instead, a ServiceWorker that specifies the
Cross-Origin-Embedder-Policy set to "require-corp" will be considered
cross-origin isolated and will be able to share the processes.

Important note:
Fresh ServiceWorker installs do not have access to the headers, and will
be put in non cross-origin isolated processes. Because of this
discrepancy, we do not enable the javascript crossOriginIsolated bit.
This will be done once this problem is addressed. Progress tracked here:
crbug.com/996511
