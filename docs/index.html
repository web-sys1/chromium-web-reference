[wptrunner] Always write text artifacts as Unicode

In crrev.com/c/3884237, the results processor was moved from a
subprocess into an in-process thread, meaning `PYTHONUTF8` mode was no
longer applied. On Windows, this meant that `open(...)`, which backs
`typ.artifacts.Artifacts.CreateArtifact(..., write_as_text=True)`,
could use a platform-dependent encoding [1].

This change fixes this issue by manually encoding text artifacts with
'utf-8', then writing to binary files. As `typ` still appears to support
python2, where `str` is `bytes`, it's not easy to fix this from `typ`'s
side currently.

This change also broadens the exception caught for `process_event(...)`
so that the auxiliary thread logs issues in general instead of crashing,
which stops result reporting.

[1]: https://docs.python.org/3/library/functions.html#open
