[json-schema-compiler] Use default init for enums

This CL is making enum initialisation uniform, by having all enum
members initialised with `()` in the initialisation list, mojo style. This
makes it possible to initialise scoped, and non-scoped enums, without
having to care about the subtleties of each case. Furthermore, `kNone`
is being explicitly initialised to 0, to match the default underlying
value.

For cases where we want to compare against the default, be it `kNone` or
a `_NONE` suffixed key, we are for now using the type's default
constructor (e.g. `T()`). This makes the reference to the default value
uniform, rather than specific to the type of enum (e.g. `kNone`, or
`_NONE`).

The function introduced to provide default initialisation, namely,
`GetEnumDefaultValue`, will be useful in the future, when we remove the
`kNone` key, as it will provide an appropriate place to decide if we
should just default initialise (i.e. `T()`), or use `absl::nullopt`.

Using default-init with enums removes the reliance on the propagation of
compiler options to generate correct `kNone` keys, which has proven to
be unreliable.
