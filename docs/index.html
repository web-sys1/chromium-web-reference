[omnibox] Suppress AIM page action in on-focus edge-case.

In order to ensure that the AIM page action doesn't get "sandwiched"
between other page actions shown in the Omnibox, this CL updates the
visibility logic such that the AIM page action gets suppressed when
the Omnibox has focus AND user input is NOT in-progress AND the Omnibox
popup is NOT open.

Please refer to the following screencasts to better understand the
visual impact on UX associated with this change:

BEFORE: http://shortn/_edyMMO2SZV
AFTER: http://shortn/_QRp5kLWfcP

NOTE: The fact that we're no longer showing the AIM page action on
internal chrome:// pages aligns with the desire from PM to prioritize
showing the page action when the popup is open. In other words, hiding
the page action on internal pages, which don't trigger the Omnibox
popup, seems like the correct behavior in this case.
