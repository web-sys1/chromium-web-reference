[Accept-CH] Add UMA to debug mismatch on offloading

This CL adds UMA metrics to investigate why the Accept-CH offloading optimization is not performing as expected.

We are observing cases where the client hints sent with the resource request do not match the client hints checked via IPC, resulting in `AreCriticalClientHintsMissing` returning false.

This UMA helps diagnose the cause by comparing the `enabled_client_hints` at resource request creation time with the `enabled_client_hints` at the time `OnAcceptCHFrameReceived` is called. It also records which specific hints are mismatched using an enumeration histogram.
