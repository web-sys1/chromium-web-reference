Get rid of SyncTest::SetEncryptionPassphraseForClient() usages

SetEncryptionPassphraseForClient() are overused and needs a lot of
boilerplate code in SyncTest. In most (except one) cases tests can
conveniently use SyncUserSettings interface and achieve the same.

The actual difference between
SyncTest::SetEncryptionPassphraseForClient() and
SyncUserSettings::SetEncryptionPassphrase() is that former mimics
passphrase being set during first sync setup, while the latter sets
passphrase at the time of call, usually after SetupSync().

The only test that needed to SetEncryptionPassphraseForClient() is
DoesNotLeakUnencryptedData, which verifies that if passphrase is set
during the first sync setup server won't see unencrypted data at all.

This CL rewrites DoesNotLeakUnencryptedData in a way it sets up sync
on its own, without relying on SetupSync() logic - probably also making
test expectations more explicit. Rest of the tests now call
SetEncryptionPassphrase() after SetupSync().

SyncTest is to be cleaned up in the follow up CL.
