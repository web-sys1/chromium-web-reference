[memories] Make cluster element delegate focus to child link

Previously, when the user tabbed over to the list of clusters, the whole
cluster element was focused, which did nothing.

Now, this cluster element correctly delegates its focus to a child
link element.

This also causes a focus-ring to be drawn around both the focused link,
as well as the selected cluster. This is to follow the a11y office
hours feedback that it's useful to indicate both which element is
selected, as well as which whole cluster.

I also tried removing the tabindex off the whole cluster element, but
that's a disaster that iron-list can't handle, so I added a comment
warning future developers of that.
