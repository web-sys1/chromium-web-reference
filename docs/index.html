[dPWA] Cleaning up intent helper & intent tab helper.

The intent picker helper and the tab helper had very complex code
paths calling back and forth from each-other, and a non-insignificant
portion of the code implemented checking for shutdown & navigation
on the web contents. By moving this logic into the tab helper & having
the WeakPtrFactory invalidate on navigation, I was able to remove
all of that code, as well as other misc cleanups.

This makes the logic much easier to follow. There are likely more
cleanups I can do, but this was a first pass. A non-goal of this
change is to move this class out of the 'ui' target, as there are very
few 'ui' dependencies here.

Follow-up patches planned:
- Moving all of the per-navigation state & logic into a separate class
  or struct to make tracking that much easier & intuitive.
- Moving the tab helper into the intent_helper directory.

This is a non-functional change, purely a refactor.
