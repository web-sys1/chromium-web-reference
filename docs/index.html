Migrate UrgentPageDiscardingPolicy to base::MemoryConsumer

All base::MemoryPressureListener implementations are being migrated to
the new memory coordinator API. This new system gives policies much more
flexibility on how memory is managed across Chrome processes.

Unlike the old system, which used a single OnMemoryPressure callback,
base::MemoryConsumer splits responsibilities into two phases:

1. OnUpdateMemoryLimit(): The consumer must cap its maximum cache size
at the greater of its current usage and the new target limit. No memory
should be evicted during this phase. This is irrelevant for
UrgentPageDiscardingPolicy, since it is not a stateful MemoryConsumer.

2. OnReleaseMemory(): The consumer should enforce the target limit
strictly, evicting any entries that no longer fit. This is where
discarding happens, but because it is non stateful, there is no target
limit to follow. This function will be called repeatedly as long as
memory pressure lasts.
