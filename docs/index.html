Change order of unique constraint on cookies SQLite table.

When I first added the top_frame_site_key, I made it the first part of
the unique constraint (and internal index) without realizing
CookieMonster only loads cookies and tracks pending cookie loading tasks
by host key only.

For now, we can keep this behavior and load all Partitioned cookies for
a given site into CookieMonster. If we find that prolific third-parties
need to load cookies across a lot of partitions, the new unique
constraint allows us to also load cookies by top_frame_site_key while
still using the index.
