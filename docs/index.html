SSO for CSSParserToken.

For CSSParserTokens that contain strings, the string is often so
short that we can store the string directly in the token instead
of a pointer to a backing string (short string optimization, or SSO).
This saves on the requirement to store a backing string _and_ the
pointer to it (although the latter is now generally cheaper than
it used to be, after the backing_string vector patch).

Style perftest PartitionAlloc usage (measured through the same
hooks as last patch):

  ECommerce       1388 →  1388 kB
  Encyclopedia    6408 →  6407 kB (  -1 kB)
  Extension       6342 →  6342 kB
  News            3932 →  3926 kB (  -6 kB)
  Search          1902 →  1899 kB (  -3 kB)
  Social1         5800 →  5751 kB ( -49 kB)
  Social2          489 →   489 kB
  Sports          3724 →  3723 kB (  -1 kB)
  Video          12183 → 12079 kB (-104 kB)

Performance is generally neutral.
