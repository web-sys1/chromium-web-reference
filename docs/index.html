Remove the Python version of StringHasher.

Instead of computing these hashes in Python, we can make parts of
StringHasher constexpr. This means that we no longer need to maintain
two parallel versions of our hashing code, which will make it easier
to change it in the future. (Well, we still need a constexpr version of
the hashing code, but it seems easier to share logic between C++ and
constexpr C++.)
