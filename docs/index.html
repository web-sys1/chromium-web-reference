Simplify WebSockets endpoint lock management

WebSocketTransportConnectSubJob needs some extra logic because there's a
window of time between when we make a LockReleaser and when we create a
lock. But WebSocketStreamSocket::Connect works just fine, so we can wrap
it earlier.

As part of this, I've merged the STATE_OBTAIN_LOCK_COMPLETE and
STATE_TRANSPORT_CONNECT states. This is a no-op, but it makes it more
obvious that we immediately take ownership of the lock when we're handed
it.

(Ideally LockEndpoint would just hand you a LockRelease, but alas
//net's calling convention is bad at returning things that aren't int.
Clearly we need to invent net::Promise!)
