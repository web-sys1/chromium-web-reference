Let the browser know when document.open() happens

document.open() will update the document's URL in the renderer, but
the browser has no way of knowing that it happened. A previous attempt
tried to generate a same-document navigation for document.open() but
was later reverted in crrev.com/c/2818538.

This CL makes document.open() tell the browser the latest URL in the
renderer without generating a navigation/updating the last committed
URL. The URL is tracked separately by the RenderFrameHost, since it's
useful to predict how future navigations will behave (e.g. whether
they will do replacement or not). This CL also makes the
has_committed_real_load_ flag in FrameTreeNode be true if a
document.open() has happened, similar to empty_document_status_ in
FrameLoader in the renderer.
