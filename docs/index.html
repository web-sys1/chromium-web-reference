Popups should run smooth scrolls on the main thread.

A WebPagePopup doesn't have a threaded input handler (see comments in
WidgetBase::InitializeCompositing), so it won't sync scroll deltas to
the main thread (which happens in TIH::ProcessCommitDeltas).

ScrollAnimator shouldn't schedule compositor animations in this case.
Make it tick the animation on the main thread instead.
