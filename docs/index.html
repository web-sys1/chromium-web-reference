[Fast Pair] Silently Write account keys on Retroactive Pair

Relevant ddocs:
 - go/cros-fp-silent-retroactive-pairing.
 - go/cros-fp-account-association-writes

On retroactive pair, before the user is prompted to save the device
to their account, the account key is written to the device and the
account association is saved locally. If the user opts to save the
device to their account, the account association is written to Footprints.

Test=
	- All unit tests pass.
	- Initial, Subsequent, and Retroactive pair scenarios all WAI
	on DUT using both a V1 and a V2 device.
        - On Retroactive Pair, without clicking 'Save' on the
        retroactive pairing notification, the account key is written
        to the peripheral. On clicking 'Save', the account association
        is written to Footprints.
