WaaP: Fix race condition in ExtensionTabsApiTest.WindowSetFocus

This CL fixes a race condition in the
`ExtensionsTabsApiTest.WindowSetFocus` interactive ui tests that occurs
when Initial WebUI is enabled.

When the feature is active, BrowserView show is deferred until the top
chrome webui finishes painting and that is not expected in this test
which assuming that calling chrome.windows.update with `focused: true`
would result in an immediate focus state change.

This CL updates the test to expect such case by using an event-driven
approach.
