[LayoutNGBlockFragmentation] Paint caret correctly when text spans multiple columns

This patch changes |ComputeNGCaretPosition()| to use |NGInlineCursor::
MoveToNextIncludingFragmentainer()| to make |ComputeNGCaretPosition()|
to work with block fragmentation.

Before this patch |ComputeNGCaretPosition()| returns null position for
|Text| laid out in multiple columns because |MoveToNext()| moves in
one fragmentainer only.
