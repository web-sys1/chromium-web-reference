[AW] DeFlake WebViewJSTest.

This test was flaky because it wasn't actually waiting on right view
conditions, but rather waiting for the java queue to go idle and then
checking all view conditions were true.

This change fixes that by using waitForView() method from ViewUtils
to wait on the right conditions we're testing for.
