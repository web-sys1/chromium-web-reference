[StyleBuilder] Remove InternalStyle call from RootElementStyle

This removes the InternalStyle() call from
StyleResolverState::RootElementStyle().

RootElementStyle() was being used from three places:

https://chromium.googlesource.com/chromium/src/+/4479f5f2a6e4f50ee9de78c5979682306e6c7e9e/third_party/blink/renderer/core/css/resolver/style_resolver_state.cc#122 [1]
https://chromium.googlesource.com/chromium/src/+/4479f5f2a6e4f50ee9de78c5979682306e6c7e9e/third_party/blink/renderer/core/css/resolver/style_resolver_state.cc#133 [2]
https://chromium.googlesource.com/chromium/src/+/4479f5f2a6e4f50ee9de78c5979682306e6c7e9e/third_party/blink/renderer/core/css/resolver/style_resolver_state.cc#259 [3]

[1] & [3] have no behaviour change - as they are being passed
InternalStyle() as |style| == |root_style| in FontSizes.

[2] I think is fine? The only main change is StyleResolverState::FontSizeConversionData() but I think when
RootElementStyle is null, we'll be using the initial-style.

Then the changes to the FontSizes constructor:
 - media_values.cc all use the same style for both arguments (should
   be fine).
 - css_container_values.cc uses the document element computed style
   (should be fine).
