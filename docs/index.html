Make room for trailing margins in column balancing.

It's not super-obvious that we should make room for such margins, but it
makes us more compatible with both Gecko and legacy Blink. And it makes
a bunch of tests pass.

The fragment size of a single-strip column in the initial column
balancing pass was largely ignored (until now), so some changes were
necessary wrt. to how we handle forced breaks (including spanners) on
the block layout algorithm side:

1. Don't include previously consumed block-size in the fragment size.
2. Don't include trailing margins in fragments to be continued.
