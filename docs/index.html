Remove pseudo-cwd value from vsh cmdline

JS code passes through --cwd=terminal:<uid> which gets translated to the
real pid by looking up ProxyProcessRegistry to map <uid> to <pid>. We
have been inadvertently passing both --cwd=terminal:<uid> and
--cwd=<pid> flags when starting vsh.
