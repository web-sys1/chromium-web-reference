Expose parseable label through AutofillShowTypePredictions

AutofillEnableSupportForParsingWithSharedLabels adds logic that
potentially splits a field's label among multiple fields. E.g. in a DOM
like
  Street name and house number: <input><input>
Autofill currently assigns the full string as the label of the first
field, and no label to the second field. With the feature, the label
is split and "Street name" and "house number" become the labels,
respectively. This is stored as a field's "parseable label".

For debugging purposes, AutofillShowTypePredictions annotates fields
with their labels/type predictions. With this CL, when
AutofillEnableSupportForParsingWithSharedLabels is enabled, fields
get annotated with their parseable label instead of their regular
label. This is consistent with the existing logic for the parseable
name (which is sanitized from the name attribute in a different way).
