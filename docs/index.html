DL: Don't double-mark ancestors as dirty when upgrading forced phase.

This patch ensures that we don't double-mark style as dirty when
upgrading the forced phase from style to style + layout. The reason
for this is that the second time we mark this, our assumptions that
we "blocked the last bit propagation" is false, since we propagated
the bit on the first time we marked style as dirty.
