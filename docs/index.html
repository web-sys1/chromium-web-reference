Fix autofill::AddressFormDataImporter public interface

This change replaces the incorrect return value which is supposed to be
the number of complete profiles for the profile extraction in
`autofill::AddressFormDataImporter`.
Although the docstring documented a different behavior, the returned
number always matched the number of all profiles that were extracted.
Thus, the interface was simplified to directly return the `std::vector`
of all extracted profiles.
