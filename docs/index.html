[Nearby][WLAN] Add mojo for opening TCP sockets in utility process

We wrap the network_context.mojom TCP creation methods in a new mojo
service, tcp_socket_factory.mojom. The NetworkContext mojo service is
meant to only be used by the browser process, not the Nearby Connections
utility process. This new service allows us communicate between
processes in the following way

Nearby Connections utility process
             ^
             | (tcp_socket_factory.mojom)
             v
      browser process
             ^
             | (network_context.mojom)
             v
     networking process

This CL is part of the effort to add WLAN as a Nearby Connections
transfer medium on Chrome OS. See http://go/cros-nearby-wlan for more
details.
