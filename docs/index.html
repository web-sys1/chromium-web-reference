CodeHealth: Switch WebApp::operator== to from make_tuple to tie.

std::make_tuple actually makes a copy of all of the arguments, which is
unexpectedly expensive for an equality check. It was done to avoid a
dangling reference to the string created by SerializeAsString().
A better solution is to go back to std::tie and define an equality
operator for the WebAppOsIntegrationState proto.

I'm a little dubious about using SerializeAsString to compare protos as
I believe this includes deprecated/unknown values but leaving it
unchanged in this CL.
