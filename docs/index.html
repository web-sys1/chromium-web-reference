Fix passing component policy from Ash to Lacros

Component policy (i.e. policy for extensions) should be passed to
Lacros at first launch. This is currently not happening because at the
time of initializing launch parameters for Lacros, component policy is
not set in EnvironmentProvider yet. This is because BrowserManager
starts observing ComponentPolicyService then continues on to
initializing Lacros launch parameters, while the notification from
ComponentPolicyService happens asynchronously.
This CL fixes this issue by caching the component policy in
ComponentPolicyService and fetching it synchronously when preparing
policies for Lacros, in addition to observing changes in the policy.
