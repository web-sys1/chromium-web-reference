Use OnWidgetDestroying() in WidgetFadeAnimator

OnWidgetClosing() is going away as an API (WidgetObservers expect this
to always be called, but it's only called if an explicit Close() request
is the reason for its closure, so it's commonly a UAF risk).

This is maybe(???) also a speculative fix for crbug.com/1245948 as it's
possible that something is requesting the closure of this widget on the
same stack that it's created but before it's been used. If that's the
case then it's possible that WidgetFadeAnimator's `widget_` is cleared
out between its construction and first call to ::FadeIn().
