vmm: set inactive timestamp on arc connection

Initialize last_peace_timestamp_ when the ARC connection is ready, so
that ArcVmmSwapScheduler gets a non-null timestamp and can enable swap.
This fixes an issue where the scheduler could not enable swap until
after an unthrottled -> throttled transition while the ARC connection is
ready.
