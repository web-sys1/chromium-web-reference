Refactor AwBackgroundTracingMetricsProvider

Currently, compression is done on the background, and some small
additional work is done when back on the main thread. This CL moves
this additional work to the background.

This is in preparation for moving some UMA metrics service code to
the background. Without this refactoring, there would be a need to
do some unnecessary thread hopping.

This CL should be a no-op.

Also adds a timer histogram, and extends expiry date.

Also changes some function inputs from references to pointers. This
is also in preparation for the aforementioned UMA changes, as a
callback will be added to delete these inputs. Otherwise, there would
be dangling references, which can be hard to reason about and protect
against. See
https://chromium-review.googlesource.com/c/chromium/src/+/4567010/comment/732a68f6_eb64c30d/
