[Masonry] Parse "infinite" keyword for item-tolerance property

The item-tolerance property is used to determine the "threshold" for
considering tracks to be the same height when placing items:
https://drafts.csswg.org/css-grid-3/#item-slack. This value can take in
Length-Percentages and Keywords.

Previously we only accounted for the "normal" keyword, but we need to
account for the `infinite` keyword for item-tolerance (listed in the
spec linked above) as well. This change adds parsing for the `infinite`
keyword, but a follow-up change is needed to actually handle the
`infinite` case when calculating item-tolerance.

A new class `ItemTolerance` was created in order to account for the
extra keyword; previously, we only had to check if the length had been
initialized, and if it hadn't, then we would use the default value of
`normal`. As a result of using a new class to handle the
`item-tolerance` property, the auto-generated code in
compute_style_base.h/cc uses the `GetItemTolerance` method, as opposed
to the `ItemTolerance` method to retrieve the `item-tolerance` property.
