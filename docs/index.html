Reland "Add tests to latency info trace events data"

This is a reland of commit ef45f32709958f5c9a560b8489b9b6951362cf39

This reland includes a fix to the test where the delay of each of the
dispatched input events (if any) is no longer logged to the test results.
Since it's unknown whether an event will have a delay, logging it to
the test results introduces a non-determistic behavior.

Fixed: 1318544
Original change's description:
> Add tests to latency info trace events data
>
> The Interactions swimlane in DevTools Performance panel was broken due
> to changes in the shape of the data of latency info trace events.
> Because of the lack of tests to the data in these events, the breakage
> in the panel wasn't noticed until more than 2 years after the first
> breaking changes landed. This CL adds tests to the generation of data
> so that we can catch future breaking changes before they cause a
> regression in the frontend.
>
> The tests added in this CL make sure that the data is dispatched in the
> way (shape) expected by the implementation of
> https://crrev.com/c/3583855 (which is based on the up-to-date shape of
> the latency info trace events).
>
