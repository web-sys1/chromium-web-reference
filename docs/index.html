Re-enable TabModelSelectorTabObserverTest tests

For some reason a change from January or earlier suddenly started
flaking the TabModelSelectorTabObserverTest.

What appears to have occurred is that deferred tab registrations from
the UI thread are racing with the test asserts which are happening on
the instrumentation thread. The easiest solution is to just wait for
the deferred operation to resolve.

In production this should never happen because all use of this observer
should take place on the UI thread.

Locally repeated 50 times with 0 flakes after this change it flaked
40% of the time before.
