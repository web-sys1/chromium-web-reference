Make MediaRecorderFromVideoElement test more resilient to init delays

MediaRecorder might take longer than 5 frames to initialize in cases
where it needs to survey codec support from accelerate encoders.
That's why this test needs to make sure that we have actual encoded
video before proceeding with the test.

In the ideal world we would use `onstart` event for that, but
unfortunately it's fired immediately and not when the recorder has
actually started. Moreover MediaRecorder can even yield one byte chunks
while waiting for accelerated encoder initialization.
