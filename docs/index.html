Fix presentation callback in ui compositor test_utils

`WaitForNextFrameToBePresented()` in ui/compositor/test/test_utils.h
uses presentation callbacks to wait for a frame to be presented.
However, the callbacks it currently uses would be called even if the
next submitted frame fails to present successfully on the screen. This
CL fixes that by using presentation callbacks that are called when a
following submitted frames gets successfully presented on the screen.
