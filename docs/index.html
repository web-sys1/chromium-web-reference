Fix coordinates used to reparent items from folder to root grid view.

This CL fixes a bug where items on non-zero'th pages of app list folder
views would have offset drag and drop coordinates when reparenting to
the root level apps grid view. This CL also updates how we populate
the app list test model as the previous model would have duplicate
app item IDs when creating items in both the root level apps grid and a
folder's apps grid.

Note that PopulatedAppListTest.ScreenRotationDuringFolderItemDrag in
had a bug where Item 2, Item 3, and Item 4 are duplicated in the folder
and the root apps grid view.
