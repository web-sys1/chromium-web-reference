ozone/wayland: fix rebinding of interfaces.

WaylandBufferManagerGpu might get AddBindingWaylandBufferManagerGpu
call again before the receiver is disconnected if an error
happened. That may result in a fatal error CHECK(!receiver_.is_bound());

To fix the problem, use mojo::ReceiverSet instead of mojo::Receiver,
and clean existing connection when BindHostInterface() is called
as WaylandBufferManagerHost can be bound several times (a callback
that notifies about destroyed channel can come later than a request
to bind the interface one more time).
