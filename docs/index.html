Fix same-version overinstalls on Mac.

OverinstallBroken exercises this codepath. It fails without this patch, which is intended to fix the issues revealed by this test:

* CopyDir fails if the bundle already exists. Removing the version directory first avoids this (and cleans up any other unexpected state in the destination).
* Failed update service proxy objects did not totally reset state on failure -- it could think it was still trying to connect after the connection attempt failed. Tracking connection-ness state separately from bound-ness of the mojo::Remote is unnecessary; removing tracking the "connecting" state gets rid of the unintended state reached here.
* After trying to change which update service exists and is active, it's a bit weird to expect the (previously failed) UpdateServiceProxy to keep working as intended. It does, but it feels conceptually weird. After a WakeCandidate, rebuild update_service_ to clean this up.

Also cleans up an IWYU violation on scoped_refptr.h
