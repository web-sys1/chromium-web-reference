Move release_fence to Overlay EndReadAccess

This CL moved release fence from SharedImageRepresentationOverlay
BeginReadAccess to EndReadAccess. Overlays sometimes are presented
across multiple frames and we need to use fence from the last frame to
correctly synchronize access.

On some platforms (e.g Android SurfaceControl) system
compositor provides us with a release fence, so it won't work with
BeginReadAccess.
