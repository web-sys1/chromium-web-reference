Optimize performance of LeakDetectionDelegateHelper.

The class does UpdateLogin() in the loop for all leaked username and
password stored. It's incorrect if it was already known to be leaked.
The timestamp should not be overridden in that case.
Another issue is degraded performance. SavedPasswordsPresenter reacts to
every such an "update". If the user has many Android credentials, the
lag is getting worse due to affiliation injections.
