BroadcastChannel: Implement Service and have Provider use it

Implements BroadcastChannelService which handles the connection
tracking that used to be handled by BroadcastChannelProvider.
This is in preparation for adding two new providers that will
leverage the same connection tracking functionality but that
will retrieve the origin differently than how it's retrieved
now (directly from the renderer via the Mojo ConnectToClient
IPC method).

This is the first of a series of incremental CLs to transition
the BroadcastChannel API to being partitioned by StorageKey
instead of origin. For more details, see the "Partitioned
BroadcastChannel Design" document:

https://docs.google.com/document/d/1CchMLU8QmDEx_tA0y57qLyQWvaTT3dlWF-4P_ncc06E/edit?usp=sharing
