[editing] (1/4) Make ComparePositions() faster

This patch is the first part of a series of patches to make
`ComparePositions()` faster.

This patch moves `ComparePositions()` related functions into the new
file "compare_positions.cc" to reduce the size of "editing_utilities.cc"
and for ease of maintenance.

The ideas of fast `ComparePositions()` are:
  * Comparing child Nodes with the same parent can be faster
    - Selecting many <span>â€™s with text can be faster.
    - This is used for comparing children in common ancestors and
      relative positions, aka before/after node, after children.
  * Container + Child Before Node and Container + Child Index are
    equivalent
    - SelectAll commands, make range (container@0,
      container@afterChildren), can be faster
    - Utilize child nodes comparison for relative positions instead of
      counting the number of children.

The series is
  1. https://crrev.com/c/4039247 Move to "compare_positions.cc"
  2. https://crrev.com/c/4040525 Move logic to functions
  3. https://crrev.com/c/4040521 CompareNodesInSameParent faster
  4. https://crrev.com/c/4040842 Relative positions faster
