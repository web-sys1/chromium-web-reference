[json-schema-compiler] Errors as base::expected

This CL adds support to returning errors through the return type, using
`base::expected`, rather than as an out param. The functions affected
are `FromValue`, and `Params::Create`.

Changes in `FromValue` aim to make it possible to eventually retire the
`Populate` altogether.

 - `FromValue(base::Value)` is added to have parity with
   `Populate(base::Value)`. This means the overload for `FromValue` has
   two function, one taking a `Value`, and another taking a `Dict`.
 - Error out params are removed in favour of `base::expected` as the way
   to communicate failure, and error description.
 - These functions are now being generated for non-top-level types as
   well.

Additionally, changes were also done to `Params::Create`, with the
compiler now generating an extra function that returns errors with
`base::expected`.

We want to migrate all the callers gradually, and once that work is
done, the existing legacy versions should be removed.

The following is a sample of how the overloads of `FromValue`, for
instance, are supposed to look like when error messages are enabled.

  // Creates a TestType object from a base::Value::Dict, or unexpected on
  // failure.
  static base::expected<TestType, std::u16string> FromValue(const base::Value::Dict& value);

  // Creates a TestType object from a base::Value, or unexpected on failure.
  static base::expected<TestType, std::u16string> FromValue(const base::Value& value);

However, for type that don't use optional, `absl::optional` is still
used, and the output looks like this:

  // Creates a TestType object from a base::Value::Dict, or nullopt on failure.
  static absl::optional<TestType> FromValue(const base::Value::Dict& value);

  // Creates a TestType object from a base::Value, or nullopt on failure.
  static absl::optional<TestType> FromValue(const base::Value& value);
