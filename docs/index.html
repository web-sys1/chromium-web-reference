lacros: Implement FileSystemProvider.ForwardOperation.

This CL continues the process of implementing the FileSystemProvider
extension API for lacros. This CL should have no functional effect on ash.

This CL adds a single method ForwardOperation to the FileSystemProvider
interface. Lacros registers itself as a FileSystemProvider with ash on
startup. When ash receives an operation, it first attempts to forward to
an ash-based extension. This covers the case where Lacros is not
running. It then forwards the operation to all crosapi clients with the
assumption that one of them will handle the event. The request manager
itself handles timeouts and missing callbacks.
