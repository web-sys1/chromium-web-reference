Make waitForStatus do what it's supposed to do

The expectation for waitForStatus is to block until a certain state is
reached. Confusingly, UrlRequest.StatusListener gets called only once.
This means that waitForStatus would unblock only if we managed to
luckily call UrlRequest#getStatus at the right point in time (i.e. when
it would return the state we're waiting for).

To circumvent that, the listener registered by waitForStatus has to
re-register itself whenever it doesn't receive the expected state.
