Speculative fix for the window dragging crash

From the crash stack, it happens because the display used to decide the
snap orientation was invalid. The display was calculated from
CursorManager so there might be a bug in the CursorManager::GetDisplay()
function. But for this particular crash, we can fix it by using the
dragged window to calculate display which should be a reliable way. The
CursorManager issue can be looked at separately.
