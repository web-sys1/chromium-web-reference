Remove `mGroupByAllowBlock` in SingleWebsiteSettings.

The variable is effectively a constant and always `true`.

More precisely, it is a remnant of an age where `AllSitesSettings` was
still integrated into `SingleCategorySettings.java`. This is no longer
the case.

Today, `mGroupByAllowBlock` is initialized to `false` in the
constructor, but then is always set to `true` on activity `onCreate`.
The one-time side effect of this is that the `mBlockListExpanded`,
`mAllowListExpanded`, `mManagedListExpanded` variables are set to
specific values once.

However, these three variables are never accessed earlier than the
activity `onStart` event. Therefore it is equivalent to set these
variables to these specific values in the constructor already.
