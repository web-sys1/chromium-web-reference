[Post Message Counter] (1) Separate categorization from processing

The previous implementation was getting out of hand. This was re-written
so that (1) the core function (RecordMessageAndCheckIfShouldSend) is
easier to read and the underlying logic of it is clear and (2) it's easy
to update the logic in ShouldSendPostMessage in the stacked CL. Before,
it would have been necessary to update logic in two places, and that's a
bad standard to set. I know this implementation is a lot longer in
linear code lines, but I don't believe it's any more inefficient. No
logic changes should have occurred.

This CL is part of a series:
(1) Separate categorization from processing
(2) Split blocking features
