Add ash-lacros plumbing for OOP video decoding

This CL implements BindStableVideoDecoderFactory methods in
LacrosService and Crosapi. Lacros-chrome uses these to ask ash-chrome
for a StableVideoDecoderFactory connection.

This CL also implements IsStableVideoDecoderFactoryAvailable() in
LacrosService. We can't use the IsAvailable() method because in order
to use it we would need to call ConstructRemote(), and this automated
flow can only maintain one mojo::Remote<StableVideoDecoderFactory>.
We would have to re-use that same mojo::Remote to create all the
StableVideoDecoders. For our use case, the Lacros browser process may
need to bind multiple independent
mojo::PendingReceiver<StableVideoDecoderFactory>, one per renderer
(even if all the corresponding  mojo::Remotes are going to be used on
the same sequence). By doing that, ash-chrome can start one utility
process per StableVideoDecoderFactory.
