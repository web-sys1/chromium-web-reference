> [Dr-Dc]:Gpu Scheduler changes to support multiple threads.
>
> Dr-Dc a.k.a direct-rendering display compositor enables use of
> 2 gpu threads instead of 1 gpu main thread.
>
> This change is to update gpu scheduler to handle multiple gpu threads.
> Primary changes are :
> 1. A sequence can be created on any thread.
> 2. Each thread will have its own priority queue to schedule sequences
> created on that thread.
> 3. GpuInProcessThreadService::CreateSequence() can be called from viz
> thread so make sure its on the gpu thread.
> 4. Update all scheduler unit tests as well as all affected unit tests
> due to scheduler changes.
>
> This change should not affect any functional or performance changes
> to current/legacy features.
