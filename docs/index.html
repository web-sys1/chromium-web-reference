Original change's description:
> [aim][popup] Add page/page handler Mojo interface and visibility events.
>
> The AIM popup hosts the <cr-composebox> which is hosted in a views
> Widget. The webcontents of the widget are always present and the
> containing Widget is shown and hidden in order to display or hide the
> composebox. Due to current limitations in WebUI <cr-composebox> is not
> notified of visibility events. This CL adds a Mojom interface to inform
> the WebUI of visibility events to allow it to react properly to being
> shown or hidden on-screen.
>
> This CL also adds a `Close()` method to the PageHandler to allow for the
> WebUI code to close the popup in response to user events.
>
> https://crrev.com/c/7138389 is a WIP CL that shows how the
> omnibox_popup_aim.mojom may evolve to accommodate passing context into
> the composebox.
