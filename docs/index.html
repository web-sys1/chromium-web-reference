Reduce the number of WindowOcclusionTracker::MaybeComputeOcclusion calls

see crbug.com/1183894 , WindowOcclusionTracker::MaybeComputeOcclusion
gets called too many times for no reason (e.g. there's 36 calls coming
from ToolbarView::Init, we should only do this once this function
returns). In my local tests it causes the number of calls to
WindowOcclusionTracker::MaybeComputeOcclusion that get past the first
early abort check to drop from 69 to 23.
