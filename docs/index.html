webview(metrics): Fix host package information in SDK runtime

The context we retrieve for the SDK runtime is a generic
sdksandbox context. This results in our metrics and minidumps
reporting a generic package name that won't help
for investigating issues.

The good news is that the uid for the SDK sandbox
process does belong to the host app and there is a deterministic
relationship between the SDK process and the host app
so we can get the host apps information from it.

The one tricky bit is that this trick doesn't apply to the
render process because it is an isolated process and so
doesn't have this same relationship.
We settled on feeding this through as a command line
switch.

I unfortunately haven't added tests because we don't
have any instrumentation tests for the SDK runtime
and this behavior is wildly environment specific.
I just tested it manually with a shell app that contains
a WebView and also confirmed that the mini dumps show
correctly in the dev UI when I induce a crash with
chrome://crash. I have also tested this against staging
crashpad and confirmed that the crashes show up there.
