Change CrasAudioHandler to use the MediaPlaybackState instead of SessionState

This CL updates CrasAudioHandler::MediaSessionInfoChanged() to use the MediaPlaybackState instead of the SessionState when determining what playback state to set. The comment in media_session.mojom says that the playback_state indicates whether audio is currently playing. The audio focus state (SessionState) may still be active even though nothing is playing.
