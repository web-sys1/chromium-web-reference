Prevent browser-initiated reloads to initial empty document

We made history navigations to initial NavigationEntry impossible in
crrev.com/c/3386503 in order to preserve previous navigation behavior.
We missed the browser-reload case, which this CL covers.

With this CL, trying to do a browser-initiated reload while on the
initial NavigationEntry will not result in a navigation, except if it's
the NavigationEntry for the synchronous about:blank. This preserves the
behavior for browser-initiated reloads before the introduction of
initial NavigationEntries:
- The initial empty document has no NavigationEntry, so
  browser-initiated reloads, which checks for NavigationEntry
  existence, can't happen
- The synchronously committed about:blank has a NavigationEntry, so
  browser-initiated reloads can happen.
This CL adds a way to explicitly track and differentiated whether an
initial NavigationEntry is for the synchronous about:blank commit or
not.

Note that this CL does not affect renderer-initiated reloads, which has
always been possible even when there is no NavigationEntry, as it is
first handled in the renderer and goes through the BeginNavigation path
without ever checking NavigationEntry existence.
