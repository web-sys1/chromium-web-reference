Delete files in DriveFS rather than moving to .Trash

DriveFS changed its trash folder path since 48 in M92 from
.Trash to .Trash-1000, and also changed how .Trash works to
match the XDG trash spec.

As a result, rather than moving <item> into .Trash, users of
DriveFS should create an entry in .Trash-1000/info/<info>.trashinfo
and move <item> to .Trash-1000/files/<item>. See http://cl/363262122

When chrome://flags/#files-trash is enabled, the JS code will take
care of creating *.trashinfo files and moving files into the correct
dirs.

When trash is not enabled, there is no need (and never has been) for
FilesApp to move files to DriveFS trash folders.  It is fine to
recursively delete the items locally.  DriveFS will keep the item
in its cloud trash which can be seen in Drive web.

This change was missed in crrev.com/c/2771402.
