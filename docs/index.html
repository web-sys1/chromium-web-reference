Update Hunspell ec597e1 -> f3f90ae

https://github.com/hunspell/hunspell/commit/f3f90ae

Mostly trivial rebases, except for a change from static_cast ->
dynamic_cast. Unfortunately, Chromium does not support
dynamic_cast, so we cannot make this change.

Examining the code, the function that uses the output of
dynamic_cast assumes the pointer is not null (i.e. it is not using
dynamic_cast to check if it can cast). Therefore, the behavior
should be the same (the cast is always successful).

We also need to incorporate a future fix via a new patch since this
uprev actually introduces an intermediate regression, which is fixed
by a later commit:
https://github.com/hunspell/hunspell/commit/73d84b6
