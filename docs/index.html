Fix PartitionAllocTest.PartialPageFreelists test for ENABLE_REF_COUNT_FOR_BACKUP_REF_PTR

Last fix did not work: https://chromium-review.googlesource.com/c/chromium/src/+/2624174

When trying to allocate 0 bytes when ENABLE_REF_COUNT_FOR_BACKUP_REF_PTR is enabled,
it gets adjusted to a size of 1. which means 1+kExtraAllocSize(48)=49, and since it doesn't match with any bucket slot size precisely, the 64 slot sized bucket gets used.
So the test should not assume that very_small_size_size+kExtraAllocSize equals the slot size.
