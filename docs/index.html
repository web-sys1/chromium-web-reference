[Profile creation] Prepare re-layouting for a back button

This CL changes the way profile picker layout changes when moving
between the WebUI screens and the Gaia sign-in screen. Instead of
destroying everything and creating new WebViews, we just swap the
WebContents and swap the visibility of the native toolbar. This results
in a much smoother UX, especially when moving back and forth between the
first profile creation screen and the Gaia sign-in screen. The
performance cost for this is keeping two WebContents in memory for the
lifetime of the signed-in profile creation, which is acceptable on
desktop.

This CL does not actually create any content for the toolbar, this will
come in the next CL. The actual toolbar view is created in this CL and
its visibility is toggled in order to allow browser tests to properly
detect the layout change (the previous way does not work after this
change).
