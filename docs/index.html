personalization: Handle pending UI for disabling daily refresh

In the previous fix for pending UI, I neglected to account for calls
to `selectGooglePhotosAlbum` with an empty `albumId`, which is how we
indicate that we want to disable Daily Refresh while keeping the
currently show wallpaper permanently.

This is the only other case where `OnWallpaperChanged` is not called.
It also requires that we fetch the daily refresh state to stop
highlighting the "Change Daily" button.

Thank you dmblack@ for catching this.
