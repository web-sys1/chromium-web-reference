Fix ServiceWorkerRegistry constructor from old registry.

Properly save data necessary for future ServiceWorkerStorage::Create
calls in ServiceWorkerRegistry c-tor from old registry.
Currently, if you construct ServiceWorkerRegistry R1 from old registry
and then construct another one R2 from R1, then R2 will be broken,
because user_data_directory_, database_task_runner_ and
quota_manager_proxy_ data are lost.
Problem was introduced in
https://chromium-review.googlesource.com/c/chromium/src/+/2477141 ,
before that, data from old ServiceWorkerStorage instance was reused.
