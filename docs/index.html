Add Android virtual files support for ArcNearbyShare and WebShareTarget

share_info_file_handler handles ShareInfo mojo message for files and
create cached files for NearbyShare integration.  We are reusing
Webshare's PrepareDirectoryTask to handle deleting files after timeout,
checking to make sure user's device has enough storage available for
streaming, and creating a hidden folder for the read-only cached files.

share_info_file_stream_adapter handles streaming content data from
Android virtual filesystems.  For WebShareTarget, streamed data will be
written to a MojoDataPipe producer endpoint.  For ArcNearbyShare,
streamed data will be written to a file descriptor.  Added basic unit
tests to stream data and write to test file.

Integrated virtual files streaming into nearby_share_session_impl and
made sure to call NearbyShare session cleanup on failure cases.

mojom changes:
- Update file size to long (int64) compatible type to match Long type
returned from OpenableColumns.SIZE in Android.
- Change Custom Tab wording to ARC Window reflecting current usage.

There are follow-up CLs for unittests (http://b/191088329) and
NearbyShare file cleanup integration (http://b/191087417).
