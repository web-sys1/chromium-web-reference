Rename onUpdateUrl to onUpdateTargetUrl

The method name onUpdateUrl in WebContentsDelegateAndroid was
ambiguous. It suggested a change in the visible URL of the web contents,
but its implementation was tied to UpdateTargetURL, which is meant to
report the URL of the link under the cursor.

To improve clarity and align the name with its actual function, this
CL renames the method to onUpdateTargetUrl in both the C++ delegate
and the Java interface.

This refactoring was requested by reviewers to be split into a separate
CL from a larger feature change. This allows for a focused review of
the API modification and its impact on existing implementers.
