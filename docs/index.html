[media] Remove coded_size from WrapSharedImage in Mappable SI pool

The MappableSharedImageVideoFramePool passes coded_size when calling
VideoFrame::WrapSharedImage to wrap a pooled SharedImage. However,
this explicit coded_size parameter is unnecessary and redundant.

The MappableSharedImageVideoFramePool enforces an exact size match for
resource reuse.

Specifically:
- IsFrameResourceCompatible requires an exact size match. [1]
- Incompatible resources are dropped from the pool in
  GetOrCreateFrameResource. [2]
- Thus, the size of shared_image is guaranteed to be exactly the same
  as the coded_size requested by the current video frame.

[1] https://source.chromium.org/chromium/chromium/src/+/main:media/video/mappable_shared_image_video_frame_pool.cc;drc=ca8ffd320c5ff5c9abf7e351d9aa4e37dd600f8c;l=235
[2] https://source.chromium.org/chromium/chromium/src/+/main:media/video/mappable_shared_image_video_frame_pool.cc;l=1224-1231;drc=409b77a78792667eb4583c52aa9faf7fa321f4b8
