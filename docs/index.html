Reland: Update PreDiscardResourceUsage when discarding multiple times

This CL relands crrev.com/c/5818211, reworking it slightly to make it
easier to support both the old and new discarding implementation. The
unit test has been moved into the TabLifecycleUnitSourceTest fixture
to allow the source to migrate the TabLifecycleUnit to newly
constructed WebContents during discard as necessary.

Currently the TabLifecycleUnit is running the older discard
implementation and this should not introduce any behavior changes.

However the UpdatePreDiscardResourceUsage() utility and associated
regression test will make it easier to handle PreDiscardResourceUsage
correctly when the new discarding approach is landed side-by-side
with the current impl.
