[zip] Use DFS instead of BFS

Changed the zip tree exploration algorithm from Breadth-First
Search (BFS) to Depth-First_Search (DFS). This has several benefits.

There is no need to create a list of all the items to zip from the
start. Entries are added to the ZIP as they are discovered, reducing the
need for string copies, memory allocations, etc.

The order of items in the resulting ZIP is more natural, and is on par
with what other archivers produce.

Knowledge of which entries are files or folders is directly used to do
the right thing, reducing the number of file system accesses.
Previously, this information wasn't recorded in the BFS-produced list,
and it had to be rediscovered during zipping at the cost of extra file
system calls.

Finally, this allows to recursively zip folders presented in the list of
items to zip.
