Fix GetProfileForLaunch to not always use active_glic

Currently if there is an active_glic set GetProfileForLaunch will
always use that profile. This is a bug since active_glic is never
reset and should be thought of as "last_used_glic".

This CL changes it so that the last used glic profile is only used if
there is an open detached window. In all other cases the profile to
use should come from the most most recent browser list or pinned
profile.
