Original change's description:
> [Lacros] Enable Mirror for Lacros's Main Profile
>
> Enable Mirror (behind a disabled flag) for Lacros's "Main Profile".
> Secondary Profiles will have DICE enabled - for the time being, until we
> can decouple Spaces' Profile Management UIs from DICE.
>
> This CL also addresses some test failures that result because of it:
> 1. Since IdentityManager (a KeyedService) now relies on a PO2TSDelegate
> that needs to talk to Chrome OS Account Manager via Mojo (using
> `LacrosChromeServiceImpl`), all tests which need `TestingProfile` end up
> needing `LacrosChromeServiceImpl`. Since almost none of them initialize
> it using `ScopedLacrosChromeServiceTestHelper`, all of them fail. To get
> around this issue, this CL adds `ScopedLacrosChromeServiceTestHelper` to
> `TestingProfile` and `BrowserWithTestWindowTest` - while ensuring that
> we do not initialize it twice. This mimics Lacros in production where
> `LacrosChromeServiceImpl` is a Singleton.
> 2. `PasswordManagerInteractiveTestWithSigninInterception` is disabled
> because it only applies to Gaia signin pages - which Lacros won't have.
> 3. Some browser_tests perform account setup via identity_test_utils. For
> the ones that end up at
> signin::<anonymous namespace>::UpdateRefreshTokenForAccount, we do not
> have any way to contact Chrome OS Account Manager because unlike Ash,
> Account Manager does not live in the same process as Lacros, and Mojo
> crosapi is disabled for browser_tests - resulting in a NOTREACHED. These
> tests have been added to
> testing/buildbot/filters/lacros.browser_tests.filter while we / EngProd
> figure out a way to do this for Lacros.
>
