[Extensions] Don't merge fetches into active request

Merging fetches to the active request requires active request to be
available (which blocks other changes) and makes unclear when merging is
allowed and when not.

This CL saves old behavior by merging requests from the queue when we
finished fetching one. Two tests are added to ensure that there was no
behavior change.

The TestMultipleManifestDownloading test had to be updated as it relied
on order stability of the fetches, and the heap used in RequestQueue<>
doesn't guarantee it.
