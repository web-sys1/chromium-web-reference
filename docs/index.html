Bypass thread check in SchemeRegistry for test-only methods.

Currently we have an issue that WTF::IsBeforeThreadCreated() always
returns true regardless of whether a child thread is already created
or not. It turns out fixing it would cause several other issues, too,
so they must be cleared first.

One of them is thread check in SchemeRegistry. It uses IsBeforeThread-
Created() to force modification of a variable to happen only before
a child thread is created. However, TestRunner may mutate the said
variable during and between tests, and that violates the expectation
and would cause an assertion failure when IsBeforeThreadCreated() is
fixed.

This CL handles this issue by marking such accesses as test-only and
allowing them to bypass the thread check, because mutations made by
TestRunner are expected and should be considered safe, even though they
may happen after a child thread is created.
