[rebaseline-cl] Fix reftest detection

Currently, `rebaseline-cl` detects if a test is a reftest by looking for
`reference_file*` artifacts. This was inadvertently broken in
crrev.com/c/3990017, which filtered out non-`actual_*` artifacts to
minimize records requested from ResultDB.

This change fixes this by using local reference files (e.g.,
`*-expected.html`) as the source of truth for reftest detection instead.
