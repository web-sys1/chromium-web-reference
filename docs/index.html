Properly invalidate for lang attribute changes

Fixes two issues:

1. Changing the lang attribute of an element may change the :lang()
   matching for any element in the subtree. There is no targeted
   invalidation for this, but in the normal case where only the lang
   attribute is used this works because the inherited -webkit-locale
   will be propagated down the style tree through inheritance and make
   sure all elements are updated.

   That is however not the case if there is an author use of the non-
   standard -webkit-locale property that does not match what the lang
   attribute is set to.

   The fix is to use a kSubtreeStyleChange to force all descendants to
   be updated even if the inheritance propagation is stopped.

2. xml_names::kLangAttr does not always match with == operator because
   QualifiedName operator== compares instances and not nsURI +
   localName. QualifiedName::Matches() has to be used to catch all
   cases.
