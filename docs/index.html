Reland: [PA] Add BUILDFLAG() guard to allocator_shim header and source code"

This CL is a reland of crrev.com/c/5009212. The reason why the CL was
reverted is that
allocator_shim_default_dispatch_to_linker_wrapped_symbols.cc was not
guarded by BUILDFLAG(USE_ALLOCATOR_SHIM). To avoid such a mistake,
add shim_headers and shim_sources to source_set("allocator_shim") and
apply the following way to initialize its sources:

   shim_headers = [ "*.h" ]
   shim_sources = [ "*.cc", "*.mm" ]
   sources = shim_headers
   if (use_allocator_shim) sources += shim_sources

This crrev.com/c/5020203 also fixes Android's all_dependent_configs,
i.e. wrap_malloc_symbols, and Apple's configs, i.e.
mac_no_default_new_delete_symbols and enable_arc. The configs needs
"if (use_allocator_shim)".

PS1 = the same as crrev.com/c/5009212.
