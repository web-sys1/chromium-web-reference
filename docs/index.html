[InputVizard] Update emission of kAndroidOSTransferredANewSequence
reason

Android platform sometimes sends ACTION_DOWN/ACTION_UP instead of
POINTER_DOWN/POINTER_UP due to a known platform issue. If we receive an
ACTION_UP while a touch sequence is still potentially active on Viz, we
should not assume that the Android OS transferred a new sequence
unexpectedly.

This change:
1. Adds a check for IsTouchSequencePotentiallyActiveOnViz() before
   emitting the kAndroidOSTransferredANewSequence reason.
2. Introduces a new enumeration histogram
   'Android.InputOnViz.Browser.NewSequenceTransferredByOS' using
   TransferredSequenceType (ActionDown, PointerDown). It is logged
   whenever an ACTION_UP is received while in the
   kConsumeEventsUntilCancel state. This helps quantify how often
   the OS transfers a different sequence than requested,
   distinguishing between new action sequences and pointer sequences.
3. Adds LINT markers to the C++ and XML enum definitions to ensure
   they remain synchronized.
