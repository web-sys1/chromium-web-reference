[Extensions] Create multi service worker per extension test.

Creates a currently passing (but should be failing) test that simulates
the issue in the linked bug.

It does this by intercepting the
`ServiceWorkerHost::DidStopServiceWorkerContext()` mojom receiver
method. This simulates that the extension renderer service worker thread
never finished terminating and informing the //extensions browser of the
worker stopping.

It is not a perfect simulation since the worker thread does actually
stop during the test, but for the //extension browser layer purposes it
seems sufficient and non-flaky.

In order to do this testing, it required adding some testing overrides
in `ServiceWorkerTaskQueue` to allow for multiple workers to be tracked
so that we can examine them in the test without crashing.

After this change, we'll be able to validate that our upcoming fix
addresses the issue encountered in the bug.

contains trivial testing related changes.
