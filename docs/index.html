Create a crash key for each feature in enable-features/disable-features

The "--enable-features" and "--disable-features" command line switches
are used heavily on ChromeOS, as a way of interacting with session
manager, and are generally so long that they are truncated by the
character limit on switches.

This CL removes the `commandline-enabled-features` and
`commandline-disabled-features` (as they could get truncated), and
instead moves each feature in its own dedicated CrashKey, named
"commandline-enabled-feature-<X>" and "commandline-disabled-feature-<X>"
(with <X> an incrementing counter starting at 1).

Note that this is on ChromeOS only.

An example crash report can be found at
http://crash-staging/browse?stbtiq=e15801b4044e9bf5#2,
and a screenshot of the crash keys can be found at
http://screen/9S8x9dq2s7RVnbX.png
