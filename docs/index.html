sharing: add share ranking skeleton

This change adds the required backend database (using LevelDb for the
underlying storage) for storing share rankings per type, and adds
functions (currently just stubs) for computing share rankings. It also
adds some tests, currently disabled, that sketch out some of the
expected behaviors of the ranking algorithm once it's implemented.

A followup CL will implement the actual ranking algorithm and add more
substantial test coverage, and a second followup CL will introduce code
on the Java side to fetch the ranking from ShareRanking, thus completing
this feature.
