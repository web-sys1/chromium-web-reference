Fix _RunValidationScript for Windows command limits

_RunValidationScript executes a script on all applicable files. When
running "git cl presubmit --all" that ends up being ~500 files and
the command line that is generated is over 43,00 characters when called
from _CheckWprShaFiles. Windows does not like that.

This change breaks up the check into multiple invocations of the command
in order to avoid these limits. This is important because running all
presubmits is the only way to systematically find bugs, and avoid being
surprised by them when submitting a change.

See crrev.com/c/3554633 for another example of fixing this issue.

These issues were found while running "git cl presubmit --all" to look
for hidden presubmit errors. These changes halt the presubmit run and
therefore hide all subsequent errors.
