[Launcher pop-in] Pin Search and Assistant to bottom initially.

Existing behavior:

Pin the Search and Assistant category to the bottom of the categories
list all the time, irrespective of timing considerations. See
crrev.com/c/3469543.

New behavior:

Pin the Search and Assistant category to the bottom of the categories
list at its time of arrival. For categories which arrive after Search
and Assistant, place these categories below Search and Assistant.

Justification:

We want to uphold the spirit of the existing behavior, while mitigating
some pop-in effects.

The existing behavior was implemented because S&A results can be easily
triggered and fairly non-specific. However, within the existing
behavior, in the following case we see a large pop-in effect with a
sometimes high degree of vertical displacement:

1) S&A arrives pre-burn-in.
2) Burn-in period elapses. S&A is pinned to the bottom.
3) A new category arrives post-burn-in. It pops-in second-from-bottom
   (i.e. above S&A, but below everything else).

(See video in bug for an example)
