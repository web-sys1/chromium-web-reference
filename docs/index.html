[editing] Don't use disconnected node in RemoveInlineStyle

ApplyStyleCommand::RemoveInlineStyle has this code:

  // If pushDownInlineStyleAroundNode has pruned start.anchorNode() or
  // end.anchorNode(), use pushDownStart or pushDownEnd instead, which
  // pushDownInlineStyleAroundNode won't prune.
  Position s = start.IsNull() || start.IsOrphan() ? push_down_start
                                                  : start;

But then it ran:

  Node* node = start.AnchorNode();

That could end up triggering a DCHECK failure. Instead, just fix the
|start| and |end| positions, and initialize |s| and |e| to them.

Bug: 1199902
