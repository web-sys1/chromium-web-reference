[Extensions] Remove bookmarkManagerPrivate permission

The bookmarkManagerPrivate permission was used by allowlisted extensions
that used the API (such as the old bookmark manager component
extension). These have since all been migrated to webui, and
https://chromium-review.googlesource.com/c/chromium/src/+/1566269
migrated tests to webui and removed the permission feature (since no
extensions then used it). However, that CL missed removing the
APIPermission entry and cleaning up the _api_features.json entries.
This also results in triggering a DCHECK when encountering an API
permission with no corresponding permission entry.

Clean these up by removing the API permission entry, marking the IDs
as deleted, and remove the _api_features.json extension entry.
