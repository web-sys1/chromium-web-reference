ozone/drm: Disable all CRTCs/connectors for tiled controller

Currently, ScreenManager::RemoveDisplayControllers() removes a
HardwareDisplayController from its mapping when a single CRTC is
removed from a HDC. In the tiled display case when multiple
CRTC/Connector pairs  may exist in a HDC, some enabled pairings may not
be disabled due to the HDC being removed so early.

This CL ensures that all CRTCs/controllers in a tiled HDC are part of
the disable commit request. Removing a tiled HDC like this is OK (and
even preferable) vs waiting for all CRTC/connectors to be removed, as
DrmGpuDisplayManager is going to create a new DrmDisplay and HDC if
there's a remaining tiled connector anyways; not erasing it just causes
confusion and the screen to freeze in occasion due to mismatched
controller mapping.
