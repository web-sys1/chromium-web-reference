Add & populate a PageEndReason metric to HistoryNavigation events for BFCache.

This page end reason is populated either when a page re-enters the
back-forward cache, or when the back forward cache observer's OnComplete
method is called.

As part of this, page_load_tracker now clears its stored page_end_reason back
to END_NONE when a page is restored - page_load_tracker won't update the
page_end_reason otherwise, and it makes sense that the old reason isn't
relevant anymore.

This patch also changes the 'ExpectMetricValueForUrl' helper in the test to
check the latest value for a given metric instead of the first, and also to
fail if no metric is found.
