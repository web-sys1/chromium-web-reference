Filter DelegatedInk metadatas with same timestamp

In order to prevent unnecessary work in the GPU process and extra calls
to the windows StartNewTrail method, filter out delegated ink metadatas
in DelegatedInkTrailPresenter::updateInkTrailStartPoint if the
timestamp of the event is the same as its predecessor.

Duplicate timestamps can be the result of the web app calling updateInkTrailStartPoint repeatedly with the same event. This results
in the same metadata being sent to the DelegatedInkPointRenderer, which
causes a drop in points being rendered until a new metadata is sent.
