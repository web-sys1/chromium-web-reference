Relax DCHECK constraint for prerender activation in background.

When a prerendered page is activated in a background tab (e.g.
ctrl+click), PageHidden() is never called because there was no
visible-to-hidden transition. In that case first_background_time_ won't
be set. Metrics observers already handle this via
GetNonPrerenderingBackgroundStartTiming() which uses activation_start
for background-activated prerenders, so allow DCHECK pass if prerender
activation in background.
