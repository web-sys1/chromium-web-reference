webhid: Add policies for managing the guard setting

This CL adds enterprise policies to enable or disable the WebHID
API for all origins or for a specific set of origins.

Three new policies are added. The DefaultWebHidGuardSetting policy
sets the default guard setting for all origins and can be configured for
Ask or Block, overriding the user settings. WebHidAskForUrls and
WebHidBlockedForUrls configure the guard setting on specific origins and
override the default guard setting.
