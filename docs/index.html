Original change's description:
> [crd host][wayland] Introduce PipewireCaptureStreamManager
>
> Before this CL, we create a single pipewire stream when creating the
> desktop environment, then use the same stream for all
> CreateVideoCapturer calls, ignoring the screen ID parameter. This won't
> work for multi-stream/multi-window.
>
> This CL introduces a PipewireCaptureStreamManager class for managing
> pipewire screencast streams, and moves stream creation logic from
> GnomeInteractionStrategy into it. It is also responsible for inferring
> the screen ID for the newly added streams. Since the ScreenCast D-BUS
> API does not give us the connector name of the new virtual monitor, we
> have to guess it by watching for changes in the display config, and
> assuming the newly seen monitors are associated with the streams that
> are being created.
>
