ozone: webpagepopups: calculate anchor for menu bounds. 4/*

Some system compositors as Wayland do not share global client
geometry and it's impossible to know if newly created child
windows (such as popups in this particular cases) are going to fit
display nicely.

Thus, there is additional information required, which is anchor
position, anchor gravity and others.

Thus, make WebPagePopup calculate anchor for new bounds and share
that data.
