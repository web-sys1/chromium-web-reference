STTS: Avoid maintaining pending entries and open all unopened entries

Currently, the Send Tab to Self (STTS) Auto-Open feature stores pending
entries in an temporary in-memory vector within the controller. This
prevents them from persisting across browser restarts.

This CL refactors the feature to use the `SendTabToSelfModel` as the
source of truth. Unopened entries targeted to the local device are now
retrieved from the model on startup or browser activation, enabling them
to persist and auto-open.

Additionally, this also updates GetUnopenedEntriesTargetedToLocalDevice
to return entries in a sorted order of shared time (oldest first).
