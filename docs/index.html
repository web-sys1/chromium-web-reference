Dispatch new events for Move operation

In CL:2852945, 2 new progress types have been added for the move
operation, kEndMove and kEndRemoveSource.

This change implements the usage of these progress types:
- kEndMove is sent upon completion of a local move (rename/update of
entry metadata).
- both kEndCopy and kEndRemoveSource are sent for a "copy + delete"
move.

The CopyOrMoveOperationDelegate defines 3 subclasses that implements the
Copy and the Move operations.
Before this change these subclasses were responsible for sending
kProgress events. kEndCopy or kError were sent by the delegate itself
upon completion of the copy.
The 3 subclasses are now responsible for sending all these progress
events, since multiple success events now potentially need to be
reported (kEndCopy + kEndRemoveSource for a
cross-filesystem move).
