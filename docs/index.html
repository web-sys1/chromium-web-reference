Migrate some #getColor() calls to go through Context.

These are the trivial changes that are simply replacing a call from a
Resources to a Context. No changes in member variables or params needed.

As part of Dynamic Colors work, more and more colors are routing through
?attrs. Often this is transitive through a @macro or color state list
We have seen several bugs where this resolution fails, because the
?attr information is not in the Resources, but instead through the
Context/Theme. Now that we only support api_level 23+,
Context#getColor() will always work.
