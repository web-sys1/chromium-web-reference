Remove out/.../enhanced_network_tts/ from gn data

mb.py enforces that BUILD.gn "data" sections should not point to
subdirectories of the output directory [1].

This BUILD.gn file contains one such reference, which blocks the use of
mb.py when building browser_tests for ChromeOS [2]. Specifically, mb.py
shows the following error [3]:

    error: gn `data` items may not list generated directories; list files in directory instead for:
    //out/bug/resources/chromeos/accessibility/enhanced_network_tts/

The recommended fix is to either list specific files that are needed
(rather than an entire directory), or to simply remove the dependency if
no longer needed.

In this case, files in the directory seems to be depended on by
enhanced_network_tts_e2e_test_base.js, so this change makes BUILD.gn
refer to the specific files rather than the overall directory.
