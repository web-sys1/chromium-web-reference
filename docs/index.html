chrome/linux: Fix icon in wayland with user data dir set

`views::Widget::InitParams.wayland_app_id` for
DesktopBrowserFrameAuraLinux was being set to the same value as
`wm_class_name` which includes the " (<usr data dir>)" suffix if
--user-data-dir is specified.
So use `wm_class_class` instead which should be used here to match with
the expected .desktop file same as x11.
