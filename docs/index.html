Fix a failing composite bgcolor animation layout test

This CL fixes a failing layout test on composite background color
animation.

The problem is in the FillRectWithPaintWorklet, when we
specify the src_rect, it should be in the Node's coordinate
system. For example, if we have a simple
.div {
  width: 200px;
  height: 200px;
  background-color: green
}
Then the dest_rect would be (8, 8, 200, 200) because the html
has a margin of 8 pixels. The src_rect should just be
(0, 0, 200, 200) which is in the Node's coordinate system.
