[omnibox] Fix overlapping label and icon in SlideAndCrossfade

When one tab has finished the animation and the trailing icon is
showing, when a user switches tabs, they see a broken overlapped state.

http://screenshot/5ySyzw4SH4vMAE7

For page actions using the SlideAndCrossfade animation strategy, this
reset updates the layer opacities (making the leading icon visible) but
did not trigger a layout update. This caused the label to remain at its
animation end position, overlapping with the newly visible leading icon.

This CL fixes the issue by explicitly calling PreferredSizeChanged() in
SlideAndCrossfadeIconLabelBubbleAnimationLayoutStrategy::ResetAnimation,
forcing a layout update when the animation is reset.

http://screencast/cast/NDYxOTY5ODUxNDY5MDA0OHxjMGFhZjY5Zi1kMw
