[omnibox] [shortcuts] Aggregate shortcuts by destination URL.

Currently, when multiple shortcuts match the user input and share the
same stripped destination URL, they are scored independently, the
highest scored shortcut is suggested (assuming provider limit capacity),
and the rest discarded.

This CL adds a feature & flag to group input-matching shortcuts by their
stripped destination URL, then score them in aggregate.

E.g., given the input 'wi', and 3 matching shortcuts: 1) 'wilson',
wikipedia.org/wilson,  2 visits, last visit 1 day ago 2) 'wiki',
wikipedia.org/wilson,  5 visits, last visit 10 days ago 3) 'wins',
wikipedia.org/winston, 7 visits, last visit 1 day ago

With the feature, the provider would score the 1st 2 shortcuts as 1
shortcut (with typed fraction `2/min(6,4)`, visit count `2+5`, and last
visit `min(1,10)`, and the resulting match would have the same score as
the match for the 3rd shortcut which has the same scoring factors .5, 7,
and 1.
