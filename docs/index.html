ChromeVox: fixes braille chorded text nav commands in content editables

The BrailleCommandHandler, when receiving an edit command, returns early
if the current node isn't editable or isn't the current editable tracked
by TextEditHandler.

However, in content editables, ChromeVox's current range is mostly on
descendants of the editable itself.

This caused the editable command handler to never get executed as
expected.

In addition, when performing a routing command on an editable, inject a
character move intent so that we treat the subsequent editing event as
character speech.
