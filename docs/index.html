[context-input] Make Omnibox AIM button use QueryContextualizer

Previously, the entrypoint was not capable of contextualization as there
were no contextual use cases for it. However, now the entry point needs
to be capable of URL-parsing made possible through the
QueryContextualizer (e.g., when users enter query text and then press
the AIM button).

Adds a QueryContextualizer delegate implementation in omnibox_edit_model
and sets up the url-opening callback to pass the contextual session
handle to the resulting web contents (if needed). The web contents will
only be created if the query_contextualizer needed it for parsed-url
uploading.

Also, fixes some issues with the object and session lifecycle of the
upload tracker and created session handles in the query_contextualizer
that were surfaced by this new entrypoint.
