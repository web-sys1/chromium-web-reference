Fix to show favicons for tabs in a saved tab groups in app menu

Favicons are now reliably shown for tabs in saved tab groups from the
app menu. Caching favicons in-memory isn't scalable if we have a lot of
saved tab groups even those that aren't loaded. Instead we load it
on-demand from the favicon service when a user is using the re-call UI.
Due to this removed the code to cache favicons from the web contents
listener since it results in only caching favicons for specific
scenarios. Note this doesn't address the issue from context menu in the
bookmark bar. That fix will happen in a follow-up CL.
