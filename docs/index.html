[Autofill] Plumb keydown events on editable elements to Autofill

Before this CL, keydown events were sent from Blink via
WebAutofillClient to AutofillAgent only for text-type <input>.

This CL extends this to also include <textarea>, and contenteditables.

The call paths are
   TextFieldInputType::DefaultEventHandler() and
   HTMLTextAreaElement::DefaultEventHandler() and
   HTMLElement::DefaultEventHandler()
-> ChromeClient::HandleKeyboardEventOnEditableElement()
-> ChromeClientImpl::HandleKeyboardEventOnEditableElement()
-> WebAutofillClient::DidReceiveKeyDown()
-> AutofillAgent::DidReceiveKeyDown()

The followup CL crrev.com/c/7984014 builds on that to observe the
keyboard shortcuts (e.g., Ctrl+Space) for AtMemory.

The reasons for implementing the shortcut handling in the renderer
are in go/atmemory-shortcut.
