webnn: Migrate gemm validation tests to WPTs

This CL adds WPT tests for gemm and removes the unit tests
`MLGraphBuilderTest.GemmTest` and `MLGraphTestMojo.GemmTest`.

Reason for creating this gemm CL: the previous CL has a failure of
creating reland because of a merge conflict （previous
CL：https://chromium-review.googlesource.com/c/chromium/src/+/5384098）

This CL also solves the WPT failure which occurred in the previous CL by
 removing 'Throw if the byte length of the output operand is too large.'
test case. In original C++ test case the input operand had a dimension
of [kSquareRootOfSizeMax,2], which varied depending on the system's
architecture (32-bit or 64-bit). However, when migrating to JavaScript,
the dimension of the input operand was changed to [kMaxUnsignedLong,2],
which used the fixed constant kMaxUnsignedLong (2^32 - 1). This constant
could potentially cause overflow errors on 32-bit systems.
