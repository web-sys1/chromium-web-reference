Regularize DecoderAdapter and DecoderStreamAdapter profile support

Rather than doing special work for DecoderStreamAdapter to determine
if a video config is supported, just do exactly what DecoderAdapter
would.  While DecoderStream should ask the decoder factory instead
of asking the GPU factories, it is expected that the DecoderFactory
doesn't actually add anything, especially if Chrome SW decoders are
turned off for RTC.

To be safe, enforce it.

Also, make DecoderStreamAdapter early-return against whatever the
gpu factory says is an unsupported config, rather than hopping
through init.  This might have some unexpected consequences, if
a client believes that it has hw support because of it, only to
have the hw decoder fall back to software asynchronously.
