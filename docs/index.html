[iOS]Remove calling #applicationDidBecomeActive when exiting safemode

There is a bug where calling #applicationDidBecomeActive when getting out
of safemode triggers a nil reference error, see crbug.com/1204496.

This is only for iOS 12.

Why calling #applicationDidBecomeActive when exiting safemode isn't
required? See crrev.com/c/2857816/3/ios/chrome/app/application_delegate/app_state.mm#b387
where we analyze the code and its usefulness.
  > Handling intents isn't needed for safemode.
  > Open a new tab if needed is already handled by the scene controller.
  > Opening the helper bubble is already handle on
    BrowserViewController#viewDidAppear.
  > Logging the startup duration metric isn't needed out of safemode. In
    fact, it should not happen.
  > This is only for iOS 12 + Post SafeMode, which is really an edgy case.
