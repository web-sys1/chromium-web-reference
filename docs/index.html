[CrOS SLP] Fix issue which shows enterprise icon by mistake

Previously, CrosNetworkConfig would default global network
configuration boolean values to false in GetBoolean(), if the key
did not exist in the global network configuration dictionary fetched
from ManagedNetworkConfigurationHandler.

If a ManagedNetworkConfigurationHandler boolean Allow*() value had a
default value of true, CrosNetworkConfig would default the value to
false nonetheless.

Defaulting to false caused an enterprise icon to show by mistake next
to the Lock SIM setting in Settings for non-managed devices.

The |allow_cellular_sim_lock| setting has a default value of true.

However, on non-managed devices, global network
configuration keys do not exist in the dictionary fetched from
ManagedNetworkConfigurationHandler (confirmed manually). As a result,
|allow_cellular_sim_lock| value was defaulted to false in
CrosNetworkConfig, along with the other boolean global network
configuration settings, and propagated to the Settings UI. The issue
with |allow_cellular_sim_lock| is that it is the only global network
configuration setting with a default of true declared in
cros_network_config.mojom, whereas the other settings had a default
of false.

Instead, the values for |allow_cellular_sim_lock| and the other
boolean global network configuration settings should default to the
intended values declared at cros_network_config.mojom [1].

[1] https://source.chromium.org/chromium/chromium/src/+/main:chromeos/services/network_config/public/mojom/cros_network_config.mojom;l=941-960?q=file:cros_network_config.mojom%20
