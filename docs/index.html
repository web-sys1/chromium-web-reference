[iOS] Fix FakeChromeIdentityService in EGTests

This patch fixes 2 issues:
 1- Send OnIdentityListChanged() when FakeChromeIdentityService is
injected
 2- Don't replace FakeChromeIdentityService by another one

1-
In EGTests, to make sure any UI loaded before the service is swapped,
OnIdentityListChanged() needs to be called when the fake service is set.
The UI loaded before the fake service is set, needs to be reloaded.
To fix it, OnIdentityListChanged(false) is added at the end of:
void ChromeAccountManagerService::OnChromeIdentityServiceDidChange().

2-
When test_switches::kSignInAtStartup is used to load identities for the
FRE, a FakeChromeIdentityService is injected.
The fake service is replaced by a new one when +[ChromeTestCase
enableMockAuthentication] is called in the setup.
All the added identities in the first fake service are lost.
