Reland "Introduce WebContentsBasedCanceller for dealing with navigation races."

This is a reland of commit a83458f63a6af178b5870d71a12f1c856f9e54fe

This fixes the BFCache test case by moving it into
FileSystemChooserBackForwardCacheBrowserTest which forces BFCache on.

PS1 of this is an unchanged reland of the original. Diff against that to
see what I have fixed.

Original change's description:
> Introduce WebContentsBasedCanceller for dealing with navigation races.
>
> This class captures the details of avoiding races with the dialog-owning
> RenderFrameHosts's active state and the containing WebContents'
> visibility.
>
> Apply this to FileSystemChooser. This fixes the bug whereby the chooser
> remains on-screen even after the document enters BFCache.
>
> This also introduces an `IsActive` check in `ShowFilePickerOnUIThread`
> to avoid triggering the `DCHECK(outermost_rfh->IsInPrimaryMainFrame())`.
> This is redundant since the AutoCancel will also check this but I don't
> think we should refactor to avoid this redundancy as it would require
> creating the AutoCancel very early and far from where it is used.
>
> This includes a hack of navigating away at the end to prevent the new
> test, DoNotDismissOnSubframeNavigation, from triggering leak detection.
> https://crrev.com/c/7122881 (which does nothing but add that test)
> demonstrates that this is a pre-existing condition. I believe the leak
> is not real and is just due to the fast-shutdown not triggering
> destruction of the main frame without changing state. The follow-on
> https://crrev.com/c/7106498 reverts the hack.
>
