[Test Automation] Method to retrieve screen bounds of a WebUI element

This CL adds test-only utility methods for retrieving the screen bounds of a WebUI element. These bounds can be used to send drag/drop, click, etc. events during an interactive UI test in order to test the behavior of a WebUI page in a live browser with real input.

Tests are provided, including tests on a live existing WebUI page, in which a button deep in the shadow DOM is located and clicked on, and the UI change resulting from the click is observed.

Note that the test which clicks a WebUI element after locating it is disabled on Wayland because all tests which inject mouse moves seem to fail on Wayland (see other disabled tests in the .filter file).
