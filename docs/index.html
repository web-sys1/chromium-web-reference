Web Share: Store file sizes in Intent

We now populate the Intent with the sizes and mime types
for the shared files from Web Share API.

Note that with files shared from Arc, the Intent already has
the size and mime type for each shared file.

Web Share Target bug fix:
For each shared file's MIME type, the first matching bucket
in the share target manifest is used. Previously, we used
the first matching bucket for the overall Intent MIME type.
https://w3c.github.io/web-share-target/level-2/#launching-the-web-share-target
