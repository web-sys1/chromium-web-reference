wayland: ack and suggest bounds upon 0x0 configure event

Apply the same logic as the one implemented for toplevel shell
surfaces, where ozone/wayland suggests an initial size (i.e:
GetBoundsInPixels()) to the Wayland compositor and sends ack_configure
when handling the first configure sequence received. Also, updates
WaylandBufferManagerTest.TestCommitBufferConditionsAckConfigured to
properly check it, otherwise it would not ack/set_geometry correctly.
This issue was found out while investigating and validating the fix for
crbug.com/1313023 [1].
