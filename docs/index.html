Fix regression in holding space drop target state.

When a user drags a file near the holding space tray in the shelf, it
displays a pin icon to let the user know that it is a drop target.
Recently the pin icon regressed such that it is no longer visible to
the user. This is a result of attempting to set the icon color before
the widget is attached and color provider information is available.
