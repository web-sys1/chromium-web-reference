[Extensions] Fix use-after-free in RequestContentScript

Now that ExtensionUserScriptLoader is destroyed when its backing
extension is unloaded, classes which store the loader's backing pointer
or try to access it in OnExtensionUnloaded can face a use-after-free, as
we shouldn't assume the order which OnExtensionUnloaded calls are made.

This CL attempts to fix this by having the RequestContentScript action
listen to its associated ExtensionUserScriptLoader, and when the action
is about to be destroyed, it calls the loader to remove its script only
if the loader still exists.
