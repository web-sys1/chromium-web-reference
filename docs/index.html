[views-ax] Expose StyledLabel as a paragraph when it hosts child views

While testing ViewsAX, we observed that the links within StyledLabels were no longer exposed to assistive technologies. This was caused by StyledLabel having a role of kStaticText, which is implied to be a leaf node, while still having children views.

This CL fixes it by instead using a role of paragraph when it has text ranges that are not just text (if it's just text, exposing the StyledLabel as a leaf text is fine because the strings get passed up to the parent). We kept the previous behavior of using a kTitleBar for dialog titles.

While working on this, we discovered this issue was already known and fixed only for the isolated_web_app_installer_view_impl.cc. Our new fix here fixes it for all StyledLabels instead, so I went ahead and resolved that bug.
