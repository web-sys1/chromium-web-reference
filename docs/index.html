base: Fix pretty printed empty dict by JSONWriter

Currently pretty printed empty dict will contain an extra line, which
looks odd, e.g.
```
   "background_contents": {
      "registered": {

      }
   },
```

With this CL, the extra line for empty dicts are removed. Non-empty
dicts are not affected:
```
   "background_contents": {
      "registered": {
      }
   },
```
