Fix WebGPU/Compat prototype.

The GL texture in SharedImageBackingD3D is now created lazily, so do so
on the Compat path by calling ProduceGLTexturePassthrough() and passing
the result to SharedImageRepresentationDawnEGLImage for ownership.
(This is similar to how SharedImageRepresentationSkiaGL works.)

This also allows SharedImageRepresentationDawnEGLImage to extract the
EGLImage from it in a platform-independent manner, once
GLImage::GetEGLImage() had been made virtual.

Finally, the EGLImage path in Dawn does not like having the chained
texture descriptor that the D3D12 path prefers, so move the Compat call
above the creation of said chain.
