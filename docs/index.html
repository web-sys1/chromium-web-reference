Better pre-match for empty :is/:where

The argument to :is/where is parsed as a <forgiving-selector-list> [1],
which can produce empty :is()/:where() pseudos that can never match
anything. Detect this situation in CollectFeaturesFromSelector, and
return kSelectorNeverMatches if appropriate.

[1] https://drafts.csswg.org/selectors/#typedef-forgiving-selector-list
