Invalidation highlights with first-line and viewport units

HighlightRegistry handles invalidation for custom highlights for most
scenarios. However, ::first-line has additional invalidation paths that
weren't accounting for highlights. In particular, ink overflow wasn't
being invalidated for first-line and the style diffs were missing the
highlight changes. This change adds additional invalidation logic for
these scenarios.

The existing WPT was only checking ::first-line, but we needed
additional logic to fix ::first-letter too, so an additional WPT was
added for ::first-letter.
