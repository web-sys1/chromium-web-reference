Improve CompositorFrame validation

Expand on the validation of CompositorFrames in SurfaceAggregator to
check render pass structure is valid. In particular, check that render
passes have unique ids and that CompositorRenderPassDrawQuads refer to a
render pass that is drawn before the quad (which is required). This also
moves the validation into ResolvedFrameData to make it easier to test.

One behaviour change here is that CompositorFrames containing a render
pass cycle are now rejected as invalid outright. Before the
CompositorRenderPassDrawQuad that embedded a later render pass would
just fail to produce anything since the render pass wouldn't be ready
yet. There shouldn't be any code producing render pass cycles so it
should be safe to reject them outright.
