[User Education] Allow promos to exempt from some UE2.5 preconditions

In some cases, we should be able to show promos even when they do not
pass certain preconditions. Examples might be:
 - A promo that does not attach to a hidable toolbar item (or to the
   overflow button) - this can show even when the toolbar collapses.
 - A promo that shows specifically when the user is entering data into
   a form - this can show even when the user has recently typed.
 - A promo that will never overlap the omnibox and/or shows when the
   omnibox is open - this can show even when the omnibox is closed.
 - A promo that shows attached to a new background window or e.g. a
   tab modal dialog that does not have focus (extrmely rare) -
   this can show even when the target window is not focused.

Now exceptions can be added on a per-promo, per-precondition basis.
This requires allowlisting, and can only be used to bypass a handful
of browser-specific preconditions.

Tests have been provided to prove this works for three of the more
common and easy-to-test cases.
