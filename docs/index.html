Disable DRI3 under rr

Unfortunately, `rr` cannot directly emulate the GPU. Toward this
end, `rr` has a number of hacks which attempt to force rendering down
the software path [0]. However, `CreateX11GbmDevice` defeats these
attempts by directly obtaining a file descriptor from X. As suggested by
`rr` [1] the solution when running under `rr` is to avoid obtaining this
direct connection the GPU and logging an error. This change allows `rr`
to be used with Chromium again.

[0] https://github.com/rr-debugger/rr/blob/4026d8d8ab57dcdb858ccddefd86e56ae7580022/src/RecordSession.cc#L2303
[1] https://github.com/rr-debugger/rr/issues/3136#issuecomment-1123926573
