[Categorical search] Disable omnibox zero-state queries

Omnibox results aren't shown in the productivity launcher's zero-state,
so we can disable omnibox zero-state queries entirely. This also fixes
a bug:

- The user opens the launcher and an omnibox zero-state search begins
- Very quickly afterwards, the user types a query
- The omnibox updates with content from the zero-state search, which
  is then incorrectly displayed in the launcher.

This is due in part to the cancellation logic within the autocomplete
controller, but we can avoid the issue by simply disabling zero-state
entirely.
