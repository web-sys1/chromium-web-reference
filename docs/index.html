[Masonry] Treat repeat(auto-fill, auto) as none in Grid

Since the CSSWG is still working out if repeat(auto-fill, auto) is a
valuable track size, focus the implementation on Masonry by treating
such track definitions as "none" for Grid.

This involved returning zero tracks in CalculateAutomaticRepetitions(),
return "none" as the computed value for Grid, and keep track of
whether a track was auto sized and a track list has an auto repeat
with an auto track size to make these adjustments more seamless.

A follow up change will be made to pass in the correct size for auto
tracks to CalculateAutomaticRepetitions() in the case of Masonry.

TODOs have been added to update Grid logic if needed depending on how
the CSSWG decides to resolve on this syntax.
