Fix :has() invalidation bug on element insertion and removal

There was a bug on scheduling :has() invalidation for element insertion
or removal when we use a direct adjacent combinator ('+') inside :has()
due to the logic of checking identifiers stored in RuleFeatureSet.

To fix the bug, StyleEngine always schedules :has() invalidation for
element insertion or removal when the parent of the inserted or removed
element has the 'ChildrenAffectedByDirectAdjacentRules' flag set.
