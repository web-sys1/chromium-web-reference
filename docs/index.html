Tighten up StyleEnvironmentVariables public methods

Having the current APIs use strings makes the proper usage less than
clean. Move the public methods to take a UADefinedVariable for setting
and removing variables to enforce the known values. Resolving still
takes a string, since it we need to try and get values from potentially
arbitrary identifiers when these resolve in the cascade.

This is in preparation for adding indices/dimensions for environment
variables per latest css-env-1 Editor's Draft:

https://drafts.csswg.org/css-env-1/

For those new types of variables, we'll have a new enum for the defined
set of variables, and methods will take the new enum value along with
the appropriate indices. Removing the string-based methods will enforce
a strongly typed interface.
