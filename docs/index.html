[iOS][EG] Fix LanguageSettingsTestCase.* failures

This change fixes failures for:

1. LanguageSettingsTestCase.testDeleteLanguage. The matcher for the
swipe-to-delete button was causing "Multiple elements were matched"
errors. This change disambiguates the delete button.

2. LanguageSettingsTestCase.testAddLanguage. The previous method of
dismissing the search bar relied on locating a specific "Cancel" button.
This approach is brittle across different iOS versions and device idioms
(e.g., iPad vs. iPhone). This CL replaces the manual dismissal with the
more robust [ChromeEarlGreyUI clearAndDismissSearchBar] helper. It also
updates the SearchBar() matcher to use accessibility traits instead of a
specific ID for more reliable detection.
