Don't avoid disambiguation dialog in CCT/WebLayer if Chrome can't handle

For external protocols, if we block the outgoing intent because it'll
show the disambiguation dialog, we end up landing on a page that just
says '"<protocol>" links are blocked'.

Given the goal of blocking the intent chooser is to avoid showing the
Chrome icon showing for navigations the CCT/WebLayer could handle, it
should be fine to show the dialog for things that are explicitly
supposed to leave the CCT/WebLayer (through the use of external
protocols).
