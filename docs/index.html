Android Tab Change Metrics

The Browser.Tabs.TabSwitch were not implemented on Android.

This refactors adjusts RenderWidgetHostViewAndroid to make use of
RenderWidgetHostViewBase::OnShowWithPageVisibility which is used by
other platforms for handling complicated tab metrics.

Android does not make use of the same Tab Controller that desktop
platforms do. Instead it has a mix of visibility signals via JNI calls,
as well as direct Navigation visibility changes when reloading tabs.
This change hooks up these sites to RequestRecordTimeToVisible to begin
recording tab changes.

Furthermore some piping DelegateFrameHostAndroid to Android's
CompositorImpl are added to make sure the requests are queued. As
ui::Compositor itself is not exposed to this portion of the code.

LOW_COVERAGE_REASON=RenderWidgetHostBase historically has low coverage. I only added two lines
