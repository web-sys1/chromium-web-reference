CrOS GMC V2: Allow multiple receivers/observers for CastMediaController

The CrOS updated Zenith media view needs to show the cast media item on
multiple UIs in the quick settings view, which requires all of them to
be able to receive media updates and send media action commands. This
needs us to turn CastMediaController to allow multiple
receivers/observers rather than deleting the old one upon new request
similar to MirroringActivity::CreateMediaController().
