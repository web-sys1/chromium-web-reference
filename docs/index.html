Add tests for GpuBenchmarking::AddCoreAnimationStatusEventListener

Add two gpu pixel integration tests for gpu benchmarking interface
AddCoreAnimationStatusEventListener. One for the overlay success case
and the other for the overlay failure case. The CALayerOverlay error
code is verified in the html and the test result is sent back to the
pixel integration test.

In ReportTimeSwapPromise, if DidNotSwapReason is COMMIT_FAILS,
core_animation_error_code_callback will keep the promise active instead
of reporting an error and breaking the promise. The new pixel tests
take this path.
Credit: The idea to KEEP_ACTIVE when COMMIT_FAILS in
web_frame_widget_impl.cc is copied from khaustovsa@'s CL which is still
under review.
