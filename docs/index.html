Invalidate counter & list marker layout on counter style rule changes

This patch implements layout invalidation on counter style changes.
In particular:

1. Each ListStyleTypeData holds a mutable reference to a CounterStyle
object, which is updated on demand. The reference can become dirty or
stale (*) after counter style updates.

2. After updating counter styles, if any list marker is holding a
dirty or stale referencing to a CounterStyle, its layout is
invalidated. This achieves partial layout invalidation.

3. The layout invalidation is done after style update, at the same
time when we are doing counter updates. Since we are already
recalculating counter text unconditionally, this also achieves
counter text update when counter styles change.

(*) "Dirty" means the counter style itself is dirtied due to an
update. "Stale" means the counter style itself is fine, but the list
marker is now referencing a different counter style due to overriding
(e.g, when a predefined counter style is overridden by author style).
