SmartCard: rename reportCancelResult to reportPlainResult

After some internal refactoring in SmartCardProviderPrivateAPI we no
longer need a 1:1 mapping between the request event and its report
result function.

Enables reuse of it by the following PS/SC functions:
 * Disconnect
 * SetAttrib (upcoming)
 * BeginTransaction (upcoming)
 * EndTransaction (upcoming)

reportDisconnectResult (and related code) was removed in favor of
reportPlainResult.

reportReleaseContextResult stays, even though it has the same signature,
as it requires special handling in the browser side.
