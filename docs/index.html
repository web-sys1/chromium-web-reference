Restrict which URLs are sent for Safe Browsing checks in the renderer process

Some URLs currently go from the renderer process to the browser process
UI thread to the browser process IO thread before being dropped due to
their scheme. We should treat these as safe in the renderer process
rather than passing them between processes.
