[iOS] Omnibox popup: pedal debouncer shouldn't prevent closure.

The pedal debouncer accidentally introduced a bug when the popup is
closing.
To close the popup, its contents are set to an empty list of
suggestions. However, if the last result before closure had a pedal,
the debouncer makes it stick around for 0.3s, effectively preventing the
popup from closing.

Add a special case for this, and a test to make sure this can't degrade.
