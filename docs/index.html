Use monitors color space when available

In crbug.com/1073962, solid colors did not match when drawn as a SolidColorDrawQuad versus when rasterized into a tile. At the time, rasterization color space conversion was performed by Skia while display color space conversion was performed by the GLRenderer. All color conversion is now performed by Skia, and whatever inconsistency resulted in these artifacts has likely gone away.
