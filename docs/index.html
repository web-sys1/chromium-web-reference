Prerender: Cancel prerendering if it wants to change WebContents-level
preferred size

RenderFrameHostImpl will cancel prerendering if it receives
the ContentsPreferredSizeChanged Mojo message.
We decide to do so because:
- We predict that this behavior will not significantly impact coverage
because renderers only send this message when running in
`PreferredSizeChanged` mode, which is enabled by extensions.
- Supporting this feature in prerendering pages would be a big project.

This CL also impacts BackForwardCache, as we use
`IsInactiveAndDisallowActivation` to check the activated state and
cancel prerendering.
