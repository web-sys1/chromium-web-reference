Unconditionally include the about:nacl page resources on Mac.

The `enable_nacl` variable is set to `true` on x86_64 builds and `false`
on arm64 (where NaCl is not available). However this breaks universal
build production, which requires that resources.pak be bit-for-bit
identical between the two single-architecture builds. This started
happening after b588b52b1563aab65ab13c084473e79e205a5730, which
corrected an erroneous use of Grit variables that previously included
the resources on all builds.

This CL bundles the resources for the about:nacl page in both x86_64 and
arm64 builds. But the page is only available on x86_64, where the WebUI
backend is actually enabled.
