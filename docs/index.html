exo: Discover scanout fourcc formats/modifiers from DRM backend

As of version 4 of the linux_dmabuf Wayland protocol, Wayland
compositors can give clients hints about which formats or modifiers are
supported by the display hardware they are currently visible on.
If clients use these hints, the chance that the DRM Ozone backend is
able to use hardware planes to directly scanout the clients buffers
increases.

Use the `display::Display` class to communicate this information to Exo
and send those formats/modifiers as part of the "scanout tranche".

supported by the compositor, so a fallback to compositing is always
possible.
