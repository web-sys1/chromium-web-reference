Refactor ChildProcessLauncher::SetProcessPriority

The ability to control the priority of a process started through the
`ChildProcessLauncher` class is currently only used for render
processes.

A future feature will also need to set the priority of the GPU process. To prepare for this, the `SetProcessPriority()` is simplified to a more
generic `SetProcessBackgrounded()` that only takes a boolean parameter,
instead of taking an instance of the renderer-specific
`ChildProcessLauncherPriority`.

The Android implementation actually needs the
`ChildProcessLauncherPriority` parameter (as most of the implementation
is on the Java side), so the original function is kept as-is for that
platform, under the more appropriate name `SetRenderProcessPriority()`.
