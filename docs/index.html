[CrOS Bluetooth] Filter dom-if and template elements in tests.

This change updates OS Settings tests to filter out dom-if and template
elements so that the tests can correctly index into their shadow root.
This change is limited in scope to the function it exists in, and only
affects how we search for sub-pages in the calling tests. Previously
we automatically selected the first child and assumed that it was the
correct element, but in some cases this will wrongly select dom-if or
template elements. This test is one of the Bluetooth revamp blockers
since the tests that call this function fail when the Bluetooth revamp
feature flag is enabled.
