Reland "Convert FrameHostMsg_SynchronizeVisualProperties to RemoteFrameHost message"

This is a reland of 28c8eb1aa6b4994e3059f3d9ade6f4a0fc65d581.

This CL is to re-land the original CL with a fix of the flaky test
failure on chromeos bots. It looks like the flaky test was caused
by missing resetting registered callback. So, this CL adds
|ScrollBubblingProxyObserver| to guarantee to reset the callback even
when the test fails or returns before the end.

Original change's description:

> Convert FrameHostMsg_SynchronizeVisualProperties to RemoteFrameHost message
>
> This CL migrates FrameHostMsg_SynchronizeVisualProperties IPC message
> to RemoteFrameHost Mojo interface. To do that, this CL moves
> FrameVisualProperties struct from //content to //blink. Then, this CL
> introduces SetSynchronizeVisualProperties function in WebRemoteFrame
> class in order to make RenderFrameProxy::SynchronizeVisualProperties
> call SynchronizeVisualProperties in blink.
