[sync] Ensure ModelType is included in crash dumps involving processor

The precise model type involved in a crash report is often insightful to
debug the crasher. However, before this patch, minidumps rarely include
ModelType, which makes investigations hard.

In this patch, `base::debug::Alias()` is used to ensure that a copy of
`ClientTagBasedModelTypeProcessor::type_` exists in the stack and
included in crash reports.
