[disk cache] Fix DCHECK failure in CacheCreator::OnIOComplete

CacheCreator::Run calls CacheCreator::OnIOComplete asynchronously with
net::ERR_IO_PENDING. This is problematic, because when
disk_cache::DelayedCacheCleanup fails in CacheCreator::OnIOComplete then
DoCallback will be called with net::ERR_IO_PENDING, leading to a DCHECK
failure.

The asynchronous call in CacheCreator::OnIOComplete is pretending a
failure, to handle `kReset` and `kResetOnError` in a unified manner.
Let's give net::ERR_FAILED instead of net::ERR_IO_PENDING.
