[profiles] Use params class for adding profile to attributes storage

This CL creates a new ProfileAttributesInitParams struct that holds all
parameters needed for creation of a ProfileAttributesEntry and replaces
all parameters of ProfileAttributesStorage::AddProfile() with a single
struct parameter.

Google C++ style guide describes advantages of this approach [1]:

"This approach has several advantages. Options are referenced by name
at the call site, which clarifies their meaning. It also reduces
function argument count, which makes function calls easier to read and
write. As an added benefit, you don't have to change call sites when you
add another option."

[1] https://google.github.io/styleguide/cppguide.html#Function_Argument_Comments
