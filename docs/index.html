[Clank] Fix a11y for required fields in Address Editor

Aligns the Address Editor with the Credit Card Editor regarding how
"required" fields are announced by screen readers on Android.
Previously, the Address Editor used an AccessibilityDelegate to override
the accessibility text to include "required". This prevented TalkBack
from reading the value entered when the field was non-empty.

This change removes the delegate and instead includes the "required"
text in the field's hint when a screen reader is active. This matches
the strategy used by the Credit Card editor and allows the native
TalkBack logic to announce the value when the field is non-empty.
