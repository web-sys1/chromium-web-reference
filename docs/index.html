[Sharesheet] Hardcoded resizing cleanup.

Slowly cleaning up the sizing! This CL removes the hardcoded resizing
from SharesheetExpandButton. It was meant to be a simple change but
Views so it's a little more complicated unfortunately. We're continuing
the saga of splitting the sharesheet up into three parts:
1. Header - Share title and content previews (Done already)
2. Body - Targets or empty state image & text (clean up TBD)
3. Footer - Margins under the body and/or the expand sharesheet button
if it's meant to be there. (Done in this CL)

Visually the sharesheet looks exactly the same. Except there was a few
pixels of text being cut off before that magically got fixed with this
refactor, probably because the hardcoded number values were a bit off.

Screenshot: http://screen/9F68n5SozfAHEGq - refer to crbug.com/1189977
for what it looked like cut off.
