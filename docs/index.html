Original change's description:
> [Snooping protection] Only (de/)configure the service when necessary.
>
> At the moment, the HPS service is running and emitting signals as long
> as the controller lives (i.e. all session). This CL ensures that the
> service is only configured and running when the user wants to be
> notified (i.e. when they are logged in and have enabled the feature via
> preference). This involves configuring and deconfiguring the service via
> DBus when it becomes available/unavailable, when the pref state changes,
> and when the session state changes.
>
