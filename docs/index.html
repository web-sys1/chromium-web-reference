[Lacros] Add test case against getLacrosInfo() in autotest_private_api

A Lacros specific test case is added to check the return object fields
returned by getLacrosInfo(). This test case is necessary to prevent
potential mismatches between autotest_private_api.cc and
autotest_private.idl, as it once manifested itself in
https://crrev.com/c/3810569.

In addition, this test case also added checks against existing fields in
the same LacrosInfo object, such as "state" ,"isKeepAlive" and
"lacrosPath". This will hopefully make future CLs less error-prone when
related Lacros changes are being made.
