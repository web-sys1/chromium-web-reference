Make BackgroundLoaderContents NOTREACHED non-fatal

This replaces a currently-fatal NOTREACHED_IN_MIGRATION() with a
base::NotFatalUntil::M132 variant. This is my best hypothesis for a
NOTREACHED attributed to
RenderFrameHostImpl::ClosePageIgnoringUnloadEvents M128, though long-tail).

The call stack in crash reports is compressed but
content::RenderFrameHostImpl::ClosePageIgnoringUnloadEvents calls into
"android_webview::AwComponentInstallerPolicy::OnCustomUninstall", except
that's an ICF'd merge of every method that just contains
NOTREACHED_IN_MIGRATION()). All we know is that
ClosePageIgnoringUnloadEvents ends up calling into some method that
contains only NOTREACHED_IN_MIGRATION().

ClosePageIgnoringUnloadEvents() however ends with a probably-tail-called
WebContentsImpl::Close() which probably tail calls
WebContentsDelegate::CloseContents(). The only CloseContents()
implementation that is just NOTREACHED_IN_MIGRATION() is in
BackgroundLoaderContents. To confirm, and probably temporarily stop
these reports, we mark that one as non-fatal which will deduplicate it
from other NOTREACHED() call sites. This'll either trigger new reports
or we'll go back to being fatal in M132 if not hit again.
