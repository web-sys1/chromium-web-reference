[CSP] Remove unused param redirect_status

The parameter `redirect_status` in the CSP code reporting violations
was originally used for security reasons to sanitize the blocked URL
in case redirects had been followed. We later changed the code to
always use the original URL (before any redirects) in CSP reporting,
so this sanitization was not needed anymore. The parameter
`redirect_status` was still being passed around at a few places,
although it was not needed. Let's clean it up.
