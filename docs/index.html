[Autofill] Pull keyboard suppression from TTF controller to delegate

This CL moves SetShouldSuppressKeyboard() calls from
TouchToFillCreditCardController to TouchToFillDelegateImpl.

This is architecturally wrong, as suppressing the keyboard is closer
to UI and an Android-specific thing and should thus live in the
controller.

However, keeping it in the controller would require significant
boilerplate code for the followup CL crrev.com/c/4266331, which
aligns the keyboard (un)suppression with asynchronous parsing.
This is because TouchToFillCreditCardController is a per-tab class.
Any solution seems to either require additional AutofillClient
functions and/or multiple AutofillManager::Observer and/or additional
member variables, which lead to a pretty complex mechanism.
