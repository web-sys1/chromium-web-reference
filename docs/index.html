Fix for uninitialized token API on lock.

When a user logs in for the first time after enabling SAML password
sync feature the token API may be not initialized. The token gets
correctly created on the login screen but the case when user only
locks their screen was not handled so far. In this cl we bootstrap
the API in reaction to a specific error condition after token fetch
attempt on the lock screen.
