WebAppProvider: Migrate profile prefs in startup sequence

We no longer create a WebAppProvider early in Profile creation.

Instead, we wait for the WebAppProvider to have been created and the
web app database to have been read, to migrate profile prefs.

Previously, in TwoClientWebAppsBMOSyncTest tests we were creating
a WebAppProvider and a FakeWebAppProvider for the same profile. Now
only a single (fake) provider is created for each profile.
