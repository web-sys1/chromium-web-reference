ozone: Ignore modifiers for video buffers

This hopefully fixes a bug on Intel devices with Gen11 GPU (chipset
Jasper Lake). If a display modeset occurs when playing a video on an
external display, sometimes the video becomes a solid color.

We suspect that disabling RBC (renderbuffer compression) for video
buffers will fix it.

GBM provides two buffer allocation functions:

    - gbm_bo_create(), whose params include gbm_bo_use_flags but no
      modifiers. GBM will choose the best modifier for the use flags.
      For video usage, GBM will exclude RBC modifiers.

    - gbm_bo_create_with_modifiers(), whose params include a list of
      modifiers but no use flags. GBM will choose the most preferred
      modifier from the list. GBM correctly prefers RBC modifiers over
      non-RBC modifiers.

DrmThread::CreateBuffer may call either allocation function. When it
calls gbm_bo_create_with_modifiers(), it provides all modifiers
supported for the format on the display, which includes RBC modifiers.
Therefore, we must redirect CreateBuffer to use gbm_bo_create() for
video buffers.
