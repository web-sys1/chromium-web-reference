Deflake InstallableManagerBrowserTest, take 2.

The tests in question appear to trigger an error of "no matching service
worker" when they flake. The flakes started a day after
https://crrev.com/c/3182724 landed, which appears to make registering a
service worker take longer. Previously, it seems that the test was
executing and beating the service worker registration; it now seems like
this is no longer reliably the case.

This CL attempts to deflake the tests by forcing them to wait for a
service worker registration to complete. The tests probably should have
been doing this from the start.

This CL also partially reverts https://crrev.com/c/3251816 to put a test
back to the way it was. That CL was a prior attempt to deflake the test
by changing the test itself to cover less, but this fix should be more
comprehensive and allow the test to return to its prior coverage. The
new operator<< method introduced in that CL is left alone as that was
useful in debugging this flake on the bots.
