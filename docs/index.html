[Nearby] Delay process shutdown to avoid thrashing

It is possible for Nearby to be transitioning states where it is not
advertising, discovering, or transferring. This is almost always
temporary since we should be background advertising at the very least.
However this was causing the process to shutdown and then re-started
quickly which can cause problems. The solution is to de-bounce the
shutdown request by waiting 15 seconds and ensuring we still want to
shutdown. Any activity like advertising, discovering or transferring
will also cancel the pending shutdown.

Unit-tests were differed (https://crbug.com/1151139) because if we
switch to the new process manager we will not try to shutdown the
process.
