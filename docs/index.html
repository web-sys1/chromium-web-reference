[editing] Restrict nbsp normalization to paste and drop

Normalize word-internal `&nbsp;` only for rich paste/drop insertions in
ReplaceSelectionCommand. This preserves the updated paste/drop spacing
behavior without rewriting explicit `&nbsp;` from insertHTML or
replacement-text flows.

Most web test expectation updates replace `&nbsp;` with plain spaces
because these paste/drop paths now normalize inter-word `&nbsp` emitted
by sanitization and smart spacing. Leading/trailing `&nbsp;` cases and
non-paste insertion paths remain unchanged.
