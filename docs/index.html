exo: Remove IME blocking feature

IME Blocking is a feature to block host-to-client IME input when
enabled. It was used exclusively by ARC but is no longer used. It was
necessary because there used to be two paths for ARC key events, namely
the Wayland path and the mojo path, but in some cases ARC wanted to
force the Wayland path to be used. With go/arc-key-event-dispatch, all
key events are sent through the Wayland path and we no longer have this
problem. See go/arc-alt-focusable-im for the detail of IME Blocking.
