[infra] Declare test targets.

To support more flexible grouping of tests when specifying tests in
starlark, declarations of the tests that know what type they are will be
used. This will allow declaring a bundle that mixes compile targets and
tests of different types rather than only being able to compose tests of
the same types into a suite.

This change adds the tests struct to the targets lib with functions for
declaring GPU telemetry tests, gtest tests, isolated script tests, junit
tests and script tests. The existing functions for declaring targets
have been renamed so that they are accessed via the isolates struct to
provide a more clear separation of the concepts. There are two
exceptions to this: the compile_target function is kept as-is since it
doesn't produce an isolate entry that can be referenced for running a
test and the junit_test function is replaced by the one on the tests
struct since it will now define a test that can be referenced by suites.
