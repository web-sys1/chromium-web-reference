Don't enforce VaapiWrapper seq. affinity in VaVDA.

The VaapiVideoDecodeAccelerator is a legacy class that is known to use
at least one of its VaapiWrappers from multiple threads. This violates
the restriction introduced by CL:2793005. Unfortunately, refactoring
this class to use the VaapiWrapper on a single sequence is non-trivial.

Thus, to avoid making major changes to a class that's on its way out, we
allow it to create VaapiWrapper instances that don't enforce sequence
affinity.
