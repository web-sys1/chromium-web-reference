Don't override error panel text

The primary and secondary strings to show in Files app feedback panels
are computed in progress_center_panel.js. However, for error panel type
the strings are overwritten with default values in the panel item.

This means that we have to re-set the strings to correct ones after
changing the type to error.

The CL removes this override and relies on proper text being set by the
caller, as is the case with other panel types.
