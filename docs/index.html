Avoid DoublePoint in MouseEvent and PointerEvent

For values needing double range/precision at the DOM API level,
use double x and y values. For internal values or API values based on
internal values, use gfx::Point or gfx::PointF.

This doesn't affect the range/precision of the DOM API values based
on layout geometries because we use LayoutUnit internally for layout
geometris.

Also fix comments about whether zooming is applied to the values.
