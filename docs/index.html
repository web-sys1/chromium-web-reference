SVG Text NG: Fix initial 'current text position' for all writing-directions

https://chromium-review.googlesource.com/c/chromium/src/+/2878237 fixed
the same issue only for the horizontal-LTR direction, and it's hard to
expand it to other directions.

We move the first text chunk to (0, 0) if x/y attribute is not
specified, instead of trying to move inline items to (0, 0) in
NGInlineLayoutAlgorithm. This approach fixes all directions.

The test succeeds with the legacy SVG text, and fails with SVG Text NG
due to other errors for vertical writing-mode processing. We'll fix it
in a later CL.
