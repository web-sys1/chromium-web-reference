Always set token if Chrome shows deep scanning prompt.

It is possible that the server returns request_deep_scan == true &&
verdict == SAFE. Chrome still shows the deep scanning prompt in this
case. However, if the user bypasses the prompt, CSBRR is not sent
because the token field is not set when the verdict is SAFE.

This CL fixes that by always setting the token if Chrome shows deep
scanning prompt.
