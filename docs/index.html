Fix feature clean-up for mojoPrivate

CL https://crrev.com/c/3707093 incorrectly removed the last two
remaining items on the mojoPrivate allowlist, i.e. setting it to an
empty allowlist, which basically allows the API to be used by *all*
extensions/apps on stable channel.
This CL cleans this up by removing that entry.

Only Imprivata in-session extensions will be allowed to use mojoPrivate.
Once Imprivata V2 is phased out (2022/2023), we can also remove that
allowlisting and potentially the entire API.
