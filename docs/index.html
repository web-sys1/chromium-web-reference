Refactor CronetTestRule

1) Plumb isTestingJavaImpl flag all the way into CronetTestFramework:
   This allows us to directly create the correct engine, instead of
   first creating a native engine (regardless of what's needed) and then
   substitute that with a Java one if required.
2) Wrap the tearDown call (inside the evaluate method) in a finally
   block, so that we cleanup resources even when a test fails
3) Create a new RequiresMinAndroidApi annotation: this will allow us to
   flag the minimum Android API level required to run the test in a
   more obvious way
4) Add a new OnlyRunJavaCronet to the list of supported annotations:
   similar to OnlyRunNativeCronet, when OnlyRunJavaCronet is specified
   the test will only run against Cronet's Java implementation
