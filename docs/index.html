[ios] enable delegated compositing using RemoteLayerAPI.

macOS' RemoteLayerAPI is based on availability of the contextWithCGSConnection ctor method in the CAContext. CGSConnection
is macOS only think and iOS has something else.

Checking WebKit, it was learnt that remoteContextWithOptions ctor
method is available on iOS, which allows us to create a CAContext
and use the same procedure as macOS uses (create a tree of CALayers,
get the context id, create a CALayerHost on the browser process side
and attach that context id).

It was also required to avoid setting flipped geometry for the
root layer as it's not required on iOS (see the comment there +
the cl [1] that disabled a flipped layer, which is not needed on
iOS).

[1] https://crrev.com/c/4334544

Test: Verified CALayers are scheduled and quads are delegated.
