[Launcher] Add persistent proto class

I want to make our handling of on-disk state more consistent across
the various classes that handle ranking. The idea is that they should
all just save state as protos into the launcher/ directory of the
cryptohome.

We have a proliferation of Read and Write methods for doing this right
now. So this CL adds a utility class called PersistentProto that acts
as a write-through cache for proto on disk. After this is submitted,
I'll migrate all of our ranking systems to use it.

This mostly isn't new code, it's a tweaked version of
//components/metrics/structured/persistent_proto. The big change is to
make this a header-only template class, rather than splitting the logic
into a cc. That was a bit of a hack and requires changing the
persistent proto class itself for each new class to be persisted.

I've had to make one code-quality compromise to make this header-only:
the code structure looks like this:

 - some enum definitions
 - an anonymous namespace containing Read and Write
 - the class itself

Ideally there wouldn't need to be an anon namespace at all, and Read
and Write could be private methods. However I need these to be free
functions due to a limitation of PostTaskAndReplyWithResult, and they
require the enums to be defined, hence the weird code structure.
