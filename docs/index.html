float: Update restore bounds after dragging a floated window.

This fixes a couple of issues regarding dragging floated windows across
displays.
  - Maximizing the window would go on the wrong display.
  - Unfloating the window would put it nearly offscreen.

This patch handles both by updating the restore bounds after completing
dragging a floating window. It shuffles some logic around so that the
initial restore bounds are usable. The initial restore bounds are needed
because sometimes restore bounds are cleared during drag, or moving
across displays makes them invalid.
