[ozone/wayland] Release capture once drag 'n drop loop ends

This problem affects particularly ozone/wayland based browsers, eg
lacros, where the aura::client::CaptureClient stays set after a drag 'n
drop operation ends.

dragging ends the user performs a click, for instance.
