[CrOS Settings] Fix activation code input positioning.

Fix activation-code-page input being in the incorrect positioning. The
input was incorrectly positioned over the 'Use camera to scan QR code'
button, blocking clicks to that button.

This was because the the activationCodeContainer's class is dynamically
calculated based on whether scanning is available or not. If it is not
available the input is positioned in the center of the page. If it is
available it is positioned at the bottom. However, because the
activationCodeContainer's class compute method was missing being
triggered by changes to qrCodeDetector_, changes to qrCodeDetector_
that were made after cameraCount_ was set would not cause the input to
be re-positioned at the bottom. This can happen in cases where
updateCameraCount_() finishes before initBarcodeDetector_() does.
