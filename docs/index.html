Prevent test-only source ID from containing uninitialized memory

The MSAN issue that caused crrev.com/c/3438825 to be reverted was the
result of SourceBuilder not explicitly initializing its source_id
field, which is a StrongAlias whose underlying type is int64_t, which
doesn't end up being initialized by StrongAlias's default constructor.
