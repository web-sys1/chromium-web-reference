Add a histogram to count how many First-Party Sets were parsed
during startup

Currently, Chrome's startup code does not record the number of
First-Party Sets that were parsed from the component. It would be
useful to store this information in a histogram so that we could
know if all clients are treating the component as empty, and compare
this with what we expect depending on how many sets we put into
the component.
