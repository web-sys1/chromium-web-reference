Fail gracefully in attribution simulation input parsing

Instead of logging fatally on finding the first error, which includes
Chromium source-line information that is unhelpful for most users, we
parse all events and write any errors into an ostream. These error
messages more helpfully contain the source or trigger index in the input
JSON that contained the error.

As a side benefit, it allows us to replace the death tests with normal
unit tests, which run much more quickly, and will allow us to fuzz the
simulator more easily.

Example output before this CL:

```
[0217/120711.011363:FATAL:attribution_simulator_input_parser.cc(45)] key not found: source_time
[...various stack addresses...]
Aborted
```

Example output after this CL:

```
sources[0]: source_time must be an integer number of seconds since the Unix epoch
sources[1]: source_type must be either "navigation" or "event"
sources[1]: source_event_id must be a uint64 formatted as a base-10 string
triggers[0]: trigger_data must be a uint64 formatted as a base-10 string
```
