Add content API to define a custom media::RendererFactory

Fuchsia needs to define a custom media::RendererFactory to be able to
map audio stream to session id for handling multizone.

This feature is client specific, which means it doesn't make sense to
add a specific API in content to handle it on Fuchsia.

Instead this CL introduces an API in ContentRendererClient to specify a
custom base RendererFactory to use for a given frame. It also removes
the global registry for FuchsiaRendererFactory and moves it to
web_engine using this new API. This fixes playing audio in Chrome on
Fuchsia.

Test: Run chrome on Fuchsia, checks that youtube plays and output sound. Run
      simple_browser on Fuchsia, checks that youtube plays and output
      sound.
