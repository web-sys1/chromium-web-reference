Consolidate caching of style invalidation tracing flag

Because the devtools.timeline.invalidationTracking tracing category is
logged on performance sensitive codepaths, we cache a pointer to the
'enabled' flag for it in several places to minimize costs when it's
disabled. This CL consolidates these cache locations to a single
instance.

A comment left in the code in 2014 called out that not sharing these
cache instances was deliberate for performance reasons. However,
recent benchmark runs do not clearly bear this out as a problem:

https://pinpoint-dot-chromeperf.appspot.com/job/148c5b22210000
blink_perf.css:
 11 improvements ranging from 1.5% to 9.1%
  6 regressions ranging from 1.6% to 4.4%
