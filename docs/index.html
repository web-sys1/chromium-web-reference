Remove unneeded test args from all runs of web_tests on bots

web_tests' generated wrapper scripts (eg: out/$BUILD_DIR/bin/run_*)
already bind build-time constants to the cmd line. This includes
path to the out dir as well as --release and --debug:
https://source.chromium.org/chromium/chromium/src/+/main:BUILD.gn;drc=39ffd2b949030f369acd58e4619645ae3695b418;l=1269-1278

And since runs of the suites on the bots use the generated wrapper
scripts, the bots don't need to pass these args. eg: For the
"blink_web_tests" suite on the "Linux Tests (dbg)(1)" bot, the "--debug"
arg is already added to its script:
https://cas-viewer.appspot.com/projects/chromium-swarm/instances/default_instance/blobs/3427bc93cb7c9f0329c313737f7c97c399eaf0c4619dcff1a24f56c9e8096147/5304?filename=run_blink_web_tests

So we can remove all such args from being specified in the bots'
test specs for web_tests.
