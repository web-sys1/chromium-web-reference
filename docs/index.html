Finish security token SAML tests with closeView

Update the mixin used for security token SAML tests (a.k.a. smart card
based login) to load the fake Gaia page that notifies Chrome with
userInfo and closeView messages. This should speed up the browser tests
by letting them not rely on the Gaia flow timeout.

Part of the fix in this commit is to send the RelayState parameter from
the fake SAML page to the fake Gaia via a POST request - previously we
were doing it via GET, which neither was understood by fake Gaia nor
corresponds to real-world SAML deployments.

Also delete a useless line in the
saml_api_login_auth_without_confirm.html test page.
