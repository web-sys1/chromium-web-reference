[iOS][FadeTruncatingLabel] Remove head truncating mode

Currently, there is some confusion about truncating mode head and tail.
The truncating mode should mean:
- Head for beginning of the string.
- Tail for trailing end of the string.
However, the implementation uses:
- Head for left edge of the UILabel.
- Tail for right edge of the UILabel.
The user of FadeTruncatingLabel then switches between head and tail depending on text direction (RTL/LTR).

This CL moves the text direction logic inside of FadeTruncatingLabel and removes FadeTruncatingMode as it's always tail of the string.
Also:
- Introduces textAlignmentFollowTextDirection that is used in popup row
  cell to keep the current behavior.
- Fixes an edge case where the gradient should not be applied.
