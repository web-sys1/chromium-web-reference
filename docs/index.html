Fix tab drag crash from libwayland: unknown object

In some tab drag corner cases, e.g. switching to target window using
Alt+Tab before completing the drop on Gnome, the compositor may not send
an `action` event on the `wl_data_offer` object. But when the drag
completed a `finish` was still being sent to the compositor regardless
which should only be sent if an `action` was received as per spec.

The compositor was sending some event on the `wl_data_offer` object even
after it was already destroyed if the `finish` was sent in that case.

While the root cause is the compositor not sending the `action` in
certain causes, ensuring `finish` is requested only if an `action` is
received to comply with the spec avoids the crash.
