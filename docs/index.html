Don't cache shader, since we need filter-quality

Image shaders now need to know their sampling options at the time
they are created. Previously, PaintShader cached its skia-shaders
at the time they were specified. However the caller doesn't provide
sampling, rather they set a filter-quality enum in the PaintFlags,
which is not visible to the shader until the actual draw. This CL
defers the creation of the skia shader until that time (the draw)
so it can see the PaintFlag's enum.

If in the future, blink specified its sampling-options when it
created its PaintShader, then this lazy construction could be removed
if desired.
