Fix crash on startup in sdk build

Before this change chrome was crashing on startup in sdk build because
of a problem caused by https://crrev.com/c/3644805

Crash error: "Tracing not initialized. Call
              perfetto::Tracing::Initialize() first."

The problem:
TracingSamplerProfilerDataSource is being used in startup trace as well
as main trace. In that CL we are calling `DataSourceProxy::Register(..)`
in the constructor of TracingSamplerProfilerDataSource. Which is
incorrect because the singleton instance of
TracingSamplerProfilerDataSource is created during startup tracing even
before perfetto::Tracing::Initialize() is called. Currently when startup
steps are completed, `TracingSamplerProfiler::RegisterDataSource` is
being called. That's where perfetto's `DataSourceProxy::Register(..)`
should have been called. This CL fixes that.

Bug: b/234107169
