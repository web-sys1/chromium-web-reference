Consolidate strong keepalive as a message port property

Current message port interface let callers indicate when to apply strong-keepalive. By having a strong-keepalive property, the message
port exposes the flag setting, and encapsulates away when and how
strong-keepalive is applied.

This enables further SW lifetime behavior improvements. One specific
upcoming change is that a long-lived message channel port should not
keep the SW alive, unless it is to a native host, or in the allowed
list from ExtendedBackgroundLifetimeForPortConnectionsToUrls policy.
To implement the change, the port needs to know if it has the
strong-keepalive property when closing.
