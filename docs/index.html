Presentational objects should not create a paragraph boundary

If an ARIA role of "none" or "presentation" is applied to an HTML element
its semantic meaning should be removed from the accessibility tree, including
any semantic contribution to the tree's text representation.
This patch ensures that the kIsLineBreakingObject attribute is not exposed on
presentational elements, fixing a related bug in Google Docs that prevented
users from navigating by paragraph in bulleted lists.

AX-Relnotes: Screen reader users can now navigate by paragraph in lists in Google Docs.
