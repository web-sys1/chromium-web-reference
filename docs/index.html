Revert "Change LPAC ACL script to work correctly on all bots."

This reverts commit 37190d5ff3757e2e3a00eccc510de7c11ef30b94.

Reason for revert: The script increased the test running time for win10 CQ builders. A better solution is implementing, see https://crbug.com/1307013

Original change's description:
> Change LPAC ACL script to work correctly on all bots.
>
> This CL changes the set_lpac_acls to set LAPC ACLs recursively
> when run from within the test_env.py script, rather than rely
> on inheritance alone.
>
> This is needed because some bots use hard links and other bots
> are using protected ACLs, neither of which will automatically
> inherit the LPAC ACE when added with icacls.
>
> The behavior for non-bot invocations such as from the
> set_appcontainer_acls target in GN are unaffected by this change.
>
> For bots with inheritable ACLs, this increases avg script execution
> time from 12s to approx 1min. However, for bots with protected ACLs,
> the worst-case script execution time drops by a factor of 3.
>
> This CL also adds some logging so execution time of the LPAC ACL
> script can be more accurately measured.
>
