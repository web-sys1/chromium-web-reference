Menulist SELECT: Update the taller OPTION behavior

We made OPTITONs taller if the user's machine was touch-capable. However
it's not a reasonable behavior if the user never uses the touch input
device.

This CL updates the behavior so that OPTIONs are taller only if the
popup is opened by the touch input device.

Implementation:
* MenuListSelectType detects the trigger event type, and passes it to
  InternalPopupMenu.

Test change:
popup-menu-appearance-coarse.html is updated for the new behavior. It
now uses chrome.gpuBenchmarking, and it seems registering 'didOpenPicker'
listener does not work well if we use chrome.gpuBenchmarking. So we
update pickerCommon.js and picker-common.js in order to call the
specified callback even if 'didOpenPicker' was already dispatched.

This CL affects only touch-capable Windows/Linux devices.
