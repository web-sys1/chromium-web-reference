[iOS] Properly set timeout for deferred PublicKeyCredential promises

The timeout property of PublicKeyCredential options is defined as:
"A numerical hint, in milliseconds, which indicates the time the calling
 web app/relying party is willing to wait for the creation/retrieval
 operation to complete. This hint may be overridden by the browser."

While we are technically allowed to always choose the timeout, this CL
sets a relatively long timeout (5 minutes) for passkey requests, and
leaves the option for sites to override this value by respecting the
hint if it is defined.
