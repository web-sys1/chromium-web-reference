Add delay for AutoScrollHandler to handle scrolled event

ARC++ ChromeVox auto scroll improvement part 5.

This CL adds a small delay in AutoScrollHandler to handle a scrolled
event. Android may invoke an animation on performing a scroll, and a
scrolled event is dispatched multiple times during the animation.
In order to wait for a ui tree stabilized, a small delay is added before
we actually handle the scrolled event.

Note that the timeout is taken from TalkBak's similar logic:
https://github.com/google/talkback/blob/6c0b475b7f52469e309e51bfcc13de58f18176ff/talkback/src/main/java/com/google/android/accessibility/talkback/interpreters/AutoScrollInterpreter.java#L42

This change also removes TODO(crbug/761415) because we found that
CollectionItemInfo is not always populated in RecyclerView and actually
TalkBack isn't using it.
