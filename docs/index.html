Fix padding calculation for wide display

This change adds a fix for the width calculation in ViewResizerUtil,
which is used for several components, including native pages like
History and Downloads. The existing calculation relies on screen width
from the configuration, which doesn't account for things like the
presence of side UI, which results in content being squeezed with
awkwardly wide padding when side UI is showing.

This change is gated by a flag, so that it can be toggled off if it
results in any regressions.
