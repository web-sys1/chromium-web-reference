Simplify ProfileHelper for testing.

This removes two methods from ProfileHelper.
- GetLockScreenIncognitoProfile: No users. Just remove.
- GetProfileByUserIdHashForTest: Used only by tests.

The latter one is more complicated and the main focus of this CL.
Despite of its name, it "Creates" a Profile instance synchronously,
and in many uses, this creation is what tests want.

Thus, this CL replaces many of them by profiles::testing::CreateProfileSync,
which is a test utility designed for the purpose.

In path_util_unittest.cc, it is used just a global getter and not the
main test target, so instead use members to keep Profile instances.
