FSA: Require user activation on renames without parent write access

Consider the following scenario:
- the site acquires a file handle via showSaveFilePicker()
- the site wants to rename this file

In this scenario, the site has write access to the file, but not to its
parent directory. After this CL, a rename in this case is allowed _only_
if the rename is in response to a user gesture.

Files within the OPFS always have write access, so this change
inherently only applies to files outside of the OPFS.
