Original change's description:
> Local filenames do a two-line middle elide on hover cards.
>
> This does two things:
> - provides more space for the filename
> - if the file is an image, makes sure the dimensions (which are placed
>   at the end, after the extension) are visible
>
> The algorithm is tailored for filenames. However the helper class that
> implements it is tied closely to RenderText and could easily be moved
> into RenderText as a precursor to creating more robust middle-elision.
> The algorithm used could very easily be adapted for more than two lines
> as well.
>
> Also of note: this algorithm respects graphemes in a way that our normal
> elision routines do not. So a multi-character grapheme will never be
> partially rendered if the entire grapheme won't fit. Also, it adds in
> some very rudimentary BIDI handling for the second line, to prevent
> unpleasant alignment of the ellipsis.
