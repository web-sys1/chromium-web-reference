Run PolicyPrefsTest.PolicyToPrefsMapping also for Lacros

Currently most of the testing for Lacros using policy_test_cases.json has been disabled due to
differences in Lacros and Ash policy handlers. This CL returns these
tests back by:

1) Introducing two new supported os types ("chromeos_lacros" and "chromeos_ash").
2) Reassigning supported os "chromeos" to "chromeos_ash" if the policy is supported on chromeos only.
It is just currently true for most of the policies, two exceptions are: LacrosSecondaryProfilesAllowed and AttestationExtensionAllowlist.
3) Reassigning supported os "chromeos" to "chromeos_ash", "chromeos_lacros" if policy is supported on chromeos and linux.
3) Adding check in test that OS is covered (chromeos is covered) instead of directly checking that policy is supported on lacros or ash.
As currently policy_templates.json has only indication that chromeos is supported, policies compiled for lacros are chromeos policies, but lot's of them do not have policy handlers in lacros. That is why tests on lacros could be skipped if policy is supported on ash.
Same will apply for ash in the future. (Currently there are no only lacros policies.)
