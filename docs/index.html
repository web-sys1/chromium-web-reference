Don't show LoginCameraTimeoutView when login extension UI closes

LoginCameraTimeoutView is always created when there are no users on the login screen. It is focused when the OOBE dialog state changes to
OobeDialogState::HIDDEN. The Imprivata login screen extension used to
send OobeDialogState::HIDDEN signal when it was closed, which
triggered the LoginCameraTimeoutView to be focused.

Sending a OobeDialogState::HIDDEN notification when the login screen
extension UI closes is risky because new features can depend on the
HIDDEN notification and assume it comes from the OOBE dialog, not the
extension UI (as it exactly happened with the LoginCameraTimeoutView).

This CL adds a new OobeDialogState, EXTENSION_LOGIN_CLOSED, in order to differentiate between the extension closing its UI and the OOBE
dialog being hidden. Using the new entry, we don't focus on the
LoginCameraTimeoutView if the extension UI closed. The CL also adds
EXTENSION_LOGIN_CLOSED to all the relevant places to show items when
the extension gets closed (which used to happen "automatically" in the
past when we were sending OobeDialogState::HIDDEN).
