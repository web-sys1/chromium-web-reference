[ios] Change global JavaScriptFeature initialisation

Use base::NoDestructor<T> variable instead of std::unique_ptr<T>
as those object don't have to be heap allocated and can instead
be allocated inline.

Remove the dispatch_once() idiom used for the initialisation as
initialisation of function scoped static variable is thread-safe
since C++11.

Implement script variable replacement using a repeating callback
to allow the feature to use the correct value even when WebClient
is updated by tests and thus avoid exposing implementation detail
to the tests.
