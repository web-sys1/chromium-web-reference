Remove wrong policy dependency for online reauth flow in dev builds

Online reauth on the lock screen always goes through
`InSessionPasswordSyncManager::CreateAndShowDialog()`. With DCHECKs
enabled this function hits NOTREACHED if
`LockScreenReauthenticationEnabled` policy is disabled. This policy -
despite the name - should only affect password sync flow, and not all
online reauth flows on the lock screen. This cl removes this
uniintendend dependency.
