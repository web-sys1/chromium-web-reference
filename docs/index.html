Deliver push messages to service worker serially

This changes the way we deliver push messages to service workers from
parallel to serial (per {origin, service worker registration id} pair)
in order for us to check visibility requirements after each delivered
push event.
Previously we would only check for visible notifications after all
in-flight push messages have been delivered.
