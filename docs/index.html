Fix usage of PostCancelableDelayedTask in PSIMemoryMetrics

PostCancelableDelayedTask() must be called from the sequence where
the task is posted. Currently, it can either be called from Start(),
which runs on the main thread, or as a continuation in
CollectEventsAndReschedule(), which runs on the background sequence.

To fix this, Start() now posts ScheduleCollector() to the background
sequence. A sequence checker was added to ensure that |last_timer_| is
always used on the right sequence.
