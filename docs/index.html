Refine ChromeContentBrowserClient::OpenExternally

See the tracking bug for general background.

Some time has passed since the main automatic redirection of
link opening from Ash to Lacros was added. This CL is a step towards
cleaning it up and minimizing the exceptional cases.

Insert various DumpWithoutCrash calls to better understand which
special handling is still necessary. Also refactor and reorder the
function to make more sense generally and to more naturally allow
a catch-all handling at the end.

This is meant as an intermediate step.

Also widen the "TryLaunchOsUrlHandler" path to non-WebUI sources.
I'm not aware of a current use case but conceptually it's
the right thing to do and we might need it in the ChromeVox
extension soon.

Also make ChromeWebUIControllerFactory::CanHandleUrl sanitize its input
to simplify call sites.
