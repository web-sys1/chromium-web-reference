[Extensions] Support parsing multiple suggestions in the omnibox API

Extend out the browser-side suggestion parsing to support multiple
suggestions being passed; this happens in the case of the
sendSuggestions() method, which is used in the onInputChanged event to
provide suggestions.

In order to facilitate this without needing to send each suggestion
separately to the isolated XML parser, we synthesize the suggestions
into a single XML document (we were already synthesizing a document out
of a single suggestion).

Given a single suggestion "my suggestion", we will parse:
<fragment>my suggestion</fragment>

Given multiple suggestions, {"first suggestion", "second suggestion"},
we will parse:
<fragment>
  <internal-suggestion>first suggestion</internal-suggestion>
  <internal-suggestion>second suggestion</internal-suggestion>
</fragment>

We then separate these suggestions back out when handling the parsed
input.

Add unit tests for the same. This CL does not yet hook this behavior to
the sendSuggestions function (that will happen in a followup), and there
should be no perceptible behavior change.
