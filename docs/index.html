PreloadServingMetrics: Introduce PreloadServingMetrics

Currently, we don't have a mechanism for collecting metrics of serving
of preloads. For prefetch, we have `PrefetchServingPageMetrics`, but
it's unmaintained and "broken" due to recent updates of prefetch stack.
For prerender, we have `PrerenderPageLoadMetricsObserver`, which
provides prerender version of FirstContentfulPoint for example, but the
capability is limited.

This CL introduces `PreloadServingMetrics`, which aims to unify serving
metrics of preloads. For more details, see
https://docs.google.com/document/d/1bBhfhO7BotUB7Myy_8mtFF_4lI5N8hUyNayV_gI019Y

For a first step, this CL introduces two components, which play core
roles:

- `PreloadServingMetrics`: Top-level log object. Associated to a
  navigation.
- `PreloadServingMetricsHolder`: `NavigationHandleUserData` that holds
  `PreloadServingMetrics`.

This CL just plumbs them in fundamenetal paths. Further extensions and
metrics follows.

This CL also introduces a feature parameter
`kPrerender2FallbackUsePreloadServingMetrics` for
`kPrerender2FallbackPrefetchSpecRules`. No behavioral change if
disabled.
