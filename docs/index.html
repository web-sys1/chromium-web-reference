Don't call WaitForRequestStart after calling WaitForSpeculativeRFH

If the speculative RFH is already created, the navigation had started
and might have moved to the next NavigationState, causing the next
WaitForRequestStart to return false since it expects the state to be
exactly STARTED.

All other tests already avoid this, and this test is just missing the
if block.
