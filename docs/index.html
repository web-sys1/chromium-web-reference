Refactor ScriptExecutor::CurrentActionData

ActionData is now owned by each action individually. Script executors
may access and write to this data, but no longer own it.

This should make this slightly more sane, but more importantly, this is
necessary for a follow-up where we want to add support for nested
actions. The current way of using ActionData would require using some
kind of stack, and it would ripple quite far. By making actions own
their data, the script executor can be ambivalent about the existence
of nested actions.
