Shutdown ChromeBrowserPolicyConnector before TaskEnvironment is deleted

ChromeBrowserPolicyConnector::Shutdown() must be called while ThreadPool
is active. Currently, TestingBrowserProcess does the shutdown on
SetLocalState(nullptr) as a workaround. For tests where this does not
work, we manually call ShutdownBrowserPolicyConnector() on TearDown()
of test fixtures.

This CL uses TaskEnvironment::DestructionObserver for
TestingBrowserProcess to shutdown objects before TaskEnvironment is
destroyed. This also removes the manual calls of
ShutdownBrowserPolicyConnector() as it is no longer needed.
