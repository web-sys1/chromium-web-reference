Partially Revert "[fuchsia] Uses file_path_watcher_linux for Fuchsia implementation."

This partially reverts commit
d2f9bef0ccb420f5485634a952a602b9ff007644.

Reason for revert: Removing use of inotify on Fuchsia
(https://crbug.com/1430566).

This change restores the specification of the file watcher
implementation to a Fuchsia-specific block but uses
`file_path_watcher_stub.cc` instead of the deleted
`file_path_watcher_fuchsia.cc` (see https://crrev.com/c/4400931).
Building the `FilePathWatcherTest` tests is once again disabled.

The file renaming in the original CL and related cleanup in the
original CL remains.

Original change's description:
> [fuchsia] Uses file_path_watcher_linux for Fuchsia implementation.
>
> The Linux inotify based file path watcher client is used with the
> Fuchsia inotify implementation. The following tests currently pass:
>
> * NewFile
> * ModifiedFile
> * DeleteDuringNotify
> * DestroyWithPendingNotification
>
> Many other tests are still failing and are being investigated. The above
> set of cases are the most important scenarios.
>
