Use VectorIcons with ColorId for aura touch selection handles.

Add vector icons for aura touch selection handles and add basic support
for following the color pipeline. For now, only use these under the CrOS
touch text editing redesign flag, but add the changes to the general
aura implementation so that it is easier to update other aura platforms
as well if it is later needed.

For TouchSelectionControllerImpl (used from views), the handles follow
the usual views color pipeline. For TouchHandleDrawableAura (used from
web content), we just make the handles follow the native theme, since it
seems a bit fiddly to get it to follow the color theme of the context
window at the moment.

Figma specs: go/next-e14s-touchtext
(Opacity, padding, CrOS-specific color not to specs yet, will work on
that in followups.)
