[Files F2] Refine focus logic for the tree widget

* Mouse interaction (click/double click/right click) on the tree
  should:
    * focus on the tree itself if the click target is outside
      the tree item (e.g. the blank area).
    * focus on tee tree item if the click target is on the tree
      item.
  To maintain this we bind the event handler on the host element
  level instead of the root level.
* When tree items get removed, the focused item inside should be
  update accordingly: set it to the selected item if exists,
  otherwise set it to null.
* Update focus related CSS to use focus-outline-visible class,
  the native :focus-visible will show focus ring even without
  keyboard interaction, which is not what we want.
