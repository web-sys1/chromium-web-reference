Implement logical COI for DIP

This CL adds an option for the content embedder to specify whether the
cross-origin isolation enabled by DocumentIsolationPolicy for an origin
will be logical or concrete. Concrete gives access to COI gated APIs
like SharedArrayBuffers, while logical does not. This is needed for
Android WebView to allow specific allow-listed origins to get access to
SABs, while preventing non allow listed origins from gaining access.
Currently, all origins are marked as having logical COI in Android
Webview (and concrete COI on all other platforms). Further work will be
needed to add the origin allow list on Android WebView.
