Prompt API: store ReceiverContext in AITextSession for cloning

After https://chromium-review.googlesource.com/c/chromium/src/+/5775466
we need to add the AITextSession into an AITextSessionSet after
creating it. The AITextSessionSet is bound to the either RFH or the
worker host. The host was stored in a ReceiverSet as the context, so
it can be accessed from the IPC method implementation.

There is one exception: the clone API is also creating new session but
it's a method from the AITextSession interface, so it cannot obtain
the host from the AIManager interface's receiver set, causing crashes
when we attempt to add the session into the set.

In this CL, AITextSession will keep a pointer to AITextSessionSet, and
will pass it to the AIManagerKeyedService::CreateTextSessionInternal().
