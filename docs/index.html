raw_ptr_unittest: Enact boilerplate reduction

Following up from change 3782443, this CL applies the pattern to all
clusters of `EXPECT_EQ(foo, RawPtrCountingImpl::foo_counter)`.

There are some caveats:
*   Single-line EXPECTations are left alone, as they would not benefit
    from this new helper matcher.
*   This CL makes the expectation-vs-reality ordering somewhat more
    uniform, since the tests were split between "regular" expectations
    and "Yoda" expectations. (This has no effect on test operation.)
*   The style guide mandates that we supply arguments to designated
    initializers in the order of declaration, which forces us to match
    the declaration order of the members of RawPtrCountingImpl. (This
    has no effect on test operation, but makes the diff a bit trickier
    to navigate.)
