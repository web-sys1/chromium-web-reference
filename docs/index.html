Add ExtraTimeMin params to fieldtrial_testing_config.json

This change updates feature params for the DnsHttpsSvcbSchemeUpgrade
experiment.

Besides adding ExtraTimeMin params, it also replaces the feature params
that were deprecated in https://crrev.com/c/3569573.

We are seeing a lot of TimedOut values in the DnsRcode histograms. We
think that the percent-based timeout logic may be too harsh when the
address queries complete very quickly. If that assumption is true, we
should be able to prevent these timeouts by setting the "extra time min"
params.
