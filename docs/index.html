Ensure ChromeMessageQueueMediator#setLayoutStateProvider is not called after destroy()

The crash stack suggests that mQueueController was null at the time when
LayoutStateObserver#onStartedShowing was called. mQueueController is set
in ctor and only reset to null in #destroy(), where mLayoutStateObserver
is removed from observer list. This suggests that setLayoutStateProvider
was called after the call to destroy().

This CL adds a check that throws an exception if setLayoutStateProvider
is called after destroy() to help identify scenarios when it happens.
