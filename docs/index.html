[fuchsia] Avoid duplicate navigation events during initialization

The previously-reported navigation state was not being stored when the
initial state was sent to a new NavigationEventListener. This would
result in a duplicate event firing as soon as the initial event had
been delivered, if the Frame was not in the "initial" navigation state.
