Deprecate XML external entity loads

I2D approved in
https://groups.google.com/a/chromium.org/g/blink-dev/c/4j2oGg2lyBk/m/FkVq-s-EBQAJ

For test cases external-entities.xml, xhtml-alternate-entities.xml and
dtd-in-source-document.xml there is a behavioral change in that the
undeclared entity is no longer reported as an error, but now as a
warning, which changes the final rendering output. This is due to the
parser logic changing when XML_PARSE_NO_XXE (no external entity loading
allowed) is specified, see [1]. The parser cannot verify if the entity
was possibly declared in an external reference. This is working as
intended and an acceptable change. For files that do not refer to
external entities in the DTD or otherwise, undeclared entities are still
reported as an error (locally tested).

In the process revert the deprecation warning: Revert "Mark externally
loaded entities in XML deprecated" This reverts commit
c5f60a036a3d005f1b1dcb79c44ed672a36bb95a.

[1]
https://source.chromium.org/chromium/chromium/src/+/main:third_party/libxml/src/parser.c;l=7371
