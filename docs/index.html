[ios/b4p] Add custom accessibility label to new feature icon

This icon has meaning, it's not just an illustration, so it needs to
be read by the screenreader. We do this by setting a custom a11y label
in TableViewSwitchCell.

- I would have preferred an approach where we just set the label on the
image and let it propagate up but that doesn't seem to happen.
- The behavior of [TableViewSwitchCell setAccessibilityLabel:] before
this CL was weird, it was a silent no-op.
- Subclassing TableViewSwitchCell just to set a custom a11y label seems
overkill. A reasonable setter is simpler.
