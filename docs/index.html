Add PERCEPTIBLE to ChildProcessImportance

ChildProcessImportance.PERCEPTIBLE represents the memory priority
between MODERATE and NORMAL.

In https://crrev.com/c/6321765, ProcessRankPolicyAndroid will boosts the
memory priority of protected tabs from NORMAL to PERCEPTIBLE.

The service binding which leads to PERCEPTIBLE_APP_ADJ (= 200) is ideal
for protected tabs. But since Android doesn't support the service
binding flag, we use not-perceptible binding as a short term solution
which leads to PERCEPTIBLE_LOW_APP_ADJ (= 250). Visible binding is too
high priority for protected tabs because it can kill perceptible apps in
PERCEPTIBLE_APP_ADJ (e.g. Youtube app playing background).

The not-perceptible binding is supported on Android Q+. On older
Android, ChildProcessImportance.PERCEPTIBLE falls back to
ChildProcessImportance.NORMAL conservatively. The setters of
ChildProcessImportance.PERCEPTIBLE need to be aware of the fallback and
should avoid setting ChildProcessImportance.PERCEPTIBLE on older
Android.
