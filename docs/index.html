Fix uploads not working on arc directories

MemoryMappedFile (posix) always uses MAP_SHARED.
Using mmap in shared mode is not supported on some file systems (e.g.,
fuse-io in direct-io mode which is, e.g., used by ARC on ChromeOS):
https://www.kernel.org/doc/Documentation/filesystems/fuse-io.txt
For READ-ONLY access the MAP_PRIVATE and MAP_SHARED mostly behave
similarly, s.t., we can retry access with MAP_PRIVATE if MAP_SHARED
fails.
So on posix systems, we create our own MemoryMappedFile that can execute
mmap calls using MAP_PRIVATE.
