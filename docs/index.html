Introduce TestPrintingContext & test UpdatePrintSettings

Ability to test out-of-process print paths benefits from having a test
abstraction layer as close as possible to where the operating system
print APIs are made.  For printing this is primarily in two classes,
PrintBackend and PrintingContext.

Introduce a new TestPrintingContext class which is analogous to the
existing TestPrintBackend class.  This will support test scenarios for
printing a document, allowing for more testing coverage of the full
printing stack.

Start off with an implementation filled in for UpdatePrinterSettings(),
which is used at the start of the document printing pipeline.  This
enables testing to be introduced which provides coverage of
PrintJobWorker::UpdatePrintSettings() for Windows, macOS, and Linux.
