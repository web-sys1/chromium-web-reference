DedicatedWorker: Enforce same-origin check for main script fetch.

A compromised renderer could potentially exfiltrate static assets from
other Isolated Web Apps (IWAs) or Extensions by requesting them as
DedicatedWorker scripts. This occurred due to a missing same-origin check
in DedicatedWorkerHostFactoryImpl combined with overly permissive URL
loader factories used for worker main resources.

This CL:
1. Implements a same-origin check in DedicatedWorkerHostFactoryImpl for
   non-data: script URLs.
2. Updates RegisterNonNetworkWorkerMainResourceURLLoaderFactories to
   accept the request_initiator's origin.
3. Passes the initiator origin and feature flag status to
   IsolatedWebAppURLLoaderFactory and ExtensionURLLoaderFactory.
4. Updates AllowExtensionResourceLoad to enforce a same-origin check
   between the script URL and the initiator when the child_id is invalid
   (browser-initiated worker fetches).
5. Guards these changes with kEnforceDedicatedWorkerSameOriginCheck
   feature flag.
