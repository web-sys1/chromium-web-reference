Android: Improve dependency_analysis for local dev

This CL focuses on improving the usage of dependency_analysis scripts as
a local build-validation tool. Oftentimes when making build changes it
is useful to visualize dep changes.

This CL removes assert failures if jars are missing. Some __compile_java
targets (like bundle_module targets and test support targets exist in
GN but do not generate a .jar as an output. These targets do not affect
the correctness of the script and should simply be ignored.

All relevant build targets (*__compile_java) are re-built each time the
script is run. This adds a few seconds to the build but is necessary to
ensure that the resulting dep graph is an accurate reflection of the
current local state.

The dependency_analysis/{git_util,subprocess_util}.py scripts have been
removed and other scripts migrated to use tools/android/python_utils.

Update docs for running dependency_analysis and the web front-end
locally.
