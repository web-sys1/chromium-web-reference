utr: Use and check for "luci-auth context" auth when running the recipe

As-is, the user needs to be authed with the following to run the utr:
- luci-auth
- luci-auth -scopes-context
- gcert (if compiling with reclient as googler)

But we can combine the first two into just one if we instead run the
rdb prefix command inside our own `luci-auth context ...`. Doing so
will make rdb re-use that context's auth instead of trying to create
its own via `luci-auth login`.

This pares that down to just two auth requirements:
- luci-auth -scopes-context
- gcert (if compiling with reclient as googler)

Which we assume is an improvement. This also clarifies to login with
a @google.com account if available.
