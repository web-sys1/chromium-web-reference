media/vaapi: refactor minimum resolution logic and add flag

VaapiWrapper may provide a minimum resolution for both encoding
and decoding during profile enumeration in VASupportedProfiles's
FillProfileInfo_Locked().

crrev.com/c/2494521 added such a minimum resolution for the case
when Intel Hybrid decoder is going to be used, which broke the
Tast bots because suddenly they saw that HW decoder was not used.

This CL adds a new kVaapiVideoMinResolutionForPerformance flag to allow
not providing such minimum resolutions, and also cleans up the logic
so that it's more readily understandable what's happening. In
particular, IIUC we don't need the Contains() query because we always
SetToMax with |min_resolution|.

Tested on samus.
