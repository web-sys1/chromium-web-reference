mojo: Fix buffer size of maps to include 8 byte alignment

Maps in mojo are encoded as 2 arrays where the first array contains all
keys and the second array contains all values. In the JS bindings, the
size allocated for the buffer was incorrectly computed as both of the
arrays which represent the map must be 8 byte aligned. This means that
if the sizeof(key_array) < (8*n) bytes, the buffer allocated to hold all
of the data will be short by (8 - (sizeof(key_array) % 8) bytes.

This change includes alignment when calculating the size of the map for
the key array portion of the encoding. The value array portion does not
need to be aligned as the `allocate` function will align the size once
the buffer is going to be allocated.
