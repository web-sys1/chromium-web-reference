Clarify -mbranch-protection to use bti+pac-ret

Guarded control stack support for clang was added back in January, which
changes -mbranch-protection=standard to also mark objects as supporting
GCS on applicable architectures.

This is undesirable for Chromium as GCS could "silently" be enabled when
GCS-capable hardware becomes available. GCS has also not been explicitly
enabled in components such as libunwind.

arm_control_flow_integrity has been adjusted to clarify that it is
bti/pac-ret that is being used. So when the time comes to enable gcs it
may done so explicitly.
