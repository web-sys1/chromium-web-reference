media/gpu/v4l2: Move ioctl error handling to called functions

The platform decoder is not expected to be able to
handle most ioctl errors. As such there is no need
to return an error value when something goes wrong.
Already the code was using LOG(FATAL) when these
ioctls failed.  Instead of having every call site
handle the error, move them into the called
functions.
