Refactor GoToOffsetWithSkipping to use iterative back/forward helpers

This CL:
1. Extracts single-step skipping logic into
   `GetIndexForGoBackWithSkipping` and
   `GetIndexForGoForwardWithSkipping`.
2. Introduces `GetIndexForOffsetWithSkipping`, which invokes these
   helpers iteratively to determine the target index.
3. Updates `GoToOffsetWithSkipping` and `CanGoToOffsetWithSkipping` to
   rely on this new iterative method.
4. Updates `GetIndexForGoBack` and `GetIndexForGoForward` to use the
   new method with from_index=GetCurrentEntryIndex(), to retain the
   current skipping logic.

Motivation:
This prepares for an upcoming ad-skipping intervention that involves a
"same-origin exclusion". In the feature, we will need to perform a
dry-run at each step of a back/forward navigation, to check if the
potential ad-skipping logic should be used or not due to a
"same-origin exclusion" (see design doc [1]). Splitting the offset
calculation into granular steps allows us to seamlessly inject this
heuristic.

[1] https://docs.google.com/document/d/1BN1c_VAM1yIIUch7ISQh1B4cOeOTV5k2FrJ1rEu0ST8/edit?resourcekey=0-DeM1fa3iJzoipqLLlQ_aUw&tab=t.0#bookmark=id.vuetpsp9ri6n
