Lacros Test: Add support to verify/close web based UIs in ash.

One of the annoying issues with lacros browser tests running
with shared ash is that some tests may open a UI in Ash (some
times by accident) and it will fail the tests running after
it, which makes the tests flaky and is hard to catch the culprit.

This cl adds the following support in InProcessBrowserTest for
lacros browser tests running with ash to prevent such issues:

1. Verify that there is no ash browser based UI such as SWA,
web UI. etc open before the test starts and after the test finishes.

2. Provide API to close all ash browser windows from lacros
browser tests.

The test should be responsible for managing the
browser windows it opens in ash, and making sure they are cleaned
up before the test tears down. Otherwise, InProcessBrowserTest will
catch the problem and fail the test.

Note this cl only fixes the issue with the web based UI. Some UI generated in ash are not web based, for example, a dialog derived
from DialogDelegateView. We will fix these UI later.
