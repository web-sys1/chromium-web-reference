[PDF Ink Signatures] Fix thumbnail updates

When strokes loaded from PDFs as ink::PartitionedMesh objects get
erased, the thumbnail image of the PDF itself has to be updated, since
these strokes are not ink::Stroke objects. To do so:

1. Change PdfInkModule to track ink::Stroke and ink::PartitionedMesh
   changes separately.
2. Ask for the PDF thumbnail image update via a new
   PdfInkModuleClient::RequestThumbnail() interface. RequestThumbnail()
   just calls the existing PDFiumEngine::RequestThumbnail() code, and
   PdfInkModule receives the thumbnail image via the callback.
3. PdfInkModule takes the thumbnail image and sends it to the WebUI. The
   "updateInk2Thumbnail" message now has an extra "isInk" field, so the
   WebUI knows which thumbnail image to update.
