Deflake RTCVideoEncoderTest.EncodeVP9TemporalLayer

RTCVideoEncoder::Impl drops the input frame and signals
WEBRTC_VIDEO_CODEC_ERROR when it runs out of free input buffers. If this
happens, we get a test failure because
RTCVideoEncoderTest.EncodeVP9TemporalLayer expects error-free encoding
of all 5 frames.

There is some indeterminism in the test due to the fact that input
buffers are returned to the "free" pool in
RTCVideoEncoder::Impl::EncodeFrameFinished(), which runs as a task
posted when the VideoFrame wrapping the input buffer gets destroyed.
Sometimes, a new Encode() will run before the input buffer is considered
"free".

The number of input buffers maintained in RTCVideoEncoder::Impl is
determined by the VEA implementation and it's normally something like 3.
The test asked for 0 input buffers (RTCVideoEncoder::Impl reserves 1
additional buffer), which made running out of free input buffers
relatively easy.
