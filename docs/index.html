[android] Use PersistableBundle in BackgroundTaskScheduler API

Internally the BackgroundTaskScheduler needed to persist any extras it
was passed. JobScheduler requires this to be PersistableBundle, and we
also have an internal proto format for storing these.

However, now that we do not support versions of Android that do not
have PersistableBundle available, we change the API to use it directly
to drop any unnecessary conversions.

Client code was also updated to use PersistableBundle across the board.

One usage of Bundle to PersistableBundle still exists, but the code
is now only used in the ChromeGcmListenerServiceImpl and therefore
the conversion is moved there. This CL could also have updated the
GCMMessage to use PersistableBundle directly, but to simplify this CL
that has been left as an exercise for the future.
