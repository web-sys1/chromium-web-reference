[Side Panel] GetEntryForKey looks at contextual then global registry

Change GetEntryForKey to look through the active contextual registry
first, then the global registry as a fallback for the entry with the
given key.

This change is required to support tab-specific extension entries in the
side panel, because extensions will be able to specify both global and
tab specific entries, with tab specific entries taking precedence over
global ones.

This change also fixes a bug in SidePanelRegistry::Deregister()
where only the active entry vs deregistered entry's id is compared.
This should compare the SidePanelEntry::Key instead.
