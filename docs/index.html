[PriceTracking] Include price track offer data to subscriptions

To make price drop notifications more accurate, we need to add offer id,
user-seen price, and country code to subscriptions.

For now, the newly added fields are only used to improve notification
accuracy, and aren't considered as part of subscription identifier. So
we only store them on the server, not in local db. Also we don't need
to get these info back from the server.
