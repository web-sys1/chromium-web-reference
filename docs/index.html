Upstream Core Runtime from Chromecast Internal

This CL upstreams the remaining parts of :core_runtime target from
chromecast internal which have not previously been upstreamed. The
following changes were made:
- BUILD target deps and public_deps were re-written due to the splitting
  of this BUILD target.
- Update RuntimeService::metrics_recorder_ to a unique_ptr<> to avoid
  refactoring required for an updated ctor. This will be updated in
  a follow-up CL
- shared_ptr usage as required for grpc channel initialization have been
  updated to `auto` instead of `shared_ptr` types. The type is
  required for interfacing with the gRPC library, so this change is
  purely to avoid presubmit requirements in Chromium.

A future CL will take care of addressing the second of these changes
above, as well as gluing the pieces together and refactoring the
CastRuntimeService class to better match the internal version.
