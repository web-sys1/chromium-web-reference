Rename some parameter names in LocalPrinterHandlerDefault callbacks.

LocalPrinterHandlerDefault callbacks for OOPPD have results that are
unions of the desired data types and result codes. e.g.
PrinterListResult is either a PrinterList or a ResultCode. Instead of
calling these unions by their desired data types, just call them
"result". This avoid situations such as one where "printer_list" is not
actually a PrinterList.
