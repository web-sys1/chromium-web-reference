[Autofill] Fail silently if items that should be removed are gone

Users are running into this NOTREACHED when they request deletion texts.

While it's not ideal that the return is reached, the behavior is
the appropriate handling of a possible race condition: if an element is
already gone, return that there is no deletion text for that element.

Same applies for the attempt to delete an element that was already
removed: if it's gone, it can't be deleted but no user data is at risk.

Updating the UI is an orthogonal issue.
