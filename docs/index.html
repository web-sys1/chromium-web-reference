Exclude OS_WIN .url shortcuts from DataTransfer.files

In windows, calls to OSExchangeData::SetURL() create both a URL, and
a FILE which is a *.url shortcut:

 [InternetShortcut]
 URL=...

This files is useful when doing a drag-drop of a link from the browser
to Windows Explorer, but when dropping in the browser, users do not
expect to have this file.

crrev.com/c/3146940 allows images dragged from a renderer to also be
dropped into a renderer. The change makes a call to GetFileContents()
and uses it to populate DataTransfer.files without realizing that
OSExchangeData may contain the *.url shortcut file.

The fix is for OSExchangeDataProviderWin which is creating this file
in SetURL() / SetFileContents() to mark that GetFileContents() should
return empty when it holds a URL shortcut by adding an empty item
with key "chromium/x-ignore-file-contents".
