Fix and re-enable ...SameDoc...NavigationWithNothingCommittedAfterCrash.

This CL removes the NOTREACHED in
NotImplementedURLLoaderFactory::CreateLoaderAndStart that used to be
flakily hit by the
BrowserTest.SameDocumentHistoryNavigationWithNothingCommittedAfterCrash
test.  The NOTREACHED was added in r794566 and 1) from the start was
focused on renderers using non-frame-specific/non-browser-provided
URLLoaderFactory (so removing the NOTREACHED shouldn't impact other,
broader scenarios) and 2) the root cause for violating this assertion is
known at this point and not _directly_ related to the scenario exercised
by the flaky test (see https://crbug.com/1184292#c4) so the NOTREACHED
is not adding much value at this point and can be temporarily taken out.

This CL re-enables the test - it shouldn't be flaky anymore after
the changes described above.
