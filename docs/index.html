webauthn: use different method for getting device name.

In Android 12, reading the Bluetooth name requires the BLUETOOTH_CONNECT
permission. Additionally, rather than just returning null when its not
available, Android kills the process.

Therefore try reading the device name from Settings. It's harmless if
it fails because we have a reasonable fallback.

(This only affects QR-linking, which is not launched.)
