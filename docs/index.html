Reland "[OOPIF PDF] Use DocumentUserData for StreamInfoHelper instead"

This is a reland of commit 5348b35305180023b5e34c2bad9fedaef907dab6

Move the CHECK_EQ() for the embedder frame's RenderFrameHost to after
verifying the stream's extension ID. The original CHECK() would cause a
crash for any other MIME handler extensions.

Original change's description:
> [OOPIF PDF] Use DocumentUserData for StreamInfoHelper instead
>
> Change StreamInfoHelper to use DocumentUserData to support multiple
> StreamInfoHelpers per WebContents. Modify the signature for
> PdfStreamDelegate::GetStreamInfo() to take a RenderFrameHost instead of
> a WebContents in order to be able to use DocumentUserData.
>
> Currently, StreamInfoHelper for ChromePdfStreamDelegate uses
> WebContentsUserData to store the PDF stream info. With the OOPIF PDF
> Viewer project, if there is a page with multiple PDF Viewers embedded,
> there will only be a single WebContents, causing collisions when
> retrieving the StreamInfoHelper.
>
