Reland "[inspector] Introduce a restricted instrumentation pause"

This is a reland of commit f96df25cdfcaff34c828612347532d9e8d672d9f

Compared to crrev.com/c/3891025, this patch only disables the
Devtools V8 session when the connection to the IO session is closed.
This fixes the situation when the IO session was closed ona V8
interrupt during painting debugger overlays and the devtools
session object was disposed while still being on stack. With this
patch, we only disable the v8 session during the v8 interrupt;
the full devtools session cleanup is performed on a normal message
loop handler.

Original change's description:
> [inspector] Introduce a restricted instrumentation pause
>
> We introduce a new instrumentation pause state in the renderer.
> The instrumentation pause is entered on script creation
> if the instrumentation breakpoints are enabled with
> Debugger.setInstrumentationBreakpoint. The purpose of the pause
> is to give the debugger a chance to insert its breakpoints
> correctly before the script runs.
>
> In this patch, we restrict the messages that are handled during
> the instrumentation pause to the ones that are handled by the
> devtools IO session (those are mostly breakpoint management
> messages). Handling of other messages are postponed until the
> instrumentation pause resumes.
>
> Without this patch, the instrumentation pause behaves the same
> as the normal pause, causing some undesirable interactions
> with layouting, navigation, etc. (e.g., see crbug.com/1304095).
>
> Note that the instrumentation breakpoints/pauses are
> an experiment in DevTools,  currently off by default.
>
