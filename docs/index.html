[COOP Isolation] Support user activation as a trigger for future isolation.

This CL adds code to detect user activation occurring on a frame that
previously triggered COOP isolation and use that to further isolate
the corresponding site for future BrowsingInstances.  For now, this
doesn't yet persist the site to disk (unlike password-triggered
isolation, for example).  That support will be added in future CLs.

Note that this is supported both for user activation in the COOP main
frame and in any of its same-origin subframes.

This user activation requirement for isolating sites in future
BrowsingInstances intends to address tracking concerns, by preventing
malicious sites from silently loading COOP sites to put them on the
isolation list and later querying that state.
