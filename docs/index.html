Update ScreenAI's handling of loaded files when service disconnects.

If the ScreenAI service gets disconnected while loading the model files,
service initialization functions are not called anymore and the handles
of the opened model files are passed to a thread that supports blocking
to close them.

`PerformOCR_AfterServiceRevive` test that was flaky because of this
issue is re-enabled.
