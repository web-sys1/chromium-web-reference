Spanify callsites of ColorProfile::Create

This is a follow-up to https://crrev.com/c/5350557, which changes
the callsite of ColorProfile::Create to accept a span instead of
a pointer and size. Ad-hoc bounds checking is error-prone, and we
should use data structures that automatically manage collection
size when possible.
