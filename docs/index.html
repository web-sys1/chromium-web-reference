Support CRD multi monitor

This adds support for multiple monitors in AuraDesktopCapturer and
DesktopDisplayInfoLoaderChromeOs.
This functionality is all hidden between a new feature flag
EnableMultiMonitorsInCrd.

With this it is possible to see the different displays in CRD (and
switch to them), but the mouse input is still messed up in these
scenarios:
   - If your secondary display has a negative offset from the primary
     display, so if the secondary display comes higher like this:
                 +-----+
         +-----+ |     |
         | Prim| | sec |
         +-----+ +-----+
     In this case selecting the secondary monitor and then going back to
     the primary monitor causes the mouse to have a wrong offset
     (meaning the actual mouse position is lower than you think it is
     through CRD).
   - You select the secondary display (because the PointTransformer
     class currently only supports the primary display).

These mouse issues will be fixed in a follow-up CL, which will not cause
any issues as all these changes are hidden behind a feature flag.
