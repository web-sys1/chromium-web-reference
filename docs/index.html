Record metrics for BackForwardCache entries

This change introduces two new UMA metrics:
- BackForwardCache.History.ForwardEntriesCount: Recorded on history back navigations, counting how many BackForwardCache entries are currently forward of the new committed entry.
- BackForwardCache.History.EntryMatch: Recorded on navigations that are not restored from BackForwardCache. It checks if the URL matches an existing BFCache entry, and whether the history index is an exact match. (Currently, this uses exact URL matching. Support for No-Vary-Search is planned for a follow-up CL.)

These metrics help understand the potential impact of keeping forward
entries in BFCache on AGSA.
