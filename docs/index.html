Fix flaky testGoBackAndGoForward

The GoBack / GoForward commands failed if Page.getNavigationHistory hit
the situation when the document was transitioning away from
kActive/kPrerendering to a yet-to-be-defined state.
This indicated an ongoing navigation. This fix solves the problem by
just retrying the GoBack / GoForwar command.
