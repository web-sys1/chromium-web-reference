Correctly detect column-spanner-ness changes.

Simply checking the column-span:all property isn't enough (e.g. removing
`float:left` from `column-span:all; float:left` may potentially turn an
object into a spanner). The check was good enough when we had a flow
thread, since then a LayoutMultiColumnSpannerPlaceholder would be
created or destroyed in such cases. But not anymore.

Add two tests. One pure crash test, and one correctness test, that
admittedly would also fail a DCHECK without this fix, but bypassing the
DCHECK would result in incorrect rendering.
