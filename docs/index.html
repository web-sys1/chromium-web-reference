Add encryption info to ArchiveAnalyzerResults

This is used to validate the password provided for an archive. Since we
currently don't support AES encryption for ZIPs, we have three possible
outcomes for passwords:
- Known correct (archive unpacks successfully)
- Known incorrect (archive fails to unpack with this password)
- Unknown (AES encryption)

We also provide is_encrypted in this field, since RAR unpacking will one
day support detection of metadata encryption, which is not currently
reflected anywhere else in the ArchiveAnalyzerResults
