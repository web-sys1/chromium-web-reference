capture_selfie_cam: Render camera frames (CL3)

This CL implements the actual rendering of the camera
video frames. The preview view hosts a window via a
NativeViewHost. This window is used to create a layer
tree frame sink, which is used to submit compositor
frames for the video frames received from the camera
device.

This CL also add a lot of test infrastructure to verify
both code paths (kSharedMemory used in the emulator, and
kGpuMemoryBuffer used on an actual device).
