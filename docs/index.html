chrome://media-app: Ignore the "-1" invalid token when showing an open file picker.

When an exception is thrown loading the "focus" file on launch,
launch.js "poisons" the FileList with a file with a "-1" token. This
ensures the list is not empty (so a "load" occurs), and ensures that
later file operations on the "error" token are rejected.

However, one operation -- opening a new file -- should still be allowed.
This is getting rejected since adding the `startIn` parameter to open
file pickers at a suggested initial folder location, based on the
currently loaded file.

Fixed: b/209342852
