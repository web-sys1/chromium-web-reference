Explicit ToGfx* functions to convert blink geometry types to gfx

They replace the previous implicit or explicit conversion operators.
They are temporary during blink -> gfx geometry type migration.

They have the following benefits:

- They will prevent redundant gfx type constructors in cases like:
    FunctionExpectingGfxRect(gfx::Rect(int_rect))
  when int_rect is converted to gfx::Rect.
  Now the above code becomes:
    FunctionExpectingGfxRect(ToGfxRect(int_rect))
  and when int_rect is converted to gfx::Rect, we will be forced to
  change the above code to
    FunctionExpectingGfxRect(gfx_rect)
  instead of possibly leaving redundant gfx::Rect() here.

- They force change of IntRect to gfx::Rect where we already have
  gfx::Rect, like
    FunctionExpectingGfxRect(IntRect(gfx_rect))
   to
    FunctionExpectingGfxRect(gfx_rect)
  or simple cases like
    FunctionExpectingGfxRect(IntRect(1, 2, 3, 4))
   to
    FunctionExpectingGfxRect(gfx::Rect(1, 2, 3, 4))

- They mark the boundary of old and new geometry type usages and we
  can count the number (currently 469) of the usages of such
  functions to estimate the amount of the remaining work.

The original conversion operators are marked 'delete' so that
developers will know the new way when they use the old way.

Also added conversion functions for FloatQuad and FloatBox which will
be migrated to gfx::QuadF and gfx::BoxF, respectively.
