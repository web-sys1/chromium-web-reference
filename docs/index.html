[Contextual Tasks] Add option to skip animation

This change introduces a mechanism to disable the opening animation of
the Contextual Tasks side panel. This is primarily intended for cases
where the panel is launched via the private API, allowing for a
faster, non-animated transition when the corresponding feature flag is
enabled.

* Introduce `kContextualTasksPrivateApiNoAnimation` feature flag and a
  helper function to check its state.
* Add a `use_no_animation` parameter to `Show` methods in
  `ContextualTasksPanelController` and
  `ContextualTasksSidePanelCoordinator` to support skipping the open
  animation.
* Update `ContextualTasksUiService::StartTaskUiInSidePanel` to
  propagate this option to the controller.
* Retrieve the flag state in the private API launch function and pass
  it to the UI service.
* Add unit tests to cover the new animation paths in the coordinator.
* Modernize the definition of
  `kContextualTasksEphemeralBrandedEntryPoint`.
