*Scan: Return batched-freeing to PCScan's sweeper

The new PCScan-in-Renderer experiment disables the thread-cache on the
main partition, which means that the PCScan's sweeper is not able to
buffer freed entries in its own thread-cache anymore.

The CL builds up a freelist in the scanner thread and appends it to the
slot-span freelist. This helps to avoid contention for
PartitionRoot::lock when sweeping is in progress.
