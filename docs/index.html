[GWP-ASan] Switch to a faster PRNG

Currently, GWP-ASan utilizes a random number generator that fetches
entropy from the operating system for each call. This approach is
computationally intensive and exceeds the needs of GWP-ASan's specific
use case.

This CL introduces a more efficient XorShift PRNG, which maintains its
state in a thread-local variable, leveraging the existing thread-local
storage code from the `SamplingState` class.

Local tests show a 2x speed improvement in
`RecordLightweightDeallocation`, and a 1.22x speedup in a pair of
`GPA::Allocate` and `GPA::Deallocate` calls.
