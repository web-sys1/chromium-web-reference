Window Placement: Fix delayed cross-screen fullscreen on Mac

This is a minimal fix for concerns raised in http://crrev.com/c/2924204

Post a task to a static helper using the `bridged_native_widget_id`:
- This avoids a potential UAF from a base::Unretained() weak pointer.
- This also uses the current `target_fullscreen_state_`, not a snapshot.

Plumb the delay TimeDelta from BrowserView to handle two scenarios:
1) Initial cross-screen fullscreen requests; works locally with 1ms.
2) Cross-screen requests while fullscreen; works locally with 1000ms.
  (exiting the current fullscreen and moving to a new screen is slow)

NOTE: This is quite hacky; we'll continue seeking a better fix.
FOLLOWUP: Log UMA when the defect occurs, to understand efficacy?
FOLLOWUP: Create base::Feature parameters to tune the delays with Finch?
