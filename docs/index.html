Fix flaky BackForwardCacheBrowserTest.VideoSuspendAndResume.

The test sometimes gets an extra "pause" event at the end. E.g.

https://ci.chromium.org/ui/p/chromium/builders/try/linux_chromium_tsan_rel_ng/1145048/test-results

I believe this is because the video is 1s long and on slow devices and configs it reaches the end before the test ends.

This CLs sets loop=true on the video to avoid that problem.
