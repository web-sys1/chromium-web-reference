[TTFPassGen] Move PasswordGenerationAgent::FocusedElementChanged call earlier

Now PasswordGenerationAgent::FocusedNodeHasChanged is called from AutofillAgent::DidCompleteFocusChangeInFrame, which happens later then the virtual keyboard is shown (when a text input element is navigated).
It feels more natural to move the FocusedNodeHasChanged call to AutofillAgent::HandleFocusChangeComplete, which happens before triggering the keyboard. This is needed for the password generation bottom sheet (the bottom sheet should be able to suppress the keyboard before it's triggered).
