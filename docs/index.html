[sync] ModelLoadManager: Add a timer to track loading datatypes

Currently, if a single datatype is stuck at loading, i.e. controller is
in MODEL_STARTING state and never goes to MODEL_LOADED nor FAILED state,
then Sync as a whole fails to start up, as
ModelLoadManager::NotifyDelegateIfReadyForConfigure is never triggered.
This CL fixes this by adding a timeout for loading datatypes. All types
which fail to load by this time are treated as having errors.
