[cleanup] Simplify PrintBackendServiceImpl invalid data handling

Error handling for invalid data coming from the browser is unnecessary.
The browser process is supposed to be trusted, so if it is driving
the PrintBackendService with invalid usage then the process should
just crash rather than return error messages.

Replace these error handling blocks with simple DCHECKs.

LOW_COVERAGE_REASON=Covered primarily by browser tests.
