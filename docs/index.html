[Lacros] Prepare for fixing Sync error from profile menu

ShowReauthForPrimaryAccountWithAuthError() previously tried to enable
sync again after fixing the account error.

This CL changes the logic so that only the error is fixed, but sync is
not re-enabled while already running.

This is causing a small behavior change on desktop as well: a regular
gaia signin tab is shown, rather that the custom Chrome sync tab.

This change is in preparation for starting the sync flow based on the
`enable_sync` value.
