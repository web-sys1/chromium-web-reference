[Bookmarks Bar] Update scene layer offset when coming on screen

This CL updates the scene layer offsets for the bookmark bar when the
bar is being animated back on screen. In some cases, BCIV is not able
to use the OffsetTag to set the correct y-position. In these cases, the
view/client must set them manually. One case is the top controls being
scrolled offscreen, and a link being pressed. The link should open a
new page and the top controls should scroll back on screen. When in
this state, in order to scroll with the rest of the top controls, the
bookmark bar needs to manually update its offset.
