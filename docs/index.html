apple: Pass -dead_strip independent of symbol level.

-dead_strip doesn't look at debug info. The comment that was here about
it requiring symbol_level 2 is wrong as far as I can tell.

This makes the mac build dead code stripping logic match Linux -- always
strip dead symbols in release builds. In particular, the CQ mac-rel bot
now strips dead symbols (like official production builds already did).

The motivation is a more uniform build configuration. Also, I was
surprised that we suddenly stopped passing -dead_strip in my local build
when I switched from symbol level 2 to 0.
