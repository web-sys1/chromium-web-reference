Original change's description:
> feed: ensure snackbar actions complete
>
> in-feed actions, like follow/unfollow, trigger snackbars.
> The follow/unfollow action doesn't actually take place until
> the snackbar is dismissed.
>
> Before this change, if the user uses pull to refresh,
> snackbar actions may not run until after the refresh request
> is sent to the server. Now, snackbars are dismissed prior
> to refresh, and we wait until any subsequent background operations
> are completed before starting the refresh.
>
> Because this change defers refresh until a bit later,
> I've update the refresh code to abort if the surface
> has already disconnected to avoid this new case.
>
> Note that this change does not resolve potential
> remaining background work when switching from main
> feed to following feed or vice-versa.
>
