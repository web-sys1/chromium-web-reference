Replace WebElements in FormTracker by (Form|Field)Ref

Currently autofill caches the elements it's tracking to be able to use
them when necessary. This is inconvenient because the elements are not
stored in weak pointers and this can cause memory leaks (more context
in the linked bug).

Autofill aims at resolving that by replacing the cached elements by
their renderer IDs.

This CL:
-Creates two new classes (Form|Field)Ref that hold a (form|field) as
well as its corresponding ID. This should ultimately be replaced by just
the form or field renderer ID.
-Adds a new feature kAutofillReplaceElementsByRendererIds controlling
whether we use the elements or the ids in (Form|Field)Ref.
-Replaces form and field used in FormTracker with FormRef and FieldRef.
-Refactors FormTracker::CanInferFormSubmitted.
