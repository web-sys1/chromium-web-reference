Files app, Image Loader, Audio and Video Player: Use chrome-extension://

This is a preparation to be able to run optimize_webui=false loading
each JS module individually.

When running individual modules and importing a module from other
extension it needs to be addressed using chrome-extension://$OTHER_ID/.
Because the module can't be loaded using directory traversal "../../".

Note this shouldn't cause changes in the normal build using rollup,
because rollup has been configured to understand such URLs.

The JS unittest are loaded in the browser tests via
chrome://file_manager_test/ and any JS module imported as part of the
dependency of the tested module, might import using chrome-extension://
so we need to enable the extensions to be able to serve such files. For
this kind of request which comes from the origin
chrome://file_manager_test/ we need to allowlist each resource in the
`web_accessible_resources` in the manifest for the extension serving the
resource.

NB: This is CL:2672988 by lucmult@ reapplied to the main branch.
