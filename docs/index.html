[memories] Simplify omnibox keyword matching to only do exact matches.

This CL makes the omnibox keyword matching only accept exact matches.
It no longer uses the QueryParser component, but instead just directly
compares keywords (including multi-word keywords) against exactly what
the user entered.

In other words: It no longer splits tokens, either in the cluster
keywords nor in the user input.

This is meant to both simplify the code, as well as reduce the amount
of overtriggering observed on the omnibox action chip for Journeys.
