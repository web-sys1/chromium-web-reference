Never return a detached AXObject for AXObjectCacheImpl::Get()

If detached nodes are returned, it can lead to crashes elsewhere in
the system, as not all AXObject methods have guards to ensure they are
not detached. It also can lead to unexpected code paths.

Whitespace nodes that are not added were being created and
detached but remained in the cache, and sometimes returned and used.
To remedy, this CL moves the whitespace pruning logic before the
AXObject is even created.
