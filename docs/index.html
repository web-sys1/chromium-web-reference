[FilesBulkPinning] Update max queue size to 5

As part of our ongoing investigation into CPU and power consumption with
the bulk pinning feature, it was identified that a queue size of 5 would
be ideal to launch with. Then we should experiment with the queue size
being updated via a finch feature param.

This pref was introduced in M117 so currently has only rolled out to
dev / canary. So far folks using this pref have been limited to the
Files team testing the queue size functionality.
