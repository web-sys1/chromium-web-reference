Make ShapeResultView::RunInfoPart::ComputeStartEnd() to support zero character run

This patch changes `ShapeResultView::RunInfoPart::ComputeStartEnd()` to
return zero length range instead of no value when specified shape result
run has no character for accepting such runs for `RunInfoPart`.

Note: This is a speculative changes for
  `CHECK_EQ(part, out->Parts().data() + out->num_parts_)`
because we have no sample pages to reproduce this.

It seems zero character runs from `ShapeResult` are not copied to
`ShapeResultView::RunInfoPart`.

This patch also removes `CHECK_GT(num_characters, 0u)` from
`RunInfoPart` constructor.
