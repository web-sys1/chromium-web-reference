[base] Fix data race in CpuReductionExperimentFilter.

The call to base::CpuReductionExperimentFilter::ShouldLogHistograms in
ChannelPosix::Write caused a data race because accesses to the
`counter_` member were not synchronized and the call could happen
concurrently from multiple threads. This CL fixes the issue by making
`counter_` an atomic variable.
