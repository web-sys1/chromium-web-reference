[Skills] Pass skill to UI controller to populate dialog fields.

https://screencast.googleplex.com/cast/NTMyNTEzODUxMjgzODY1NnxlNGVhMWM1NC03Ng

This change adds the plumbing to pre-fill fields in the Skills dialog.
The SkillsUiTabController now passes the Skill data to the SkillsUI
controller when opening the dialog.

Because the dialog takes a moment to load, we have to store the skill
data temporarily in SkillsUI until the Mojo handler is ready. To make
this efficient, Iâ€™ve updated the code to move the data (using std::move)
from the TabController, through the UI, and into the final Handler. This
transfers the strings directly instead of making multiple copies in
memory.

Key changes:
 * The SkillsUI acts as a temporary mailbox for the Skill data.
 * The SkillsDialogHandler takes ownership of this data during construction.
 * Updated GetInitialSkill to return this stored data to the frontend, replacing placeholder logic for edit and remix workflows.
