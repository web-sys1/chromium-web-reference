RawDraw: improve scrolling performance for therecord.com

Before compositing a tile, sync token associated to the tile's backing
should be waited on, so SkiaRenderer can access paint ops in the backing
on the compositor thread, so SkiaOutputSurfaceImpl::FlushGpuTask() is
called with wait_for_tasks_finished = true. However we just need to make
sure the sync token is released, and we don't need to wait on all tasks
are finished. So this CL will allows we start compositing a tile
earlier by just waiting on sync tokens but not tasks completion.
