Relocate DeclarativePerformanceObserver feature flag

This CL moves the 'DeclarativePerformanceObserver' feature flag from the
network service ('services/network') to Blink, leveraging Blink's
built-in 'base::Feature' auto-generation in
'runtime_enabled_features.json5'.

Since the network service now parses the 'Performance-Observer' header
unconditionally, the feature flag is no longer needed there. Relocating
it to Blink aligns with Chromium's architectural standards for Web
Platform features.

By setting 'base_feature_status: "enabled"' and
'copied_from_base_feature_if: "overridden"', the generated
'blink::features::kDeclarativePerformanceObserver' serves as a
killswitch (enabled by default in C++) without enabling the feature by
default in the renderer process.

Furthermore, NavigationRequest gating is updated so that:
1. If kill-switched OFF via Finch or command line, observer creation is
   completely disabled.
2. Otherwise, valid Origin Trial tokens enable the feature
   per-navigation.
3. Explicit command-line overrides (--enable-features) enable the
   feature globally for testing and development.
StoragePartitionImpl uses the killswitch flag to conditionally
initialize DeclarativePerformanceObserverStore.

Key changes:
- Configured 'runtime_enabled_features.json5' with base_feature_status
  and copied_from_base_feature_if options for Origin Trial killswitch
  behavior.
- Updated NavigationRequest policy gating logic to support both Origin
  Trial token validation and explicit developer flag overrides while
  honoring the killswitch.
- Updated DeclarativePerformanceObserverOriginTrialTest to verify normal
  Origin Trial gating, killswitch behavior, and the explicit flag
  override behavior (testing that overrides activate the observer
  without an Origin Trial token).
- Removed manual C++ flag definitions from 'third_party/blink'.
- Removed manual mapping from 'content/child/runtime_features.cc'.
- Removed C++ flag definitions from 'services/network'.
- Updated all browser-side and test references to use the new
  auto-generated 'blink::features::kDeclarativePerformanceObserver'.
