ServiceWorker: Start the event immediately if its type is same as the type of the currently running events

This CL makes an event start immediately after its creation if its type
is the same as the events currently running.

The behavior slightly changes by this change.
Previously, if the events come like [online1, offline1, online2], the
order of execution is 1) online1 2) offline1 3) online2.
Now, if the events come like [online1, offline1, online2], the order of
execution is 1) online1 2) online2 3) offline1 if online1 is running
when online2 comes to the queue.

Also, this CL renames `id_event_map_` to `all_events_`.
