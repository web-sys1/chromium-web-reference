PageLoadMetrics: manage the PageLoadTracker per-page manner

This patch changes MetricswebContentsObserver to construct
PageLoadTracker in per-page manner. `committed_load_` is now
renamed to `primary_page_` as now we have multiple committed loads.
Other committed loads for non-primary pages are stored in
`active_pages_` instead. We clear inner page trackers when
the primary page navigation happens.

This is the first step to support FencedFrames in the
PageLoadMetrics as described in the design doc below.
https://docs.google.com/document/d/1sxzBBR4MZXvLFelN8EygSq5F8hxiaw-JeoiJtFxSu_w/edit?usp=sharing
