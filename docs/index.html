Restrict timeline-scope to <dashed-ident>

Apparently, timeline-scope was neglected when we made the decision
to make timeline names dashed idents; it still accepts non-dashed
idents in Blink.

This CL aligns our behavior with the spec, and also with Safari.

I'm confident that this bugfix has ~zero compat risk, since both
scroll timeline names and view timeline names only accept dashed idents.
In other words, if there are cases of timeline-scope:undashed out there,
they cannot match any named timelines.
