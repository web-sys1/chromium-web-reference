[SelectMenu] Correctly update document for SelectMenu mutation observer.

Prior to this CL, we were using SynchronousMutationObserver::DidMoveTreeToNewDocument
to update the document for the SelectMenu mutation observer.

This behavior is not correct, since DidMoveTreeToNewDocument is called
from HeapObserverSet::ForEachObserver[1] that doesn't allow adding or
removing observers while iterating.

This CL fixes the issue by using instead Element::DidMoveToNewDocument
to set the new document for the SelectMenu mutation observer.

[1]: https://source.chromium.org/chromium/chromium/src/+/main:third_party/blink/renderer/platform/heap_observer_set.h;l=65;
