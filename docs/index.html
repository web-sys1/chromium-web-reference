[FilesDarkMode] Fix splitter icon hover color

Issue: when the splitter button in the side nav is clicked, the hover
will disappear first and then the ripple color effect shows. Check this
demonstration video: http://shortn/_9k14NWSCfd (from 0.04)

When the splitter button is clicked and long pressed (`:active`), the
background color is a layered color: the `ripple-color` layers on
top of the `hover-bg-color`, so to make sure we can simulate the
`:active` style of the button when the splitter bar is clicked, we need
to override the background color:
* When both the splitter bar and the splitter button are active (this
activeness is triggered by clicking the splitter button), layered color
shows, when the mouse is moved away from the button without releasing
(`:active:not(:hover)`), hover color disappears but paper ripple
still shows, we need to make the background color to be the hover
color (--cros-ripple-color) to maintain a layered color.
* When the splitter bar is active but the splitter button is not active,
both hover color and the ripple color disappear, to mimic the `:active`
style, we need to set the background color directly to the layered
color (--cros-icon-button-pressed-color) to maintain a layered color.

Demo: http://shortn/_stw43uaFAz (when the color ripple effect is
started, the hover bg color is still existed)
