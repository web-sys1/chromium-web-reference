tracing: Fix data race with the SystemTracingService mojo remote.

The race happens because we bind the mojo::SharedRemote on the IO
thread, and the mojo::SharedRemote::is_bound() is checked on the
Perfetto task runner. An internal pointer is read/assigned on different
threads and leads to a potential race.
Fix the race by binding the remote on the Perfetto task runner when the
pending remote is received.
