[Sync] Remove obsolete LOG(FATAL) from SyncSetupChecker

Sync doesn't require decryption passphrase to be set in order to
complete first setup, removed LOG(FATAL) was likely helpful for some
legacy sync implementation (at least Directory, but probably even before
it).

These days, SyncSetupChecker completes successfully when passphrase is
needed, but not specified and this is intended behavior.
