[Nearby Onboarding] Dark mode support for visibility setting text

This fix lets the device visibility text inherit the cros_styles colors
to be both dark and light compliant for one-page onboarding. This only
affects onboarding when using Nearby Share from the Sharesheet or
chrome://nearby. This fix does not cause a regression when using
one-page onboarding from the Settings or System Tray.
