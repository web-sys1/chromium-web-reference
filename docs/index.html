Tweak how missing colors are handled for keyboard backlight color

When KeyboardBacklightController was written, kInvalidColor was used
to indicate that wallpaper color was not available. Now,
WallpaperCalculatedColor is represented as an optional where its
missing if its not available. A previous change was made to handle
this but it was incomplete.  This matches the semantics of the
original implementation.
