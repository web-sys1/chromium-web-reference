Don't crash when breaking before a repeated footer.

Of course, repeated footers should normally be ... well, repeated, and
we should never break before them. But if the table starts "too" late in
a fragmentainer, the footer might not fit. Apparently there was no test
coverage for this at all.

This reverts https://crrev.com/c/7532680 . We need to jump directly even
deeper into the fragmentation machinery. BreakBeforeChildIfNeeded()
didn't handle null builders properly, and there's no need for it to
handle that. After all, inserting a break does require a builder. Jump
directly into MovePastBreakpoint() instead, and break on our own if it
returns false. This time it's even more similar to how floats are
treated.
