[ZPS] Ensure that cache persistence prefs are cleared in `ClearCache()`.

The current implementation of the in-memory ZPS caching service has a
bug whereby the persistence-related prefs are NOT being properly cleared
when the user clears their browsing history.

This CL aims to eliminate this bug by properly clearing the
persistence-related user prefs when `ClearCache()` is called. This fix
will ensure that all data related to the in-memory cache is properly
emptied out when the user opts to clear their browsing history.
