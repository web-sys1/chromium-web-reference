[ODPPC] Explicitly set the trigger_notification.. field on issues

For now, all places in Chrome which compute password issues set this
field to false. In order to avoid an large change, the
InsecurityMetadata constructor had a default trigger_notification
param.

As of now, the trigger_notification field is only set to true outside of
Chrome (e.g. by the proactive check on Android). Chrome only which syncs
it down and persists it as-is. (That change was made in
https://crrev.com/c/4414899).

In order to prevent that this field is reset accidentally in the
future, all places should set it explicitly.
