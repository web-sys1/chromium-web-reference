Prevent AudioBuffer::ReadFrames CHECK failure

This CL changes WebAudioSourceProviderImpl::ProvideInput to verify that
the number of channels in the given AudioBus matches the number
expected. If not, it simply outputs silence. This may be the case due
to a race condition during configuration change, and audio output should
resume afterwards.
