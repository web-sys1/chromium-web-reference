cros: Force update token handle after online Gaia auth

Token handle is currently not updated, even after an online Gaia auth,
if the previous token handle is still valid.

This is a bug because the token handle can then point to an older LST,
and will be eventually invalidated by a Gaia cron job. This would cause
users to receive an incorrect notification that their current token is
invalid, when in reality an older token of theirs has been invalidated.

Fix this by force updating token handle when the user goes through an
online Gaia sign-in.
In combination with https://crrev.com/c/2659047, these CLs guarantee
that token handle will always be tied to the LST. Whenever LST changes,
in-session or at sign-in screen, we will force update the token handle.
