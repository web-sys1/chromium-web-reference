[Files app] Refactor DiskMountManager::MountPointMap -> MountPoints

Changed the type of the DiskMountManager::MountPointMap collection from
a map<string, MountInfo> to a set<MountInfo> indexed by mount path.

Previously, the mount path string was duplicated and stored twice (once
as key of the map entry, and once as part of the associated MountInfo
value). Now, with the set, there is no such data duplication, and the
code using this collections looks simpler and more natural.
