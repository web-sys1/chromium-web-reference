Original change's description:
> Add forced enrollment to screen result
>
> The enrollment screen was returning "completed" if the back action was
> triggered without forced re-enrollment being on. Semantically, this
> does not make any sense because clicking "back" should take a user back
> to the screen that would have been shown instead of enrollment screen,
> which is packaged license.
>
> Add a new enum to explicitly encode going back to auto re-enrollment
> check for forced re-enrollment and a normal back that goes to the
> packaged license.
>
> This does change behavior in the sense that packaged license is shown if
> one goes back from the enrollment screen instead of instantely going to
> the login screen. Also, "OnEnrollmentDone" isn't triggered anymore
> every time someone leaves the enrollment screen without enrolling.
