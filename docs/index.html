Refactor CompositorFrameSinkSupport to use FrameSinkThrottler

Extract throttling state and logic from CompositorFrameSinkSupport into
a new FrameSinkThrottler helper class. This encapsulates the logic for
calculating the effective begin frame interval based on multiple inputs,
including explicit throttle requests and video cadence signals.

While this is primarily a refactor, it introduces a functional shift in
how conflicting throttle intervals are resolved. Previously, throttling
was managed via a "last writer wins" approach where incremental updates
directly mutated the interval. This was prone to execution-order race
conditions (e.g., global throttling overriding a screen capture
exclusion). This CL shifts to a deterministic model where the effective
interval is strictly calculated based on current inputs.

Other notable changes:
- Add RecurseChildren and RecurseParents helpers to FrameSinkManagerImpl to facilitate safe hierarchy traversal.
- Add VideoCadenceThrottlingTest and RecurseFrameSinks tests.
