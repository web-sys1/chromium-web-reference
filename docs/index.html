mac: Always do Developer ID certificate reauthorization via the stub

In preparation of changing the code signing certificate, and with
https://chromium-review.googlesource.com/c/3469261/ now providing a
viable stub signed with the old Developer ID certiciate, nothing blocks
switching the Developer ID certificate used to sign the rest of Chrome
(except for the stub) to a new certificate issued in the same Team ID
that we are currently using (EQHXZ8M8AV). In order to allow that change
to be made, reauthorization can no longer be done in the application
itself, as once the certificate change is complete, the application will
lose access to Keychain items whose access control lists specify access
only by code signed with the old Developer ID certificate.
