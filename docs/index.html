arc-a11y: reset pointer to a notification window on surface removal

This is a speculative fix for Chrome crash.

There is a chance that after a notification surface is removed and
before ArcAccessibilityTreeTracker::OnNotificationStateChanged is
called, an accessibility event with notification window can be delivered
to ArcAccessibilityHelperBridge.

In order to prevent invalid access to the pointers on the above case,
with this change the pointer to a window is reset on the notification
surface removal.
