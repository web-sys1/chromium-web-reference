[memories] Pass incomplete visits to backend and sort results

This CL does two things:

 1. Also passes incomplete visits to the backend. Without this, the UI
    looks strangely devoid of all the open tabs, inconsistent with how
    History works. We pass the incomplete visits directly from memory
    because we want to avoid hitting the already contentious History DB
    thread.

 2. This CL also clarifies that the backend can take visits in any
    order, and return clusters in any order. This CL adds a sort step
    after getting the clusters from the backend.

    We made this choice, because it's not necessary for the backend to
    sort the output, since we will be persisting it to a database
    anyways. And moreover, the correct order may change after the user
    deletes constituent visits from the cluster.
