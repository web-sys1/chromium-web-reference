Disallow legacy language values in script elements

i.e. disallows

- <script language="javascript1.6">
- <script language="javascript1.7">

which aligns with the spec and Firefox.

According to the comment at
MIMETypeRegistry::IsLegacySupportedJavaScriptLanguage()
dating back at least 2007 in WebKit [1],
"javascript1.6" and "javascript1.7" are allowed because
Mozilla 1.8 allowed them, but now Firefox doesn't
allow these values [2].

This CL also removes non-WPT tests that expect these
legacy languages to be executed, as the test cases are
covered by WPTs like [2].

[1] https://source.chromium.org/chromium/chromium/src/+/f3b8edb28bfe485a95727eb9ad967227b0938811
[2] https://wpt.fyi/results/html/semantics/scripting-1/the-script-element/script-type-and-language-js.html
