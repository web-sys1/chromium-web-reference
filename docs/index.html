Add FCP and DCL milestones to AbandonedPageLoadMetricsObserver

This CL adds UMAs for FCP and DCL milestones to
AbandonedPageLoadMetricsObserver. Basically these two milestones are
reached after the ParseStart, which is implemented in the previous CL.

However, we can't perfectly guarantee these events are sent from the
renderer to the browser by the exact chronological order. So this CL
also includes changes on `latest_loading_milestone_`. After this CL,
`latest_loading_milestone_` always contains the latest loading milestone
in terms of the taken time from the navigation start, rather than latest
milestone that browser received order.
