Fix a crash when activating a prerendered page which did a same-doc navi

This is a speculative fix for a crash when activating a prerendered
page which did a same-doc navigation.

A crash happens in HistoryService:

> DCHECK_EQ(add_page_args.url, add_page_args.redirects.back());
(See bug 1211274 for details)

It seems a prerender activation doesn't update a NavigationRequest's
redirect chain correctly. This CL clears a redirect chain so that the
activation won't confuse the History Service or any other clients.

A redirect chain seems one of the parameters which is out of sync after
https://crrev.com/c/2674430.

Confirmed that a crash won't happen in the repro case in bug 1211274.
