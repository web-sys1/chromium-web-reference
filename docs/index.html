Extract a credential finder out of SecurePaymentConfirmationAppFactory

This CL refactors SecurePaymentConfirmationAppFactory to extract
the credential finding logic into a new class,
SecurePaymentConfirmationCredentialFinder. This simplifies the
tests for SecurePaymentConfirmationAppFactory, which no longer need
to care about setting up the underlying user database and/or
credential store APIs, and will also allow us to more easily change
the logic for credential finding in the future (e.g., to use *both*
the credential store APIs and the database together).

Changes in SecurePaymentConfirmationAppFactory unittests:
  - Functionality from test subclasses moved to the top-level
    SecurePaymentConfirmationAppFactoryTest class.
  - Tests now use a mock SecurePaymentConfirmationCredentialFinder
    instead of talking to mock database or authenticators.
  - Third-party payment bit related tests move to unittests for the
    new credential finder class.
  - AppDisabledIfCredentialStoreAPIsUnavailable test changed to be
    more generic, renamed to AppDisabledIfCredentialFetchingFails.
  - SecureConfirmationPaymentRequest_PaymentInstrumentDetails moved
    to the base test class.
  - Fallback_NoAuthenticator renamed to match its actual
    implementation, that there is no *user-verifying platform*
    authenticator available.
