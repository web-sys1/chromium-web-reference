Always check OOP print feature when calling RegisterSystemPrintClient().

As is, https://crrev.com/1002819 is incomplete and missed this check for
RegisterSystemPrintClient() in the system print dialog code path. This
puts the browser process in a bad state, and the system print dialog no
longer works correctly.

Fix this by consistently checking the OOP print feature before calling
RegisterSystemPrintClient(). Add DCHECKs to RegisterSystemPrintClient()
and UnregisterSystemPrintClient() to sanity check that they are being
called only when appropriate.
