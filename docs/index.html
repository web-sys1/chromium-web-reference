Fix Vulkan layout change ordering in RasterDecoder

GrContext::setBackendTextureState are ordered relative to other flush
calls, not the draw commands. So SkSurface::flush() will apply layout
that is necessary for drawing and transition of the images to the
desired end state needs to happen after it, but before submit call.

This CL splits FlushAndSubmitIfNecessary() to FlushSurface() and
SubmitIfNecessary() calls and fixes the ordering problem.
