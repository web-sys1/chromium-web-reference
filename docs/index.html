[Side panel] Combobox displays at most one extension entry (1 if global)

This CL implements the following behavioral changes:

Display only one entry in the side panel’s combobox if there is both a
matching contextual and global SidePanelEntry for a given extension for
the current tab.

When an extension’s entry (global or contextual) is registered, check
if another entry already exists for that extension, if so do not add
the item to the combobox model

When an extension’s contextual entry is deregistered check if a global
entry exists for that extension and if so, do not remove the item from
the combobox model

Changes in common:
 - For the combobox model, AddItem(s) does not add an item if it has
   the same key as another item in the combobox's entries.
 - The SidePanelRegistry that broadcasted the OnEntryWillDeregister
   event is also passed in it. This is needed for the third behavioral
   change.
