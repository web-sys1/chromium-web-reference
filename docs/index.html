Zero score on embeddings of low-word-count passages in similarity search

This CL creates a new SearchMinimumWordCount parameter for the
HistoryEmbeddings feature, and uses it as a threshold for embeddings
to be considered in the nearest-neighbor search. Word counts for
passages are stored with their embeddings to avoid a major performance
penalty on the search. Note, existing data is compatible due to the
proto defaulting behavior, so the database doesn't need to be rolled.
But existing embeddings will have passage_word_count = 0 and thus will
only show up in search if the new parameter is forced to 0 -- so to
see search results with the default of 3, visit pages again first.
