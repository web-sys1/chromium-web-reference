[TTFCC] Fix flakiness of touch to fill for credit cards render tests

The reason for the flakiness was the following: there are tests which
take the full screenshot of Chrome browser app. They were needed to test
that the bottom sheet height is correct in half-shown state. But the
problem was that the progress bar of the Chrome app turned out to be in
different states in the render tests, which caused failures. The fix is
the following: we no longer take the screenshot of the full Chrome
window for the test, instead we only screenshot the parent view group of
the bottom sheet. This gives us all the necessary information to test
the BS height not including the flaky progress bar.
