Implement UMA reliability metrics for persistent store

This CL adds database read reliability and TTL expiration tracking to
ensure data health and log arrival monitoring:
- Storage.DeclarativePerformanceObserver.ExpiredReportsCount: Logs the
  number of reports deleted due to the 7-day Time-To-Live (TTL)
  constraint.
- Storage.DeclarativePerformanceObserver.ReadReportResult: Logs the
  success and detailed failures (DB init, SQL, or JSON parsing failures)
  when loading stored early-failure reports for delivery.

This CL also refactors and cleans up the persistent store and its unit
tests by removing out-of-sync `CREATE TABLE` manual definitions.
