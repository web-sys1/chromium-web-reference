capture: Set correct texture target for 0c mf win video capture

After crrev.com/c/3159003, SCANOUT_CPU_READ_WRITE DXGI GMBs are marked
as unsupported since you can't use the DXGI GMB factory for that anyway,
so that we use shared memory GMBs for one copy canvas capture instead.

A side effect of that change was that the renderer uses GL_TEXTURE_2D
instead of the correct GL_TEXTURE_EXTERNAL_OES texture target which
causes a Skia assert to fail and rendering corruption as well.

This CL makes the renderer use GL_TEXTURE_EXTERNAL_OES explicitly. Long
term, we probably want to make the shared image the source of truth
instead about texture target of the renderer making an educated guess.
