Fix minimizeApp back press handler is initialized later than others

It seems minimizeApp is initialized slightly later than text bubble.
That means there is a small chance that a back press will be performed
to dismiss a text bubble while minimizeApp handler is not ready yet.
This CL makes initializing back press handler earlier.

Another possibility is some handlers cause app to be closed, which
removes minimizeApp handler. This CL adds an internal variable to
check that and stop triggering asserts if it has been destroyed.

This CL also polishes to cache finch param (no need to query every
time) and correctly check the max value of histogram.
