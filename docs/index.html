Add FLAG_WIN_SHARE_DELETE to FileURLLoader to avoid file locking

This CL adds base::File::FLAG_WIN_SHARE_DELETE when opening files in
FileURLLoader.

Currently, on Windows, opening a file via file:// URL locks the file,
preventing the user from deleting or renaming it while the page is
loading. Adding this flag allows shared deletion, improving the user
experience and aligning with behavior in other parts of the codebase
(e.g., net::FileStream).
