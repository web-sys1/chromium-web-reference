sql: Clean up Database::Open*() methods.

This CL cleans up the following aspects of our sql::Database::Open*()
method implementations.

1. The interface between Open() / OpenInMemory() / OpenTemporary() and
   OpenInternal() is cleaned up. The distinction between on-disk
   persistent / in-memory / temporary is encoded in explicit mode flags,
   not just in the database path. All mode flags are represented by an
   enum class.

2. OpenTemporary() is restricted to the sql::Recovery code. Its
   semantics are subtle, and not a good fit for Chrome features.

3. All methods with arguments have their preconditions DCHECKed.

4. Variable names and comments are aligned to SQLite's terminology in
   https://www.sqlite.org/rescode.html. TL;DR: SQLite result codes
   include success values, like SQLITE_OK.  SQLite error codes are a
   subset of result codes that signal failure.

5. Minor errors in comments or diagnostic strings are fixed.

This CL should not introduce any functional changes.
