Fix old search results briefly appearing on new search

In the productivity launcher:
1. Open the launcher and type "a"
2. Close the launcher
3. Open the launcher and type "b"
The "a" results will briefly flash on screen before being updated.

Cause: the search model holds on to the "a" results even after the
launcher is closed. This wasn't an issue until we introduced a
burn-in period before displaying the "b" results, leaving enough
time for the stale results to be shown.

This clearing isn't done when changing a query, only when going from
zero-state to a search.
