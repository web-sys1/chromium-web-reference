[Profile creation] Move some Switch* functions appropriately

This CL simplifies the ProfilePicker interface by removing three static
functions and replaces them by a more direct communication between the
controller and the delegate for the sign-in flow.

As part of the change, an existing function FinishSignedInCreationFlow()
gets exposed publicly and gets renamed to FinishAndOpenBrowser() to make
it clearer for the caller (profile_picker_view_sync_delegate.cc).

The CL introduces no behavioral difference: the implementation of the
previous static functions is equivalent to using a WeakPtr (except for
the rare case of deleting and recreating the picker view; for this case
it anyway feels safer to drop the calls to an invalidated WeakPtr than
to forward them to the new instance).
