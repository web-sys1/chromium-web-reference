Add test wait exiting condition to deflake

The test adds 3 image in turn. Each subsequent image is larger than the
previous one. Hence there would be 3 LCP performance timeline entries
emitted and the last one is the one that get recorded as the LCP.

The cause of the test failure is probably when the recording to LCP
happens, the last LCP candidate update IPC hasnâ€™t reached the browser
side yet.

This CL de-flakes the test by add test waiter expectations that it waits
for 3 LCP updates each of which has the LCP value larger than the one in
the previous update. This way, it ensures the last LCP update happens
before the test waiter exits the waits.
