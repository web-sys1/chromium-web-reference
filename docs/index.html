Fix crash/hang in ui::AXNodeData::IsIgnored from AXPosition

With the following change:
https://chromium-review.googlesource.com/c/chromium/src/+/2648678
AXPositions were allowed to traverse into empty objects.

There is a separate restriction under IsValid() that specifically
checks if a node is a child of an empty object, and considers these
nodes to be invalid, and converts them to NullPositions.

This creates problems when converting between TextPositions and
TreePositions, as the constructor will fail the IsValid check and
return a NullPosition. At this point, the anchor object is lost and
we end up crashing or hanging.

This change removes this restriction from the IsValid check and updates
some test expectations to reflect the new behavior.
