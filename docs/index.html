[Signin][Android] Add missing histogram coverage for Seamless Signin

Introduces AccountPickerDismissalLogger to encapsulate the logic for
logging metrics when the account picker bottom sheet is dismissed.
This simplifies AccountPickerBottomSheetCoordinator and
SeamlessSigninCoordinator by removing duplicated metric logging code
and improves test coverage for dismissal scenarios.

For consistency and readability between seamless flow and account
picker flow, AccountConsistencyPromoAction.DISMISSED_BUTTON is logged
in the mediator (see #logDismissedButtonClick())

Note that AccountPickerBottomSheetTest contains coverage for
DISMISSED_BUTTON and
getWebSigninAccountPickerActiveDismissalCount.
