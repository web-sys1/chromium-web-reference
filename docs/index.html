Fix a dangling pointer found by PA-E

If sdk_delegate_ and group_data_model_ are both non-null when the
method DataSharingServiceImpl::SetSDKDelegate(...) is called with
null then sdk_delegate_ would be deleted before group_data_model_.

As group_data_model_ has a raw_ptr<...> pointing to sdk_delegate_,
this lead to the crash found in https://crbug.com/445761354. Fix
this by swapping the member variable and the local variable, in
order to extend the lifetime of the DataSharingSDKDelegate until
the end of the current scope (instead of destroying it during the
assignation).
