Move call site of BrowserSessionRestoreDataLoaded

After recent refactoring attempt for LoginUnlockThroughputRecorder, it
turned out that BrowserSessionRestoreDataLoaded can be called when
actually all restoring windows has not been reported with
AddScheduledRestoreWindow.

This CL moves the call site of BrowserSessionRestoreDataLoaded to a more
appropriate place where listing windows should be completed for all
sessions, i.e. for browser and for apps. In addition, this CL changes it
to take the list of session windows all at once.
