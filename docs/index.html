Add UMA to track received GetCookiesString IPCs

We have anecdotal evidence that some sites very frequently call
GetCookiesString. We are working on a project to reduce these IPCs and
we would like to understand what is the distribution of these calls in
the field.

The UMA is collected every 30 seconds if a NetworkService is created.

Note on changed tests:
It turns out that some Attribution tests use the NetworkService (and
thus start the periodic timer to update the UMA) and use
TaskEnvironment::FastForwardBy with huge time deltas (100 days). This
now means that they will run the UMA update code time_delta / 30s times
(for 100 days that is 288000 times). This caused them to time out.
