Tagged PDFs: Move heuristic tree construction to a dedicated class

This change extracts the heuristic-based accessibility tree construction
logic from PdfAccessibilityTreeBuilder into a new
PdfAccessibilityTreeBuilderHeuristic class, placed in its own files.

The main builder now contains only the generic node creation
functionality and input data, while all heuristic-specific logic is
isolated. No functional changes.

This refactoring lays the groundwork for structure-tree integration,
coming in the following commit, by separating the two approaches.
