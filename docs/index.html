fileSystemProvider: keep service worker alive while FSP request is in progress

When a request is dispatched to a service worker in a Manifest V3
extension implementing a fileSystemProvider, keep the service worker
alive until the request is handled or aborted.

This is done by notifying ServiceWorkerLifetimeManager when the service
worker's request is sent, dispatched, or finished in (at appropriate
points in Ash and Lacros), which will adjust the service worker's
keep-alive state.

Note that the initialisation order of RequestManager and
RequestDispatcherImpl has been reordered in ProvidedFileSystem and
ExtensionProvider: RequestDispatcherImpl needs to outlive
RequestManager, because the request manager may access the dispatcher in
order to abort outstanding operation on destruction.
