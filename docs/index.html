Don't store in NoVarySearchCache entries that won't be written to disk cache

Previously, entries with Cache-Control: no-store could be written to the
NoVarySearchCache even though they would never be written to disk. Since
persistence isn't implemented yet, this had no security or privacy
implications, but was inefficient.

Delay inserting an entry into the NoVarySearchCache until the final
decision has been made to write the disk cache entry to disk.

In addition, stop keeping entries that have failed the simple cache
memory hint check in the NoVarySearchCache. Since these are going to be
deleted from the disk cache and refetched anyway, there is no benefit in
retaining the NoVarySearchCache entry. When the new response arrives it
will be added to the NoVarySearchCache as usual.
