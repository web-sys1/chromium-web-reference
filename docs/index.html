Fix Coop/Coep sanity check for same-document navigations.

We verify that COOP and COEP state match before committing a frame.
By that, we mean that if COOP was augmented with COEP, COEP was
necessarily set. These values are potentially wrong for
same-document navigations.

In practice it is a bit hard to trigger a mismatch because:
- Main frame will simply compare default COOP against default COEP,
  so it has to be a subframe.
- The check is in CommitNavigation(), but same-document render
  navigation simply go through DidCommit(), so it has to be a
  browser initiated navigation, and we're using history navigation.

This patch modifies the check to account for same-document navigations
and adds a reproducer test.
