[Sync] Optimize handling bookmarks reordering for moving single bookmark

Android platform uses bookmarks reordering to move just one bookmark.
Prior to current CL, this resulted in updating unique positions (and
hence reuploading) for all the bookmarks in the same folder. This CL
introduces an algorithm to detect those cases. This change should decrease the number of bookmark commits.

The algorithm tries to restore unique positions for individual bookmarks which were placed between two other bookmarks. If there are more than one bookmark have been moved into one place, the algorithm falls back to updating all the following unique positions.
