[code cache] Fix sending wasm generated code to code cache

Wasm code is generated on a background thread and a task is posted to
the main thread to send the data to the GeneratedCodeCache. To
achieve this the CodeCacheHost interface was obtained from the
current execution context but was used later when the task actually
executed. Though it is not ensured that the CodeCacheHost interface
would be alive when the task executes. Hence instead of using the
CodeCacheHost interface, use the execution_context as a weak
persistent pointer and obtain the code cache host interface if the
execution_context is still valid.
