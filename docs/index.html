[Viz] Clarify check of texture size in ImageContextImpl

ImageContextImpl checks whether the texture size is equal to that of
the GPU resource size when running with the validating decoder. After
digging, this check turned out to have been added in [1]. This CL
streamlines the logic of the check and adds comments clarifying its
purpose as well as the questions that need to be answered to determine
whether it is still relevant. There is no functional change.

[1] https://chromium-review.googlesource.com/c/chromium/src/+/1593102
