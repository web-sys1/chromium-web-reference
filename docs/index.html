[Downloads] Add failure toast for copy download link button

The "copy" button on chrome://downloads allows the user to copy the
download's URL to their clipboard. This uses the JS call
navigator.clipboard.writeText(), which (for security reasons) may be
prohibited e.g. if the page is not focused. A normal user would not
typically encounter this, but this happens sometimes if the button is
clicked programmatically.

This CL adds explicit handling of this exception state, and adds a
"toast" to give the user UI feedback when the copy operation failed.
