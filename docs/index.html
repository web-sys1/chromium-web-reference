[Extensions] Add extension host info to getUserAndExtensionSitesByEtld

This CL adds granted extension host permissions to SiteGroups.

New fields added:
runOnAllSitesCount: # extensions that have been granted access on all
sites.

numExtensions (SiteGroup): # extensions that are allowed to run on at
least one site for this SiteGroup's eTLD+1.

numExtensions (SiteInfo): # extensions that are allowed to run on this
site.

Processing extensions and their granted sites can be divided into a few
cases:
  1) If the extension has been granted access to run on all sites,
     increment runOnAllSitesCount by 1. Note that no numExtensions
     fields for SiteGroup and SiteInfo are updated for these extensions.
  2) If the extension has been granted access on specific sites:
     a) If the site is not in a user specified list
        (permitted/restricted), then increment numExtensions for that
        site (SiteInfo) and its eTLD+1s (SiteGroup)
     b) If the site is in a user specified list, then do not change
        anything. Note that the numExtensions count is not incremented
        for the site's eTLD+1 unless another site with the same eTLD+1
        is granted for the extension

Example. Say we have 3 extensions: A, B and C, with the following host
permissions:

A: <all_urls>
B: google.ca, example.com
C: google.ca, images.google.ca, a.example.com

a.example.com is a user permitted site.

Then the JSON response for getUserAndExtensionSitesByEtld() will return:

{
  runOnAllSitesCount: 1,
  siteGroups: [{
    etldPlusOne: google.ca,
    numExtensions: 2,
    sites: [{
      numExtensions: 2,
      site: google.ca
    }, {
      numExtensions, 1,
      site: images.google.ca
    }]
  }, {
    etldPlusOne: example.com,
    numExtensions: 1,
    sites: [{
      siteList: PERMITTED,
      numExtensions: 0,
      site: a.example.com
    }, {
      numExtensions: 1,
      site: example.com
    }]
  }]
}
