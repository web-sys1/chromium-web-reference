Update Password views to use BubbleAnchor

This change updates several `passwords views`-based UI components to use
`views::BubbleAnchor` instead of a raw `views::View` for their
`anchor_view`.

This migration of UI components from using views::View to
views::BubbleAnchor as anchor_view will enable better code reuse and
WebUI compatibility as views::BubbleAnchor supports ui::TrackedElement, enabling WebUI anchoring.

Key changes:

1. Refactor anchor handling to replace views::View with
views::BubbleAnchor in several password views.
2. Update PasswordBubbleViewBase to accept views::BubbleAnchor and manage anchoring for view based approach.
