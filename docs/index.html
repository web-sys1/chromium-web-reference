All blocked extensions are getting enable on browser restart,
and extensions will be enabled till browser makes call
to safebrowsing service.

Reason:
On extension_service start, it loads all the extensions and correctly
mark blocklist status based on preference file. After loading extensions,
it re-validate the blocklist status from v4_local_database_manager,
which gives all blocklisted ids, from local state.
Till this time init of local db is not yet done. So extension_service
gets empty blocklist extensions. As it gets empty list, it evaluates
all blocklisted extensions as unblocklisted, and mark each of them as
unblocklisted, which enables all these extensions.

These will be disabled again, after the first call to safebrowsing
service (~ 5 min after launch of browser), which gives the state of
thsese extensions as blocklisted.

Fix:
Remove the call to update blocklist state based on
v4_loca_databse_manager in init sequence, as service is able to mark
correct state based on preference file,
It can ignore re-evaluating of the state.
