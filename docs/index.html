float: Fix crash with multitask nudge when window is mostly offscreen

There is logic to close the multitask nudge when updating if the
window is dragged across displays. The crash was happening because we
tried to create the nudge on a window which is on primary display, but
its bounds were on the secondary display.

This CL removes a check that was added before the aforementioned
logic. It also first checks to see if the anchor is visible on the
same display as the window, so we stop trying to create the nudge in
the first place.

Test: manual, ash_unittests MultitaskMenuNudgeControllerTest.NoCrashAfterActivatingMostlyOffscreenWindowMultidisplay
