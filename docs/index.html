Throttle a display locked frame, even if it needs IntersectionObserver

If an entire frame is display locked because it's an iframe whose owner
<iframe> element is locked in the parent document then...

- We should not run lifecycle updates, even if the frame requires an
IntersectionObserver update.

- IntersectionObserver should be allowed to run, even if the document
is dirty (because it skipped lifecycle updates).

It's OK to do this because when the IntersectionObserver algorithm runs,
it will skip all geometry work due to the display lock, and just
generate a "not intersecting" notification if one is needed.
