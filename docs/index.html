Prevent metric tests from writing to registry on Windows

Adds RegistryOverrideManager to metrics tests that affect the Windows
registry. In particular, tests that call SetCollectStatsConsent,
either directly or indirectly (through
ChangeMetricsReportingState/ChangeMetricsReportingStateWithReply).

This was causing flakiness with some tests on Windows since they
sometimes started with metrics reporting enabled, and sometimes with
metrics reporting disabled. This is because the default state of
metrics reporting is decided by some registry value ("usagestats").
