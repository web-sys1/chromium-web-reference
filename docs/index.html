[MV][RQ] Extract and improve mechanism fetching favicons.

This change extracts the mechanism that retrieves favicons to a
separate component that is now shared between all Suggestion
elements. The component caches information what kind of icon is
available for any URL (but not the icon itself) for the duration of
the Omnibox interaction.

The component introduces multi-step backoff in order to make an
attempt to retrieve lower quality favicon if better quality favicon
is not available, and allows the caller to fall back to using
generated icon if that is desired.
