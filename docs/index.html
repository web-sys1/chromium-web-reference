[A11y] Do not remove subtrees for SetChildTreeId()

As of CL:5137975, changes to the "hidden by child tree" state now
propagates to descendants automatically.

The previous implementation for "hidden by child tree" changes is no
longer necessary. It removed the subtree of AXObjects in order to force
the AXObject for each node to reflect the new "hidden by child tree"
state (and dependent hidden/ignored/included states).
That implementation can now be removed, because merely kicking off
the OnInheritedCachedValuesChanged() is now sufficient to cause
the state changes to be inherited by all descendants.
