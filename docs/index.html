[omnibox][rich-autocompletion] Extend counterfactual logging.

Currently, we slice core omnibox metrics by whether rich ac triggered.

This CL adds 3 rich ac histograms. These are not used to slice omnibox
metrics.

1) Omnibox.RichAutocompletion.Triggered
   Which types, if any, of rich ac triggered. E.g. title prefix or URL
   non-prefix. Logged once per session for each type of rich
   autocompletion that triggered or 0 times if no rich autocompletion
   triggered.

2) Omnibox.RichAutocompletion.Triggered.Any
   Whether any rich ac triggered. Logged exactly once per session.

3) Omnibox.SuggestionUsed.RichAutocompletion
   Which type, if any, of rich ac was used. Logged once or 0 times per
   session.

See the descriptions in `omnibox/histograms.xml` for more details on
when each is logged.

The motive is to better understand how often each type of rich ac
triggers or is used. We usually run experiments with multiple types of
rich ac enabled and don't have any ability to distinguish the effects of
each type.
