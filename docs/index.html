Make `AllAssertionsPassed` aware of frame lifecycle state.

Sometimes TestRecipeReplayer::AllAssertionsPassed would attempt to
`EvalJs` in a frame that was unloading or in back/forward cache, this
would cause `EvalJs` to fail. This became more noticable when `unload`
event handlers were disabled in all tests and more pages entered
back/forward-cache.

Now, a frame that is not in the `kActive` state will be considered to
fail assertions without attempting to `EvalJs`.

With this test, the previous error does not appear.
