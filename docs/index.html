media/gpu/v4l2: Split upstream compatible and ChromeOS specific code

Various parts of the V4L2 acceleration code conform to upstream
specifications. However, there are some parts that are ChromeOS
specific. These include support for pre-upstream, pre-finalized
stateless decoders, and DMAbuf offsets used in the encoders.

Split the V4L2 code along the "upstream compatible" line, such that
the upstream compatible subset can be built for standard Linux.

Currently this involves either building natively with a sufficiently
recent set of kernel headers, or unpacking the latest (5.17.x as of
this writing) linux-libc-dev package into the sysroots by hand:

    wget http://ftp.us.debian.org/debian/pool/main/l/linux/linux-libc-dev_5.17.1-1~exp1_amd64.deb
    dpkg-deb -X linux-libc-dev_5.17.1-1~exp1_amd64.deb build/linux/debian_bullseye_amd64-sysroot/
    wget http://ftp.us.debian.org/debian/pool/main/l/linux/linux-libc-dev_5.17.1-1~exp1_arm64.deb
    dpkg-deb -X linux-libc-dev_5.17.1-1~exp1_arm64.deb build/linux/debian_bullseye_arm64-sysroot/
    wget http://ftp.us.debian.org/debian/pool/main/l/linux/linux-libc-dev_5.17.1-1~exp1_armhf.deb
    dpkg-deb -X linux-libc-dev_5.17.1-1~exp1_armhf.deb build/linux/debian_bullseye_arm-sysroot/
    wget http://ftp.us.debian.org/debian/pool/main/l/linux/linux-libc-dev_5.17.1-1~exp1_i386.deb
    dpkg-deb -X linux-libc-dev_5.17.1-1~exp1_i386.deb build/linux/debian_bullseye_i386-sysroot/

For arm64, the amd64 headers also need to be updated, as V8 snapshot is
currently built for amd64, and for some reason the V4L2 stuff is built,
even though it is not used. Same goes for armhf and i386.

This allows building V4L2 decoders for VP8, H.264 and VP9 videos on for
standard Linux, which would allow partners and the broader ecosystem to
test and compare Chromium V4L2 decoder implementations against others,
such as Gstreamer or FFmpeg.

This would also allow further development to actually make V4L2 usable
on Linux. Currently other parts of the media pipeline, such as buffer
allocation, still depend on things not normally found on standard Linux.
