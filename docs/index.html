Listen to FaviconDriverObserver to update Return to Recent Tab tile

Upon cold start, the favicon associated with the current page in
a WebState is not available and has to be fetched from the db. Thus,
the favicon will not be available when the Return to Recent Tab tile
is first configured. A fetch needs to be executed and a Start Surface
object needs to listen to FaviconDriverObserver to received the fetched
favicon.

StartSurfaceRecentTabBrowserAgent is a good fit for this observation
to be able to then send the favicon to ContentSuggestionsMediator.
The name of the browser agent observer is changed to reflect this new
additional purpose.
