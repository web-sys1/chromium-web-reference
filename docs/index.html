Prerender: Remove Prerender2AllowActivationInBackground flag

This change tweaks the visibility check for prerender activation to
eliminate the need for the `Prerender2AllowActivationInBackground` flag.

Previously, the flag was used to allow prerender activation in hidden
tabs, primarily to work around race conditions specific to Speculation
Rules target_hint WPTs on macOS. This flag is no longer necessary.

The visibility check condition is now loosened specifically for the
`target_hint` case. Activation is now permitted if either the
prerendering `web_contents` or its `initiator_web_contents` is visible.
This ensures correct behavior when prerendering in a new tab
(`target_hint: "_blank"`), where the new tab's visibility might
initially be hidden while the initiator tab remains visible.

The associated virtual test suites are also removed as they are no
longer required.
