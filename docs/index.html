[editing] Make InsertParagraphSeparatorCommand work with fallback content

This patch changes `InsertParagraphSeparatorCommand::
CloneHierarchyUnderNewBlock()` to handle fallback content of <object>
to avoid `DCHECK(container->GetLayoutObject())` in
`CompositeEditCommand::AppendBlockPlaceholder()` where we insert a
placeholder into cloned subtree, e.g. <object><b><br></b></object>.

Note: the fallback content of <object> is populated by plugin update
timer or `HTMLImageLoader` afer layout, aka post layout.
