Added new view for optional toolbar button

ToolbarPhone has an optional button that shows up between the omnibox
and the tab switcher button. This button is mainly used to surface
features on a user-by-user basis (a.k.a. Magic Toolbar), it contains a
show/hide animation.

The contextual page action (CPA) project aims to surface features based
on the current website. With the limited real estate on the toolbar we
decided to replace the Magic Toolbar action with a contextual action
when loading compatible websites. We are aiming to experiment with two
main variations, one that swaps the icons in place and another one that
exposes the new action on an action chip which then colapses back into
a regular icon.

This CL adds a new view OptionalButtonView which shows a toolbar
button whose icon can be updated with one of two transitions: swapping
or expanding as an action chip. A follow-up CL (https://chromium-review.googlesource.com/c/chromium/src/+/3698772) creates MVC classes
and a final CL (https://chromium-review.googlesource.com/c/chromium/src/+/3697050/) integrates this
component into ToolbarPhone.
