[iOS] Stop Credential Provider coordinators when possible

This CL adds the CredentialResponseHandler, which is essentially a
wrapper around ASCredentialProviderExtensionContext to force all
interaction to happen through the parent
CredentialProviderViewController, allowing it to stop necessary coordinators when the CPE is about to dismiss.

While doing this, I also noticed that the password field on the
new password page had autocorrect turned on, which is probably not
the best behavior for a password being entered.
