Make image elements compatible with MISU.

<img> elements are ineligible for incremental style updates on inline
style (MISU), since they have custom style callbacks. However, in most cases
(i.e., when the image is not replaced by fallback content), the custom
style behaves identically to regular style. Thus, only set the custom
style flag if we actually have custom style, leaving it eligible for
MISU in all other cases. This helps ~7% on MotionMark's “leaves” benchmark.

All the heavy lifting (profiling and identifying the issue) by vmiura@.
