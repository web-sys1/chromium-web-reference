Improve notification logic for cellular policy application

This CL has two parts:

Part one: MNCH triggers ESim installation:

Move the responsibility of triggering new cellular policy application
from PolicyApplicator to ManagedNetworkConfigurationHandler (MNCH).
PolicyApplicator only reports back a set of newly found cellular policy
network configs in its OnPoliciesApplied signal to MNCH.
MNCH then triggers the ESim installation.

Rationale: PolicyApplicator can report back that it's done after the
parts that are fast (non-cellular policy application) are done.

Part two: MNCH ensures that PoliciesApplied is never called on its
observes before non-cellular policy application is done.

The state of policy application for cellular/non-cellular will be made more explicit in a follow-up CL.
