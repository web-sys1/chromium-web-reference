[a11y] Generate popup lifecycle events for dynamic web menus

Dynamic role=menu nodes need the same popup lifecycle as native menus:
MENU_POPUP_START when they enter the exposed accessibility tree, and
MENU_POPUP_END when they leave it.

Previously, dynamically-created web menus could emit a close event on
delete without a matching open event, and hide/delete sequences could
either drop the close when the target had become ignored or produce a
duplicate close. This CL makes AXEventGenerator emit MENU_POPUP_START
when a non-ignored menu is created or becomes unignored, and emit
MENU_POPUP_END when an exposed menu becomes ignored. AXTreeManager keeps
the delete-time close path for menus that are still exposed, but skips
it for ignored menus so hide-then-delete only closes once.

BrowserAccessibilityManager now allows generated MENU_POPUP_END to be
dispatched for the ignored menu target because that event represents the
menu leaving the exposed tree.

This makes Chromium views and web content behave the same way, and meets
the spec: https://www.w3.org/TR/core-aam-1.2/#mapping_events_menus

There's a remaining issue I'll investigate as part of the ViewsAX bug fixing effort where menu events in UIA are duplicated.

AT verification: Validated that NVDA and Narrator behave correctly, but unable to confidently review the behavior with JAWS as is often the case in local builds. I'll test this change with JAWS in Canary.

There's a known issue about menu closed event being duplicated on UIA only but I couldn't observe negative consequences from this unexpected behavior, so I'll investigate in a follow-up change.

AX-Relnotes: Dynamic web menus now emit menu open and close accessibility events consistently.
