test: fix ProfileKeyedService tests in branded builds

in OptimizationGuideKeyedService::Initialize() , in branded builds,
optimization_guide_fetching_enabled is true, eventually
KAnonymityServiceFactory creates the service, which the test doesn't
expect. In non-branded builds, this code path isn't taken.

This CL adds an expectation for KAnonymityServiceFactory in branded
builds for
ProfileKeyedServiceBrowserTest.GuestProfileOTR_NeededServices and
ProfileKeyedServiceBrowserTest.GuestProfileParent_NeededServices.

KAnonymityService is required by the Protected Audience API to
to determine which ads are k-anonymous enough to be shown to the user,
for both guest and OTR profiles.

The CL also removes a TODO for crbug.com/1418376.
