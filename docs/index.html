Make UnretainedWrapper::GetPtrType obey raw_ptr_traits::IsSupportedType

GetPtrType shouldn't be raw_ptr<T>, when T isn't supported by raw_ptr.
It simply won't compile.

To be fair, those same cases wouldn't compile in the current code for
another reason, but this adds clarity, which allows to strengthen
a static_assert.

Update comments while at it.

This document explains the allowed type combinations in the current
implementation:
https://docs.google.com/document/d/1dLM34aKqbNBfRdOYxxV_T-zQU4J5wjmXwIBJZr7JvZM/edit
