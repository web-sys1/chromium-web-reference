Allow PostLayout when simplified layout copies layout boundaries

When subtree-layout requires to rebuild spines,
|RebuildFragmentTreeSpine| skips rebuilding the containing
block of the subtree layout root if |NeedsLayout|.

When laying out this containing block, if simplified layout is
chosen, it may copy out-of-date fragments that were not
rebuilt.

This patch allows PostLayout in such case.
