[DNS] Add revocation type

Add revocation type to PermissionsData to indicate the reason why
permissions were revoked. It is especially relevant now with the
addition of disruptive notification revocation.

Previously the code checked if there are notifications in the revoked
permissions list and distinguished between unused permissions and
abusive notifications based on that. There is no way to know whether the
notifications were revoked because they are disruptive or abusive
without the additional field. To simplify the UI logic, also use the
revocation type for unused permissions and combinations of unused permissions and notification revocations.
