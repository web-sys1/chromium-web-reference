[GPU][Android] Move AHBFormat-related functions into the AHB SI factory

ahardwarebuffer_utils.h exposes functions that take in ResourceFormat
and convert it to AHBFormat. The only consumer of these functions is the
AHB SharedImageBacking factory. This CL folds the functions into the
implementation file of that factory so that they don't grow further
usage (note: it doesn't make sense to convert them to take in SIF at
this time as their only callsites are currently tied to ResourceFormat).
