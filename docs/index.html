Merge prompt-based tests into base class; add explicit tests for
implicit grants

This makes each test fixture explicitly spell out the features it relies
on, instead of making assumptions about the default state of the
feature params.

It also rewrites the core test cases to not rely on any features
Chromium provides that aren't guaranteed by the Storage Access API
spec - namely, implicit grants, First-Party Sets integration, and
interaction with storage.

Finally, it adds an explicit test for the behavior of implicit
grants, since none of the existing tests exercised that behavior.
