Remove logs accumulation

The autofill logging had the following behavior:
Any time one or more internals pages were open, the LogRouter would
keep a log (no pun intended) of log messages. This means if two
internals pages were open and you refreshed one, the latter could get
all the log messages since the time at least one internals page was
open. When the last internals page was closed, the accumulated logs were
wiped and new log messages were ignored until a new internals page was
opened.
This effectively ensured that any two internals pages of the same type
(e.g. two chrome://autofill-internals pages) would show the same content.

With this CL, the behavior changes such that an internals page is a
true event logger: It only keeps events from the time that it is open
and never shows events from before it was opened.

This is the foundation for a follow-up CL in which we can pause logging,
which is desirable because internals pages can consume quite a lot of
memory - especially if you forget closing them.
