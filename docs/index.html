[WebOTP] Fix double counting on Blink.Sms.BackendAvailability

Currently when we launch the one-time permission for the new backend,
we first record an "AVAILABLE" sample and call `onPermissionDone` to ask
for permission. Upon user grants such permission we call `listen` which
triggers another recording for the same OTP request.

Note that this should only affect non-Chrome browsers because Chrome is
allowlisted such that the one-time permission is auto granted.
