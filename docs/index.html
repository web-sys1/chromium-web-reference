Web app icon generator deals with non-BMP chars in URL and app name.

Fixes a bug where if the first letter of the app name or URL is a
non-BMP Unicode character (i.e. U+10000 or larger, such as emoji and
certain other characters), it would be displayed as a question mark in a
diamond, due to incorrectly taking only the first UTF-16 code unit,
rather than the whole first code point.

Had to change some test values to explicitly unsigned to avoid new
compiler warnings caused by changing char16_t to char32_t. (I don't know
why these weren't warning before.)
