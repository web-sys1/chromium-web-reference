[AIS] Launch app if it's already installed for an install request

This CL causes AppInstallService::InstallApp() requests to launch
an already installed app with the same PackageId if present instead
of showing install dialogs.

This CL includes some test refactor churn from making
OpenApplicationCallback return void to make it compatible with
RepeatingTestFuture (and nothing was using the return value anyway).
