[Extensions] Test extension fetch() requests with header modifications.

Before this change there wasn't a test that explicitly confirmed the
behavior of fetch() requests from extension contexts that attempt to
overwrite forbidden and non-forbidden headers.

After this change we confirm the current behavior of privileged contexts
(background scripts, extension resource scripts) and non-privileged
contexts (content scripts).

Currently the behavior between these contexts is the same: non-forbidden
headers can be set by the extension, but setting forbidden headers are
overwritten by the browser before sending. This complies with the
current web API spec for fetch(). In a future CL, we'll make it so that
privileged contexts can set forbidden headers.
