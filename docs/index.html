fileSystemProvider: improve error handling and cleanup formatting

Cleanup error handling and formatting to be consistent in all tests.

- Use catchError in all tests to catch expected errors: using try blocks
  would lead to confusing failure reports, as chrome.test.{fail,assert*}
  throw exceptions themselves, so in a structure like this:

    try {
      actionThatThrows();
      chrome.test.fail('should throw');
    } catch (error) {
      chrome.test.assertEq('SomeError', error.name);
      chrome.test.succeed();
    }

  chrome.test.assertEq line would run on a 'chrome.test.failure' string
  if actionThatThrows does not throw.
- Group the code in all the tests in three blocks (setup, action, check
  results).
- Use string interpolation to construct entry paths everywhere.
