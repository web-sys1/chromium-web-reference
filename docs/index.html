[webauthn] Allow attestation under testing API

If the WebAuthn testing API is enabled and the UI is disabled, chrome
thinks that requests for WebAuthn attestation are erroneous cryptotoken
requests, resulting in a NOTREACHED() and disallowing attestation. This
regressed when we removed the VirtualAuthenticatorRequestDelegate [1]
which used to always allow attestation.

Fix this bug by always allowing attestation if the virtual environment
is enabled and the UI is disabled. If the UI is enabled, chrome will use
the usual logic and prompt the user for attestation to match real
authenticators.

[1] https://chromium-review.googlesource.com/c/chromium/src/+/3630567
