Merge paint worklet animation flag checks

Previously, code shared by native paint worklet animations composited clip paths and composited bgcolor animations would depend on checking their respective individual feature flags.

This change creates a new function in NativePaintImageGenerator that includes both flags, and replaces all common code feature flag checks not dependent on checking for the presence of an animation or property value with instances of this check.

Importantly, this fixes a bug where composited clip paths was inadvertently dependent on composited bgcolor animations. This created a gap in test coverage as composited bgcolor animations are enabled by default in tests.
