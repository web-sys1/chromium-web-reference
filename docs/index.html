[Build] Optimize lookup_dep.py

If chrome_apk is built and lookup_dep.py is run on a class referenced by
chrome_apk (vs a test class) this CL decreases the lookup_dep.py runtime
from 40s to 5s.

This CL:
- Runs list_java_targets.py without --build and stops script if there is
a match. If there is no match list_java_targets.py is run a second time
with --build This optimization yields most of the performance win.
- Avoids reading the same .build_config file twice - Once in
lookup_dep.py and once in list_java_targets.py
- Adds a cache for the namelist of .jar files.

BUG=1195121
