CCA: Use CancelableEvent for reprocess event

We used to use WaitableEvent for reprocess event but unfortunately it
won't reject the promise when signalling with error, causing the error
being handled unexpectedly.

This CL changes to use CancelableEvent so the error can be thrown
properly when the returned promise is awaited.
