WebUI: Change some @override notation to non-jsdoc

TypeScript compiler after version 4.3 starts to transform the @override
jsdoc to override keyword in .d.ts files. This cause problem in some
files since TypeScript does NOT transform the @implements jsdoc to
"implements" keyword for interface in .d.ts files, and causing tsc to
complain about unknown overridden method.

To unblock upreving the TypeScript compiler to latest version, this
pre-cursor commit change those @override jsdoc to a simple comment for
documentation purpose, so the override keyword can be added later
if/when those files are migrated to TypeScript. Since for Closure
Compiler the usage of @override is optional, this shouldn't cause
problem when these files are used by Closure Compiler.
