multiword: dismiss on cursor move

Currently multi word suggestions are not dismissed if the user's current
cursor location moves away from the "end of the text" they are currently
editing.

This CL fixes this behavior by dismissing any suggestion currently shown
if the user's cursor moves away from the end of the current text.
