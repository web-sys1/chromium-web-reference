Composite property tree nodes for animations only when relevant property is animating.

Prior to this change, an animation of transform, opacity, filter, or
backdrop-filter would cause the creation of property tree nodes (in both
the paint property tree and the cc property tree) for all of them
(though only a single effect property node for both opacity and
backdrop-filter).

With this change, an animation of transform only causes the creation of
a transform node (in the transform tree), an animation of filter only
causes the creation of a filter node (in the effect tree), and an
animation of opacity or backdrop-filter only causes the creation of an
effect node (in the effect tree).

This provides the basis on which we can separate the transform node into
multiple transform nodes, for efficient implementation and animation of
individual transform properties, without causing a performance
regression.
