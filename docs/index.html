[Autofill] Refactor AsyncContext and async ML calls

This CL refactors the asynchronous ML model calls in AutofillManager
in two ways:

1. It restructures ApplyMlModel().

   Old behaviour: ApplyMlModel() was called twice, once synchronously
   and once asynchronously, to run the Autofill and the PWM models,
   respectively. The second call was done by the first one.

   New behaviour: RunMlModels() is called asynchronously. It
   triggers two asynchronous model runs, for Autofill and PWM.

   Motivation: The threading through of the AsyncContext is easier to
   follow with new structure. The new name is to avoid confusion: after
   the followup CLs, RunMlModels() won't "apply" the predictions to the
   FormStructure (that'll be done in ParseFormsAsyncCommon()).

2. It threads through an AsyncContext through all asynchronous calls.

   Old behaviour: Only the heuristics used AsyncContext.

   Future behaviour: Also the ML predictions will be stored in
   AsyncContext. AsyncContext::current_stage indicates whether we're
   running the Autofill or the PWM model.

   Motivation: In a followup CL, the final response callback
   (update_cache()) will apply the heuristic and ML predictions to
   the FormStructure.
