Run deduplication routine twice per milestone

AutofillSilentlyRemoveQuasiDuplicates modifies how the deduplication
routine behaves. Since the deduplication routine is only run once per
milestone, this would force us to stay on every channel for at least one
milestone during the rollout to get meaningful data.

This CL thus relaxes the "once per milestone" condition and allows
triggering the deduplication routine a second time for users enrolled in
the experiment. This is done by adding a new pref, which tracks whether
the deduplication logic ran a second time.

This will prevent us from using the existing deduplication metrics (e.g.
number of profiles removed) for the rollout, since the baseline is
different. A follow-up will thus implement feature specific metrics.
