[sync] Modernize Android tests to avoid setting sync-requested to false

Outside tests, no Java code invokes SyncService.setSyncRequested() with
false. This is because the UI only allows one joint operation, which is
to disable sync and sign out.

However, several tests exercise a different scenario, nowadays
unreachable outside tests, which is the case where sync is turned off
without signing out the user.

In follow-up patches, the APIs that make this unrealistic flow
possible are going away, so it is necessary to modernize the tests
first.
