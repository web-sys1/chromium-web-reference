iwa: Create a separate Manager for IWA policy

As it was discussed in [1] we should create a separate manager for
enterprise policy installed IWAs.
In this CL we just aim to establish the IsolatedWebAppPolicyManager.

This CL aims to keep the changes to the logic as little as possible to
make subsequent CL conceptually simpler. To do that we:
- renamed the old IsolatedWebAppPolicyManager to BulkIwaInstaller;
- extracted the IWA logic from the WebAppPolicyManager to the newly
  created IsolatedWebAppPolicyManager.

No new functionality was added.

In its current state the IsolatedWebAppPolicyManager can do one time
installation of the IWAs in the Managed Guest Session.
