Fix the issue that APK download reports are not sent.

Download reports are not being sent since we switched the APK check
from MIME type to file extension. To fix the issue in this CL:
* Move the file extension check from OnDownloadCreated to
OnDownloadUpdated because the file path is empty when the download
item is created.
* Check GetFileNameToReportUser instead of GetTargetFilePath because
for android Q+, the file path will be a content URI that doesn't end
with ".apk".
* Still send the report if the MIME type is APK but the file extension
is not APK so that we don't lose any report that is sent before the
change.
