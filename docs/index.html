Add a GuestOS Installer WebUI with a stub implementation

Right now it will just display a "This site can't be reached" error
page because the implementation doesn't assign any resources to the
WebUI.

The intended flow is that a GuestOS implementation will call
GuestOSInstallerDialog::Show with an appropriate URL (starting with
kChromeUIGuestOSInstallerUrl). In chrome_web_ui_controller_factory.cc
this URL is mapped to GuestOSInstallerUI as the C++-side WebUI
delegate, which has a static ::Create function that gets the URL as
input. It can then use this URL to decide which installer
implementation to forward too, and the URL can contain parameters for
further GuestOS-specific controls.
